ক্রিপ্টো ওয়ালেট অ্যাপ ডেভেলপমেন্টে সিকিউরিটি সবচেয়ে গুরুত্বপূর্ণ বিষয়। একটি ছোট ভুলও ব্যবহারকারীর সম্পদ হারানোর কারণ হতে পারে। এই পাঠে আমরা ওয়ালেট অ্যাপের নিরাপত্তা নিশ্চিত করার জন্য প্রয়োজনীয় বেস্ট প্র্যাক্টিস, রিয়েল লাইফ উদাহরণ, এবং সহজ অ্যানালজি নিয়ে আলোচনা করবো।

---

## কেন সিকিউরিটি এত গুরুত্বপূর্ণ?

ধরা যাক, আপনার ওয়ালেট একটি ডিজিটাল ব্যাংক। এখানে টাকা, চাবি, এবং দরজা—সবকিছুই ডিজিটাল। যদি দরজা দুর্বল হয়, চোর সহজেই ঢুকে পড়তে পারে। তাই, ওয়ালেট অ্যাপের প্রতিটি অংশে শক্ত নিরাপত্তা দরকার।

---

## বেস্ট প্র্যাক্টিস: ধাপে ধাপে

### ১. **প্রাইভেট কী/সিড ফ্রেইজ নিরাপদে সংরক্ষণ**
- কখনোই plaintext বা localStorage-এ রাখবেন না।
- OS-level secure enclave, hardware wallet, বা encrypted storage ব্যবহার করুন।
- ব্যবহারকারীর সিড/প্রাইভেট কী export করার সময় সতর্কতা দেখান।

### ২. **ইনপুট ভ্যালিডেশন ও স্যানিটাইজেশন**
- ইউজার ইনপুট (address, amount, memo) সবসময় যাচাই করুন।
- ফরমে XSS, SQL Injection, বা Buffer Overflow-এর ঝুঁকি থাকলে তা রোধ করুন।

### ৩. **ফিশিং ও স্ক্যাম প্রোটেকশন**
- dApp বা ওয়েবসাইটের URL যাচাই করুন।
- অজানা/ভুয়া ওয়েবসাইটে অটো-সাইনিং বন্ধ রাখুন।
- ইউজারকে সতর্কবার্তা দেখান (যেমন: "আপনি অজানা dApp-এ কানেক্ট করছেন")।

### ৪. **MFA/2FA ইন্টিগ্রেশন**
- এক্সচেঞ্জ বা অ্যাকাউন্টে Multi-Factor Authentication (MFA) বা Two-Factor Authentication (2FA) যুক্ত করুন।
- OTP, biometric, বা hardware token ব্যবহার করুন।

### ৫. **ডেটা এনক্রিপশন**
- সংবেদনশীল ডেটা (private key, seed, PIN) সবসময় এনক্রিপ্টেড ফরমে রাখুন।
- AES, RSA, বা OS-level encryption ব্যবহার করুন।

### ৬. **সিকিউর API/নেটওয়ার্ক কমিউনিকেশন**
- HTTPS ছাড়া কোনো API কল করবেন না।
- Certificate pinning, CORS policy, এবং rate limiting ব্যবহার করুন।

### ৭. **কোড অডিট ও পেন-টেস্টিং**
- রেগুলার কোড অডিট করুন (নিজে বা থার্ড পার্টি দিয়ে)।
- Penetration testing করে দুর্বলতা খুঁজে বের করুন।

### ৮. **Error Handling ও Logging**
- Error message-এ কখনো sensitive data দেখাবেন না।
- Logging-এ private key, seed, বা password না রাখুন।

## রিয়েল লাইফ উদাহরণ

- **Ledger/Trezor:** Hardware wallet-এ private key কখনোই device-এর বাইরে যায় না।
- **MetaMask:** Seed phrase export করার সময় বড় সতর্কবার্তা দেখায়।
- **Phantom:** dApp connect করার সময় ইউজারকে স্পষ্টভাবে অনুমতি চায়।

## অ্যানালজি: ডিজিটাল দুর্গ

আপনার ওয়ালেট অ্যাপ যেন একটি দুর্গ—প্রতি দরজায় পাহারা, প্রতি জানালায় তালা। দুর্বলতা থাকলে চোর ঢুকে পড়তে পারে। তাই, প্রতিটি সিকিউরিটি লেয়ার গুরুত্বপূর্ণ।

---

## কোড স্নিপেট: এনক্রিপশন (JavaScript)

```js
import crypto from 'crypto';

const secret = 'user-private-key';
const password = 'strong-password';
const encrypted = crypto.createCipher('aes-256-cbc', password).update(secret, 'utf8', 'hex');
console.log('Encrypted:', encrypted);
```
