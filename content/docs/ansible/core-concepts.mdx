আনসিবল প্লেবুকের মূল ভিত্তি—টাস্ক, হ্যান্ডলার, ভেরিয়েবল ও মডিউল। এই পাঠে আমরা এগুলোর কাজ, উদাহরণ, বাস্তব ব্যবহার ও অ্যানালজি নিয়ে আলোচনা করব।

## ১. টাস্ক (Task)

টাস্ক হলো প্লেবুকের একেকটি ধাপ, যেখানে নির্দিষ্ট কাজ (module) এক্সিকিউট হয়।

### উদাহরণ:
```yaml
- name: Apache install
  apt:
    name: apache2
    state: present
```

### বাস্তব উদাহরণ:

আপনি চান ১০০টি সার্ভারে Apache install করতে—একটি টাস্ক লিখে, সব সার্ভারে একসাথে install হবে।

### অ্যানালজি:

টাস্ক হলো, শিক্ষক ক্লাসে "বই খুলো" বলেন—সব ছাত্র বই খুলে।

## ২. হ্যান্ডলার (Handler)

হ্যান্ডলার হলো বিশেষ টাস্ক, যা তখনই রান হয় যখন কোনো টাস্ক "notify" করে। সাধারণত সার্ভিস restart, reload ইত্যাদির জন্য ব্যবহৃত হয়।

### উদাহরণ:
```yaml
- name: Change config file
  copy:
    src: config.conf
    dest: /etc/app/config.conf
  notify: Restart app

handlers:
  - name: Restart app
    service:
      name: app
      state: restarted
```

### বাস্তব উদাহরণ:

Config file পরিবর্তন হলে, app restart করতে হবে—হ্যান্ডলার দিয়ে automate করা যায়।

### অ্যানালজি:

হ্যান্ডলার হলো, শিক্ষক বলেন "যদি কেউ দেরি করে, তাকে নোট দিতে হবে"—শুধু দেরি করলে নোট দেয়া হয়।

## ৩. ভেরিয়েবল (Variable)

ভেরিয়েবল দিয়ে ডাইনামিক মান প্লেবুকে ব্যবহার করা যায়—host, user, path, version ইত্যাদি।

### উদাহরণ:
```yaml
vars:
  app_version: 2.0

- name: Install app
  apt:
    name: myapp-{{ app_version }}
    state: present
```

### বাস্তব উদাহরণ:

আপনি চান, dev server-এ app v1.0, prod server-এ v2.0 install হবে—ভেরিয়েবল দিয়ে সহজে manage করা যায়।

### অ্যানালজি:

ভেরিয়েবল হলো, শিক্ষক ছাত্রদের নাম ধরে ডাকেন—"রাকিব, বই খুলো"।

## ৪. মডিউল (Module)

মডিউল হলো আনসিবল-এর বিল্ট-ইন বা কাস্টম ফাংশন, যা টাস্কে ব্যবহার হয়—apt, yum, copy, file, user, service ইত্যাদি।

### উদাহরণ:
```yaml
- name: Create user
  user:
    name: devuser
    state: present
```

### বাস্তব উদাহরণ:

আপনি ৫০টি সার্ভারে user add করতে চান—user module দিয়ে automate করা যায়।

### অ্যানালজি:

মডিউল হলো, শিক্ষক "বই খুলো" বলেন—বই module, "user add" module, "service restart" module।

## ৫. সংক্ষেপে

- **টাস্ক:** নির্দিষ্ট কাজ
- **হ্যান্ডলার:** notify হলে বিশেষ কাজ
- **ভেরিয়েবল:** ডাইনামিক মান
- **মডিউল:** বিল্ট-ইন/কাস্টম ফাংশন

সবকিছু মিলিয়ে আনসিবল প্লেবুককে শক্তিশালী, পুনঃব্যবহারযোগ্য ও ডাইনামিক করা যায়।

পরবর্তী পাঠে ইডেমপোটেন্সি ও YAML বেসিক নিয়ে আলোচনা হবে।
