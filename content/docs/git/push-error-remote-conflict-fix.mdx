রিমোট রিপোজিটরিতে কোড পুশ করার সময় অনেক সময় এরর বা কনফ্লিক্ট দেখা দেয়। এই পাঠে আমরা সাধারণ পুশ এরর ও রিমোট কনফ্লিক্টের কারণ, ডায়াগনোসিস ও সমাধান শিখব।

## ১. সাধারণ পুশ এরর

### (ক) `rejected` (non-fast-forward)

- **কারণ:** রিমোটে আপনার লোকাল ব্রাঞ্চের চেয়ে নতুন কমিট আছে।
- **সমাধান:**

```sh
git pull --rebase
# কনফ্লিক্ট হলে সমাধান করুন, তারপর
#
git push
```

### (খ) `error: failed to push some refs`

- **কারণ:** সাধারণত রিমোটে নতুন কমিট, বা প্রোটেক্টেড ব্রাঞ্চে পুশ করার চেষ্টা।
- **সমাধান:**
  - রিমোটে নতুন কমিট থাকলে `git pull --rebase` করুন।
  - ব্রাঞ্চ প্রোটেক্টেড হলে, অনুমতি চেক করুন বা PR সাবমিট করুন।

### (গ) `permission denied`/`authentication failed`

- **কারণ:** SSH/GitHub token/পাসওয়ার্ড সমস্যা।
- **সমাধান:**
  - SSH key/Token আপডেট করুন।
  - রিমোট URL চেক করুন: HTTPS/SSH ঠিক আছে কিনা।

## ২. রিমোট কনফ্লিক্ট ফিক্স

- **কারণ:** একই ফাইলে লোকাল ও রিমোটে পরিবর্তন হয়েছে।
- **সমাধান:**

```sh
git pull --rebase
# কনফ্লিক্ট হলে ফাইল ঠিক করে
#
git add <file>
git rebase --continue
# সব ঠিক হলে
#
git push
```

- প্রয়োজনে `git merge origin/main` ব্যবহার করতে পারেন।

## ৩. ফোর্স পুশ কখন?

- শুধুমাত্র নিজের ব্রাঞ্চে, এবং টিমের অনুমতি থাকলে:

```sh
git push --force-with-lease
```

- সতর্ক থাকুন: অন্যের কাজ ওভাররাইট হতে পারে!

## ৪. বেস্ট প্র্যাকটিস

- পুশের আগে সবসময় `git pull --rebase` করুন।
- কনফ্লিক্ট হলে ধাপে ধাপে সমাধান করুন, কখনোই অপ্রয়োজনে force push করবেন না।
- রিমোটে সমস্যা হলে URL, পারমিশন, ও ব্রাঞ্চ প্রোটেকশন চেক করুন।

---

**সংক্ষেপে:**
- পুশ এরর হলে প্রথমে কারণ বুঝুন, তারপর `git pull --rebase` দিয়ে সমাধান করুন।
- কনফ্লিক্ট হলে ফাইল ঠিক করে `git add` ও `git rebase --continue` দিন।
- ফোর্স পুশ খুব সাবধানে ব্যবহার করুন।
