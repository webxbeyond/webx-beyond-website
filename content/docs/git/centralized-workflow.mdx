## সেন্ট্রালাইজড ওয়ার্কফ্লো ব্যাখ্যা

সেন্ট্রালাইজড ওয়ার্কফ্লো (Centralized Workflow) হল গিট ব্যবহারের একটি সহজ ও প্রচলিত প্যাটার্ন, যেখানে একটি প্রধান (central) রিপোজিটরি থাকে এবং সবাই সেই রিপোজিটরিতে কাজ করেন।

### কিভাবে কাজ করে?
- টিমের সবাই একই রিমোট রিপোজিটরি (সাধারণত `origin`) ব্যবহার করেন।
- প্রত্যেকে লোকাল রিপোতে কাজ করেন, পরিবর্তন কমিট করেন।
- কাজ শেষ হলে সবাই একই ব্রাঞ্চে (সাধারণত `main` বা `master`) পুশ করেন।

### ধাপসমূহ
1. **রিমোট ক্লোন করুন:**
   ```bash
   git clone <repo-url>
   ```
2. **লোকাল চেঞ্জ করুন ও কমিট করুন:**
   ```bash
   git add .
   git commit -m "আপনার বার্তা"
   ```
3. **রিমোটে পুশ করুন:**
   ```bash
   git push origin main
   ```

### কনফ্লিক্ট ম্যানেজমেন্ট
- একাধিক ইউজার একসাথে একই ফাইলে কাজ করলে কনফ্লিক্ট হতে পারে।
- পুশ করার আগে `git pull` দিয়ে সর্বশেষ আপডেট নিয়ে নিন:
  ```bash
  git pull origin main
  ```

### সুবিধা
- সহজ ও সরল ওয়ার্কফ্লো
- ছোট টিম বা একক ডেভেলপারের জন্য উপযুক্ত

### সীমাবদ্ধতা
- বড় টিমে কনফ্লিক্ট বেশি হয়
- একই ব্রাঞ্চে সবাই কাজ করলে কোড ভেঙে যাওয়ার ঝুঁকি

### কখন ব্যবহার করবেন?
- ছোট টিম, সিম্পল প্রজেক্ট, বা গিটে নতুনদের জন্য

### উপসংহার
সেন্ট্রালাইজড ওয়ার্কফ্লো গিটের বেসিক ওয়ার্কফ্লো, যা দ্রুত শুরু করতে ও শেখার জন্য আদর্শ। তবে বড় টিমে ব্রাঞ্চিং ও আরও অ্যাডভান্সড ওয়ার্কফ্লো বিবেচনা করা উচিত।
