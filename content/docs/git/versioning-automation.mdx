## কমিট মেসেজ দিয়ে ভার্সনিং অটোমেশন

কমিট মেসেজের কনভেনশন (বিশেষ করে কনভেনশনাল কমিট) ফলো করলে স্বয়ংক্রিয়ভাবে ভার্সন বাম্প, চেঞ্জলগ জেনারেশন ও রিলিজ অটোমেশন করা যায়। এটি আধুনিক সফটওয়্যার ডেভেলপমেন্টে খুবই জনপ্রিয় ও কার্যকর।

### কিভাবে কাজ করে?
- প্রতিটি কমিট মেসেজে টাইপ (feat, fix, breaking change ইত্যাদি) উল্লেখ করুন
- টুল (যেমন semantic-release, standard-version) কমিট মেসেজ দেখে স্বয়ংক্রিয়ভাবে ভার্সন, ট্যাগ ও চেঞ্জলগ তৈরি করে

### জনপ্রিয় টুল
- [semantic-release](https://semantic-release.gitbook.io/semantic-release/)
- [standard-version](https://github.com/conventional-changelog/standard-version)

### উদাহরণ: semantic-release
1. **প্রজেক্টে ইনস্টল করুন:**
   ```bash
   npm install --save-dev semantic-release
   ```
2. **CI-তে কনফিগার করুন:**
   - GitHub Actions, GitLab CI, ইত্যাদিতে semantic-release রান করুন
3. **কনভেনশনাল কমিট মেসেজ ব্যবহার করুন:**
   - `feat: নতুন ফিচার`
   - `fix: বাগ ফিক্স`
   - `BREAKING CHANGE: ...`
4. **রিলিজ হলে:**
   - ভার্সন বাম্প, ট্যাগ, চেঞ্জলগ, npm/pypi/publish সব স্বয়ংক্রিয়

### সুবিধা
- ম্যানুয়াল ভার্সন বাম্প ও চেঞ্জলগ লেখার ঝামেলা নেই
- রিলিজ প্রসেস দ্রুত ও নির্ভুল
- টিমে কনসিসটেন্ট ইতিহাস

### টিপস
- টিমে কনভেনশনাল কমিট বাধ্যতামূলক করুন
- CI/CD পাইপলাইনে semantic-release বা standard-version যোগ করুন

### উপসংহার
কমিট মেসেজ দিয়ে ভার্সনিং ও রিলিজ অটোমেশন করলে সফটওয়্যার ডেভেলপমেন্ট আরও পেশাদার, দ্রুত ও নির্ভুল হয়।
