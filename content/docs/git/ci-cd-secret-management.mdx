## সিক্রেট ম্যানেজমেন্ট কী?

সিক্রেট মানে—API key, token, password ইত্যাদি সংবেদনশীল তথ্য, যা কোডে হার্ডকোড না করে নিরাপদে সংরক্ষণ ও ব্যবহারের নিয়ম।

## গিটহাব অ্যাকশনসে সিক্রেট

- **Settings → Secrets and variables → Actions**-এ গিয়ে সিক্রেট যোগ করুন
- ওয়ার্কফ্লো YAML-এ `${{ secrets.MY_SECRET }}` দিয়ে ব্যবহার করুন

```yaml
steps:
  - name: Use secret
    run: echo ${{ secrets.MY_SECRET }}
```

## গিটল্যাব সিআই/সিডি-তে সিক্রেট

- **Settings → CI/CD → Variables**-এ সিক্রেট যোগ করুন
- `.gitlab-ci.yml`-এ `$MY_SECRET` হিসেবে ব্যবহার করুন

```yaml
script:
  - echo $MY_SECRET
```

## বেস্ট প্র্যাকটিস
- কখনোই সিক্রেট/পাসওয়ার্ড কোডবেসে রাখবেন না
- শুধুমাত্র প্রয়োজনীয় পারমিশন দিন
- সিক্রেট রোটেট (পরিবর্তন) করুন
- প্রোডাকশন ও ডেভেলপমেন্ট সিক্রেট আলাদা রাখুন
- সিক্রেট ব্যবহারের জন্য environment variable ব্যবহার করুন

## সিক্রেট স্ক্যানিং
- গিটহাব ও গিটল্যাব উভয়েই সিক্রেট স্ক্যানিং ফিচার দেয়, ভুলে পুশ হলে সতর্ক করে
- [git-secrets](https://github.com/awslabs/git-secrets), [truffleHog](https://github.com/trufflesecurity/trufflehog) ইত্যাদি টুল দিয়ে লোকাল স্ক্যান করুন
