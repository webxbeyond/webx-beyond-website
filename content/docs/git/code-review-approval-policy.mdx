প্রফেশনাল সফটওয়্যার ডেভেলপমেন্টে কোড রিভিউ ও অ্যাপ্রুভাল পলিসি মানা অত্যন্ত গুরুত্বপূর্ণ। এতে কোডের মান, নিরাপত্তা ও টিম কমিউনিকেশন উন্নত হয়। এই পাঠে GitHub/GitLab-এ কোড রিভিউ ও অ্যাপ্রুভাল পলিসি সেটআপের কৌশল আলোচনা করা হয়েছে।

## ১. কোড রিভিউ কী ও কেন?
- অন্য ডেভেলপারদের কোড চেক করা, ফিডব্যাক দেওয়া ও বাগ/ইমপ্রুভমেন্ট খুঁজে বের করা।
- কোডের মান, কনভেনশন, নিরাপত্তা ও পারফরম্যান্স নিশ্চিত হয়।

## ২. Pull/Merge Request ও রিভিউ
- নতুন ফিচার/বাগফিক্স ব্রাঞ্চে কাজ শেষে Pull Request (GitHub) বা Merge Request (GitLab) সাবমিট করুন।
- টিমের নির্দিষ্ট সদস্য/রিভিউয়ার অ্যাসাইন করুন।
- রিভিউয়াররা কমেন্ট, সাজেশন, চেঞ্জ রিকোয়েস্ট দিতে পারেন।
- সব ফিডব্যাক ঠিক হলে অ্যাপ্রুভাল দিয়ে মার্জ করা হয়।

## ৩. অ্যাপ্রুভাল পলিসি সেটআপ (GitHub)
1. Settings > Branches > Branch protection rules-এ যান।
2. "Require pull request reviews before merging" এনাবল করুন।
3. "Require x approvals" (যেমন ১ বা ২) সেট করুন।
4. "Dismiss stale pull request approvals when new commits are pushed" এনাবল করুন।
5. "Require review from Code Owners" (প্রয়োজনে) এনাবল করুন।

## ৪. অ্যাপ্রুভাল পলিসি (GitLab)
1. Settings > General > Merge request approvals-এ যান।
2. "Number of required approvals" সেট করুন।
3. নির্দিষ্ট ইউজার/গ্রুপকে অ্যাপ্রুভার হিসেবে অ্যাড করুন।

## ৫. বেস্ট প্র্যাকটিস
- সব ব্রাঞ্চে মার্জের আগে কমপক্ষে ১ রিভিউ ও অ্যাপ্রুভাল বাধ্যতামূলক করুন।
- কোড স্টাইল, টেস্ট, সিকিউরিটি চেকলিস্ট ফলো করুন।
- রিভিউয়াররা গঠনমূলক ফিডব্যাক দিন, টিম কালচার পজিটিভ রাখুন।
- Code Owners ফাইল দিয়ে স্পেসিফিক ফাইল/ডিরেক্টরির জন্য রিভিউয়ার নির্ধারণ করুন।

---

**সংক্ষেপে:**
- কোড রিভিউ ও অ্যাপ্রুভাল পলিসি মানলে কোডবেস নিরাপদ, মানসম্মত ও টিমওয়ার্ক উন্নত হয়।
- GitHub/GitLab-এ সহজেই এসব পলিসি সেটআপ করা যায়।
