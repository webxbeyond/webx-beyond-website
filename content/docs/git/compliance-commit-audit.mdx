কমপ্লায়েন্স ও নিরাপত্তার জন্য গিট রিপোজিটরির কমিট হিস্টরি অডিট করা জরুরি। এতে কে, কখন, কী পরিবর্তন করেছে, কোনো সিক্রেট/সংবেদনশীল ডেটা কমিট হয়েছে কিনা, এসব যাচাই করা যায়। এই পাঠে কমিট হিস্টরি অডিটের কৌশল ও টুল দেখানো হয়েছে।

## ১. কমিট হিস্টরি দেখা

```sh
git log --pretty=fuller
```
- কে, কখন, কী কমিট করেছে বিস্তারিত দেখা যায়।

## ২. নির্দিষ্ট ফাইল/কোডের হিস্টরি

```sh
git log -- <file>
git blame <file>
```
- কোন লাইনে কে পরিবর্তন এনেছে, তা দেখা যায়।

## ৩. সিক্রেট/সংবেদনশীল ডেটা খোঁজা
- পুরনো কমিটে পাসওয়ার্ড/API key আছে কিনা খুঁজতে:

```sh
git log -S 'password'
git log -G 'AKIA[0-9A-Z]{16}'
```
- `-S` দিয়ে নির্দিষ্ট স্ট্রিং, `-G` দিয়ে regex খোঁজা যায়।

## ৪. টুল দিয়ে অডিট

### (ক) git-secrets
- কমিটে সিক্রেট আছে কিনা চেক করতে:

```sh
git secrets --scan-history
```

### (খ) truffleHog
- গভীরভাবে সিক্রেট/কনফিডেনশিয়াল ডেটা খুঁজতে:

```sh
pip install truffleHog
trufflehog git <repo-url>
```

### (গ) git-verify-commit
- কমিট সাইনড ও অথেন্টিক কিনা যাচাই করতে:

```sh
git log --show-signature
```

## ৫. কমপ্লায়েন্স রিপোর্ট তৈরি
- অডিটের ফলাফল ডকুমেন্ট করুন (কে, কখন, কী পরিবর্তন, কোনো সিক্রেট আছে কিনা)।
- প্রয়োজনে কমিট রিমুভ/রিরাইট করুন (BFG, git filter-branch)।

## ৬. বেস্ট প্র্যাকটিস
- নিয়মিত অডিট করুন, বিশেষ করে রিলিজের আগে।
- সিক্রেট পাওয়া গেলে দ্রুত রিমুভ ও রিভোক করুন।
- কমিট সাইনিং ও রিভিউ বাধ্যতামূলক করুন।

---

**সংক্ষেপে:**
- `git log`, `git blame`, git-secrets, truffleHog দিয়ে কমিট হিস্টরি অডিট করুন।
- কমপ্লায়েন্স নিশ্চিত করতে নিয়মিত অডিট ও রিপোর্ট তৈরি করুন।
