# ফেচ বনাম পুল বনাম পুশ (`git fetch`, `git pull`, `git push`)

রিমোট রিপোজিটরির সাথে কাজ করতে গেলে তিনটি কমান্ড সবচেয়ে বেশি ব্যবহৃত হয়: `git fetch`, `git pull`, ও `git push`। এদের কাজ ও পার্থক্য বুঝে নেওয়া জরুরি।

## ১. `git fetch`

- রিমোট রিপোজিটরিতে নতুন কোনো কমিট বা ব্রাঞ্চ এলে, সেগুলো লোকাল রিপোতে আপডেট করে কিন্তু আপনার ওয়ার্কিং ডিরেক্টরি বা লোকাল ব্রাঞ্চে কোনো পরিবর্তন আনে না।
- নিরাপদ, কারণ এটি শুধু ডাউনলোড করে, মর্জ বা চেঞ্জ করে না।

```sh
git fetch origin
```

## ২. `git pull`

- `git fetch` + `git merge` একসাথে।
- রিমোট রিপো থেকে আপডেট এনে সরাসরি আপনার লোকাল ব্রাঞ্চে মার্জ করে দেয়।
- দ্রুত আপডেট পেতে ভালো, তবে কনফ্লিক্ট হতে পারে।

```sh
git pull origin main
```

## ৩. `git push`

- আপনার লোকাল কমিটগুলো রিমোট রিপোজিটরিতে আপলোড করে।
- সাধারণত, টিমে কাজ করলে সবাইকে আপডেট জানাতে ব্যবহৃত হয়।

```sh
git push origin main
```

## তুলনামূলক টেবিল

| কমান্ড        | কাজ                | লোকাল চেঞ্জ | রিমোট চেঞ্জ |
|---------------|--------------------|-------------|-------------|
| git fetch     | শুধু আপডেট আনে    | না          | না          |
| git pull      | আপডেট আনে ও মার্জ | হ্যাঁ        | না          |
| git push      | আপলোড করে        | না          | হ্যাঁ        |

## কখন কোনটা ব্যবহার করবেন?
- শুধু আপডেট দেখতে চাইলে: `git fetch`
- সরাসরি লোকাল ব্রাঞ্চ আপডেট করতে চাইলে: `git pull`
- নিজের কাজ রিমোটে পাঠাতে চাইলে: `git push`

## সংক্ষেপে
- `git fetch` = শুধু ডাউনলোড
- `git pull` = ডাউনলোড + মার্জ
- `git push` = আপলোড

এগুলো বুঝে ব্যবহার করলে টিমে কাজ করা সহজ হবে এবং কনফ্লিক্ট কম হবে।
