# রিবেসিং এর মৌলিক বিষয় (`git rebase`)

গিটে রিবেস (rebase) হলো একটি ব্রাঞ্চের কমিটগুলোকে অন্য ব্রাঞ্চের সর্বশেষ কমিটের পরে "পুনরায় লেখা"। এতে ইতিহাস আরও পরিষ্কার ও সরল হয়।

## রিবেস কী?
- `git rebase` কমান্ড দিয়ে আপনি আপনার কাজের শাখার (feature branch) কমিটগুলোকে মূল শাখার (main/master) সর্বশেষ কমিটের পরে নিয়ে যেতে পারেন।
- এতে কমিট হিস্টরি লিনিয়ার হয়, মার্জ কমিট কম হয়।

## কখন রিবেস করবেন?
- যখন চান আপনার ব্রাঞ্চের কাজ মূল ব্রাঞ্চের সর্বশেষ আপডেটের পরে যুক্ত হোক।
- মার্জ কমিট এড়াতে ও ইতিহাস পরিষ্কার রাখতে।

## রিবেসের ধাপ

1. মূল ব্রাঞ্চে যান ও আপডেট নিন:
   ```sh
   git checkout main
   git pull
   ```
2. ফিচার ব্রাঞ্চে যান:
   ```sh
   git checkout feature/login
   ```
3. রিবেস করুন:
   ```sh
   git rebase main
   ```

## কনফ্লিক্ট হলে
- কনফ্লিক্ট হলে গিট বার্তা দেখাবে।
- কনফ্লিক্ট ঠিক করে ফাইল স্টেজ করুন:
  ```sh
  git add <file>
  ```
- তারপর রিবেস চালিয়ে যান:
  ```sh
  git rebase --continue
  ```

## সংক্ষেপে
- `git rebase` দিয়ে কমিটগুলোকে নতুন বেসে নিয়ে যান
- ইতিহাস পরিষ্কার ও মার্জ কমিট কম হয়
- কনফ্লিক্ট হলে ঠিক করে `git rebase --continue` দিন

রিবেস বুঝলে গিটে ইতিহাস আরও সুন্দর ও সহজে ম্যানেজ করা যায়।
