## রিফলগ (reflog) কী?

রিফলগ হলো—HEAD ও ব্রাঞ্চের সব মুভমেন্টের লোকাল হিস্টরি। ভুলে কমিট হারিয়ে গেলে বা ব্রাঞ্চ ডিলিট হলে এখান থেকে উদ্ধার করা যায়।

## হারানো কমিট খুঁজে বের করা
```sh
git reflog
```
- এখানে সব HEAD মুভমেন্টের লিস্ট পাবেন।
- কাঙ্ক্ষিত কমিটের হ্যাশ কপি করুন।

## হারানো কমিট বা ব্রাঞ্চ উদ্ধার

### ১. হারানো কমিট থেকে নতুন ব্রাঞ্চ
```sh
git checkout -b rescue-branch <commit-hash>
```

### ২. হারানো ফাইল ফিরিয়ে আনা
```sh
git checkout <commit-hash> -- path/to/file
```

## রিফলগ ক্লিনআপ
- রিফলগ লোকাল রিপোতে থাকে, গার্বেজ কালেকশন (git gc) দিলে পুরনো এন্ট্রি মুছে যেতে পারে

## সতর্কতা
- রিফলগ শুধু লোকাল রিপোতে থাকে, রিমোটে থাকে না
- গার্বেজ কালেকশন বা রিপো ক্লোন করলে পুরনো রিফলগ হারিয়ে যেতে পারে
