রিভার্স প্রক্সি হলো এমন একটি নেটওয়ার্ক ডিভাইস বা সার্ভার, যা ক্লায়েন্টের অনুরোধ গ্রহণ করে এবং সেগুলোকে এক বা একাধিক ব্যাকএন্ড সার্ভারে ফরওয়ার্ড করে। এটি ওয়েব অ্যাপ্লিকেশন, API, মাইক্রোসার্ভিস আর্কিটেকচারে অত্যন্ত গুরুত্বপূর্ণ।

## রিভার্স প্রক্সি কী?
- **Reverse Proxy** হলো একটি ইন্টারমিডিয়েট সার্ভার, যা ক্লায়েন্ট ও ব্যাকএন্ড সার্ভারের মাঝে অবস্থান করে।
- ক্লায়েন্ট সরাসরি ব্যাকএন্ড সার্ভারের সাথে যোগাযোগ না করে, রিভার্স প্রক্সির মাধ্যমে অনুরোধ পাঠায়।

## কেন রিভার্স প্রক্সি দরকার?
- **লোড ব্যালান্সিং:** একাধিক সার্ভারে ট্রাফিক ভাগ করে দেয়, ফলে পারফরম্যান্স ও স্কেলেবিলিটি বাড়ে।
- **সিকিউরিটি:** ব্যাকএন্ড সার্ভার হাইড করা যায়, DDoS/বট আক্রমণ থেকে সুরক্ষা পাওয়া যায়।
- **SSL Termination:** SSL/TLS এনক্রিপশন/ডিক্রিপশন রিভার্স প্রক্সিতে হয়, ব্যাকএন্ডে লোড কমে।
- **ক্যাশিং:** কমন কনটেন্ট ক্যাশ করে দ্রুত রেসপন্স দেয়।
- **কম্প্রেশন ও অপ্টিমাইজেশন:** ডেটা কম্প্রেস, ইমেজ অপ্টিমাইজ ইত্যাদি করা যায়।

## NGINX ও HAProxy

### NGINX
- ওপেন সোর্স ওয়েব সার্ভার ও রিভার্স প্রক্সি।
- লোড ব্যালান্সিং, SSL termination, ক্যাশিং, স্ট্যাটিক ফাইল সার্ভিং ইত্যাদিতে জনপ্রিয়।
- কনফিগারেশন সহজ, পারফরম্যান্স খুব ভালো।

#### উদাহরণ: NGINX রিভার্স প্রক্সি কনফিগ
```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### HAProxy
- ওপেন সোর্স লোড ব্যালান্সার ও রিভার্স প্রক্সি।
- TCP ও HTTP লেভেলে লোড ব্যালান্সিং, হাই অ্যাভেইলেবিলিটি, হেলথ চেক ইত্যাদিতে ব্যবহৃত হয়।
- বড় স্কেল ও হাই পারফরম্যান্সের জন্য উপযোগী।

#### উদাহরণ: HAProxy কনফিগ
```
frontend http-in
    bind *:80
    default_backend servers

backend servers
    server server1 127.0.0.1:3000 check
    server server2 127.0.0.1:3001 check
```

## রিভার্স প্রক্সি বনাম ফরওয়ার্ড প্রক্সি
- **Reverse Proxy:** ক্লায়েন্টের অনুরোধ ব্যাকএন্ড সার্ভারে ফরওয়ার্ড করে, ব্যাকএন্ডকে হাইড করে।
- **Forward Proxy:** ক্লায়েন্টের অনুরোধ ইন্টারনেট বা এক্সটার্নাল সার্ভারে পাঠায়, ক্লায়েন্টকে হাইড করে।

## বাস্তব উদাহরণ
- **ওয়েব অ্যাপ্লিকেশন:** একাধিক অ্যাপ সার্ভার, স্ট্যাটিক ফাইল, API ইত্যাদির জন্য NGINX/HAProxy ব্যবহার।
- **SSL Termination:** SSL সার্টিফিকেট NGINX-এ, ব্যাকএন্ডে প্লেইন HTTP।
- **লোড ব্যালান্সিং:** একাধিক সার্ভারে ট্রাফিক ভাগ করে পারফরম্যান্স বাড়ানো।

## চ্যালেঞ্জ ও বেস্ট প্র্যাকটিস
- **কনফিগারেশন সিকিউর রাখা**
- **হেলথ চেক ও মনিটরিং চালু রাখা**
- **লগিং ও অডিটিং**
- **SSL/TLS সঠিকভাবে কনফিগার করা**

## উপসংহার
রিভার্স প্রক্সি (NGINX, HAProxy) ওয়েব ও API আর্কিটেকচারে অপরিহার্য। সিকিউরিটি, স্কেল, পারফরম্যান্স ও মেইনটেনেন্স সহজ করতে এগুলোর ব্যবহার দিন দিন বাড়ছে।
