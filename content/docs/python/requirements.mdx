পাইথন প্রজেক্টে ডিপেন্ডেন্সি ম্যানেজমেন্টের জন্য `requirements.txt` ফাইল অত্যন্ত গুরুত্বপূর্ণ। এটি প্রজেক্টের সব প্রয়োজনীয় প্যাকেজ ও তাদের ভার্সন সংরক্ষণ করে, যাতে অন্য কেউ বা অন্য পরিবেশে সহজে সেটআপ করতে পারে। এই পাঠে আমরা জানবো:
- requirements.txt কী, কেন দরকার
- কিভাবে তৈরি ও ব্যবহার করবেন
- রিয়েল-লাইফ উদাহরণ ও analogies
- Best practices

## ১. requirements.txt কী?

এটি একটি সাধারণ টেক্সট ফাইল, যেখানে প্রতিটি লাইনে একটি প্যাকেজের নাম ও ভার্সন থাকে।

### উদাহরণ:
```
requests==2.31.0
flask>=2.0.0
numpy
```

## ২. কেন দরকার?

- প্রজেক্টের সব ডিপেন্ডেন্সি এক জায়গায় রাখা যায়
- অন্য কেউ সহজে সেটআপ করতে পারে
- ভার্সন কনফ্লিক্ট এড়ানো যায়
- ডিপ্লয়মেন্ট ও CI/CD-তে অপরিহার্য

### Analogies: বাজারের লিস্ট

যেমন বাজারে যাওয়ার আগে লিস্ট বানান—কী কী লাগবে, কতটা লাগবে। requirements.txt-ও ঠিক তেমন—প্রজেক্টে কী কী প্যাকেজ লাগবে, সব লিখে রাখেন।

## ৩. requirements.txt তৈরি

### ম্যানুয়ালি:
নতুন ফাইল খুলে প্রয়োজনীয় প্যাকেজ লিখুন।

### pip freeze দিয়ে:
```bash
pip freeze > requirements.txt
```
এতে বর্তমান এনভায়রনমেন্টের সব প্যাকেজ ও ভার্সন চলে আসবে।

## ৪. requirements.txt থেকে ইন্সটল

```bash
pip install -r requirements.txt
```

## ৫. ভার্সন কন্ট্রোল ও আপডেট

- নির্দিষ্ট ভার্সন লিখলে (==) কোড রিপ্রোডিউসেবল হয়
- `>=, <=` দিয়ে মিনিমাম/ম্যাক্সিমাম ভার্সন নির্ধারণ করা যায়
- requirements.txt আপডেট করতে নতুন প্যাকেজ যোগ করুন

## ৬. রিয়েল-লাইফ উদাহরণ: টিম প্রজেক্ট

ধরা যাক, টিমে ৫ জন কাজ করছেন। requirements.txt শেয়ার করলে সবাই একই প্যাকেজ/ভার্সন ব্যবহার করতে পারে, কোডে সমস্যা কমে যায়।

## ৭. Best Practices

- requirements.txt ভার্চুয়াল এনভায়রনমেন্টে তৈরি করুন
- নির্দিষ্ট ভার্সন লিখুন
- অপ্রয়োজনীয় প্যাকেজ বাদ দিন
- requirements.txt ফাইল ভার্সন কন্ট্রোলে (git) রাখুন
- pip freeze > requirements.txt দিয়ে আপডেট করুন

## ৮. সংক্ষেপে

- requirements.txt ডিপেন্ডেন্সি ম্যানেজমেন্ট সহজ করে
- pip install -r requirements.txt দিয়ে দ্রুত সেটআপ
- টিমওয়ার্ক, ডিপ্লয়মেন্ট, CI/CD-তে অপরিহার্য

এভাবেই requirements.txt ব্যবহার করে আপনার পাইথন প্রজেক্টকে আরও সংগঠিত, নিরাপদ ও মেইনটেনেবল করতে পারেন।
