## `|`, `>`, `>>`, `<`

লিনাক্স টার্মিনালের সবচেয়ে শক্তিশালী ফিচারগুলোর মধ্যে পাইপিং (|) ও রিডিরেকশন `>`, `>>`, `<` অন্যতম। এগুলোর মাধ্যমে আপনি একাধিক কমান্ডকে একসাথে ব্যবহার করতে, আউটপুট ফাইল/কমান্ডে পাঠাতে, ও ইনপুট ফাইল থেকে নিতে পারেন।

---

## ১. পাইপিং (|)

`|` (pipe) দিয়ে এক কমান্ডের আউটপুট সরাসরি আরেক কমান্ডের ইনপুট হিসেবে পাঠানো যায়।

```bash
command1 | command2
```

**উদাহরণ:**

```bash
ls -l | grep ".txt"
cat access.log | sort | uniq -c | sort -nr
```

---

## ২. আউটপুট রিডিরেকশন: > ও >>

- `>` : আউটপুট ফাইল-এ লিখে, আগের কনটেন্ট ওভাররাইট করে
- `>>` : আউটপুট ফাইল-এ অ্যাপেন্ড (শেষে যোগ) করে

**উদাহরণ:**

```bash
echo "Hello World" > hello.txt      # নতুন ফাইল/ওভাররাইট
ls -l > filelist.txt                # ফোল্ডারের কনটেন্ট ফাইলে
cat file1.txt file2.txt >> all.txt  # দুই ফাইলের কনটেন্ট এক ফাইলে যোগ
```

---

## ৩. ইনপুট রিডিরেকশন: `<`

`<` দিয়ে কোনো ফাইলের কনটেন্ট কমান্ডের ইনপুট হিসেবে পাঠানো যায়।

**উদাহরণ:**

```bash
sort < unsorted.txt
```

এখানে, `unsorted.txt`-এর কনটেন্ট sort কমান্ডে যাবে।

---

## ৪. কম্বিনেশন: পাইপিং ও রিডিরেকশন একসাথে

```bash
cat file.txt | grep "error" > errors.txt
```

এখানে, file.txt থেকে "error" খুঁজে errors.txt-তে লিখবে।

---

## ৫. স্ট্যান্ডার্ড ইনপুট, আউটপুট, ও এরর

- stdin (0): ইনপুট
- stdout (1): আউটপুট
- stderr (2): এরর

**রিডিরেকশন নম্বর:**
- `command > file 2>&1` : আউটপুট ও এরর দুটোই ফাইলে
- `command 2> error.txt` : শুধু এরর ফাইলে

---

## ৬. প্র্যাকটিক্যাল টিপস

- পাইপিং দিয়ে একসাথে অনেক ফিল্টার/প্রসেস করুন
- > দিয়ে রিপোর্ট/লগ ফাইলে আউটপুট রাখুন
- >> দিয়ে লগ অ্যাপেন্ড করুন
- < দিয়ে ফাইল থেকে ইনপুট নিন
- 2>&1 দিয়ে আউটপুট ও এরর একসাথে ধরুন

১. একটি ফাইলের কনটেন্ট grep দিয়ে ফিল্টার করে নতুন ফাইলে লিখুন
২. sort < file.txt > sorted.txt চালান
৩. পাইপিং ও রিডিরেকশন একসাথে ব্যবহার করুন
