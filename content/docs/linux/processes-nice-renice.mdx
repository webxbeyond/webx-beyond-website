লিনাক্সে একসাথে অনেক প্রসেস চলতে পারে, কিন্তু কোন প্রসেস বেশি CPU পাবে, সেটা নির্ধারণ করা যায় nice ও renice কমান্ড দিয়ে। এখানে আমরা nice ও renice-এর কাজ, সিনট্যাক্স, উদাহরণ, ও বেস্ট প্র্যাকটিস জানব।

---

## ১. প্রসেস প্রায়োরিটি ও nice value

- **nice value**: -20 (সবচেয়ে বেশি প্রায়োরিটি) থেকে 19 (সবচেয়ে কম)
- ডিফল্ট: 0
- কম value = বেশি প্রায়োরিটি

---

## ২. নতুন প্রসেসের প্রায়োরিটি সেট: nice

### বেসিক সিনট্যাক্স:
```bash
nice -n value command
```

### উদাহরণ:
```bash
nice -n 10 tar -czf backup.tar.gz /home/anis
```
- এখানে tar কমান্ড কম প্রায়োরিটিতে চলবে

### ডিফল্ট nice value (0):
```bash
nice command
```

### কম প্রায়োরিটি (বেশি value):
- ব্যাকগ্রাউন্ড/লং রানিং টাস্কে ব্যবহার করুন

### বেশি প্রায়োরিটি (কম value):
- -20 থেকে 0 দিতে root পারমিশন লাগে
```bash
sudo nice -n -5 make
```

---

## ৩. চলমান প্রসেসের প্রায়োরিটি চেঞ্জ: renice

### বেসিক সিনট্যাক্স:
```bash
renice -n value -p PID
```

### উদাহরণ:
```bash
renice -n 5 -p 1234
```
- 1234 PID-এর প্রসেসের nice value 5 হবে

### একাধিক PID:
```bash
renice -n 10 -p 1234 5678
```

### ইউজার/গ্রুপ অনুযায়ী:
```bash
sudo renice -n 0 -u anis
```

---

## ৪. কোন প্রসেসের nice value কত?

```bash
ps -o pid,ni,comm -p 1234
```
- `ni` কলামে nice value

---

## ৫. বেস্ট প্র্যাকটিস ও টিপস

- লোডেড সিস্টেমে লং রানিং টাস্কে বেশি nice value দিন
- root ছাড়া -ve nice value দেয়া যায় না
- ভুল হলে renice দিয়ে ঠিক করুন
- man page পড়ুন: `man nice`, `man renice`

---

## ৬. চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. নতুন প্রসেস কম প্রায়োরিটিতে চালান
২. চলমান প্রসেসের প্রায়োরিটি চেঞ্জ করুন
৩. ps/top/htop দিয়ে nice value দেখুন

---

## উপসংহার

nice ও renice—এই দুইটি টুল দিয়ে আপনি লিনাক্সে প্রসেসের প্রায়োরিটি কাস্টমাইজ করতে পারবেন। স্মার্টলি ব্যবহার করুন, সিস্টেম পারফরম্যান্স অপ্টিমাইজ করুন।

**প্র্যাকটিস করুন, প্রসেস কন্ট্রোলের আর্ট শিখুন!**
