# su ও sudo ব্যবহার: সুপারপাওয়ার কমান্ড

লিনাক্সে অনেক কাজ আছে যেগুলো সাধারণ ইউজার হিসেবে করা যায় না—শুধুমাত্র অ্যাডমিন (root) পারমিশন লাগে। এই কাজগুলো করার জন্য `su` ও `sudo` কমান্ড ব্যবহার করা হয়। এখানে আমরা এই দুই কমান্ডের পার্থক্য, ব্যবহার, কনফিগারেশন ও নিরাপত্তা নিয়ে বিস্তারিত জানব।

---

## ১. su (Substitute User/Switch User)

`su` কমান্ড দিয়ে আপনি অন্য ইউজার (সাধারণত root) হিসেবে টার্মিনালে লগইন করতে পারেন।

### বেসিক ব্যবহার:

```bash
su
```

- শুধু `su` দিলে root ইউজারে স্যুইচ করবে (root পাসওয়ার্ড চাইবে)

```bash
su anis
```

- `anis` ইউজারে স্যুইচ করবে

### exit দিয়ে আগের ইউজারে ফিরে আসুন:

```bash
exit
```

### su - (লগইন শেল সহ)

```bash
su -
```

- সম্পূর্ণ নতুন শেল (root-এর environment সহ) পেতে

---

## ২. sudo (Superuser Do)

`sudo` কমান্ড দিয়ে আপনি সাধারণ ইউজার হিসেবেই থাকবেন, কিন্তু নির্দিষ্ট কমান্ড অ্যাডমিন পারমিশন নিয়ে চালাতে পারবেন।

### বেসিক ব্যবহার:

```bash
sudo apt update
```

- `apt update` কমান্ডটি root পারমিশনে চলবে

### sudo দিয়ে যেকোনো কমান্ড:

```bash
sudo command arguments
```

### sudo -i (root shell)

```bash
sudo -i
```

- root shell খুলবে (root environment সহ)

---

## ৩. sudoers ফাইল ও পারমিশন কনফিগার

`/etc/sudoers` ফাইল দিয়ে নির্ধারণ করা হয় কে কে sudo ব্যবহার করতে পারবে।

### visudo দিয়ে এডিট করুন:

```bash
sudo visudo
```

- এখানে ইউজার/গ্রুপ নির্দিষ্ট করে sudo পারমিশন দিন

```
# anis ইউজারকে সবকিছু sudo করার অনুমতি
anis ALL=(ALL:ALL) ALL
```

---

## ৪. sudo গ্রুপে ইউজার যুক্ত করা

উবুন্টুতে `sudo` গ্রুপে ইউজার থাকলেই সে sudo চালাতে পারবে:

```bash
sudo usermod -aG sudo anis
```

---

## ৫. নিরাপত্তা ও বেস্ট প্র্যাকটিস

- সবসময় sudo ব্যবহার করুন, su দিয়ে root shell ওপেন না করাই ভালো
- sudo কমান্ডের লগ রাখা হয় `/var/log/auth.log`-এ
- sudo টাইমআউট: একবার পাসওয়ার্ড দিলে কিছুক্ষণ আর চাইবে না
- sudoers ফাইল ভুল এডিট করলে সিস্টেমে প্রবেশে সমস্যা হতে পারে, visudo ব্যবহার করুন

---

## ৬. sudo ও su-এর পার্থক্য

| বৈশিষ্ট্য | su | sudo |
|-----------|----|------|
| ইউজার স্যুইচ | হ্যাঁ | না (কমান্ড-ভিত্তিক) |
| পাসওয়ার্ড | root-এর | নিজের |
| লগিং | না | হ্যাঁ |
| নিরাপত্তা | কম | বেশি |

---

## ৭. চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. su দিয়ে root ইউজারে যান ও ফাইল তৈরি করুন
২. sudo দিয়ে প্যাকেজ ইন্সটল করুন
৩. sudoers ফাইল এডিট করুন
৪. sudo গ্রুপে নতুন ইউজার যুক্ত করুন

---

## উপসংহার

`su` ও `sudo`—দুইটি লিনাক্স অ্যাডমিনিস্ট্রেশনের জন্য অপরিহার্য টুল। নিরাপদ ও স্মার্টভাবে ব্যবহার করুন, সিস্টেম সিকিউর রাখুন!
