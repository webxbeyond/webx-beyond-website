ডাটাবেজ ব্যাকআপ মানে—ডাটার কপি সংরক্ষণ করা, যাতে হারিয়ে গেলে বা সমস্যা হলে আগের অবস্থায় ফিরিয়ে আনা যায়। রিস্টোর মানে—সংরক্ষিত ব্যাকআপ থেকে ডাটাবেজ পুনরুদ্ধার করা।

## কেন ব্যাকআপ দরকার?
- হার্ডওয়্যার/সফটওয়্যার সমস্যা
- ভুল ডিলিট/আপডেট
- হ্যাকিং/র‍্যানসমওয়্যার
- ডাটা মাইগ্রেশন

## ব্যাকআপের ধরন
- **Full Backup:** পুরো ডাটাবেজের কপি
- **Incremental Backup:** শেষ ব্যাকআপের পর পরিবর্তিত ডাটা
- **Differential Backup:** শেষ Full Backup থেকে পরিবর্তিত ডাটা

## ব্যাকআপ পদ্ধতি

### MySQL
```bash
mysqldump -u username -p database_name > backup.sql
```

### PostgreSQL
```bash
pg_dump -U username dbname > backup.sql
```

### MongoDB
```bash
mongodump --db dbname --out /backup/folder
```

### Cloud Database (AWS RDS)
- Automated snapshot, manual backup

## রিস্টোর পদ্ধতি

### MySQL
```bash
mysql -u username -p database_name < backup.sql
```

### PostgreSQL
```bash
psql -U username dbname < backup.sql
```

### MongoDB
```bash
mongorestore /backup/folder
```

## বাস্তব উদাহরণ

> "আমরা প্রোডাকশন ডাটাবেজে প্রতিদিন automated backup চালাই। একবার ডাটা ভুলে ডিলিট হলে, ব্যাকআপ থেকে restore করে আগের অবস্থায় ফিরিয়ে এনেছি।"

## টিপস
- Automated backup schedule করুন (daily/weekly)
- ব্যাকআপ ফাইল নিরাপদ স্থানে রাখুন (cloud, external drive)
- রিস্টোর প্রক্রিয়া টেস্ট করুন
- ব্যাকআপ retention policy ঠিক করুন

## সংক্ষেপে
- ব্যাকআপ ছাড়া ডাটাবেজ নিরাপদ নয়
- Automated, secure, tested backup policy রাখুন
- রিস্টোর প্রক্রিয়া জানা ও টেস্ট করা জরুরি
