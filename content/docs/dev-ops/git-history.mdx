গিট ইতিহাস মানে হলো—প্রজেক্টে কে, কখন, কী পরিবর্তন করেছে, তার সম্পূর্ণ রেকর্ড। ইতিহাস দেখে আপনি আগের কমিট, পরিবর্তন, ও প্রয়োজন হলে আগের অবস্থায় ফিরে যেতে পারেন।

## গিট ইতিহাস দেখার কমান্ড

### কমিট ইতিহাস দেখুন
```bash
git log
```

- এতে সব কমিট, তার SHA, লেখক, তারিখ, ও ম্যাসেজ দেখা যাবে।

### সংক্ষেপে ইতিহাস দেখুন
```bash
git log --oneline
```

### নির্দিষ্ট ফাইলের ইতিহাস
```bash
git log path/to/file
```

### গ্রাফ আকারে ইতিহাস
```bash
git log --graph --oneline --all
```

## আগের অবস্থায় ফিরে যাওয়া

### আগের কমিটে চেকআউট
```bash
git checkout <commit-SHA>
```
- এতে আপনি আগের কমিটের অবস্থায় চলে যাবেন (detached HEAD)।

### আগের কমিটে নতুন ব্রাঞ্চ
```bash
git checkout -b old-state <commit-SHA>
```
- এতে আগের অবস্থায় নতুন ব্রাঞ্চ তৈরি হবে।

### আগের কমিটে রিপোজিটরি ফিরিয়ে আনা (reset)
```bash
git reset --hard <commit-SHA>
```
- সতর্কতা: এতে বর্তমান ব্রাঞ্চের সব পরিবর্তন হারিয়ে যাবে, আগের অবস্থায় ফিরে যাবে।

## বাস্তব উদাহরণ

> "আমরা টিমে কাজ করি, হঠাৎ কোনো বাগ/সমস্যা হলে, git log দেখে আগের কমিটে ফিরে যাই। কখনো আগের ফিচার দরকার হলে, সেই কমিটে নতুন ব্রাঞ্চ তৈরি করি।"

## টিপস
- `git log`-এ SHA কপি করে, আগের অবস্থায় যেতে পারেন।
- ভুল হলে, reset করার আগে সবকিছু ব্যাকআপ রাখুন।
- Visual tool (GitHub, GitKraken, VS Code) দিয়েও ইতিহাস দেখা যায়।

## সংক্ষেপে
- গিট ইতিহাসে সব পরিবর্তন, কমিট, ও আগের অবস্থায় ফিরে যাওয়ার উপায় থাকে।
- টিমওয়ার্ক, বাগ ফিক্স, ও ফিচার রিকভার—সবকিছুতে ইতিহাস জানা জরুরি।
- সতর্কভাবে reset/checkout ব্যবহার করুন।
