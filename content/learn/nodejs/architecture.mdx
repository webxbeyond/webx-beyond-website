Node.js হচ্ছে একটি ওপেন-সোর্স, ক্রস-প্ল্যাটফর্ম রানটাইম এনভায়রনমেন্ট যা JavaScript কে সার্ভার-সাইডে চালাতে সক্ষম করে। এটি V8 JavaScript ইঞ্জিনের উপর ভিত্তি করে তৈরি, যা Google Chrome-এ ব্যবহৃত হয়।

## Node.js এর মূল বৈশিষ্ট্য

### ১. ইভেন্ট-ড্রিভেন ও নন-ব্লকিং I/O
Node.js এর সবচেয়ে বড় বৈশিষ্ট্য হচ্ছে এর ইভেন্ট-ড্রিভেন ও নন-ব্লকিং I/O মডেল। এর ফলে একাধিক ক্লায়েন্টের অনুরোধ একসাথে দ্রুত প্রসেস করা যায়, সার্ভার "ব্লক" হয় না।

**রিয়েল লাইফ উদাহরণ:**
ধরা যাক, আপনি একটি চা দোকান চালাচ্ছেন। একজন কাস্টমার চা অর্ডার দিল, আপনি চা বানাতে শুরু করলেন। এসময় আরেকজন কাস্টমার এলেন, আপনি তার অর্ডারও নিলেন এবং চা বানাতে শুরু করলেন। Node.js ঠিক এমনভাবেই কাজ করে—একজনের কাজ শেষ না হলেও অন্যজনের কাজ শুরু হয়, কেউ "ব্লক" হয় না।

### ২. সিঙ্গেল থ্রেডেড কিন্তু Highly Scalable
Node.js একক থ্রেডে চলে, কিন্তু ইভেন্ট লুপ ও অ্যাসিনক্রোনাস অপারেশন ব্যবহার করে হাজার হাজার অনুরোধ একসাথে হ্যান্ডল করতে পারে।

### ৩. ক্রস-প্ল্যাটফর্ম
Windows, Linux, Mac—সব প্ল্যাটফর্মেই Node.js চলে।

### ৪. npm (Node Package Manager)
বিশ্বের সবচেয়ে বড় ওপেন-সোর্স লাইব্রেরি ইকোসিস্টেম।

### ৫. JSON ও API ফ্রেন্ডলি
Node.js JSON-ভিত্তিক ডেটা প্রসেসিং ও API ডেভেলপমেন্টের জন্য আদর্শ।

## Node.js এর আর্কিটেকচার

Node.js এর আর্কিটেকচার মূলত চারটি অংশে বিভক্ত:

1. **V8 JavaScript Engine**
2. **libuv**
3. **Event Loop**
4. **C++ Bindings & APIs**

### ১. V8 JavaScript Engine
Google Chrome-এর V8 ইঞ্জিন Node.js-এ ব্যবহৃত হয়। এটি JavaScript কে দ্রুত মেশিন কোডে রূপান্তর করে।

### ২. libuv
libuv হচ্ছে C-তে লেখা একটি লাইব্রেরি, যা ইভেন্ট-ড্রিভেন, নন-ব্লকিং I/O, ফাইল সিস্টেম, TCP/UDP, DNS ইত্যাদি হ্যান্ডল করে।

### ৩. Event Loop
Event Loop হচ্ছে Node.js-এর "হৃদপিণ্ড"। এটি অ্যাসিনক্রোনাস টাস্ক, কলব্যাক, টাইমার, I/O ইভেন্ট ইত্যাদি প্রসেস করে।

**রিয়েল লাইফ উদাহরণ:**
একটি রেস্টুরেন্টে একজন ওয়েটার অর্ডার নিচ্ছেন, রান্না হচ্ছে, খাবার পরিবেশন হচ্ছে—সবকিছুই "ইভেন্ট"। ওয়েটার (Event Loop) সবকিছু সমন্বয় করেন।

### ৪. C++ Bindings & APIs
Node.js-এর অনেক "কোর" ফিচার (যেমন ফাইল সিস্টেম, নেটওয়ার্কিং) C++ দিয়ে লেখা এবং JavaScript API-এর মাধ্যমে এক্সেস করা যায়।

## Node.js আর্কিটেকচারের ফ্লো

1. ইউজার অনুরোধ পাঠায় (HTTP, TCP, ইত্যাদি)
2. Event Loop অনুরোধ গ্রহণ করে
3. যদি I/O অপারেশন হয়, libuv সেটি অ্যাসিনক্রোনাসভাবে প্রসেস করে
4. কাজ শেষ হলে Callback/Event Trigger হয়
5. রেসপন্স ইউজারকে পাঠানো হয়

## কেন Node.js?

- **রিয়েল-টাইম অ্যাপ:** চ্যাট, গেম, লাইভ আপডেট
- **API সার্ভার:** REST, GraphQL
- **স্ট্রিমিং:** ভিডিও, ফাইল
- **IoT:** সেন্সর ডেটা প্রসেসিং

## ছোট্ট প্রজেক্ট উদাহরণ: HTTP সার্ভার

```js
// server.js
const http = require('http');
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello, Node.js World!');
});
server.listen(3000, () => {
  console.log('Server running at http://localhost:3000/');
});
```

এখানে, একাধিক ক্লায়েন্ট একসাথে সার্ভারে রিকোয়েস্ট পাঠাতে পারে, Node.js ইভেন্ট-লুপের মাধ্যমে সবকিছু হ্যান্ডল করে।

## উপসংহার

Node.js-এর ইভেন্ট-ড্রিভেন, নন-ব্লকিং আর্কিটেকচার এবং V8 ইঞ্জিনের কারণে এটি দ্রুত, স্কেলেবল ও API-ফ্রেন্ডলি। পরবর্তী পাঠে V8 ইঞ্জিন ও অপ্টিমাইজেশন নিয়ে বিস্তারিত আলোচনা হবে।
