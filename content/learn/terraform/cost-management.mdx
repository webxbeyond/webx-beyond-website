Terraform দিয়ে ইনফ্রা তৈরি করার সময় কস্ট (ব্যয়) কন্ট্রোল ও অপ্টিমাইজ করা খুব গুরুত্বপূর্ণ। এখানে `terraform plan` দিয়ে কস্ট এস্টিমেট, অপ্টিমাইজেশন টিপস—সবকিছু হাতে-কলমে উদাহরণসহ ব্যাখ্যা করা হয়েছে।

---

## কস্ট এস্টিমেট কী?

- Terraform plan/apply করার আগে জানতে চান—কত খরচ হবে
- কিছু provider (AWS, Azure, GCP) কস্ট এস্টিমেট API দেয়
- কস্ট প্লাগইন/টুল দিয়ে আরও ডিটেইল এস্টিমেট করা যায়

---

## terraform plan দিয়ে কস্ট এস্টিমেট

- `terraform plan` কমান্ডে সরাসরি কস্ট দেখায় না, কিন্তু কী রিসোর্স তৈরি হবে—তা দেখে কস্ট অনুমান করা যায়
- কস্ট এস্টিমেট প্লাগইন/টুল ব্যবহার করুন

**Infracost:**
- Infracost হচ্ছে ওপেন সোর্স টুল, Terraform plan থেকে কস্ট এস্টিমেট দেখায়

**ইনস্টল ও ব্যবহার:**
```sh
npm install -g infracost
infracost breakdown --path .
```

- Infracost API key লাগবে (ফ্রি)
- রিপোর্টে দেখাবে—নতুন/চেঞ্জড রিসোর্সে কত খরচ হবে

---

## অপ্টিমাইজেশন টিপস

- অপ্রয়োজনীয় রিসোর্স তৈরি করবেন না
- instance size, storage, network—সবকিছু মিনিমাইজ করুন
- autoscaling, spot instance, reserved instance ব্যবহার করুন
- কস্ট tag/label দিন—billing/report সহজ হবে
- নিয়মিত কস্ট রিপোর্ট/monitor করুন

---

## হাতে-কলমে: কস্ট ট্যাগিং

```hcl
resource "aws_instance" "web" {
  ami           = "ami-123456"
  instance_type = "t3.micro"
  tags = {
    Name = "web"
    CostCenter = "dev"
  }
}
```

## সমস্যা ও সমাধান

**সমস্যা:** কস্ট বেশি আসছে
- **সমাধান:** instance/resource size, টাইপ, region চেক করুন

**সমস্যা:** কস্ট ট্র্যাক করা যাচ্ছে না
- **সমাধান:** tag/label ব্যবহার করুন, কস্ট টুল ইন্টিগ্রেট করুন
