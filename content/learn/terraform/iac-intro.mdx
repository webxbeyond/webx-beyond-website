Infrastructure as Code (IaC) মানে—ইনফ্রাস্ট্রাকচার (যেমন সার্ভার, নেটওয়ার্ক, ডাটাবেস, ক্লাউড রিসোর্স) কোড দিয়ে ডিফাইন ও ম্যানেজ করা। আগে যেখানে ম্যানুয়ালি কনফিগারেশন করতে হত, এখন কোড লিখে সবকিছু অটোমেট করা যায়।

## কেন IaC দরকার?

- **ম্যানুয়াল কাজ কমানো:** একবার কোড লিখে বারবার একই ইনফ্রা তৈরি করা যায়।
- **রিপ্রোডিউসিবল:** একই কোড দিয়ে একাধিক এনভায়রনমেন্ট (dev, staging, prod) তৈরি করা যায়।
- **ভার্সন কন্ট্রোল:** Git-এ কোড রাখলে কে কখন কী চেঞ্জ করেছে ট্র্যাক করা যায়।
- **অটোমেশন:** CI/CD pipeline-এ অটোমেটেড ডিপ্লয়মেন্ট সম্ভব।
- **ডকুমেন্টেশন:** কোডই ডকুমেন্টেশন—কী আছে, কীভাবে তৈরি হয়, সব স্পষ্ট।

## IaC কিভাবে কাজ করে?

IaC টুল দিয়ে আপনি YAML, JSON, HCL বা অন্য কোনো ডিক্লারেটিভ ল্যাঙ্গুয়েজে ইনফ্রার কাঠামো লিখবেন। টুলটি সেই কোড পড়ে ক্লাউড/অন-প্রেম/হাইব্রিড ইনফ্রা তৈরি করবে।

### উদাহরণ (মানুয়াল বনাম IaC)

**ম্যানুয়াল:**
- AWS কনসোলে গিয়ে EC2 instance launch
- Security group, disk, network ম্যানুয়ালি সেটআপ
- ভুল হলে আবার শুরু

**IaC:**
- কোডে লিখুন—"একটা EC2 instance, এই কনফিগে"
- এক কমান্ডে তৈরি, পরিবর্তন, ডিলিট—সব সম্ভব

```hcl
resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

## IaC টুলের ধরন

- **Terraform:** ওপেন সোর্স, ক্লাউড-অ্যাগনস্টিক, ডিক্লারেটিভ
- **AWS CloudFormation:** শুধু AWS, YAML/JSON
- **Ansible:** কনফিগারেশন + ইনফ্রা, YAML, Agentless
- **Pulumi:** কোডে (TypeScript, Python, Go) ইনফ্রা
- **Chef/Puppet:** কনফিগারেশন ম্যানেজমেন্ট

## কেন টেরাফর্ম?

- **একই কোডে মাল্টি-ক্লাউড:** AWS, Azure, GCP, DigitalOcean, আরও অনেক কিছু
- **ডিক্লারেটিভ:** কী চাই, শুধু সেটাই লিখুন—কীভাবে হবে, সেটা টেরাফর্ম ঠিক করবে
- **স্টেট ম্যানেজমেন্ট:** কী আছে, কী চেঞ্জ হবে—সব ট্র্যাক
- **মডিউলার:** কোড রিইউজ, কমিউনিটি মডিউল
- **ওপেন সোর্স ও শক্তিশালী কমিউনিটি**
- **সহজ সিনট্যাক্স (HCL):** নতুনদের জন্যও সহজ

### টেরাফর্মের সুবিধা vs অন্যান্য IaC টুল

| স্টেট ম্যানেজমেন্ট     | ✅         | ✅             | ❌       | ✅      |

## সহজ উদাহরণ: টেরাফর্ম দিয়ে AWS EC2

```hcl
provider "aws" {
  region = "ap-south-1"
}

resource "aws_instance" "my_server" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

- প্রথমে provider ব্লকে AWS region দিন
- resource ব্লকে EC2 instance কনফিগার করুন
- `terraform init` → `terraform plan` → `terraform apply` কমান্ডে ইনফ্রা তৈরি

## সংক্ষেপে

- IaC মানে—ইনফ্রা কোডে, ম্যানুয়াল নয়
- টেরাফর্ম—সবচেয়ে জনপ্রিয়, মাল্টি-ক্লাউড, সহজ
- একবার শিখলে—ক্লাউড, অন-প্রেম, হাইব্রিড—সব জায়গায় কাজে লাগবে
