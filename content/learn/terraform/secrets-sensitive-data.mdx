Terraform-এ সিক্রেট (secret) ও সেনসিটিভ ডাটা (যেমন: পাসওয়ার্ড, API key) নিরাপদে ম্যানেজ করা খুব গুরুত্বপূর্ণ। এখানে বেস্ট প্র্যাকটিস, env var, Vault—সবকিছু হাতে-কলমে উদাহরণসহ ব্যাখ্যা করা হয়েছে।

---

## কেন সেনসিটিভ ডাটা আলাদা গুরুত্ব পায়?

- পাসওয়ার্ড, টোকেন, API key—এসব লিক হলে নিরাপত্তা ঝুঁকি
- state ফাইলে সেনসিটিভ ডাটা থাকতে পারে
- Terraform apply করলে আউটপুটে সেনসিটিভ ডাটা দেখা যেতে পারে

---

## বেস্ট প্র্যাকটিস

- সেনসিটিভ ডাটা কখনোই কোডে হার্ডকোড করবেন না
- `.tfvars` বা environment variable ব্যবহার করুন
- আউটপুটে sensitive=true দিন
- state ফাইল version control-এ রাখবেন না

---

## Environment Variable দিয়ে সেনসিটিভ ডাটা

Terraform-এ ইনপুট ভ্যারিয়েবল environment variable দিয়ে সেট করা যায়:

**variable.tf:**
```hcl
variable "db_password" {
  description = "Database password"
  type        = string
  sensitive   = true
}
```

**CLI-তে:**
```sh
export TF_VAR_db_password="mysecretpassword"
terraform apply
```

---

## আউটপুটে সেনসিটিভ ডাটা লুকানো

```hcl
output "db_password" {
  value     = var.db_password
  sensitive = true
}
```
- sensitive=true দিলে apply করার পর আউটপুটে ডাটা দেখা যাবে না

---

## HashiCorp Vault দিয়ে সিক্রেট ম্যানেজ

Vault হচ্ছে সিক্রেট ম্যানেজমেন্ট টুল।

**Vault provider দিয়ে ডাটা ফেচ:**
```hcl
provider "vault" {
  address = "https://vault.example.com"
}

data "vault_generic_secret" "db" {
  path = "secret/data/db"
}

variable "db_password" {
  default = data.vault_generic_secret.db.data["password"]
  sensitive = true
}
```

---

## state ফাইলে সেনসিটিভ ডাটা

- state ফাইলে sensitive=true দিলেও ডাটা encrypt হয় না
- রিমোট ব্যাকএন্ডে encryption ও access control ব্যবহার করুন

---

## টিপস ও বেস্ট প্র্যাকটিস

- সেনসিটিভ ডাটা কখনোই কোডে/গিটে রাখবেন না
- Vault বা ক্লাউড সিক্রেট ম্যানেজার ব্যবহার করুন
- আউটপুটে sensitive=true দিন
- state ফাইল encrypt ও access control দিন

---

## সমস্যা ও সমাধান

**সমস্যা:** সেনসিটিভ ডাটা apply আউটপুটে দেখা যাচ্ছে
- **সমাধান:** output-এ sensitive=true দিন

**সমস্যা:** state ফাইলে ডাটা encrypt হয়নি
- **সমাধান:** রিমোট ব্যাকএন্ডে encryption এনাবল করুন
