নেমিং কনভেনশন হলো কোডে ভ্যারিয়েবল, ফাংশন, ক্লাস, কনস্ট্যান্ট ইত্যাদির নামকরণের নির্দিষ্ট নিয়ম। ভালো নেমিং কনভেনশন কোডকে আরও রিডেবল, মেইনটেইনেবল ও টিমওয়ার্ক-ফ্রেন্ডলি করে তোলে। এই অধ্যায়ে আমরা নেমিং কনভেনশনের গুরুত্ব, বিভিন্ন স্টাইল, ভাষাভিত্তিক উদাহরণ, ভালো অভ্যাস ও অনুশীলন জানব।

---

## ১. নেমিং কনভেনশন কী?

- কোডে নামকরণের নির্দিষ্ট নিয়ম
- টিম ও প্রজেক্টভিত্তিক স্টাইলগাইড

---

## ২. নেমিং কনভেনশনের গুরুত্ব

- কোড সহজে পড়া ও বোঝা যায়
- বাগ কমে
- টিমওয়ার্ক ও স্কেলিং সহজ

---

## ৩. জনপ্রিয় নেমিং স্টাইল

1. **snake_case:** ছোট হাতের অক্ষর, শব্দের মাঝে আন্ডারস্কোর (Python)
2. **camelCase:** প্রথম শব্দ ছোট, পরের শব্দ বড় হাতের অক্ষর (JavaScript, Java)
3. **PascalCase:** প্রতিটি শব্দ বড় হাতের অক্ষর (Class, Constructor)
4. **UPPER_CASE:** কনস্ট্যান্ট বা ম্যাক্রো

---

## ৪. Python-এ নেমিং কনভেনশন

- ভ্যারিয়েবল/ফাংশন: snake_case
- ক্লাস: PascalCase
- কনস্ট্যান্ট: UPPER_CASE

### উদাহরণ:
```python
user_name = "Rahim"
def calculate_area():
    pass
class StudentProfile:
    pass
MAX_SIZE = 100
```

---

## ৫. JavaScript-এ নেমিং কনভেনশন

- ভ্যারিয়েবল/ফাংশন: camelCase
- ক্লাস/কনস্ট্রাক্টর: PascalCase
- কনস্ট্যান্ট: UPPER_CASE

### উদাহরণ:
```js
let userName = "Rahim";
function calculateArea() {}
class StudentProfile {}
const MAX_SIZE = 100;
```

---

## ৬. ভালো নেমিংয়ের কৌশল

- অর্থবহ ও স্পষ্ট নাম দিন
- সংক্ষিপ্ত কিন্তু অর্থপূর্ণ রাখুন
- একধরনের স্টাইল পুরো কোডবেসে বজায় রাখুন
- এক অক্ষরের নাম এড়িয়ে চলুন (i, j ছাড়া)
- বুলিয়ান ভ্যারিয়েবলে is/has/should prefix ব্যবহার করুন
