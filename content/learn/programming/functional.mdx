ফাংশনাল প্রোগ্রামিং (Functional Programming) হলো প্রোগ্রামিংয়ের একটি প্যারাডাইম, যেখানে গণিতের ফাংশনের মতো নির্দিষ্ট ইনপুট থেকে নির্দিষ্ট আউটপুট পাওয়া যায় এবং স্টেট পরিবর্তন বা সাইড-ইফেক্ট এড়িয়ে কোড লেখা হয়। এই অধ্যায়ে আমরা ফাংশনাল প্রোগ্রামিংয়ের ধারণা, বৈশিষ্ট্য, মূল কনসেপ্ট (pure function, immutability, first-class function, higher-order function, recursion), উদাহরণ, সুবিধা-অসুবিধা, তুলনা ও ভালো অভ্যাস জানব।

---

## ১. ফাংশনাল প্রোগ্রামিং কী?

- গণিতের ফাংশনের মতো নির্ভরযোগ্য
- স্টেট পরিবর্তন ও সাইড-ইফেক্ট এড়িয়ে চলা
- ফাংশনকে ফার্স্ট-ক্লাস সিটিজেন হিসেবে দেখা

---

## ২. মূল বৈশিষ্ট্য

1. **Pure Function:** একই ইনপুটে সবসময় একই আউটপুট, কোনো সাইড-ইফেক্ট নেই
2. **Immutability:** ডাটা পরিবর্তন না করে নতুন ডাটা তৈরি
3. **First-Class Function:** ফাংশনকে ভ্যারিয়েবল, আর্গুমেন্ট, রিটার্ন ভ্যালু হিসেবে ব্যবহার
4. **Higher-Order Function:** ফাংশনকে আর্গুমেন্ট হিসেবে নেওয়া বা ফাংশন রিটার্ন করা
5. **Recursion:** লুপের বদলে ফাংশনের নিজেকে কল করা

---

## ৩. উদাহরণ

### Pure Function (Python):
```python
def add(a, b):
    return a + b
```

### Immutability (JavaScript):
```js
const arr = [1,2,3];
const newArr = [...arr, 4]; // arr অপরিবর্তিত
```

### First-Class & Higher-Order Function (JavaScript):
```js
function greet(name) {
  return "Hello, " + name;
}
function processUser(name, fn) {
  return fn(name);
}
console.log(processUser("Rahim", greet));
```

### Recursion (Python):
```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n-1)
```

## ৪. ফাংশনাল প্রোগ্রামিং ভাষার উদাহরণ

- Haskell, Lisp, Erlang, Clojure, F#, Scala

## ৫. সুবিধা

- কোড সহজে টেস্ট ও রিইউজ করা যায়
- কনকারেন্সি ও প্যারালাল প্রোগ্রামিং সহজ
- বাগ কম হয়, কারণ স্টেট পরিবর্তন কম
- ছোট ছোট ফাংশনে বড় সমস্যা ভাগ করা যায়

---

## ৬. অসুবিধা

- নতুনদের জন্য ধারণা কঠিন
- পারফরম্যান্স কিছু ক্ষেত্রে কম (অনেক রিকারশন)
- বড় প্রজেক্টে ডিবাগিং কঠিন হতে পারে
