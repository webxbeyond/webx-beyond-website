প্রোগ্রামিংয়ে ভেরিয়েবল, ফাংশন বা অবজেক্ট কোথায় এবং কতক্ষণ বিদ্যমান থাকবে—এটা নির্ধারণ করে স্কোপ (Scope) ও লাইফটাইম (Lifetime)। এই অধ্যায়ে আমরা স্কোপ ও লাইফটাইমের ধারণা, ধরন, উদাহরণ, বিভিন্ন ভাষায় ব্যবহার, সমস্যা ও ভালো অভ্যাস জানব।

---

## ১. স্কোপ (Scope) কী?

- স্কোপ মানে কোনো ভেরিয়েবল/ফাংশন কোথা থেকে অ্যাক্সেস করা যাবে
- স্কোপ নির্ধারণ করে, কোন অংশে কোন নাম বৈধ

### স্কোপের ধরন:
- **লোকাল স্কোপ (Local):** ফাংশন/ব্লকের ভিতরে
- **গ্লোবাল স্কোপ (Global):** পুরো প্রোগ্রামে
- **নন-লোকাল/এনক্লোজিং স্কোপ:** নেস্টেড ফাংশনে (Python: LEGB রুল)
- **ব্লক স্কোপ:** {} ব্লকের ভিতরে (C, Java, JS-এ let/const)

---

## ২. লাইফটাইম (Lifetime) কী?

- লাইফটাইম মানে কোনো ভেরিয়েবল/অবজেক্ট কতক্ষণ মেমোরিতে থাকবে
- স্কোপ শেষ হলে সাধারণত লাইফটাইমও শেষ হয়

---

## ৩. উদাহরণ: স্কোপ

### Python:
```python
global_var = 10

def func():
    local_var = 5
    print(global_var)  # গ্লোবাল অ্যাক্সেস
    print(local_var)   # লোকাল অ্যাক্সেস

func()
# print(local_var)  # এরর, কারণ লোকাল স্কোপের বাইরে
```

### C:
```c
int global_var = 10;

void func() {
    int local_var = 5;
    printf("%d\n", global_var);
    printf("%d\n", local_var);
}
```

---

## ৪. ব্লক স্কোপ

### C/Java:
```c
if (1) {
    int x = 5;
    printf("%d\n", x);
}
// printf("%d\n", x);  // এরর, x ব্লকের বাইরে নেই
```

### JavaScript (let/const):
```js
if (true) {
  let x = 10;
  console.log(x);
}
// console.log(x); // ReferenceError
```

## ৫. নেস্টেড স্কোপ ও LEGB রুল (Python)

- **Local → Enclosing → Global → Built-in**

### উদাহরণ:
```python
def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)
    inner()
    print(x)
outer()
```

---

## ৬. লাইফটাইম উদাহরণ

### Python:
```python
def foo():
    x = 5  # x-এর লাইফটাইম foo() চলাকালীন
    print(x)
foo()
# x আর নেই
```

### C:
```c
void foo() {
    int x = 5; // x-এর লাইফটাইম foo() চলাকালীন
}
```

---

## ৭. গ্লোবাল ও লোকাল কিওয়ার্ড (Python)

- **global:** ফাংশনের ভিতর থেকে গ্লোবাল ভেরিয়েবল পরিবর্তন করতে
- **nonlocal:** নেস্টেড ফাংশনে enclosing স্কোপের ভেরিয়েবল পরিবর্তন করতে

### উদাহরণ:
```python
global_var = 0

def func():
    global global_var
    global_var = 5
```

## ৮. স্কোপ ও লাইফটাইম সমস্যা

- একই নামের ভেরিয়েবল লোকাল ও গ্লোবাল স্কোপে থাকলে কনফিউশন
- মেমোরি লিক (C/C++-এ ম্যানুয়াল মেমোরি ম্যানেজমেন্ট)
- ব্লক স্কোপ না থাকলে (var, Python) ভুল হতে পারে
