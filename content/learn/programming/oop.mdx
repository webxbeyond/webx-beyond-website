অবজেক্ট-ওরিয়েন্টেড প্রোগ্রামিং (OOP) হলো প্রোগ্রামিংয়ের একটি শক্তিশালী প্যারাডাইম, যেখানে ডাটা ও ফাংশন একত্রে অবজেক্ট আকারে ব্যবহৃত হয়। এই অধ্যায়ে আমরা OOP-এর ধারণা, বৈশিষ্ট্য, মূল ধারণা (ক্লাস, অবজেক্ট, ইনহেরিটেন্স, পলিমরফিজম, ইনক্যাপসুলেশন, অ্যাবস্ট্রাকশন), উদাহরণ, সুবিধা-অসুবিধা, তুলনা ও ভালো অভ্যাস জানব।

---

## ১. OOP কী?

- ডাটা ও ফাংশন একত্রে অবজেক্ট আকারে
- ক্লাস দিয়ে অবজেক্ট তৈরি
- বাস্তব জগতের ধারণা (real-world modeling)

---

## ২. OOP-এর মূল বৈশিষ্ট্য

1. **ক্লাস (Class):** অবজেক্ট তৈরির ব্লুপ্রিন্ট
2. **অবজেক্ট (Object):** ক্লাসের ইনস্ট্যান্স
3. **ইনক্যাপসুলেশন (Encapsulation):** ডাটা ও মেথড একত্রে
4. **ইনহেরিটেন্স (Inheritance):** একটি ক্লাস থেকে আরেকটি ক্লাসের বৈশিষ্ট্য পাওয়া
5. **পলিমরফিজম (Polymorphism):** একই ইন্টারফেসে ভিন্ন আচরণ
6. **অ্যাবস্ট্রাকশন (Abstraction):** জটিলতা লুকিয়ে সহজ ইন্টারফেস

## ৩. উদাহরণ

### Python:
```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def greet(self):
        print(f"Hello, I am {self.name}")

p = Person("Rahim", 25)
p.greet()
```

### Java:
```java
class Person {
    String name;
    int age;
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    void greet() {
        System.out.println("Hello, I am " + name);
    }
}
Person p = new Person("Rahim", 25);
p.greet();
```

## ৪. ইনহেরিটেন্স উদাহরণ

### Python:
```python
class Animal:
    def speak(self):
        print("Animal sound")

class Dog(Animal):
    def speak(self):
        print("Woof!")

d = Dog()
d.speak()  # Woof!
```

## ৫. পলিমরফিজম উদাহরণ

### Python:
```python
class Shape:
    def area(self):
        pass
class Circle(Shape):
    def area(self):
        return 3.14 * 5 * 5
class Square(Shape):
    def area(self):
        return 4 * 4
shapes = [Circle(), Square()]
for s in shapes:
    print(s.area())
```

## ৬. OOP-এর সুবিধা

- কোড রিইউজ, মডুলারিটি
- ডাটা হাইডিং ও সিকিউরিটি
- বড় প্রজেক্টে টিমওয়ার্ক সহজ
- বাস্তব জগতের সমস্যা সমাধানে উপযোগী

---

## ৭. অসুবিধা

- ছোট প্রজেক্টে জটিলতা বাড়ে
- বেশি অ্যাবস্ট্রাকশন ও ইনহেরিটেন্সে বাগের ঝুঁকি
- পারফরম্যান্স কিছুটা কম (কিছু ক্ষেত্রে)

---

## ৮. প্রসিডিউরাল বনাম OOP

| বৈশিষ্ট্য         | প্রসিডিউরাল | OOP           |
|-------------------|-------------|---------------|
| ডাটা ও ফাংশন      | আলাদা       | একত্রে (অবজেক্ট)|
| রিইউজ             | কম          | বেশি          |
| মডেলিং            | গাণিতিক/স্টেপ | বাস্তব জগত   |

## ৯. ভালো অভ্যাস

- ক্লাস ও অবজেক্টের নাম অর্থবহ দিন
- ইনক্যাপসুলেশন বজায় রাখুন (প্রাইভেট/পাবলিক)
- ইনহেরিটেন্স বুঝে ব্যবহার করুন
- কম্পোজিশন (composition) প্রয়োজনে ব্যবহার করুন
