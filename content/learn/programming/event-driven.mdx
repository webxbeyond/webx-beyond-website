ইভেন্ট-ড্রিভেন ও রিঅ্যাক্টিভ প্রোগ্রামিং আধুনিক সফটওয়্যার ডিজাইনের গুরুত্বপূর্ণ প্যারাডাইম। এই অধ্যায়ে আমরা ইভেন্ট-ড্রিভেন ও রিঅ্যাক্টিভ প্রোগ্রামিংয়ের ধারণা, বৈশিষ্ট্য, উদাহরণ, সুবিধা-অসুবিধা, তুলনা ও ভালো অভ্যাস জানব।

---

## ১. ইভেন্ট-ড্রিভেন প্রোগ্রামিং কী?

- ইভেন্ট (ঘটনা) ঘটলে নির্দিষ্ট কোড চলে
- ইউজার ইন্টারঅ্যাকশন, হার্ডওয়্যার সিগন্যাল, টাইমার ইত্যাদি ইভেন্ট হতে পারে
- GUI, গেম, সার্ভার, নেটওয়ার্ক প্রোগ্রামিংয়ে বহুল ব্যবহৃত

---

## ২. ইভেন্ট-ড্রিভেন প্রোগ্রামিংয়ের বৈশিষ্ট্য

- Event loop: ইভেন্টের জন্য অপেক্ষা করে
- Callback/Handler: ইভেন্ট ঘটলে নির্দিষ্ট ফাংশন চলে
- Asynchronous execution: একাধিক কাজ একসাথে

---

## ৩. উদাহরণ (JavaScript)

```js
document.getElementById('btn').addEventListener('click', function() {
  alert('Button clicked!');
});
```

### Python (Tkinter):
```python
import tkinter as tk
def on_click():
    print('Button clicked!')
root = tk.Tk()
btn = tk.Button(root, text='Click', command=on_click)
btn.pack()
root.mainloop()
```

## ৪. রিঅ্যাক্টিভ প্রোগ্রামিং কী?

- ডাটা স্ট্রিম ও চেঞ্জের প্রতি রেসপন্স
- Observable, Observer, Subscription, Operator ইত্যাদি কনসেপ্ট
- UI, ডাটা ফ্লো, লাইভ আপডেট, রিয়েল-টাইম অ্যাপ্লিকেশনে ব্যবহৃত

---

## ৫. রিঅ্যাক্টিভ প্রোগ্রামিংয়ের বৈশিষ্ট্য

- ডাটা স্ট্রিম হিসেবে চিন্তা
- ইভেন্ট/ডাটা চেঞ্জে অটোমেটিক রেসপন্স
- Declarative কোডিং স্টাইল

---

## ৬. উদাহরণ (RxJS - JavaScript)

```js
import { fromEvent } from 'rxjs';
const btn = document.getElementById('btn');
fromEvent(btn, 'click').subscribe(() => {
  alert('Button clicked (Reactive)!');
});
```

### Python (RxPY):
```python
from rx import from_list
from rx.operators import map
source = from_list([1,2,3,4])
source.pipe(
    map(lambda x: x * 2)
).subscribe(print)
```

## ৭. সুবিধা

- ইউজার ইন্টারঅ্যাকশন ও রিয়েল-টাইম ডাটা সহজে হ্যান্ডল
- কোড মডুলার ও স্কেলেবল
- অ্যাসিনক্রোনাস ও কনকারেন্ট টাস্ক সহজ

## ৮. অসুবিধা

- ডিবাগিং ও ট্রেসিং কঠিন
- জটিল ইভেন্ট চেইনে বাগ খুঁজে পাওয়া কঠিন
- নতুনদের জন্য ধারণা কঠিন

## ৯. অন্যান্য প্যারাডাইমের সাথে তুলনা

| বৈশিষ্ট্য         | ইভেন্ট-ড্রিভেন/রিঅ্যাক্টিভ | প্রসিডিউরাল/OOP/ফাংশনাল |
|-------------------|-----------------------------|-------------------------|
| Execution Flow    | ইভেন্ট/ডাটা চেঞ্জে নির্ভর  | নির্দিষ্ট স্টেপ/ফাংশন   |
| Async Handling    | সহজ                        | তুলনামূলক কঠিন         |
| UI/Real-time      | খুব উপযোগী                  | সীমিত                  |

## ১০. ভালো অভ্যাস

## ১১. অনুশীলন

## ১২. উপসংহার

ইভেন্ট-ড্রিভেন ও রিঅ্যাক্টিভ প্রোগ্রামিং আধুনিক সফটওয়্যারে ইউজার ইন্টারঅ্যাকশন, রিয়েল-টাইম ডাটা ও স্কেলেবিলিটি সহজ করে। এই ধারণা ভালোভাবে আয়ত্ত করলে জটিল অ্যাপ্লিকেশনও সহজে ডিজাইন করা যায়।
