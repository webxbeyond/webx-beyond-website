## সরাসরি ও পরোক্ষ

রিকারশন (Recursion) হলো এমন একটি টেকনিক, যেখানে কোনো ফাংশন নিজেকে (বা অন্য কোনো ফাংশনকে) কল করে। এটি জটিল সমস্যাকে ছোট ছোট অংশে ভাগ করে সমাধান করতে সাহায্য করে। এই অধ্যায়ে আমরা রিকারশনের ধারণা, ধরন, গঠন, উদাহরণ, সমস্যা, ভালো অভ্যাস ও অনুশীলন জানব।

---

## ১. রিকারশন কী?

- কোনো ফাংশন নিজেকে (বা অন্য ফাংশনকে) কল করলে তাকে রিকারশন বলে
- প্রতিটি রিকার্সিভ কলের জন্য নতুন ফাংশন ফ্রেম তৈরি হয়
- বেস কেস (base case) না থাকলে ইনফিনিট রিকারশন হয়

---

## ২. রিকারশনের ধরন

### ১. সরাসরি রিকারশন (Direct Recursion)
- ফাংশন নিজেই নিজেকে কল করে

### ২. পরোক্ষ রিকারশন (Indirect Recursion)
- একটি ফাংশন অন্য ফাংশনকে কল করে, এবং সেই ফাংশন আবার প্রথম ফাংশনকে কল করে

---

## ৩. রিকারশনের গঠন

### বেস কেস ও রিকার্সিভ কেস
- **বেস কেস:** কখন রিকারশন থামবে
- **রিকার্সিভ কেস:** ফাংশন নিজেকে কল করবে

### উদাহরণ (Python): সরাসরি রিকারশন
```python
def factorial(n):
    if n == 0:
        return 1  # বেস কেস
    return n * factorial(n - 1)  # রিকার্সিভ কেস

print(factorial(5))  # 120
```

### উদাহরণ (C):
```c
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
```

## ৪. পরোক্ষ রিকারশন উদাহরণ

### Python:
```python
def funcA(n):
    if n <= 0:
        return
    print("A", n)
    funcB(n - 1)

def funcB(n):
    if n <= 0:
        return
    print("B", n)
    funcA(n - 1)

funcA(3)
```

## ৫. রিকারশন বনাম ইটারেশন

- রিকারশন: ফাংশন নিজেকে কল করে
- ইটারেশন: লুপ ব্যবহার করে
- অনেক সমস্যা রিকারশন ও ইটারেশন—দুইভাবেই সমাধান করা যায়

---

## ৬. রিকারশনের সমস্যা ও সতর্কতা

- বেস কেস না থাকলে ইনফিনিট রিকারশন (Stack Overflow)
- বেশি ডিপ রিকারশন হলে পারফরম্যান্স সমস্যা
- প্রতিটি কলের জন্য মেমোরি লাগে

---

## ৭. ভালো অভ্যাস

- বেস কেস স্পষ্ট রাখুন
- রিকারশন প্রয়োজন ছাড়া ব্যবহার করবেন না
- Tail Recursion (সমর্থিত ভাষায়) ব্যবহার করুন
- ডিবাগিং সহজ রাখতে print/log ব্যবহার করুন

২. পরোক্ষ রিকারশনের উদাহরণ দিন
৩. রিকারশন ও ইটারেশনের পার্থক্য লিখুন

## ৯. উপসংহার

রিকারশন জটিল সমস্যা সহজে সমাধান করতে সাহায্য করে। তবে বেস কেস ও পারফরম্যান্সের বিষয়গুলো মাথায় রেখে ব্যবহার করা উচিত।
