## টোকেন, লিটারাল, ডেলিমিটার, কমেন্ট

প্রোগ্রামিং ভাষার মৌলিক গঠন বোঝার জন্য লেক্সিক্যাল উপাদানগুলো জানা অত্যন্ত গুরুত্বপূর্ণ। এগুলো হচ্ছে সেই ক্ষুদ্রতম অংশ, যেগুলো একত্রিত হয়ে একটি প্রোগ্রাম গঠন করে। এই অধ্যায়ে আমরা টোকেন, লিটারাল, ডেলিমিটার এবং কমেন্ট সম্পর্কে বিস্তারিত জানব।

---

## ১. টোকেন (Token)

**টোকেন** হলো প্রোগ্রামিং ভাষার ক্ষুদ্রতম অর্থবহ একক। যখন কোনো সোর্স কোড কম্পাইলার বা ইন্টারপ্রিটার দ্বারা প্রসেস হয়, তখন প্রথমেই কোডটি টোকেনে বিভক্ত হয়।

### টোকেনের ধরনসমূহ:
- **কীওয়ার্ড (Keyword):** নির্দিষ্ট অর্থবহ সংরক্ষিত শব্দ (যেমন: `if`, `else`, `for`, `while`)
- **আইডেন্টিফায়ার (Identifier):** ভেরিয়েবল, ফাংশন, ক্লাস ইত্যাদির নাম (যেমন: `total`, `sum`, `Student`)
- **লিটারাল (Literal):** সরাসরি মান (যেমন: সংখ্যা, স্ট্রিং, বুলিয়ান)
- **অপারেটর (Operator):** গাণিতিক, লজিক্যাল বা তুলনামূলক চিহ্ন (যেমন: `+`, `-`, `*`, `==`)
- **ডেলিমিটার (Delimiter):** কোডের বিভিন্ন অংশ আলাদা করার চিহ্ন (যেমন: `;`, `{}`, `()`, `,`)
- **কমেন্ট (Comment):** কোডে ব্যাখ্যা বা নোট (যা এক্সিকিউশনে প্রভাব ফেলে না)

#### উদাহরণ (C ভাষা):
```c
int sum = 10 + 20; // এখানে প্রতিটি অংশ একটি টোকেন
```
- `int` → কীওয়ার্ড
- `sum` → আইডেন্টিফায়ার
- `=` → অপারেটর
- `10`, `20` → লিটারাল
- `+` → অপারেটর
- `;` → ডেলিমিটার
- `// ...` → কমেন্ট

---

## ২. লিটারাল (Literal)

**লিটারাল** হচ্ছে সোর্স কোডে সরাসরি ব্যবহৃত মান। এগুলো পরিবর্তন হয় না।

### লিটারালের ধরন:
- **ইন্টিজার লিটারাল:** যেমন `5`, `-10`, `100`
- **ফ্লোটিং পয়েন্ট লিটারাল:** যেমন `3.14`, `-0.5`
- **স্ট্রিং লিটারাল:** যেমন `'hello'`, `"Bangla"`
- **বুলিয়ান লিটারাল:** যেমন `true`, `false`
- **নাল/নান লিটারাল:** যেমন `null`, `None`, `undefined`

#### উদাহরণ (Python):
```python
x = 42          # ইন্টিজার লিটারাল
pi = 3.1416     # ফ্লোট লিটারাল
name = "রহিম"   # স্ট্রিং লিটারাল
is_valid = True # বুলিয়ান লিটারাল
```

## ৩. ডেলিমিটার (Delimiter)

**ডেলিমিটার** হচ্ছে কোডের বিভিন্ন অংশকে আলাদা করার জন্য ব্যবহৃত চিহ্ন।

### সাধারণ ডেলিমিটার:
- **সেমিকোলন (`;`):** স্টেটমেন্টের শেষ
- **কমা (`,`):** তালিকায় আইটেম আলাদা করা
- **ব্র্যাকেটস:**
  - গোল ব্র্যাকেট `()` → ফাংশন কল, গ্রুপিং
  - কার্লি ব্রেস `{}` → ব্লক শুরু ও শেষ
  - স্কয়ার ব্র্যাকেট `[]` → অ্যারে, লিস্ট

#### উদাহরণ (JavaScript):
```js
function add(a, b) {
  return a + b;
}
```
এখানে `()`, `{}`, `,`, `;` — সবই ডেলিমিটার।

## ৪. কমেন্ট (Comment)

**কমেন্ট** হচ্ছে কোডের মধ্যে লেখকের নোট বা ব্যাখ্যা, যা কম্পাইলার/ইন্টারপ্রিটার উপেক্ষা করে।

### কমেন্টের ধরন:
- **সিঙ্গেল লাইন কমেন্ট:**
  - C, C++, Java, JavaScript: `// ...`
  - Python: `# ...`
- **মাল্টি-লাইন কমেন্ট:**
  - C, Java, JavaScript: `/* ... */`
  - Python: `''' ... '''` বা `""" ... """`

#### কমেন্ট লেখার উদ্দেশ্য:
- কোড ব্যাখ্যা করা
- ভবিষ্যতে কোড বুঝতে সহায়তা
- অস্থায়ীভাবে কোড নিষ্ক্রিয় রাখা

#### উদাহরণ:
```python
# এটি একটি সিঙ্গেল লাইন কমেন্ট
print("Hello")  # ইনলাইন কমেন্ট

'''
এটি
মাল্টি-লাইন
কমেন্ট
'''
```

## ৫. লেক্সিক্যাল অ্যানালাইসিস (Lexical Analysis)

প্রোগ্রামিং ভাষার কম্পাইলার বা ইন্টারপ্রিটার প্রথমে সোর্স কোডকে টোকেনে ভাগ করে। এই ধাপকে **লেক্সিক্যাল অ্যানালাইসিস** বলে।

- **লেক্সার/স্ক্যানার:** এই কাজটি করে
- **ভুল টোকেন:** যদি অজানা বা অবৈধ চিহ্ন থাকে, তাহলে লেক্সার ত্রুটি দেখায়

#### উদাহরণ:
```python
x = 5 + 3
```
এখানে লেক্সার কোডটিকে টোকেনে ভাগ করবে: `x`, `=`, `5`, `+`, `3`
