# HMAC: এনক্রিপশনের পাশাপাশি ডেটার সততা যাচাই

## পরিচিতি

এনক্রিপশন আমাদের ডেটা গোপন রাখে, কিন্তু ডেটা ঠিক আছে কিনা, কেউ বদলেছে কিনা—তা যাচাই করার জন্য দরকার "সততা" যাচাই। এই কাজের জন্য ব্যবহৃত হয় **HMAC** (Hash-based Message Authentication Code)।

### সহজ উদাহরণ: সিল করা চিঠি

ধরা যাক, আপনি একটি গুরুত্বপূর্ণ চিঠি পাঠাচ্ছেন। চিঠির উপর একটি বিশেষ সিল দিলেন, যা শুধু প্রাপক ও আপনি জানেন। কেউ যদি চিঠি বদলায়, সিল আর মিলবে না।
- HMAC হচ্ছে সেই "ডিজিটাল সিল"—যা ডেটার সততা ও অথেন্টিকেশন নিশ্চিত করে।

## HMAC কীভাবে কাজ করে?
- HMAC দুইটি জিনিস ব্যবহার করে: একটি "গোপন কী" ও একটি "হ্যাশ ফাংশন" (যেমন SHA-256)।
- ডেটা ও কী একসাথে হ্যাশ করে একটি ইউনিক "কোড" তৈরি হয়—এটাই HMAC।
- প্রাপকও যদি একই কী ও ডেটা ব্যবহার করে, একই HMAC পাবে।
- কেউ যদি ডেটা বদলায় বা কী না জানে, HMAC মিলবে না।

## HMAC-এর ব্যবহার
- API authentication (যেমন: Google, AWS)
- Secure communication (SSL/TLS)
- ফাইল বা মেসেজের সততা যাচাই
- পাসওয়ার্ড ম্যানেজার

## HMAC কেন নিরাপদ?
- শুধু হ্যাশ ফাংশন নয়, "গোপন কী"ও লাগে
- ডেটা বদলালে বা কী ভুল হলে HMAC মিলবে না
- "replay" বা "tampering" attack-এর বিরুদ্ধে শক্তিশালী

## মূল পয়েন্ট
- HMAC = Hash + গোপন কী
- ডেটার সততা ও অথেন্টিকেশন নিশ্চিত
- API, SSL, ফাইল—সবখানে ব্যবহৃত

## অনুশীলন

১. একটি টেক্সট নিন (যেমন: "hmacwebx") ও একটি গোপন কী দিন (যেমন: "12345")।
   - HMAC generator দিয়ে HMAC-SHA256 বের করুন।
   - টেক্সট বা কী পরিবর্তন করে আবার HMAC দেখুন—পুরোপুরি বদলে যাবে।

২. API authentication-এ HMAC কীভাবে ব্যবহৃত হয়, তা জানতে Google-এ "HMAC API authentication" সার্চ করুন।

৩. SSL/TLS-এ HMAC কীভাবে ডেটার সততা নিশ্চিত করে, তা জানুন।

## উপসংহার

HMAC আমাদের ডেটার সততা ও নিরাপত্তার জন্য অপরিহার্য। এনক্রিপশনের পাশাপাশি HMAC ব্যবহার করলে ডেটা শুধু গোপন নয়, নিরাপদও থাকে।
