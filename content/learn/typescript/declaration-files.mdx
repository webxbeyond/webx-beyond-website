## পরিচিতি

ধরা যাক, আপনি একটি বড় লাইব্রেরি ব্যবহার করছেন, কিন্তু তার ডকুমেন্টেশন বাংলায় নেই। আপনি নিজের মতো করে "নোট" লিখে রাখলেন, যাতে অন্যরা সহজে বুঝতে পারে। TypeScript-এ ".d.ts" ফাইল মানে, টাইপ ডেফিনিশন—যা JS লাইব্রেরির জন্য টাইপ তথ্য দেয়। "Declaration Merging" মানে, একাধিক টাইপ ডেফিনিশন একত্রে মিশে যায়।

## অ্যানালজি

"লাইব্রেরির নোট"—বিদ্যমান বইয়ের জন্য নিজের মতো করে নোট লিখে রাখা (type definition)। একাধিক নোট একত্রে মিশে গেলে (declaration merging)।

## বিস্তারিত ব্যাখ্যা

### `.d.ts` ফাইল কী?
- টাইপ ডেফিনিশন ফাইল
- JS লাইব্রেরির জন্য টাইপ তথ্য দেয়
- উদাহরণ:
  ```typescript
  // math-lib.d.ts
  declare module "math-lib" {
    export function add(a: number, b: number): number;
  }
  ```
- DefinitelyTyped-এ হাজারো `.d.ts` ফাইল আছে

### Declaration Merging
- একাধিক টাইপ ডেফিনিশন একত্রে মিশে যায়
- Interface, namespace, function, class—সব merge হতে পারে
- উদাহরণ:
  ```typescript
  interface User {
    name: string;
  }
  interface User {
    age: number;
  }
  // User: { name: string; age: number }
  ```

### কখন ব্যবহার করবেন?
- যখন JS লাইব্রেরির টাইপ দরকার
- যখন টাইপ ডেফিনিশন extend করতে হবে

## মূল পয়েন্ট

- `.d.ts` ফাইল: JS লাইব্রেরির জন্য টাইপ
- Declaration Merging: একাধিক টাইপ একত্রে
- Maintainable ও scalable কোডের জন্য গুরুত্বপূর্ণ
