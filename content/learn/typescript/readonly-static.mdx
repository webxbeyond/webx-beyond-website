## পরিচিতি

ধরা যাক, আপনি একটি লাইব্রেরি পরিচালনা করছেন। লাইব্রেরির নাম কখনোই পরিবর্তন হয় না (readonly), কিন্তু লাইব্রেরির মোট বই সংখ্যা সব সদস্যের জন্য একই (static)। TypeScript-এ "readonly" মানে, একবার সেট হলে আর পরিবর্তন করা যায় না; "static" মানে, ক্লাসের সব অবজেক্টের জন্য একই ভ্যালু।

## অ্যানালজি

"লাইব্রেরির নাম ও মোট বই সংখ্যা"—নাম কখনোই বদলায় না (readonly), আর মোট বই সংখ্যা সব সদস্যের জন্য একই (static)।

## বিস্তারিত ব্যাখ্যা

### Readonly প্রপার্টি
- একবার সেট হলে আর পরিবর্তন করা যায় না
- উদাহরণ:
  ```typescript
  class Book {
    readonly isbn: string;
    constructor(isbn: string) {
      this.isbn = isbn;
    }
  }
  const b = new Book("123-456");
  // b.isbn = "789-101"; // ভুল
  ```
- Immutable ডাটা মডেল তৈরিতে গুরুত্বপূর্ণ

### Static প্রপার্টি
- ক্লাসের সব অবজেক্টের জন্য একই ভ্যালু
- ক্লাস লেভেলে অ্যাক্সেস করা যায়
- উদাহরণ:
  ```typescript
  class Library {
    static totalBooks: number = 0;
    addBook() {
      Library.totalBooks++;
    }
  }
  const l1 = new Library();
  l1.addBook();
  const l2 = new Library();
  l2.addBook();
  console.log(Library.totalBooks); // 2
  ```
- Utility বা constant value রাখতে কাজে লাগে

### কখন ব্যবহার করবেন?
- যখন ডাটা পরিবর্তন করা যাবে না (readonly)
- যখন ক্লাস লেভেলে ভ্যালু দরকার (static)

## মূল পয়েন্ট

- Readonly: একবার সেট, আর পরিবর্তন নয়
- Static: ক্লাস লেভেলে, সব অবজেক্টের জন্য একই
- Immutable ও utility design-এর জন্য গুরুত্বপূর্ণ
