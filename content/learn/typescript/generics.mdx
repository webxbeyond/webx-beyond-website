## পরিচিতি

ধরা যাক, আপনি একটি ব্যাগ তৈরি করছেন, যেখানে যেকোনো কিছু রাখা যায়—বই, জামা, খাবার, ইত্যাদি। ব্যাগের ডিজাইন এমন, যাতে ভেতরে কী আছে সেটা আগে থেকে জানা যায় না, কিন্তু ব্যাগটি নিরাপদ ও ব্যবহারযোগ্য। TypeScript-এ "জেনেরিকস" ঠিক এমন—একটি ফাংশন, ক্লাস বা ইন্টারফেস, যা বিভিন্ন টাইপের ডাটা নিয়ে কাজ করতে পারে, কিন্তু টাইপ সেফটি বজায় রাখে।

## অ্যানালজি

"ব্যাগের ডিজাইন"—একই ব্যাগ, কিন্তু কখনো বই, কখনো জামা, কখনো খাবার রাখা যায়। জেনেরিকস মানে, একই কোড, কিন্তু বিভিন্ন টাইপের ডাটা নিয়ে কাজ করতে পারে।

## বিস্তারিত ব্যাখ্যা

- **জেনেরিকস কী?**
  - কোডকে flexible ও reusable করার উপায়
  - টাইপ সেফটি বজায় রেখে বিভিন্ন টাইপে কাজ করা
  - উদাহরণ:
    ```typescript
    function identity<T>(value: T): T {
      return value;
    }
    const num = identity<number>(42); // 42
    const str = identity<string>("hello"); // "hello"
    ```

- **কেন ব্যবহার করবেন?**
  - কোড পুনঃব্যবহারযোগ্য (reusable) হয়
  - টাইপ সেফটি বজায় থাকে
  - API flexible হয়

- **জেনেরিকস কোথায় ব্যবহার হয়?**
  - ফাংশন, ক্লাস, ইন্টারফেস
  - উদাহরণ:
    ```typescript
    interface Box<T> {
      content: T;
    }
    const box: Box<string> = { content: "TypeScript" };
    ```

- **জেনেরিকস-এর নিয়ম**
  - `<T>` বা অন্য টাইপ প্যারামিটার ব্যবহার
  - ডিক্লারেশন ও ব্যবহার দুই জায়গায় টাইপ নির্ধারণ

## মূল পয়েন্ট

- জেনেরিকস: flexible, reusable, type-safe কোড
- ফাংশন, ক্লাস, ইন্টারফেসে ব্যবহার হয়
- টাইপ প্যারামিটার দিয়ে ডাটা টাইপ নির্ধারণ
