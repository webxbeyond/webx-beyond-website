## পরিচিতি

ধরা যাক, আপনি একটি নতুন যন্ত্র আবিষ্কার করেছেন, কিন্তু তার ব্যবহারবিধি কেউ জানে না। আপনি নিজেই "ম্যানুয়াল" লিখে দিলেন, যাতে সবাই বুঝতে পারে। TypeScript-এ "কাস্টম টাইপ ডেফিনিশন" মানে, নিজের মতো করে `.d.ts` ফাইল লিখে JS লাইব্রেরির জন্য টাইপ তথ্য তৈরি করা।

## অ্যানালজি

"নতুন যন্ত্রের ম্যানুয়াল"—নিজে লিখে সবাইকে বুঝিয়ে দেওয়া। কাস্টম টাইপ ডেফিনিশন মানে, নিজের মতো করে টাইপ তথ্য তৈরি করা।

## বিস্তারিত ব্যাখ্যা

### কাস্টম টাইপ ডেফিনিশন কী?
- JS লাইব্রেরির জন্য নিজে টাইপ তথ্য তৈরি
- `.d.ts` ফাইল লিখে
- উদাহরণ:
  ```typescript
  // awesome-lib.d.ts
  declare module "awesome-lib" {
    export function magic(a: string): number;
  }
  ```
- TypeScript-এ import করলে টাইপ-সেফ কোড পাওয়া যায়

### কিভাবে লিখবেন?
- লাইব্রেরির API দেখে টাইপ ঠিক করুন
- `declare module`, `declare function`, `declare class` ব্যবহার
- DefinitelyTyped-এ contribute করতে পারেন

### Integration Tips
- টাইপ চেকিং নিশ্চিত করুন
- ডকুমেন্টেশন ও usage উদাহরণ দিন
- Project structure ঠিক রাখুন

### কখন ব্যবহার করবেন?
- যখন কোনো লাইব্রেরির টাইপ নেই
- যখন নিজের JS কোড টাইপ-সেফ করতে চান

## মূল পয়েন্ট

- কাস্টম টাইপ ডেফিনিশন: নিজের মতো করে টাইপ তথ্য
- `.d.ts` ফাইল: JS লাইব্রেরির জন্য
- Maintainable ও scalable কোডের জন্য গুরুত্বপূর্ণ
