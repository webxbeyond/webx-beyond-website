## পরিচিতি

ধরা যাক, আপনি একটি পার্টির জন্য অতিথিদের তালিকা করছেন। কেউ কেউ RSVP করে, কেউ করেন না—তাদের উপস্থিতি "অপশনাল"। আবার কেউ কেউ একাধিক অতিথি নিয়ে আসেন—তাদের জন্য "রেস্ট" (বাকি) অতিথিদের নাম রাখতে হয়। TypeScript-এ "অপশনাল প্যারামিটার" মানে কোনো প্যারামিটার না দিলেও চলে, আর "রেস্ট প্যারামিটার" মানে একাধিক মান একসাথে নেওয়া যায়।

## অ্যানালজি

"পার্টির অতিথি তালিকা"—অপশনাল প্যারামিটার হচ্ছে এমন অতিথি, যিনি আসতে পারেন, না-ও পারেন। রেস্ট প্যারামিটার হচ্ছে এমন অতিথি, যিনি একাধিক বন্ধু নিয়ে আসেন—তাদের নাম একসাথে রাখতে হয়।

## বিস্তারিত ব্যাখ্যা

- **অপশনাল প্যারামিটার**
  - ফাংশনের কোনো প্যারামিটার না দিলেও চলে
  - `?` চিহ্ন দিয়ে নির্ধারণ করা হয়
  - উদাহরণ:
    ```typescript
    function greet(name?: string): string {
      return `Hello, ${name || "Guest"}!`;
    }
    ```

- **রেস্ট প্যারামিটার**
  - একাধিক মান একসাথে নেওয়া যায়
  - `...` চিহ্ন দিয়ে নির্ধারণ
  - উদাহরণ:
    ```typescript
    function sum(...numbers: number[]): number {
      return numbers.reduce((acc, curr) => acc + curr, 0);
    }
    ```

- **কেন ব্যবহার করবেন?**
  - ফাংশন flexible ও পুনঃব্যবহারযোগ্য হয়
  - কোড পরিষ্কার ও maintainable

## মূল পয়েন্ট

- অপশনাল প্যারামিটার: না দিলেও চলে
- রেস্ট প্যারামিটার: একাধিক মান একসাথে নেওয়া যায়
- টাইপ সেফটি ও কোড স্পষ্টতার জন্য গুরুত্বপূর্ণ
