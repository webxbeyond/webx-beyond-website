# অবজেক্ট টাইপ ও টাইপ অ্যালিয়াস

## প্রস্তাবনা

Google, Netflix, Uber—production app-এ complex data model—type safety, maintainability—essential। TypeScript-এর object type ও type alias—codebase-এ strong foundation—industry standard।

## কেন গুরুত্বপূর্ণ?

- **Type safety:** Object/type alias—complex data model-এ bug কমানো, early error detect।
- **Maintainability:** Type alias—reusable, readable code—easy refactor, long-term maintain।
- **Productivity:** IntelliSense, auto-complete—fast coding, error avoid।

## বাস্তব উদাহরণ

## মূল ধারণা

### ১. Object টাইপ

- **Object type:** Multiple property—complex data—type define করতে।
- **Usage:**
  ```typescript
  let user: { name: string; age: number; isActive: boolean } = {
    name: "Anis",
    age: 25,
    isActive: true
  };
  ```

#### সহজ এনালজি

- "Object type—একটি ফাইল, যেখানে অনেক তথ্য (property) একসাথে রাখা যায়।"

### ২. Type Alias

- **Type alias:** Complex/reusable type—একটি নাম দিয়ে define করা।
- **Usage:**
  ```typescript
  type Product = {
    name: string;
    price: number;
    tags?: string[];
  };

  let item: Product = {
    name: "Laptop",
    price: 999,
    tags: ["electronics", "tech"]
  };
  ```

#### সহজ এনালজি

- "Type alias—একটি shortcut—বড় address-এর বদলে ছোট নাম ব্যবহার করা।"

### ৩. Optional Property

- **Optional property:** Object/type alias-এ property না থাকলেও চলবে—`?` দিয়ে define।
- **Usage:**
  ```typescript
  type User = {
    name: string;
    email?: string;
  };
  ```

### ৪. Practical Example

```typescript
// Type alias
type Order = {
  id: number;
  product: string;
  quantity: number;
  delivered?: boolean;
};

let order1: Order = {
  id: 101,
  product: "Book",
  quantity: 2
};
```
