## পরিচিতি

ধরা যাক, আপনি একটি ফ্যাক্টরি চালাচ্ছেন—প্রতিটি পণ্য তৈরি হওয়ার পর অটোমেটেড মেশিন দিয়ে চেক করা হয়, কোনো ত্রুটি থাকলে সঙ্গে সঙ্গে বাতিল। TypeScript-এ CI/CD-তে টাইপ চেকিং ঠিক এমনই—প্রতিটি কোড চেঞ্জ, কমিট, বিল্ড, ডিপ্লয়—সবকিছু অটোমেটেডভাবে টাইপ চেক হয়, ভুল থাকলে ডিপ্লয় হয় না।

## অ্যানালজি

CI/CD-তে টাইপ চেকিং-কে ভাবুন "ফ্যাক্টরির অটোমেটেড কোয়ালিটি চেক" হিসেবে—মানুষের ভুল কমে যায়, কোয়ালিটি নিশ্চিত হয়।

## বিস্তারিত ব্যাখ্যা

### ১. CI/CD কী?
- Continuous Integration (CI): কোড চেঞ্জ, কমিট, বিল্ড—সবকিছু অটোমেটেড
- Continuous Deployment (CD): কোড বিল্ড, টেস্ট, ডিপ্লয়—সবকিছু অটোমেটেড
- Quality, reliability, automation

### ২. টাইপ চেকিং CI/CD-তে
- কোড push/commit হলে অটোমেটেড টাইপ চেক
- ভুল টাইপ, missing type, error—সবকিছু আগেই ধরা পড়ে
- Build fail হলে ডিপ্লয় হয় না

### ৩. কোড উদাহরণ: GitHub Actions

```yaml
# .github/workflows/type-check.yml
name: TypeScript Type Check
on: [push, pull_request]
jobs:
  type-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npm run type-check
```

### ৪. package.json-এ type-check স্ক্রিপ্ট

```json
{
  "scripts": {
    "type-check": "tsc --noEmit"
  }
}
```

## সুবিধা

- কোডের quality ও reliability অটোমেটেডভাবে নিশ্চিত
- ভুল টাইপ, missing type—production-এ যাওয়ার আগেই ধরা পড়ে
- বড় টিমে collaboration ও confidence বাড়ে

## ব্যবহার

- CI/CD pipeline-এ type-check স্ক্রিপ্ট যোগ করুন
- Pull request, push—সবকিছুতে টাইপ চেক
- Build fail হলে ডিপ্লয় বন্ধ

## গুরুত্বপূর্ণ পয়েন্ট

- type-check স্ক্রিপ্টে `tsc --noEmit` ব্যবহার করুন
- CI/CD pipeline-এ টাইপ চেকিং must-have
- Quality ও reliability বজায় রাখতে অটোমেশন করুন
