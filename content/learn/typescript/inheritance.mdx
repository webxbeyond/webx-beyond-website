## পরিচিতি

ধরা যাক, আপনি একটি পরিবারে জন্মেছেন। পরিবারের কিছু বৈশিষ্ট্য (property) আপনার মধ্যে এসেছে—এটাই "ইনহেরিট্যান্স"। আবার, আপনি নিজের মতো করে কিছু কাজ করেন—এটাই "মেথড ওভাররাইডিং"। TypeScript-এ "ইনহেরিট্যান্স" মানে, একটি ক্লাস থেকে আরেকটি ক্লাসে বৈশিষ্ট্য পাওয়া; "মেথড ওভাররাইডিং" মানে, সাবক্লাসে নিজের মতো করে মেথড লিখে ফেলা।

## অ্যানালজি

"পরিবারের বৈশিষ্ট্য ও নিজস্বতা"—বাবা-মায়ের বৈশিষ্ট্য পেয়েছেন (ইনহেরিট্যান্স), কিন্তু নিজের মতো করে কিছু কাজ করেন (ওভাররাইডিং)।

## বিস্তারিত ব্যাখ্যা

### ইনহেরিট্যান্স
- একটি ক্লাস থেকে আরেকটি ক্লাসে প্রপার্টি ও মেথড পাওয়া
- `extends` কীওয়ার্ড দিয়ে
- উদাহরণ:
  ```typescript
  class Animal {
    move() {
      console.log("Moving...");
    }
  }
  class Dog extends Animal {
    bark() {
      console.log("Woof!");
    }
  }
  const d = new Dog();
  d.move(); // "Moving..."
  d.bark(); // "Woof!"
  ```

### মেথড ওভাররাইডিং
- সাবক্লাসে প্যারেন্ট ক্লাসের মেথড নিজের মতো করে লেখা
- `override` কীওয়ার্ড (TypeScript 4.3+) বা একই নামের মেথড
- উদাহরণ:
  ```typescript
  class Bird extends Animal {
    override move() {
      console.log("Flying...");
    }
  }
  const b = new Bird();
  b.move(); // "Flying..."
  ```

### কখন ব্যবহার করবেন?
- যখন কোড পুনঃব্যবহারযোগ্য করতে হবে
- যখন সাবক্লাসে আলাদা আচরণ দরকার

## মূল পয়েন্ট

- ইনহেরিট্যান্স: বৈশিষ্ট্য পাওয়া
- ওভাররাইডিং: নিজের মতো করে মেথড লেখা
- DRY (Don't Repeat Yourself) ও maintainable কোডের জন্য গুরুত্বপূর্ণ
