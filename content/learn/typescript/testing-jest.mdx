## পরিচিতি

ধরা যাক, আপনি একটি গাড়ি তৈরি করছেন—প্রতিটি যন্ত্রাংশ আলাদা আলাদা টেস্ট করে নিশ্চিত হন, সবকিছু ঠিকঠাক আছে। TypeScript-এ Jest দিয়ে ইউনিট টেস্টিং ঠিক এমনই—প্রতিটি ফাংশন, ক্লাস, কম্পোনেন্ট আলাদা করে টেস্ট করে, কোডের গুণগত মান ও reliability নিশ্চিত করা যায়।

## অ্যানালজি

Jest দিয়ে ইউনিট টেস্টিং-কে ভাবুন "গাড়ির পার্টস টেস্টিং ল্যাব" হিসেবে—প্রতিটি পার্ট আলাদা করে টেস্ট হয়, কোনো সমস্যা থাকলে সঙ্গে সঙ্গে ধরা পড়ে।

## বিস্তারিত ব্যাখ্যা

### ১. Jest কী?
- JavaScript/TypeScript-এর জন্য জনপ্রিয় টেস্টিং ফ্রেমওয়ার্ক
- ফাস্ট, simple, zero-config
- Assertion, mocking, coverage—all-in-one

### ২. TypeScript-এ Jest সেটআপ
- `jest`, `ts-jest`, `@types/jest` npm থেকে ইনস্টল
- `jest.config.js` বা `jest.config.ts` দিয়ে কনফিগার
- `tsconfig.json`-এ টেস্ট ফোল্ডার/ফাইল অন্তর্ভুক্ত

### ৩. কোড উদাহরণ

```typescript
// sum.ts
export function sum(a: number, b: number): number {
  return a + b;
}

// sum.test.ts
import { sum } from "./sum";

test("sum adds two numbers", () => {
  expect(sum(2, 3)).toBe(5);
});
```

### ৪. Mocking ও Coverage
- Mocking: External dependency, API call, DB—সবকিছু mock করা যায়
- Coverage: কত শতাংশ কোড টেস্ট হয়েছে, রিপোর্ট পাওয়া যায়

## সুবিধা

- কোডের reliability ও maintainability বাড়ে
- বাগ, ভুল, regression—সবকিছু আগেই ধরা যায়
- বড় প্রজেক্টে confidence ও quality বজায় থাকে

## ব্যবহার

- ফাংশন, ক্লাস, কম্পোনেন্ট—সবকিছুর ইউনিট টেস্ট
- Mocking ও coverage রিপোর্ট
- Continuous Integration (CI)-তে অটোমেটেড টেস্ট

## গুরুত্বপূর্ণ পয়েন্ট

- টেস্ট ফাইলের নাম `.test.ts` বা `.spec.ts` দিন
- Assertion ও mocking ভালোভাবে ব্যবহার করুন
- Coverage রিপোর্ট দেখুন

## অনুশীলন

1. একটি ফাংশনের জন্য Jest টেস্ট লিখুন
2. Mocking ব্যবহার করে external dependency টেস্ট করুন
3. Coverage রিপোর্ট জেনারেট করুন
