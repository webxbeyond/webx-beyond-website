## পরিচিতি

ধরা যাক, আপনি একটি গাড়ি ভাড়া দিতে চান, কিন্তু শর্ত—গাড়ির লাইসেন্স থাকতে হবে। TypeScript-এ "Generic Constraints" মানে, জেনেরিক টাইপে শর্ত দেওয়া; "extends" দিয়ে ঠিক করা হয়, কোন টাইপের উপর ভিত্তি করে কাজ হবে।

## অ্যানালজি

"গাড়ি ভাড়া ও লাইসেন্স"—সব গাড়ি ভাড়া দেওয়া যাবে না, শুধু যাদের লাইসেন্স আছে। Generic Constraints মানে, জেনেরিক টাইপে শর্ত দেওয়া।

## বিস্তারিত ব্যাখ্যা

### Generic Constraints কী?
- জেনেরিক টাইপে শর্ত দেওয়া
- `extends` দিয়ে নির্দিষ্ট টাইপের উপর ভিত্তি করে
- উদাহরণ:
  ```typescript
  function getName<T extends { name: string }>(obj: T): string {
    return obj.name;
  }
  getName({ name: "Rahim", age: 20 }); // "Rahim"
  getName({ age: 20 }); // ভুল
  ```
- Type-safe ও flexible API তৈরিতে গুরুত্বপূর্ণ

### Advanced Usage
- Multiple constraints
- keyof, union, intersection
- উদাহরণ:
  ```typescript
  function getProp<T, K extends keyof T>(obj: T, key: K) {
    return obj[key];
  }
  ```

### কখন ব্যবহার করবেন?
- যখন জেনেরিক টাইপে শর্ত দরকার
- যখন টাইপ-সেফ কোড লিখতে চান

## মূল পয়েন্ট

- Generic Constraints: জেনেরিক টাইপে শর্ত
- extends: নির্দিষ্ট টাইপের উপর ভিত্তি করে
- Type-safe ও maintainable কোডের জন্য গুরুত্বপূর্ণ
