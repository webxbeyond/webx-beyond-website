## পরিচিতি

ধরা যাক, আপনি একটি বড় অফিসে কাজ করেন। অফিসের প্রতিটি ডিপার্টমেন্টের নাম ও তথ্য আছে। "Keyof" মানে, সব ডিপার্টমেন্টের নাম; "Lookup" মানে, নির্দিষ্ট ডিপার্টমেন্টের তথ্য; "Indexed Access" মানে, নির্দিষ্ট তথ্য বের করা। TypeScript-এ এগুলো টাইপ-লেভেলে ডাটা access ও validate করার জন্য ব্যবহৃত হয়।

## অ্যানালজি

"অফিসের ডিপার্টমেন্ট ও তথ্য"—সব নাম (keyof), নির্দিষ্ট তথ্য (lookup), নির্দিষ্ট অংশ (indexed access)।

## বিস্তারিত ব্যাখ্যা

### Keyof টাইপ
- অবজেক্ট টাইপের সব key বের করে
- উদাহরণ:
  ```typescript
  type User = { name: string; age: number };
  type UserKeys = keyof User; // "name" | "age"
  ```

### Lookup টাইপ
- নির্দিষ্ট key-এর টাইপ বের করা
- Syntax: `T[K]`
- উদাহরণ:
  ```typescript
  type NameType = User["name"]; // string
  ```

### Indexed Access টাইপ
- Array বা অবজেক্টের নির্দিষ্ট অংশ বের করা
- উদাহরণ:
  ```typescript
  type Arr = [string, number];
  type First = Arr[0]; // string
  ```

### Advanced Usage
- Dynamic key access
- Generic function-এ ব্যবহার
- উদাহরণ:
  ```typescript
  function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
  }
  ```

### কখন ব্যবহার করবেন?
- যখন টাইপ-লেভেলে ডাটা access দরকার
- যখন flexible ও type-safe কোড চাই

## মূল পয়েন্ট

- Keyof: সব key
- Lookup: নির্দিষ্ট key-এর টাইপ
- Indexed Access: নির্দিষ্ট অংশ
- Type-safe ও maintainable কোডের জন্য গুরুত্বপূর্ণ
