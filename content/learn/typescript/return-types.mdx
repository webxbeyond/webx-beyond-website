## পরিচিতি

ধরা যাক, আপনি একটি চিঠি লিখছেন। কিছু চিঠি পাঠানোর পর উত্তর আসে (রিটার্ন টাইপ), কিছু চিঠি পাঠানোর পর কোনো উত্তর আসে না (void), আবার কিছু চিঠি পাঠানোর পর কখনোই উত্তর আসবে না—কারণ চিঠি হারিয়ে গেছে (never)। TypeScript-এ ফাংশনের "রিটার্ন টাইপ" ঠিক এমনই—কিছু ফাংশন মান ফেরত দেয়, কিছু দেয় না, কিছু কখনোই ফেরত দেয় না।

## অ্যানালজি

"চিঠি ও উত্তর"—রিটার্ন টাইপ হচ্ছে চিঠির উত্তর, void হচ্ছে কোনো উত্তর নেই, never হচ্ছে কখনোই উত্তর আসবে না (যেমন চিঠি হারিয়ে গেছে)।

## বিস্তারিত ব্যাখ্যা

- **রিটার্ন টাইপ**
  - ফাংশন কী ধরনের মান ফেরত দেবে তা নির্ধারণ
  - উদাহরণ:
    ```typescript
    function add(a: number, b: number): number {
      return a + b;
    }
    ```

- **void**
  - ফাংশন কোনো মান ফেরত দেয় না
  - উদাহরণ:
    ```typescript
    function log(message: string): void {
      console.log(message);
    }
    ```

- **never**
  - ফাংশন কখনোই মান ফেরত দেয় না (যেমন: error throw, infinite loop)
  - উদাহরণ:
    ```typescript
    function throwError(): never {
      throw new Error("Something went wrong");
    }
    function infinite(): never {
      while (true) {}
    }
    ```

- **কেন ব্যবহার করবেন?**
  - টাইপ সেফটি, কোড স্পষ্টতা, ভুল কমানো
  - API, error handling, complex logic—সবখানে গুরুত্বপূর্ণ

## মূল পয়েন্ট

- রিটার্ন টাইপ: ফাংশন কী ফেরত দেবে তা নির্ধারণ
- void: কোনো মান ফেরত দেয় না
- never: কখনোই মান ফেরত দেয় না
- টাইপ সেফটি ও কোড স্পষ্টতার জন্য গুরুত্বপূর্ণ
