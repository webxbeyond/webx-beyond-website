## পরিচিতি

ধরা যাক, আপনি একটি হাসপাতালের ওষুধের তালিকা তৈরি করছেন—প্রতিটি রোগের জন্য নির্দিষ্ট ওষুধ, কোনো রোগ বাদ গেলে ভুল চিকিৎসা হতে পারে। TypeScript-এ Exhaustiveness চেকিং ঠিক এমনই—সবকিছু কভার হয়েছে কিনা, কোনো "কেস" বাদ পড়েছে কিনা, তা নিশ্চিত করে।

## অ্যানালজি

Exhaustiveness চেকিং-কে ভাবুন "চেকলিস্ট" হিসেবে—প্রতিটি আইটেম টিক দেওয়া হয়েছে কিনা, কোনো কিছু বাদ পড়লে warning আসে।

## বিস্তারিত ব্যাখ্যা

### ১. Exhaustiveness চেকিং কী?
- Union টাইপ, Enum, Discriminated Union—সবকিছু কভার হয়েছে কিনা চেক
- Switch/case বা if/else-এ সবকিছু handle হয়েছে কিনা
- কোনো কেস বাদ পড়লে TypeScript error দেখায়

### ২. `never` টাইপ
- TypeScript-এর special টাইপ, কোনো value নেই
- Exhaustiveness চেকিং-এ `never` ব্যবহার করে "unreachable code" বা "missing case" ধরানো যায়

### ৩. কোড উদাহরণ

```typescript
// Discriminated Union
interface Cat {
  type: "cat";
  meow: () => void;
}
interface Dog {
  type: "dog";
  bark: () => void;
}
interface Bird {
  type: "bird";
  chirp: () => void;
}

type Animal = Cat | Dog | Bird;

function handleAnimal(animal: Animal) {
  switch (animal.type) {
    case "cat":
      animal.meow();
      break;
    case "dog":
      animal.bark();
      break;
    case "bird":
      animal.chirp();
      break;
    default:
      // Exhaustiveness check
      const _exhaustive: never = animal;
      throw new Error("Unknown animal type");
  }
}
```

### ৪. ভুল কেস ধরার সুবিধা
- নতুন টাইপ যোগ করলে, switch/case-এ handle না করলে error
- বড় কোডবেসে ভুল বা missing case runtime-এ বাগ আনার আগেই ধরা যায়

## সুবিধা

- সবকিছু handle হয়েছে কিনা নিশ্চিত
- বাগ, ভুল, missing case—runtime-এ আসার আগেই error
- বড় প্রজেক্টে reliability ও maintainability বাড়ে

## ব্যবহার

- Discriminated Union, Enum, Union টাইপে Exhaustiveness চেক
- Switch/case-এ default-এ `never` টাইপ ব্যবহার
- নতুন টাইপ যোগ করলে, সবকিছু handle হয়েছে কিনা চেক

## গুরুত্বপূর্ণ পয়েন্ট

- Exhaustiveness চেকিং বড় কোডবেসে must-have
- `never` টাইপ দিয়ে missing case ধরানো যায়
- TypeScript-এর type system runtime error কমায়

## অনুশীলন

1. একটি Discriminated Union টাইপ তৈরি করুন
2. Switch/case-এ সবকিছু handle করুন, default-এ `never` টাইপ ব্যবহার করুন
3. নতুন টাইপ যোগ করে দেখুন, error আসে কিনা
