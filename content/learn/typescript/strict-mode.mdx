## পরিচিতি

ধরা যাক, আপনি একটি ব্যাংকের vault পরিচালনা করছেন—প্রতিটি প্রবেশ, এক্সিট, ট্রান্সাকশন, সবকিছু কড়া নিরাপত্তায় রাখতে হয়। TypeScript-এর `strict` মোড ও `noImplicitAny` ঠিক এমনই—কোডের প্রতিটি অংশে "টাইপ সেফটি" নিশ্চিত করে, যেন কোনো "অনিশ্চিত" বা "অজানা" টাইপ ঢুকে না পড়ে।

## অ্যানালজি

`strict` মোডকে ভাবুন "বিমানবন্দরের সিকিউরিটি চেক" হিসেবে—প্রতিটি ব্যাগ, যাত্রী, ডকুমেন্ট কড়া চেক হয়, কোনো সন্দেহজনক কিছু থাকলে সঙ্গে সঙ্গে আটকানো হয়।

## বিস্তারিত ব্যাখ্যা

### ১. `strict` মোড
- TypeScript-এর সবচেয়ে কড়া টাইপ চেকিং
- `tsconfig.json`-এ `strict: true` দিলে অনেকগুলো সাব-ফ্ল্যাগ একসাথে চালু হয়
- টাইপ ইনফারেন্স, null safety, function parameter, property initialization—সবকিছুতে কড়া চেক

### ২. `noImplicitAny`
- কোনো ভ্যারিয়েবল, প্যারামিটার, রিটার্ন টাইপ "any" হলে error
- টাইপ না দিলে TypeScript "any" ধরে নেয়, strict মোডে এটা আটকানো হয়

### ৩. tsconfig.json সেটআপ

```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true
  }
}
```

### ৪. কোড উদাহরণ

```typescript
// strict মোড ছাড়া
function add(a, b) {
  return a + b;
}
// এখানে a, b-এর টাইপ "any"—বাগের সম্ভাবনা

// strict মোডে
function add(a: number, b: number): number {
  return a + b;
}
// টাইপ সেফ, ভুল হলে error
```

## সুবিধা

- টাইপ সেফটি সর্বোচ্চ
- বাগ, runtime error, ভুল টাইপ—সবকিছু কমে যায়
- বড় প্রজেক্টে maintainability ও reliability বাড়ে

## ব্যবহার

- tsconfig.json-এ strict ও noImplicitAny চালু করুন
- প্রতিটি ভ্যারিয়েবল, প্যারামিটার, রিটার্ন টাইপ নির্ধারণ করুন
- Null safety, property initialization, function typing—সবকিছুতে টাইপ দিন

## গুরুত্বপূর্ণ পয়েন্ট

- strict মোড চালু করলে অনেক সাব-ফ্ল্যাগ (noImplicitAny, strictNullChecks, ইত্যাদি) চালু হয়
- কোডে টাইপ না দিলে error আসবে—এটা ভালো practice
- বড় প্রজেক্টে strict মোড must-have

## অনুশীলন

1. tsconfig.json-এ strict ও noImplicitAny চালু করুন
2. টাইপ ছাড়া কোনো ভ্যারিয়েবল/ফাংশন লিখে error দেখুন
3. সবকিছুতে টাইপ দিয়ে strict মোডে কোড রান করুন
