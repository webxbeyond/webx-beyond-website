## ভূমিকা

TypeScript দিয়ে বড় কোডবেস ম্যানেজ করা অনেকটা বিশাল লাইব্রেরি চালানোর মতো—যেখানে হাজারো বই, বিভাগ, এবং রেফারেন্স থাকে। যদি সবকিছু একসাথে রাখেন, খুঁজে পাওয়া, আপডেট করা, এবং রক্ষণাবেক্ষণ কঠিন হয়ে যায়। ঠিক তেমনই, TypeScript-এ "Project References" বড় কোডবেসকে ছোট ছোট অংশে ভাগ করে, সহজে ম্যানেজ করার সুযোগ দেয়।

## অ্যানালজি: "লাইব্রেরির বিভাগ"

ধরা যাক, আপনি একটি বিশাল লাইব্রেরি চালাচ্ছেন। সেখানে প্রতিটি বিভাগ আলাদা—বিজ্ঞান, সাহিত্য, ইতিহাস। প্রতিটি বিভাগে আলাদা রেফারেন্স, ক্যাটালগ, এবং মেইন লাইব্রেরির সাথে সংযোগ। এতে বই খুঁজে পাওয়া, আপডেট করা, এবং নতুন বিভাগ যোগ করা সহজ হয়। TypeScript-এর Project References ঠিক এই লাইব্রেরির বিভাগগুলোর মতো—প্রতিটি সাব-প্রজেক্ট আলাদা, কিন্তু একসাথে কাজ করে।

## বিস্তারিত আলোচনা

### Project References কী?
- TypeScript-এর একটি ফিচার, যা বড় কোডবেসকে ছোট ছোট সাব-প্রজেক্টে ভাগ করতে দেয়
- প্রতিটি সাব-প্রজেক্টের আলাদা `tsconfig.json` থাকে
- একাধিক সাব-প্রজেক্ট একে অপরকে রেফারেন্স করতে পারে
- বিল্ড টাইমে শুধু পরিবর্তিত অংশগুলো কম্পাইল হয়

### সুবিধা
- **ইনক্রিমেন্টাল বিল্ড:** শুধু পরিবর্তিত সাব-প্রজেক্ট কম্পাইল হয়, পুরো কোডবেস নয়
- **স্কেলেবিলিটি:** বড় টিমে আলাদা সাব-প্রজেক্টে কাজ করা সহজ
- **রক্ষণাবেক্ষণ:** কোডবেস ছোট ছোট অংশে ভাগ থাকায় রক্ষণাবেক্ষণ সহজ
- **ডিপেন্ডেন্সি ম্যানেজমেন্ট:** কোন সাব-প্রজেক্ট কোনটা ব্যবহার করছে, সহজে ট্র্যাক করা যায়

### কিভাবে কাজ করে?
1. প্রতিটি সাব-প্রজেক্টে আলাদা `tsconfig.json` তৈরি করুন
2. মেইন প্রজেক্টের `tsconfig.json`-এ `references` ফিল্ডে সাব-প্রজেক্টের path দিন
3. `tsc --build` কমান্ড দিয়ে পুরো কোডবেস বিল্ড করুন
4. শুধু পরিবর্তিত সাব-প্রজেক্ট বিল্ড হবে, বাকিগুলো স্কিপ হবে

### উদাহরণ
```json
// packages/core/tsconfig.json
{
  "compilerOptions": { "composite": true },
  "include": ["src"]
}

// packages/utils/tsconfig.json
{
  "compilerOptions": { "composite": true },
  "include": ["src"]
}

// tsconfig.json (root)
{
  "files": [],
  "references": [
    { "path": "./packages/core" },
    { "path": "./packages/utils" }
  ]
}
```

## বুলেট পয়েন্টস
- Project References বড় কোডবেসকে ছোট অংশে ভাগ করে
- ইনক্রিমেন্টাল বিল্ডে সময় ও রিসোর্স বাঁচে
- প্রতিটি সাব-প্রজেক্টে আলাদা `tsconfig.json`
- ডিপেন্ডেন্সি ও রেফারেন্স ট্র্যাকিং সহজ
- Monorepo ও enterprise কোডবেসে অপরিহার্য

## প্র্যাকটিকাল এক্সারসাইজ

1. **Monorepo তৈরি করুন:**
   - দুইটি সাব-প্রজেক্ট (core, utils) তৈরি করুন
   - প্রতিটির জন্য আলাদা `tsconfig.json` লিখুন
   - রুট `tsconfig.json`-এ references যোগ করুন
   - `tsc --build` চালিয়ে দেখুন কিভাবে incremental build হয়

2. **ডিপেন্ডেন্সি ম্যাপ করুন:**
   - কোন সাব-প্রজেক্ট কোনটা ব্যবহার করছে, diagram আঁকুন

3. **রিফ্যাক্টরিং:**
   - একটি সাব-প্রজেক্টে পরিবর্তন করুন, শুধু সেটি বিল্ড হচ্ছে কিনা দেখুন
