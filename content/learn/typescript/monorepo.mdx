## পরিচিতি

ধরা যাক, আপনি একটি বড় কোম্পানির অফিস পরিচালনা করছেন—একই বিল্ডিংয়ে HR, Accounts, IT, Sales, Marketing—সব ডিপার্টমেন্ট আলাদা, কিন্তু একসাথে কাজ করে। Monorepo-তে TypeScript ব্যবহার করলে অনেকগুলো প্রজেক্ট, লাইব্রেরি, অ্যাপ—সবকিছু একসাথে, টাইপ সেফ ও স্কেলেবলভাবে ম্যানেজ করা যায়।

## অ্যানালজি

Monorepo-কে ভাবুন "বড় অফিস বিল্ডিং" হিসেবে—প্রতিটি ডিপার্টমেন্টের নিজস্ব কাজ, কিন্তু কমন রিসোর্স, কমিউনিকেশন, ও সিকিউরিটি সিস্টেম শেয়ার করা। TypeScript Monorepo-তে প্রতিটি প্যাকেজ/প্রজেক্টের টাইপ সেফটি ও শেয়ারিং সহজ হয়।

## বিস্তারিত ব্যাখ্যা

### ১. Monorepo কী?
- একাধিক অ্যাপ/লাইব্রেরি/প্যাকেজ একসাথে একটি রিপোজিটরিতে
- Dependency, কোড শেয়ারিং, বিল্ড, টেস্ট—সবকিছু সহজ

### ২. Nx ও Turborepo
- Nx: Monorepo management tool, TypeScript-friendly
- Turborepo: High-performance build system, Monorepo-তে স্কেলিং সহজ
- দুইটাতেই TypeScript integration, lint/test/build automation

### ৩. TypeScript Monorepo Structure
- `packages/` বা `apps/` ফোল্ডারে আলাদা আলাদা প্রজেক্ট
- Root-level tsconfig, প্রতিটি প্যাকেজে আলাদা tsconfig
- Dependency graph, type references

```text
monorepo/
  apps/
    web/
    api/
  packages/
    ui/
    utils/
  tsconfig.json
  package.json
```

### ৪. TypeScript Project References
- বড় কোডবেসে টাইপ চেকিং ও বিল্ড টাইম অপ্টিমাইজ
- প্রতিটি প্যাকেজে `tsconfig.json`-এ `references` ব্যবহার

```json
// apps/web/tsconfig.json
{
  "extends": "../../tsconfig.json",
  "references": [
    { "path": "../../packages/ui" },
    { "path": "../../packages/utils" }
  ]
}
```
