## পরিচিতি

ধরা যাক, আপনি একটি ফরম ডিজাইন করছেন। কখনো সব ফিল্ড দরকার, কখনো কিছু ফিল্ড, আবার কখনো নতুন ফিল্ড যোগ করতে হয়। TypeScript-এ "Mapped টাইপ" ও "Utility টাইপ" ঠিক এমন—একটি টাইপ থেকে নতুন টাইপ তৈরি করা, যাতে flexibility ও type-safety বজায় থাকে।

## অ্যানালজি

"ফরম ডিজাইন"—একই ফরম, কিন্তু কখনো সব ফিল্ড, কখনো কিছু ফিল্ড, আবার কখনো নতুন ফিল্ড যোগ। Mapped টাইপ মানে, পুরনো টাইপ থেকে নতুন টাইপ বানানো; Utility টাইপ মানে, shortcut দিয়ে কাজ সহজ করা।

## বিস্তারিত ব্যাখ্যা

### Mapped টাইপ
- একটি টাইপের সব প্রপার্টি নিয়ে নতুন টাইপ তৈরি
- উদাহরণ:
  ```typescript
  type ReadonlyUser = {
    readonly [K in keyof User]: User[K];
  };
  ```
- Custom mapped টাইপ দিয়ে flexible টাইপ বানানো যায়

### Utility টাইপ
- TypeScript built-in shortcut টাইপ
- সাধারণ Utility টাইপ:
  - **`Partial<T>`**: সব প্রপার্টি optional
    ```typescript
    type User = { name: string; age: number };
    const u: Partial<User> = {};
    ```
  - **`Pick<T, K>`**: নির্দিষ্ট কিছু প্রপার্টি
    ```typescript
    type User = { name: string; age: number };
    const u: Pick<User, "name"> = { name: "Rahim" };
    ```
  - **`Record<K, T>`**: key-value টাইপ
    ```typescript
    const scores: Record<string, number> = { math: 90, english: 85 };
    ```
- আরও আছে: Omit, Required, Readonly, Exclude, Extract

### কখন ব্যবহার করবেন?
- যখন টাইপ থেকে নতুন টাইপ বানাতে হবে
- যখন shortcut দিয়ে কোড সহজ করতে হবে

## মূল পয়েন্ট

- Mapped টাইপ: পুরনো টাইপ থেকে নতুন টাইপ
- Utility টাইপ: shortcut দিয়ে কাজ সহজ
- Type-safe ও maintainable কোডের জন্য গুরুত্বপূর্ণ
