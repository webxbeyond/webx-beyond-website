# Mapped টাইপ ও Utility টাইপ (Partial, Pick, Record)

## প্রস্তাবনা

Google-এর API, Netflix-এর ডেটা মডেল, Uber-এর রাইড অপশন—সবখানে dynamic, type-safe data transformation দরকার। TypeScript-এর Mapped টাইপ ও Utility টাইপ—complex workflow-এ powerful tool, industry-standard solution।

## কেন গুরুত্বপূর্ণ?

- **Dynamic Data Transformation:** Real-world app-এ property add/remove/change—mapped type handle করে।
- **Code Reusability:** Utility type—common pattern—reuse করা যায়।
- **Type Safety:** Compile time-এ error detect—runtime bug কমায়।

## বাস্তব উদাহরণ

- **Google:** API response—optional property—Partial utility type দিয়ে handle।
- **Netflix:** Movie/Series—specific property—Pick utility type দিয়ে select।
- **Uber:** Ride option—dynamic key-value—Record utility type দিয়ে manage।

## মূল ধারণা

### ১. Mapped টাইপ (নকশা টাইপ)

- **সংজ্ঞা:** Type-এর property-কে loop করে নতুন টাইপ তৈরি করা।
- **Usage:**
  ```typescript
  type ReadonlyUser<T> = {
    readonly [K in keyof T]: T[K];
  };

  type User = { name: string; age: number };
  type ROUser = ReadonlyUser<User>;
  // ROUser: { readonly name: string; readonly age: number }
  ```

#### সহজ এনালজি

- "Mapped টাইপ—একটি কপি মেশিন—পুরনো ডকুমেন্ট থেকে নতুন ডকুমেন্ট তৈরি।"

### ২. Utility টাইপ

#### Partial

- **সংজ্ঞা:** সব property optional করে দেয়।
- **Usage:**
  ```typescript
  type User = { name: string; age: number };
  type PartialUser = Partial<User>;
  // PartialUser: { name?: string; age?: number }
  ```

#### Pick

- **সংজ্ঞা:** নির্দিষ্ট property select করে নতুন টাইপ তৈরি।
- **Usage:**
  ```typescript
  type User = { name: string; age: number; email: string };
  type NameEmail = Pick<User, "name" | "email">;
  // NameEmail: { name: string; email: string }
  ```

#### Record

- **সংজ্ঞা:** নির্দিষ্ট key ও value টাইপ দিয়ে object তৈরি।
- **Usage:**
  ```typescript
  type RideStatus = "pending" | "active" | "completed";
  type RideInfo = Record<RideStatus, { fare: number }>;
  // RideInfo: { pending: { fare: number }, active: { fare: number }, completed: { fare: number } }
  ```

### ৩. কখন ব্যবহার করবেন?

- **Mapped টাইপ:**
  - Property transformation দরকার হলে
  - Custom utility type তৈরি করতে
- **Partial:**
  - Optional property দরকার হলে
  - API response/model flexible রাখতে
- **Pick:**
  - Specific property select করতে
  - Data privacy/security নিশ্চিত করতে
- **Record:**
  - Dynamic key-value object তৈরি করতে
  - Config/status mapping handle করতে

### ৪. Common Pitfalls

- Mapped টাইপ—key miss করলে—type error
- Partial—unexpected undefined value
- Pick—property miss করলে—runtime bug
- Record—key/value mismatch

### ৫. Best Practices

- Mapped টাইপ—key/property clear define করুন
- Utility type—proper use/documentation
- Type safety—runtime bug কমাতে
- Documentation/readme—clear রাখুন

### ৬. Industry Trends

- API design—Partial/Record—flexible response

- Type safety—production workflow-এ default
