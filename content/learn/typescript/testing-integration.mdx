## পরিচিতি

ধরা যাক, আপনি একটি গাড়ি তৈরি করেছেন—প্রতিটি পার্ট আলাদা টেস্ট করেছেন, এবার সব পার্ট একসাথে লাগিয়ে পুরো গাড়ি চালিয়ে দেখছেন। Integration Testing ঠিক এমনই—সব ফাংশন, ক্লাস, কম্পোনেন্ট একসাথে কাজ করছে কিনা, তা যাচাই করা। TypeScript-এ টাইপ মকস দিয়ে external dependency, API, DB—সবকিছু "নকল" করে টেস্ট করা যায়।

## অ্যানালজি

Integration Testing-কে ভাবুন "গাড়ির ফাইনাল টেস্ট ড্রাইভ" হিসেবে—সব পার্ট একসাথে, কোনো সমস্যা থাকলে সঙ্গে সঙ্গে ধরা পড়ে। টাইপ মকস মানে "নকল পার্ট"—আসল পার্টের মতো আচরণ করে, কিন্তু আসল নয়।

## বিস্তারিত ব্যাখ্যা

### ১. Integration Testing কী?
- একাধিক ফাংশন, ক্লাস, কম্পোনেন্ট একসাথে টেস্ট
- External dependency (API, DB, Service) mock করে টেস্ট
- End-to-end flow, data passing, error handling—সবকিছু যাচাই

### ২. টাইপ মকস
- TypeScript-এ interface/class-এর "নকল" version তৈরি
- External dependency, API call, DB—সবকিছু mock করা যায়
- Mock object/interface দিয়ে টেস্ট environment তৈরি

### ৩. কোড উদাহরণ

```typescript
// userService.ts
export interface User {
  id: string;
  name: string;
}
export function getUser(id: string): Promise<User> {
  // ...real API call
}

// userService.test.ts
import { getUser, User } from "./userService";

test("getUser returns user", async () => {
  // Mock API
  const mockGetUser = jest.fn().mockResolvedValue({ id: "1", name: "Anis" });
  const user: User = await mockGetUser("1");
  expect(user.name).toBe("Anis");
});
```

### ৪. Integration Flow
- Multiple function/class/component একসাথে টেস্ট
- Mock object/interface দিয়ে external dependency simulate
- Data flow, error, edge case—সবকিছু যাচাই

## সুবিধা

- End-to-end reliability
- External dependency, API, DB—সবকিছু টেস্ট
- বড় প্রজেক্টে confidence ও quality বজায় থাকে

## ব্যবহার

- Integration test ফোল্ডার/ফাইল তৈরি করুন
- Mocking ও assertion ভালোভাবে ব্যবহার করুন
- Edge case ও error flow টেস্ট করুন

## গুরুত্বপূর্ণ পয়েন্ট

- Integration test ও unit test আলাদা রাখুন
- Mock object/interface টাইপ সেফ করুন
- Coverage রিপোর্ট দেখুন

## অনুশীলন

1. একটি integration test লিখুন, যেখানে একাধিক ফাংশন/ক্লাস একসাথে টেস্ট হয়
2. টাইপ মকস দিয়ে external dependency mock করুন
3. Edge case ও error flow টেস্ট করুন
