## পরিচিতি

ধরা যাক, আপনি একটি রেসিপি লিখছেন। প্রতিটি উপাদানের পরিমাণ, ধরন—সবকিছু স্পষ্টভাবে লিখতে হয়, যাতে রান্না ঠিকমতো হয়। TypeScript-এ "ফাংশন টাইপ অ্যানোটেশন" হচ্ছে সেই রেসিপির উপাদানের ধরন নির্ধারণ, আর "ডিফল্ট প্যারামিটার" হচ্ছে এমন উপাদান, যা না দিলেও রান্না ঠিকমতো হবে কারণ তার জন্য ডিফল্ট মান নির্ধারিত আছে।

## অ্যানালজি

"রেসিপি ও উপাদান"—ফাংশনের প্যারামিটার হচ্ছে রেসিপির উপাদান, টাইপ অ্যানোটেশন হচ্ছে উপাদানের ধরন, আর ডিফল্ট প্যারামিটার হচ্ছে এমন উপাদান, যা না দিলেও রান্না হবে কারণ তার জন্য ডিফল্ট মান আছে।

## বিস্তারিত ব্যাখ্যা

- **ফাংশন টাইপ অ্যানোটেশন**
  - ফাংশনের প্যারামিটার ও রিটার্ন টাইপ নির্ধারণ
  - উদাহরণ:
    ```typescript
    function add(a: number, b: number): number {
      return a + b;
    }
    ```
  - সুবিধা: টাইপ mismatch হলে error, কোড স্পষ্ট

- **ডিফল্ট প্যারামিটার**
  - কোনো প্যারামিটার না দিলে ডিফল্ট মান ব্যবহার
  - উদাহরণ:
    ```typescript
    function greet(name: string = "Guest"): string {
      return `Hello, ${name}!`;
    }
    ```
  - সুবিধা: কোড সহজ, ভুল কম

- **কেন ব্যবহার করবেন?**
  - টাইপ সেফটি, স্পষ্টতা, কোড maintainability
  - ডিফল্ট মান দিয়ে ফাংশন আরও flexible

## মূল পয়েন্ট

- ফাংশন টাইপ অ্যানোটেশন: প্যারামিটার ও রিটার্ন টাইপ নির্ধারণ
- ডিফল্ট প্যারামিটার: না দিলে ডিফল্ট মান ব্যবহার
- টাইপ সেফটি ও কোড স্পষ্টতার জন্য গুরুত্বপূর্ণ
