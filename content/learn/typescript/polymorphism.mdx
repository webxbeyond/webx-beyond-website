## পরিচিতি

ধরা যাক, আপনি একজন শিল্পী। একই "ব্রাশ" দিয়ে কখনো ছবি আঁকেন, কখনো দেয়াল রঙ করেন, আবার কখনো কাগজে স্কেচ করেন। "ব্রাশ" একটাই, কিন্তু ব্যবহার ভিন্ন—এটাই "পলিমরফিজম"। TypeScript-এ "পলিমরফিজম" মানে, একই ইন্টারফেস বা ক্লাস, কিন্তু ভিন্ন ভিন্ন ইমপ্লিমেন্টেশন; "ডিজাইন প্যাটার্ন" মানে, সমস্যার জন্য পরীক্ষিত সমাধান।

## অ্যানালজি

"একই ব্রাশ, ভিন্ন ব্যবহার"—একই টুল, কিন্তু ভিন্ন কাজে। পলিমরফিজম মানে, একই ইন্টারফেস/ক্লাস, ভিন্ন ইমপ্লিমেন্টেশন।

## বিস্তারিত ব্যাখ্যা

### পলিমরফিজম
- একই ইন্টারফেস/ক্লাস, ভিন্ন ইমপ্লিমেন্টেশন
- কোড flexible ও reusable হয়
- উদাহরণ:
  ```typescript
  interface Shape {
    area(): number;
  }
  class Circle implements Shape {
    constructor(public radius: number) {}
    area() {
      return Math.PI * this.radius * this.radius;
    }
  }
  class Square implements Shape {
    constructor(public side: number) {}
    area() {
      return this.side * this.side;
    }
  }
  function printArea(shape: Shape) {
    console.log(shape.area());
  }
  printArea(new Circle(5));
  printArea(new Square(4));
  ```

### ডিজাইন প্যাটার্ন
- সমস্যার জন্য পরীক্ষিত সমাধান
- OOP-এ জনপ্রিয় কিছু প্যাটার্ন:
  - Singleton
  - Factory
  - Observer
  - Strategy
- উদাহরণ (Factory Pattern):
  ```typescript
  class Car {
    drive() { console.log("Driving car..."); }
  }
  class Bike {
    ride() { console.log("Riding bike..."); }
  }
  function vehicleFactory(type: "car" | "bike") {
    if (type === "car") return new Car();
    else return new Bike();
  }
  const v = vehicleFactory("car");
  v.drive?.();
  ```

### কখন ব্যবহার করবেন?
- যখন flexible ও reusable কোড দরকার
- যখন বড় প্রজেক্টে structure দরকার

## মূল পয়েন্ট

- পলিমরফিজম: একই ইন্টারফেস/ক্লাস, ভিন্ন ইমপ্লিমেন্টেশন
- ডিজাইন প্যাটার্ন: পরীক্ষিত সমাধান
