## মূল ধারণা

### ১. পলিমরফিজম (Polymorphism)

- **সংজ্ঞা:** Multiple object/class—same interface/contract—different behavior implement।
- **Usage:**
  ```typescript
  interface Animal {
    speak(): void;
  }

  class Dog implements Animal {
    speak() {
      console.log("Dog barks");
    }
  }

  class Cat implements Animal {
    speak() {
      console.log("Cat meows");
    }
  }

  function makeSound(animal: Animal) {
    animal.speak();
  }

  makeSound(new Dog()); // Dog barks
  makeSound(new Cat()); // Cat meows
  ```

#### সহজ এনালজি

- "পলিমরফিজম—একটি রিমোট—একই বোতাম, আলাদা ডিভাইস—আলাদা কাজ।"

### ২. ডিজাইন প্যাটার্ন (Design Pattern)

- **সংজ্ঞা:** Common code structure—problem solve—industry-standard solution।
- **Strategy Pattern:** Multiple algorithm—runtime-এ select করা যায়।
  ```typescript
  interface SearchStrategy {
    search(query: string): string;
  }

  class GoogleSearch implements SearchStrategy {
    search(query: string) {
      return `Google searching for ${query}`;
    }
  }

  class BingSearch implements SearchStrategy {
    search(query: string) {
      return `Bing searching for ${query}`;
    }
  }

  function runSearch(strategy: SearchStrategy, query: string) {
    return strategy.search(query);
  }

  runSearch(new GoogleSearch(), "TypeScript"); // Google searching for TypeScript
  runSearch(new BingSearch(), "TypeScript"); // Bing searching for TypeScript
  ```

#### সহজ এনালজি

### ৩. কখন ব্যবহার করবেন?

- Multiple object/class—same contract—different behavior দরকার হলে
- Common workflow—extensible/maintainable structure দরকার হলে
- Large codebase—debug/refactor সহজ করতে

### ৪. Common Pitfalls

- Contract/interface miss—unexpected behavior
- Wrong pattern implement—complexity বাড়ে
- Polymorphic method/property—name mismatch—bug

### ৫. Best Practices

- Interface/contract—clear define করুন
- Design pattern—problem-specific implement করুন
- Documentation/readme—clear রাখুন
- Team/project standard—follow করুন

### ৬. Industry Trends

- Large-scale app—polymorphism/design pattern—flexible workflow
- Maintainability—pattern-based structure—easy debug/refactor
- Extensibility—polymorphic contract—easy feature add
