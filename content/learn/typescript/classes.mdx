## পরিচিতি

ধরা যাক, আপনি একটি স্কুল পরিচালনা করছেন। স্কুলের কিছু তথ্য সবার জন্য উন্মুক্ত (public), কিছু তথ্য শুধু শিক্ষকদের জন্য (protected), আর কিছু তথ্য শুধুমাত্র প্রধান শিক্ষকের জন্য (private)। TypeScript-এ "ক্লাস" মানে, একটি ব্লুপ্রিন্ট বা ছাঁচ, আর "অ্যাক্সেস মডিফায়ার" দিয়ে ঠিক করা হয়, কে কোন তথ্য দেখতে বা পরিবর্তন করতে পারবে।

## অ্যানালজি

"স্কুলের তথ্য"—সবাই দেখতে পারে এমন তথ্য (public), কিছু সীমিত তথ্য (protected), আর কিছু একান্ত ব্যক্তিগত তথ্য (private)।

## বিস্তারিত ব্যাখ্যা

### ক্লাস কী?
- অবজেক্ট তৈরির ছাঁচ
- প্রপার্টি ও মেথড থাকে
- উদাহরণ:
  ```typescript
  class Student {
    name: string;
    constructor(name: string) {
      this.name = name;
    }
    greet() {
      console.log(`Hello, ${this.name}`);
    }
  }
  ```

### অ্যাক্সেস মডিফায়ার
- **public**: সবার জন্য উন্মুক্ত
- **private**: শুধু ক্লাসের ভেতরে
- **protected**: ক্লাস ও সাবক্লাসে
- উদাহরণ:
  ```typescript
  class Teacher {
    public name: string;
    private salary: number;
    protected subject: string;
    constructor(name: string, salary: number, subject: string) {
      this.name = name;
      this.salary = salary;
      this.subject = subject;
    }
    getInfo() {
      return `${this.name} teaches ${this.subject}`;
    }
  }
  ```

### কখন ব্যবহার করবেন?
- যখন অবজেক্ট-ভিত্তিক ডিজাইন দরকার
- যখন ডাটা হাইড বা সীমিত করতে হবে

## মূল পয়েন্ট

- ক্লাস: অবজেক্ট তৈরির ছাঁচ
- public/private/protected: ডাটা অ্যাক্সেস নিয়ন্ত্রণ
- Type-safe ও maintainable কোডের জন্য গুরুত্বপূর্ণ
