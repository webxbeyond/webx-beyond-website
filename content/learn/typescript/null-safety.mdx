## টাইপস্ক্রিপ্টে "অদৃশ্য বাগ" ধরার কৌশল

## পরিচিতি

ধরা যাক, আপনি একটি হাসপাতালের ডেটাবেস পরিচালনা করছেন—প্রতিটি রোগীর তথ্য, রিপোর্ট, ওষুধ, সবকিছু সঠিকভাবে রাখতে হয়। কোনো তথ্য "missing" বা "null" থাকলে ভুল চিকিৎসা হতে পারে। TypeScript-এর নাল সেফটি ও `strictNullChecks` ঠিক এমনই—কোডে "অদৃশ্য" বা "missing" ডাটা থাকলে সঙ্গে সঙ্গে error দেখায়, যাতে runtime-এ বাগ না আসে।

## অ্যানালজি

নাল সেফটি ও `strictNullChecks`-কে ভাবুন "হাসপাতালের চেকলিস্ট" হিসেবে—প্রতিটি রোগীর রিপোর্টে কোনো ফিল্ড missing থাকলে warning আসে, ভুল চিকিৎসা আটকানো যায়।

## বিস্তারিত ব্যাখ্যা

### ১. নাল সেফটি
- TypeScript-এ `null` ও `undefined` টাইপ আলাদা
- টাইপে `null`/`undefined` explicitly না দিলে, সেই value assign করা যাবে না
- টাইপ সেফটি বজায় থাকে

### ২. `strictNullChecks`
- `tsconfig.json`-এ `strictNullChecks: true` দিলে, টাইপে `null`/`undefined` না থাকলে সেই value assign করলে error
- Optional chaining, nullish coalescing—সবকিছুতে কড়া চেক

### ৩. tsconfig.json সেটআপ

```json
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}
```

### ৪. কোড উদাহরণ

```typescript
// strictNullChecks ছাড়া
let name: string;
name = null; // কোনো error নেই

// strictNullChecks চালু
let name: string;
name = null; // Error: Type 'null' is not assignable to type 'string'

// টাইপে null/undefined দিলে
let name: string | null;
name = null; // No error
```

### ৫. Optional chaining ও Nullish coalescing

```typescript
const user = { profile: { name: "Anis" } };
console.log(user.profile?.name); // Optional chaining
console.log(user.profile?.age ?? "Not set"); // Nullish coalescing
```

## সুবিধা

- "অদৃশ্য" বা "missing" ডাটা runtime-এ বাগ আনার আগেই error
- কোডের reliability ও maintainability বাড়ে
- বড় প্রজেক্টে data integrity বজায় থাকে

## ব্যবহার

- tsconfig.json-এ strictNullChecks চালু করুন
- টাইপে null/undefined handle করুন
- Optional chaining ও nullish coalescing ব্যবহার করুন

## গুরুত্বপূর্ণ পয়েন্ট

- strictNullChecks চালু করলে টাইপে null/undefined না থাকলে assign করা যাবে না

- Null safety বড় প্রজেক্টে must-have
