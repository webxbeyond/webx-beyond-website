# নাল সেফটি ও `strictNullChecks`

## প্রস্তাবনা

Google-এর Search Engine, Netflix-এর Recommendation System, Uber-এর Ride Calculation—সবখানে null/undefined value নিয়ে bug, crash, বা unpredictable behavior হতে পারে। TypeScript-এর নাল সেফটি ও `strictNullChecks`—industry-standard, যা robust, readable, এবং maintainable app build করতে সাহায্য করে।

## কেন গুরুত্বপূর্ণ?

- **Type Safety:** Null/undefined—runtime bug, crash বাড়ায়; strictNullChecks—type error ধরতে সাহায্য করে
- **Developer Experience:** Early error—fast fix, productivity boost
- **Maintainability:** Large-scale app—easy refactor ও extend

## বাস্তব উদাহরণ

- **Google:** Search Engine—strictNullChecks-এ null bug-free algorithm
- **Netflix:** Recommendation—null safety-এ predictable logic
- **Uber:** Ride Calculation—strictNullChecks-এ safe workflow

### ২. `strictNullChecks` কী?

- **সংজ্ঞা:** TypeScript-এর strictNullChecks—null/undefined value-কে type-level-এ check করে
- **Usage:**
  ```json
  // tsconfig.json
  {
    "compilerOptions": {
      "strictNullChecks": true
    }
  }
  ```
- **Effect:**
  - Variable/property—null/undefined assign করলে error
  - Type-safe codebase

#### Example:
```typescript
let name: string;
name = null; // Error
name = undefined; // Error
let name2: string | null = null; // OK
```

### ৪. Common Pitfalls

- strictNullChecks—legacy code-এ error বাড়াতে পারে
- Optional chaining—type miss করলে runtime bug
- Over-strict—flexibility কমে যায়

### ৫. Best Practices

- strictNullChecks—new project-এ default রাখুন

- Optional chaining/nullish coalescing—properly use করুন
- Documentation/readme—clear রাখুন
- Team/project standard—follow করুন
