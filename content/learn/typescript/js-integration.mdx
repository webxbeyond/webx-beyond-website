## পরিচিতি

ধরা যাক, আপনি একটি কিচেন চালাচ্ছেন। নিজের কিছু সরঞ্জাম আছে, আবার বাজার থেকে নতুন সরঞ্জাম কিনে আনেন। TypeScript-এ "থার্ড-পার্টি JS লাইব্রেরি" মানে, বাইরের কোড—যা আপনার প্রজেক্টে যুক্ত করেন। TypeScript-এ এসব লাইব্রেরি ব্যবহার করতে হলে টাইপ ডেফিনিশন দরকার হয়, যাতে কোড টাইপ-সেফ থাকে।

## অ্যানালজি

"কিচেনের সরঞ্জাম"—নিজের, আর বাইরের। বাইরের সরঞ্জাম আনলে, কিভাবে ব্যবহার করবেন, সেটা জানতে হয় (type definition)।

## বিস্তারিত ব্যাখ্যা

### থার্ড-পার্টি JS লাইব্রেরি কী?
- বাইরের কোড, npm থেকে ইনস্টল
- TypeScript-এ ব্যবহার করতে হলে টাইপ ডেফিনিশন দরকার
- DefinitelyTyped-এ অনেক লাইব্রেরির টাইপ আছে
- উদাহরণ:
  ```bash
  npm install lodash
  npm install --save-dev @types/lodash
  ```
  ```typescript
  import _ from "lodash";
  _.chunk([1,2,3,4], 2);
  ```

### টাইপ ডেফিনিশন কিভাবে পাবেন?
- লাইব্রেরির সাথে built-in থাকতে পারে
- না থাকলে, @types/ প্যাকেজ ইনস্টল
- নিজে লিখতে হলে `.d.ts` ফাইল তৈরি

### Integration Tips
- টাইপ চেকিং নিশ্চিত করুন
- লাইব্রেরির ডকুমেন্টেশন দেখুন
- TypeScript config (`tsconfig.json`) ঠিক রাখুন

### কখন ব্যবহার করবেন?
- যখন বাইরের JS লাইব্রেরি দরকার
- যখন টাইপ-সেফ কোড লিখতে চান

## মূল পয়েন্ট

- থার্ড-পার্টি লাইব্রেরি: বাইরের কোড
- টাইপ ডেফিনিশন: টাইপ-সেফ কোডের জন্য
- Integration: ডকুমেন্টেশন ও config ঠিক রাখা
