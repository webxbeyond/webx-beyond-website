## পরিচিতি

ধরা যাক, আপনি একটি দোকানে কাজ করেন। একই নামের পণ্য, কিন্তু ভিন্ন ভিন্ন প্যাকেজ—কখনো ছোট, কখনো বড়, কখনো গিফট প্যাক। ক্রেতা যখন "চকলেট" চায়, আপনি তার চাহিদা অনুযায়ী প্যাকেজ দেন। TypeScript-এ "ফাংশন ওভারলোডিং" ঠিক এমন—একই ফাংশন, কিন্তু ভিন্ন ভিন্ন প্যারামিটার বা রিটার্ন টাইপ অনুযায়ী আলাদা আচরণ।

## অ্যানালজি

"দোকানের পণ্য ও প্যাকেজ"—একই নামের পণ্য, কিন্তু ভিন্ন ভিন্ন প্যাকেজ। ফাংশন ওভারলোডিং মানে, একই ফাংশন নাম, কিন্তু ভিন্ন ভিন্ন ইনপুট/আউটপুট অনুযায়ী আলাদা কাজ।

## বিস্তারিত ব্যাখ্যা

- **ফাংশন ওভারলোডিং কী?**
  - একাধিক ফাংশন সিগনেচার, কিন্তু একটাই ইমপ্লিমেন্টেশন
  - ইনপুট/আউটপুট টাইপ অনুযায়ী আলাদা আচরণ
  - উদাহরণ:
    ```typescript
    function greet(name: string): string;
    function greet(age: number): string;
    function greet(value: any): string {
      if (typeof value === "string") {
        return `Hello, ${value}!`;
      } else if (typeof value === "number") {
        return `You are ${value} years old.`;
      }
      return "Unknown";
    }
    ```

- **কেন ব্যবহার করবেন?**
  - API flexible ও user-friendly হয়
  - কোড maintainable ও reusable

- **ওভারলোডিং-এর নিয়ম**
  - একাধিক সিগনেচার লিখতে হয়
  - ইমপ্লিমেন্টেশন সবশেষে একবারই লিখতে হয়

## মূল পয়েন্ট

- ফাংশন ওভারলোডিং: একই নাম, ভিন্ন ইনপুট/আউটপুট
- API flexible ও user-friendly হয়
- টাইপ সেফটি ও কোড স্পষ্টতার জন্য গুরুত্বপূর্ণ
