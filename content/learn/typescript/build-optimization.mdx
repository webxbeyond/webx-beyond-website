# বিল্ড পারফরম্যান্স টিউনিং ও `tsc` অপশন

## প্রস্তাবনা

ধরা যাক, Netflix-এর মতো একটি বড় কোডবেস—হাজার হাজার ফাইল, প্রতিদিন শত শত কোড চেঞ্জ।
এখানে বিল্ড টাইম যদি বেশি হয়, তাহলে ডেভেলপারদের কাজ ধীর হয়ে যায়, ডিপ্লয় স্লো হয়, এবং overall productivity কমে যায়।

এজন্য TypeScript-এ **বিল্ড পারফরম্যান্স টিউনিং** ও `tsc` (TypeScript Compiler) অপশনগুলো জানা জরুরি—যাতে বড় প্রজেক্টেও দ্রুত, efficient বিল্ড করা যায়।

---

## কেন গুরুত্বপূর্ণ?

- **Speed:** বড় কোডবেসে বিল্ড টাইম কমানো productivity বাড়ায়।
- **Scalability:** কোডবেস বড় হলে efficient বিল্ড ছাড়া maintain করা কঠিন।
- **Continuous Integration:** CI/CD pipeline-এ বিল্ড টাইম কম হলে deploy দ্রুত হয়।
- **Developer Experience:** দ্রুত বিল্ড মানে দ্রুত ফিডব্যাক, কম frustration।

---

## মূল ধারণা

### ১. TypeScript Compiler (`tsc`) কী?

- **tsc:** TypeScript কোডকে JavaScript-এ রূপান্তর করার জন্য compiler।
- **Config:** `tsconfig.json` ফাইলে বিল্ড অপশন কনফিগার করা হয়।
- **CLI:** `npx tsc` বা `tsc` কমান্ড দিয়ে বিল্ড চালানো হয়।

### ২. বিল্ড পারফরম্যান্স টিউনিং কীভাবে করবেন?

#### এনালজি:
> যেমন, বড় ফ্যাক্টরিতে efficient মেশিন ব্যবহার করলে উৎপাদন দ্রুত হয়—তেমনি efficient বিল্ড অপশন ব্যবহার করলে কোড দ্রুত কম্পাইল হয়।

#### গুরুত্বপূর্ণ `tsc` অপশনসমূহ:

- **incremental:** আগের বিল্ডের ডেটা রেখে শুধু পরিবর্তিত ফাইল কম্পাইল করে।
- **composite:** বড়/মাল্টি-প্যাকেজ প্রজেক্টে dependency management সহজ করে।
- **skipLibCheck:** লাইব্রেরি টাইপ চেকিং স্কিপ করে—বিল্ড দ্রুত হয়।
- **isolatedModules:** প্রতিটি ফাইল আলাদাভাবে কম্পাইল—বিল্ড দ্রুত হয়, especially bundlers-এর জন্য।
- **exclude/include:** কোন ফাইল/ফোল্ডার কম্পাইল হবে, সেটি নির্ধারণ করে।

## বাস্তব উদাহরণ

### উদাহরণ ১: `tsconfig.json`-এ পারফরম্যান্স অপশন

```json
{
  "compilerOptions": {
    "incremental": true,
    "skipLibCheck": true,
    "isolatedModules": true,
    "outDir": "dist",
    "target": "ES2020"
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}
```

### উদাহরণ ২: Monorepo-তে `composite` ও Project References

```json
{
  "compilerOptions": {
    "composite": true,
    "declaration": true
  },
  "references": [
    { "path": "../shared" },
    { "path": "../api" }
  ]
}
```

### উদাহরণ ৩: CLI-তে দ্রুত বিল্ড

```bash
npx tsc --incremental --skipLibCheck
```

## কখন ব্যবহার করবেন?

- **বড় কোডবেসে:** হাজার+ ফাইল, মাল্টি-প্যাকেজ, monorepo—efficient বিল্ড অপরিহার্য।
- **CI/CD pipeline-এ:** বিল্ড টাইম কমাতে পারফরম্যান্স অপশন ব্যবহার করুন।
- **Development-এ:** দ্রুত ফিডব্যাক পেতে incremental বিল্ড চালান।

---

## সাধারণ ভুলত্রুটি

- **সব ফাইল একসাথে কম্পাইল করা:** incremental বা exclude ব্যবহার না করলে বিল্ড স্লো হয়।
- **skipLibCheck না ব্যবহার করা:** third-party লাইব্রেরি টাইপ error-এ বিল্ড fail হতে পারে, অথচ production-এ impact নেই।
- **outDir না দেওয়া:** compiled JS ফাইল source-এর সাথে মিশে যায়—maintain করা কঠিন।

---

## সেরা অনুশীলন (Best Practices)

- **incremental ও skipLibCheck ব্যবহার করুন।**
- **outDir ও exclude ঠিকভাবে কনফিগার করুন।**
- **Monorepo-তে composite ও references ব্যবহার করুন।**
- **CI/CD pipeline-এ দ্রুত বিল্ডের জন্য পারফরম্যান্স অপশন চালু রাখুন।**
