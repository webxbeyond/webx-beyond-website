## পরিচিতি

ধরা যাক, আপনি একটি বড় ফ্যাক্টরি চালাচ্ছেন—প্রতিটি মেশিন, লাইন, প্রসেস অপ্টিমাইজ করলে উৎপাদন দ্রুত হয়, খরচ কমে। TypeScript-এ বিল্ড পারফরম্যান্স টিউনিং ও `tsc` অপশন ঠিক এমনই—বড় কোডবেস দ্রুত, দক্ষ, ও resource-friendly কম্পাইল হয়।

## অ্যানালজি

বিল্ড পারফরম্যান্স টিউনিং-কে ভাবুন "ফ্যাক্টরির মেশিন অপ্টিমাইজেশন" হিসেবে—প্রতিটি মেশিনের speed, efficiency, energy consumption টিউন করলে output বাড়ে, downtime কমে।

## বিস্তারিত ব্যাখ্যা

### ১. `tsc` (TypeScript Compiler) কী?
- TypeScript কোডকে JavaScript-এ রূপান্তর করে
- টাইপ চেকিং, lint, transpile—all-in-one

### ২. বিল্ড পারফরম্যান্স টিউনিং
- বড় কোডবেসে বিল্ড টাইম, resource usage, efficiency বাড়ানো
- Incremental build, project references, exclude/include, outDir—সবকিছু optimize

### ৩. গুরুত্বপূর্ণ `tsc` অপশন

```json
{
  "compilerOptions": {
    "incremental": true, // শুধু পরিবর্তিত ফাইল কম্পাইল
    "tsBuildInfoFile": "./.tsbuildinfo", // বিল্ড cache
    "outDir": "./dist", // output directory
    "rootDir": "./src", // source directory
    "exclude": ["node_modules", "dist"], // অপ্রয়োজনীয় ফোল্ডার বাদ
    "composite": true, // project references
    "declaration": true, // .d.ts ফাইল জেনারেট
    "sourceMap": true // debug-friendly
  }
}
```

### ৪. Incremental Build
- শুধু পরিবর্তিত ফাইল কম্পাইল
- বিল্ড টাইম অনেক কমে যায়

### ৫. Project References
- বড় কোডবেসে dependency graph তৈরি
- একাধিক প্রজেক্ট/প্যাকেজ একসাথে efficiently কম্পাইল

### ৬. Exclude/Include
- অপ্রয়োজনীয় ফাইল/ফোল্ডার বাদ দিলে বিল্ড দ্রুত হয়

## সুবিধা
