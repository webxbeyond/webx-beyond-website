# Node.js-এ TypeScript (Express, Fastify)

## প্রস্তাবনা

Google-এর API, Netflix-এর Backend, Uber-এর Real-time Service—সবখানে scalable, maintainable, এবং bug-free server দরকার। Node.js-এ TypeScript—modern backend-এ industry-standard, যা robust, readable, এবং type-safe API তৈরি করে।

## মূল ধারণা

### ১. Express-এ TypeScript

- **সংজ্ঞা:** Express—popular Node.js framework—type-safe API build করা যায়
- **Usage:**
  ```typescript
  import express, { Request, Response } from 'express';
  const app = express();

  app.get('/user/:id', (req: Request, res: Response) => {
    const id: string = req.params.id;
    res.json({ id });
  });

  app.listen(3000, () => console.log('Server running'));
  ```

#### সহজ এনালজি

- "Express-এ TypeScript—একটি traffic police—সব request/response type-check করে।"

### ২. Fastify-তে TypeScript

- **সংজ্ঞা:** Fastify—high-performance Node.js framework—type-safe API build করা যায়
- **Usage:**
  ```typescript
  import fastify from 'fastify';
  const app = fastify();

  app.get<{ Params: { id: string } }>('/user/:id', (request, reply) => {
    const id = request.params.id;
    reply.send({ id });
  });

  app.listen({ port: 3000 }, err => {
    if (err) throw err;
    console.log('Server running');
  });
  ```

#### সহজ এনালজি

- "Fastify-তে TypeScript—একটি security gate—সব data type-check করে।"

### ৩. Typed Middleware

- **সংজ্ঞা:** Middleware—type-safe logic add করা যায়
- **Usage:**
  ```typescript
  import { Request, Response, NextFunction } from 'express';

  function logger(req: Request, res: Response, next: NextFunction) {
    console.log(req.method, req.url);
    next();
  }
  ```

### ৪. কখন ব্যবহার করবেন?

### ৫. Common Pitfalls

- Wrong type—unexpected bug/error
- Optional/nullable param—type miss করলে runtime bug
- Middleware—type mismatch

### ৬. Best Practices
