## পরিচিতি

ধরা যাক, আপনি একটি নতুন অফিসে কাজ শুরু করেছেন। অফিসে কাজ করার জন্য আপনাকে প্রথমে ডেস্ক, কম্পিউটার, সফটওয়্যার—সবকিছু সেটআপ করতে হয়। ঠিক তেমনি, TypeScript দিয়ে কাজ শুরু করতে হলে প্রথমেই সেটআপ ও কনফিগারেশন করতে হয়, যাতে আপনার কোড "অফিসের নিয়ম" মেনে চলে।

## অ্যানালজি

"অফিসের নিয়ম-কানুন"—TypeScript-এর tsconfig.json হচ্ছে সেই নিয়ম-কানুনের বই, যেখানে বলা আছে কীভাবে কোড লিখবেন, কোন ফোল্ডারে কাজ করবেন, কোন ফাইল কম্পাইল হবে, ইত্যাদি।

## বিস্তারিত ব্যাখ্যা

- **TypeScript ইনস্টল**
  - Node.js ইনস্টল করুন (https://nodejs.org/)
  - টার্মিনালে লিখুন: `npm install -g typescript`
  - ইনস্টল হয়েছে কিনা দেখতে: `tsc --version`

- **প্রজেক্টে TypeScript যোগ করা**
  - নতুন ফোল্ডার তৈরি করুন: `mkdir my-ts-project && cd my-ts-project`
  - `tsc --init` চালিয়ে tsconfig.json তৈরি করুন
  - `.ts` ফাইল তৈরি করে কোড লিখুন

- **tsconfig.json কী?**
  - এটি TypeScript কম্পাইলারের কনফিগারেশন ফাইল
  - এখানে কম্পাইলার অপশন, ফোল্ডার, ফাইল, strictness, ইত্যাদি নির্ধারণ করা হয়

- **গুরুত্বপূর্ণ tsconfig.json অপশন**
  - `target`: কোন JS ভার্সনে কম্পাইল হবে (যেমন ES6)
  - `module`: কোন মডিউল সিস্টেম (যেমন commonjs, esnext)
  - `strict`: টাইপ চেকিং কড়া হবে কিনা
  - `include`/`exclude`: কোন ফোল্ডার/ফাইল কম্পাইল হবে/হবে না

- **কনফিগারেশন টিপস**
  - ছোট প্রজেক্টে ডিফল্ট tsconfig.json যথেষ্ট
  - বড় প্রজেক্টে strict mode, path alias, incremental build ব্যবহার করুন

## মূল পয়েন্ট

- TypeScript ইনস্টল করতে npm ব্যবহার করুন
- tsconfig.json হচ্ছে TypeScript-এর নিয়ম-কানুনের বই
- strictness, target, module, include/exclude—সবকিছু এখানে নির্ধারণ হয়
