## পরিচিতি

ধরা যাক, আপনি একটি বড় বই লিখেছেন—সব অধ্যায় একসাথে পাঠালে পাঠক ক্লান্ত হয়ে যায়। তাই বইয়ের অধ্যায় আলাদা আলাদা পাঠানো হয়, অপ্রয়োজনীয় অংশ বাদ দেওয়া হয়। TypeScript-এ কোড স্প্লিটিং ও ট্রি-শেকিং ঠিক এমনই—অ্যাপের প্রয়োজনীয় অংশ আলাদা করে, অপ্রয়োজনীয় কোড বাদ দিয়ে, দ্রুত ও ছোট বানানো হয়।

## অ্যানালজি

কোড স্প্লিটিং-কে ভাবুন "বইয়ের অধ্যায় আলাদা আলাদা পাঠানো" হিসেবে—প্রয়োজনীয় অংশ পাঠানো হয়, অপ্রয়োজনীয় অংশ বাদ। ট্রি-শেকিং মানে "গাছ ঝাঁকিয়ে শুকনো পাতা ফেলা"—অপ্রয়োজনীয় কোড বাদ দেওয়া।

## বিস্তারিত ব্যাখ্যা

### ১. কোড স্প্লিটিং
- অ্যাপের বড় অংশ আলাদা আলাদা bundle/চাঙ্কে ভাগ
- প্রয়োজন অনুযায়ী লোড হয় (lazy loading)
- Initial load দ্রুত, resource usage কম

### ২. ট্রি-শেকিং
- Unused/অপ্রয়োজনীয় কোড বাদ দেওয়া
- Production build ছোট, দ্রুত
- Modern bundler (Webpack, Vite, Rollup) tree-shaking support

### ৩. TypeScript-এ কোড স্প্লিটিং
- Dynamic import (`import()`), lazy loading
- React, Next.js, Vite—সবকিছুতে কোড স্প্লিটিং

```typescript
// Dynamic import
const UserComponent = React.lazy(() => import('./UserComponent'));
```

### ৪. TypeScript-এ ট্রি-শেকিং
- ES Module (`export/import`) ব্যবহার করুন
- Side-effect-free কোড লিখুন
- Production build-এ unused code বাদ

```typescript
// Only used function will be included
export function add(a: number, b: number) { return a + b; }
export function unused() { /* ... */ }
```

### ৫. Bundler config
- Webpack/Vite/Rollup-এ tree-shaking ও code splitting config
- Production build-এ optimization

## সুবিধা

- অ্যাপ দ্রুত লোড হয়
- Production bundle ছোট
- Resource usage ও performance বাড়ে

## ব্যবহার

- Dynamic import ও lazy loading
- ES Module ও side-effect-free কোড
- Bundler config-এ optimization

## গুরুত্বপূর্ণ পয়েন্ট

- Modern bundler ব্যবহার করুন
- Production build-এ optimization চালু করুন
- Unused code ও side-effect-free কোড লিখুন

## অনুশীলন

1. একটি React/Vite/Next.js অ্যাপে dynamic import ও lazy loading ব্যবহার করুন
2. Production build-এ bundle size চেক করুন
3. Unused code বাদ দিতে tree-shaking config দেখুন
