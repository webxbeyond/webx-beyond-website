## ভূমিকা

বড় এন্টারপ্রাইজ অ্যাপ্লিকেশনে একাধিক সার্ভিস একসাথে কাজ করে—এগুলোকে বলে "মাইক্রোসার্ভিস"। এই সার্ভিসগুলো একে অপরের সাথে যোগাযোগ করে API, gRPC, বা ইভেন্ট-ড্রিভেন প্যাটার্নে। TypeScript দিয়ে টাইপ-সেফ gRPC ও ইভেন্ট-ড্রিভেন সিস্টেম তৈরি করলে, ডাটা ও কমিউনিকেশন আরও নিরাপদ, maintainable, এবং স্কেলেবল হয়।

## অ্যানালজি: "ট্রেন স্টেশন ও সিগনাল"

ধরা যাক, একটি বড় শহরে অনেকগুলো ট্রেন স্টেশন আছে। প্রতিটি স্টেশন আলাদা, কিন্তু ট্রেন, সিগনাল, ও রুট দিয়ে একে অপরের সাথে যোগাযোগ করে। যদি সিগনাল ভুল হয়, দুর্ঘটনা হতে পারে। ঠিক তেমনই, মাইক্রোসার্ভিসে টাইপ-সেফ কমিউনিকেশন না হলে, ডাটা ভুল, বাগ, ও সমস্যা হতে পারে।

## বিস্তারিত আলোচনা

### মাইক্রোসার্ভিস কী?
- বড় অ্যাপ্লিকেশনকে ছোট ছোট সার্ভিসে ভাগ করা হয়
- প্রতিটি সার্ভিস আলাদা, কিন্তু একে অপরের সাথে যোগাযোগ করে
- স্কেল, রক্ষণাবেক্ষণ, ও ডিপ্লয়মেন্ট সহজ হয়

### gRPC কী?
- Google-এর তৈরি একটি হাই-পারফরম্যান্স RPC (Remote Procedure Call) ফ্রেমওয়ার্ক
- Protobuf দিয়ে ডাটা স্ট্রাকচার ও API contract define করা হয়
- টাইপ-সেফ, ফাস্ট, এবং language-agnostic
- TypeScript-এ gRPC client/server তৈরি করা যায়

### ইভেন্ট-ড্রিভেন সিস্টেম
- সার্ভিসগুলো ইভেন্ট (যেমন: order_created) পাঠায় ও গ্রহণ করে
- Loose coupling, scalability, ও async communication সহজ হয়
- TypeScript দিয়ে ইভেন্ট টাইপ, হ্যান্ডলার, ও ডাটা সেফটি নিশ্চিত করা যায়

### TypeScript-এ টাইপ-সেফ কমিউনিকেশন
- Protobuf বা custom টাইপ দিয়ে ডাটা স্ট্রাকচার define করুন
- gRPC tools দিয়ে TypeScript টাইপ/ইন্টারফেস জেনারেট করুন
- ইভেন্ট-ড্রিভেন সিস্টেমে ইভেন্ট টাইপ ও হ্যান্ডলার লিখুন
- সার্ভিসগুলোতে টাইপ-সেফ API ও ইভেন্ট ব্যবহার করুন

### উদাহরণ
```protobuf
// Protobuf (user.proto)
message User {
  string id = 1;
  string name = 2;
}
service UserService {
  rpc GetUser (UserRequest) returns (User);
}
```

```typescript
// TypeScript (auto-generated)
export interface User {
  id: string;
  name: string;
}

// gRPC client
const user = await userService.getUser({ id: '123' });

// ইভেন্ট-ড্রিভেন
interface OrderCreatedEvent {
  orderId: string;
  userId: string;
}
function handleOrderCreated(event: OrderCreatedEvent) {
  // ...business logic
}
```

## বুলেট পয়েন্টস
- মাইক্রোসার্ভিসে টাইপ-সেফ কমিউনিকেশন নিরাপদ ও maintainable
- gRPC contract Protobuf দিয়ে লিখুন, TypeScript টাইপ জেনারেট করুন
- ইভেন্ট-ড্রিভেন সিস্টেমে ইভেন্ট টাইপ ও হ্যান্ডলার লিখুন
- স্কেল, পারফরম্যান্স, ও async communication সহজ হয়
- বড় টিমে কমিউনিকেশন ও ডিবাগিং সহজ হয়

## প্র্যাকটিকাল এক্সারসাইজ

1. **Protobuf contract লিখুন:**
   - একটি সার্ভিসের জন্য Protobuf ফাইল লিখুন
   - gRPC tools দিয়ে TypeScript টাইপ জেনারেট করুন

2. **ইভেন্ট টাইপ ও হ্যান্ডলার লিখুন:**
   - একটি ইভেন্ট (যেমন: order_created) এর জন্য টাইপ ও হ্যান্ডলার লিখুন

3. **মাইক্রোসার্ভিস কমিউনিকেশন:**
   - দুইটি সার্ভিসে টাইপ-সেফ API ও ইভেন্ট-ড্রিভেন কমিউনিকেশন ইমপ্লিমেন্ট করুন
