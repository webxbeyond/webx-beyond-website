## TypeScript-এ "পরিষ্কার" ও "শৃঙ্খলিত" কোড

## পরিচিতি

ধরা যাক, আপনি একটি অফিস পরিচালনা করছেন—সবকিছু পরিষ্কার, সাজানো, নিয়মিত চেক হয়। TypeScript-এ লিন্টার ও ফরম্যাটার ঠিক এমনই—কোডের ভুল, অনিয়ম, স্টাইল mismatch, সবকিছু অটোমেটেডভাবে ঠিক করে, কোডকে "পরিষ্কার" ও "শৃঙ্খলিত" রাখে।

## অ্যানালজি

লিন্টার ও ফরম্যাটার-কে ভাবুন "অফিসের ক্লিনার ও অডিটর" হিসেবে—কোডে কোথাও ধুলো, ভুল, অনিয়ম থাকলে সঙ্গে সঙ্গে পরিষ্কার ও রিপোর্ট আসে।

## বিস্তারিত ব্যাখ্যা

### ১. লিন্টার (ESLint)
- কোডের ভুল, টাইপ mismatch, bad practice—সবকিছু চেক
- Custom rule, plugin, config—সবকিছু কাস্টমাইজ
- TypeScript integration, auto-fix

### ২. ফরম্যাটার (Prettier)
- কোডের স্টাইল, spacing, indentation—সবকিছু অটোমেটেড
- Opinionated, zero-config
- ESLint-এর সাথে integration

### ৩. সেটআপ
- `eslint`, `@typescript-eslint/parser`, `@typescript-eslint/eslint-plugin`, `prettier` npm থেকে ইনস্টল
- `.eslintrc`, `.prettierrc` দিয়ে কনফিগার
- VSCode extension দিয়ে auto-format

### ৪. কোড উদাহরণ

```json
// .eslintrc.json
{
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": ["plugin:@typescript-eslint/recommended"],
  "rules": {
    "semi": ["error", "always"],
    "quotes": ["error", "double"]
  }
}

// .prettierrc
{
  "singleQuote": true,
  "semi": true
}
```
