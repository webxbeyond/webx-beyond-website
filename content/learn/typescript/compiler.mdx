## পরিচিতি

ধরা যাক, আপনি একটি গল্প লিখেছেন, কিন্তু সেটি প্রকাশ করার আগে একজন সম্পাদক সেটি যাচাই করে, ভুল ঠিক করে, এবং ছাপার উপযোগী করে তোলে। TypeScript কম্পাইলার (tsc) হচ্ছে সেই "সম্পাদক"—যে আপনার কোডকে যাচাই করে, টাইপের ভুল ধরিয়ে দেয়, এবং JavaScript-এ রূপান্তর করে দেয়।

## অ্যানালজি

"সম্পাদক ও ছাপা"—TypeScript কোড হচ্ছে আপনার খসড়া গল্প, আর tsc হচ্ছে সেই সম্পাদক, যে গল্পটি যাচাই করে, ঠিক করে, এবং ছাপার জন্য প্রস্তুত করে (JavaScript-এ রূপান্তর)।

## বিস্তারিত ব্যাখ্যা

- **TypeScript কম্পাইলার (tsc) কী?**
  - এটি TypeScript কোডকে JavaScript-এ রূপান্তর করে।
  - টাইপ চেকিং, সিনট্যাক্স চেকিং, এবং কনফিগারেশন অনুযায়ী কোড কম্পাইল করে।

- **tsc ব্যবহার**
  - টার্মিনালে লিখুন: `tsc`
  - `.ts` ফাইলকে `.js` ফাইলে রূপান্তর করে
  - `tsc filename.ts` — নির্দিষ্ট ফাইল কম্পাইল
  - `tsc` — পুরো প্রজেক্ট কম্পাইল (tsconfig.json থাকলে)

- **বিল্ড প্রক্রিয়া**
  - কোড লিখুন `.ts` ফাইলে
  - `tsc` চালিয়ে JavaScript-এ রূপান্তর করুন
  - আউটপুট ফোল্ডার (যেমন `dist/` বা `build/`) নির্ধারণ করতে tsconfig.json-এ `outDir` ব্যবহার করুন

- **কম্পাইলার অপশন**
  - `watch`: পরিবর্তন হলে অটো-কম্পাইল (`tsc --watch`)
  - `incremental`: দ্রুত বিল্ডের জন্য
  - `sourceMap`: ডিবাগিংয়ের জন্য

- **বিল্ড স্ক্রিপ্ট**
  - `package.json`-এ স্ক্রিপ্ট যোগ করুন:
    ```json
    "scripts": {
      "build": "tsc"
    }
    ```
  - এরপর `npm run build` চালিয়ে বিল্ড করুন

## মূল পয়েন্ট

- tsc হচ্ছে TypeScript-এর কম্পাইলার, যা কোডকে JavaScript-এ রূপান্তর করে
- টাইপ চেকিং, সিনট্যাক্স চেকিং, এবং কনফিগারেশন অনুযায়ী কম্পাইল
- বিল্ড স্ক্রিপ্ট দিয়ে সহজে বিল্ড করা যায়
