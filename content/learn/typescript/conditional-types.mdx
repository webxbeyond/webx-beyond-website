# Conditional টাইপ ও infer

## প্রস্তাবনা

Google-এর API, Netflix-এর ডেটা প্রসেসিং, Uber-এর রাইড অপশন—সবখানে dynamic, context-aware type transformation দরকার। TypeScript-এর Conditional টাইপ ও infer—complex, intelligent type workflow—industry-standard solution।

## কেন গুরুত্বপূর্ণ?

- **Dynamic Type Logic:** Real-world app-এ type conditionally change—conditional type handle করে।
- **Type Inference:** infer—type extract/derive—complex workflow-এ essential।
- **Type Safety:** Compile time-এ error detect—runtime bug কমায়।

## মূল ধারণা

### ১. Conditional টাইপ (শর্তাধীন টাইপ)

- **সংজ্ঞা:** TypeScript-এ type conditionally change—`extends` ও ternary operator
- **Usage:**
  ```typescript
  type IsString<T> = T extends string ? true : false;
  type A = IsString<string>; // true
  type B = IsString<number>; // false
  ```

#### সহজ এনালজি

- "Conditional টাইপ—একটি ট্রাফিক সিগনাল—শর্ত অনুযায়ী পথ নির্ধারণ।"

### ২. infer কী?

- **সংজ্ঞা:** TypeScript-এর infer—type extract/derive—conditional type-এর মধ্যে
- **Usage:**
  ```typescript
  type ElementType<T> = T extends (infer U)[] ? U : T;
  type A = ElementType<string[]>; // string
  type B = ElementType<number>; // number
  ```

#### সহজ এনালজি

- "infer—একটি ডিটেক্টর—বড় বাক্স থেকে নির্দিষ্ট জিনিস বের করে আনে।"

### ৩. কখন ব্যবহার করবেন?

- Complex type transformation দরকার হলে
- Type extract/derive করতে
- API response/model—dynamic type logic handle করতে

### ৪. Common Pitfalls

- Conditional type—wrong logic—unexpected value
- infer—type miss করলে—type error
- Type nesting—complexity বাড়ে

### ৫. Best Practices

- Type nesting—simple রাখুন
- Documentation/readme—clear রাখুন
