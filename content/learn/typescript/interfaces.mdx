## পরিচিতি

ধরা যাক, আপনি একটি গাড়ি কোম্পানির ডিজাইনার। কোম্পানি আপনাকে বলে, "গাড়ির চাকা, ইঞ্জিন, দরজা থাকতে হবে"—কিন্তু কেমন হবে, সেটা আপনি ঠিক করবেন। TypeScript-এ "অ্যাবস্ট্রাক্ট ক্লাস" মানে, কিছু ফিচার বাধ্যতামূলক, কিছু ফিচার নিজের মতো করে ইমপ্লিমেন্ট করতে হবে; "ইন্টারফেস" মানে, শুধু কাঠামো, কোনো ইমপ্লিমেন্টেশন নেই।

## অ্যানালজি

"গাড়ির ডিজাইন"—কিছু অংশ বাধ্যতামূলক (abstract), কিছু অংশ নিজের মতো করে (interface)।

## বিস্তারিত ব্যাখ্যা

### অ্যাবস্ট্রাক্ট ক্লাস
- কিছু মেথড/প্রপার্টি বাধ্যতামূলক, কিছু ইমপ্লিমেন্টেশন থাকতে পারে
- `abstract` কীওয়ার্ড দিয়ে
- উদাহরণ:
  ```typescript
  abstract class Vehicle {
    abstract move(): void;
    start() {
      console.log("Starting vehicle...");
    }
  }
  class Car extends Vehicle {
    move() {
      console.log("Car is moving...");
    }
  }
  ```

### ইন্টারফেস
- শুধু কাঠামো, কোনো ইমপ্লিমেন্টেশন নেই
- `interface` কীওয়ার্ড দিয়ে
- উদাহরণ:
  ```typescript
  interface Drivable {
    move(): void;
  }
  class Bike implements Drivable {
    move() {
      console.log("Bike is moving...");
    }
  }
  ```

### কখন ব্যবহার করবেন?
- অ্যাবস্ট্রাক্ট ক্লাস: কিছু ডিফল্ট আচরণ + বাধ্যতামূলক ফিচার
- ইন্টারফেস: শুধু কাঠামো, multiple class-এ implement

## মূল পয়েন্ট

- অ্যাবস্ট্রাক্ট ক্লাস: partial implementation + বাধ্যতামূলক ফিচার
- ইন্টারফেস: শুধু কাঠামো, কোনো ইমপ্লিমেন্টেশন নেই
- Flexible ও maintainable কোডের জন্য গুরুত্বপূর্ণ
