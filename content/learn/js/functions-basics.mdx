জাভাস্ক্রিপ্টে ফাংশন হলো কোডের "রেসিপি"। ফাংশন দিয়ে কোড পুনরায় ব্যবহার, অর্গানাইজ, এবং ডাটা প্রসেস করা যায়। এই পাঠে আমরা দেখব ফাংশন ডিক্লারেশন, এক্সপ্রেশন, এবং ডিফল্ট প্যারামিটার।

## ১. ফাংশন ডিক্লারেশন

- সরাসরি ফাংশন তৈরি
- Hoisting হয় (উপরে উঠে যায়)

```js
function greet(name) {
  return "Hello, " + name;
}
console.log(greet("Anis")); // "Hello, Anis"
```

## ২. ফাংশন এক্সপ্রেশন

- ভ্যারিয়েবলে ফাংশন সংরক্ষণ
- Hoisting হয় না

```js
const greet = function(name) {
  return "Hi, " + name;
};
console.log(greet("Rahim")); // "Hi, Rahim"
```

### Arrow Function (ES6)

- সংক্ষিপ্ত সিনট্যাক্স

```js
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5
```

## ৩. ডিফল্ট প্যারামিটার

- ফাংশনে প্যারামিটার না দিলে ডিফল্ট ভ্যালু

```js
function multiply(a, b = 2) {
  return a * b;
}
console.log(multiply(5)); // 10
console.log(multiply(5, 3)); // 15
```

## ৪. ফাংশন Return ও Arguments

- `return` দিয়ে ফাংশন থেকে ভ্যালু ফেরত
- `arguments` object (ডিক্লারেশন/এক্সপ্রেশন)

```js
function sum() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}
console.log(sum(1, 2, 3)); // 6
```

## ৫. বাস্তব উদাহরণ ও Analogies

- **ফাংশন = রান্নার রেসিপি**: একবার লিখে বারবার ব্যবহার
- **ডিক্লারেশন = রেসিপি বই**: আগে থেকেই আছে, যেকোনো সময় ব্যবহার
- **এক্সপ্রেশন = কাগজে লিখে রাখা রেসিপি**: আগে লিখতে হয়, পরে ব্যবহার
- **ডিফল্ট প্যারামিটার = "লবণ স্বাদমতো"**: না দিলে ডিফল্ট

## ৬. সাধারণ ভুল ও সমাধান

- **Hoisting বিভ্রান্তি**: ডিক্লারেশন আগে ব্যবহার করা যায়, এক্সপ্রেশন যায় না
- **Arrow function-এ arguments object নেই**
- **ডিফল্ট প্যারামিটার ভুলভাবে ব্যবহার**

## ৭. চ্যালেঞ্জ

```js
// নাম না দিলে "Guest" দেখাবে
function welcome(name = "Guest") {
  return "Welcome, " + name;
}
console.log(welcome()); // "Welcome, Guest"
```

## উপসংহার

ফাংশন ডিক্লারেশন, এক্সপ্রেশন, এবং ডিফল্ট প্যারামিটার আয়ত্ত করলে, জাভাস্ক্রিপ্টে কোডিং হবে আরও শক্তিশালী ও পুনরায় ব্যবহারযোগ্য।
