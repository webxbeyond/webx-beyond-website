## ১. Object.assign: অবজেক্ট কপি ও মার্জ

`Object.assign` মূলত এক বা একাধিক অবজেক্টকে মার্জ বা কপি করার জন্য ব্যবহৃত হয়।

### উদাহরণ:
```js
const user = { name: "Anis", age: 25 };
const extra = { country: "Bangladesh" };
const merged = Object.assign({}, user, extra);
console.log(merged); // { name: "Anis", age: 25, country: "Bangladesh" }
```

#### রিয়েল লাইফ অ্যানালজি:
ধরা যাক, আপনি একটি ফর্ম ফিলআপ করছেন। আপনার বেসিক ইনফো আছে, পরে নতুন ইনফো যোগ করতে হচ্ছে। `Object.assign` ঠিক সেই কাজটাই করে—পুরনো ডাটা রেখে নতুন ডাটা যোগ করে।

#### সতর্কতা:
- এটি "shallow copy" করে, অর্থাৎ nested অবজেক্ট কপি হয় না, reference থাকে।
- প্রথম প্যারামিটার (target) পরিবর্তিত হয়, তাই নতুন অবজেক্ট তৈরিতে `{}` ব্যবহার করা ভালো।

## ২. ডেস্ট্রাকচারিং: অবজেক্ট থেকে ডাটা এক্সট্রাক্ট

ডেস্ট্রাকচারিং হলো অবজেক্ট বা অ্যারে থেকে নির্দিষ্ট প্রপার্টি/ভ্যালু সহজে বের করে আনা।

### উদাহরণ:
```js
const person = { name: "Rafi", age: 30, job: "Engineer" };
const { name, job } = person;
console.log(name); // "Rafi"
console.log(job); // "Engineer"
```

#### অ্যানালজি:
এটা অনেকটা কাগজের ফাইল থেকে নির্দিষ্ট পৃষ্ঠা বের করে নেওয়ার মতো। পুরো ফাইল না নিয়ে, শুধু দরকারি অংশটা নিয়ে কাজ করা যায়।

#### Nested Destructuring:
```js
const user = { info: { name: "Sara", city: "Dhaka" }, age: 22 };
const { info: { city } } = user;
console.log(city); // "Dhaka"
```

## ৩. স্প্রেড ও রেস্ট অপারেটর (...)

### স্প্রেড:
স্প্রেড অপারেটর (`...`) দিয়ে অবজেক্ট বা অ্যারে "unpack" করা যায়—মানে, তার ভেতরের ডাটা আলাদা করে নেওয়া যায়।

#### উদাহরণ:
```js
const base = { a: 1, b: 2 };
const extended = { ...base, c: 3 };
console.log(extended); // { a: 1, b: 2, c: 3 }
```

#### অ্যানালজি:
এটা অনেকটা ব্যাগ খুলে সব জিনিস টেবিলে ছড়িয়ে দেওয়া—তারপর নতুন কিছু যোগ করা।

### রেস্ট:
রেস্ট অপারেটর (`...`) দিয়ে অবজেক্ট বা অ্যারে থেকে বাকি ডাটা "collect" করা যায়।

#### উদাহরণ:
```js
const { a, ...others } = { a: 1, b: 2, c: 3 };
console.log(a); // 1
console.log(others); // { b: 2, c: 3 }
```

#### অ্যানালজি:
এটা অনেকটা একটা বাক্স থেকে কিছু জিনিস বের করে নেওয়া, বাকিগুলো আলাদা বাক্সে রেখে দেওয়া।

## ৪. ব্যবহারিক টিপস ও চ্যালেঞ্জ

- ডেস্ট্রাকচারিং ও স্প্রেড/রেস্ট অপারেটর কোডকে সংক্ষিপ্ত, পরিষ্কার ও মেইনটেইনেবল করে তোলে।
- Nested অবজেক্টে কাজ করার সময় সতর্ক থাকুন—shallow copy vs deep copy পার্থক্য বুঝুন।
- `Object.assign` vs স্প্রেড: স্প্রেড অপারেটর ES6+, তাই পুরনো ব্রাউজারে সমস্যা হতে পারে।

### চ্যালেঞ্জ:
একটি অবজেক্টে user info আছে—name, age, address, email।
- শুধু name ও email বের করুন (ডেস্ট্রাকচারিং)।
- address বাদে বাকি ডাটা নতুন অবজেক্টে রাখুন (রেস্ট)।
- নতুন অবজেক্টে country: "Bangladesh" যোগ করুন (স্প্রেড)।
