জাভাস্ক্রিপ্টে স্কোপ মানে ভ্যারিয়েবল কোথায় "দেখা" যাবে, আর ক্লোজার মানে ফাংশন তার বাইরের ভ্যারিয়েবল "মনে রাখে"। এই পাঠে আমরা দেখব স্কোপের ধরন, ক্লোজার কী, এবং বাস্তব উদাহরণ।

## ১. স্কোপের ধরন

### গ্লোবাল স্কোপ
- স্ক্রিপ্টের বাইরে ডিক্লেয়ার করা ভ্যারিয়েবল
- সব জায়গা থেকে অ্যাক্সেস করা যায়

```js
var globalVar = "I am global";
function show() {
  console.log(globalVar); // "I am global"
}
```

### ফাংশন স্কোপ
- ফাংশনের ভিতরে ডিক্লেয়ার করা ভ্যারিয়েবল
- শুধু ফাংশনের ভিতরে দেখা যায়

```js
function test() {
  var localVar = "I am local";
  console.log(localVar); // "I am local"
}
// console.log(localVar); // Error
```

### ব্লক স্কোপ
- `{}` এর ভিতরে ডিক্লেয়ার করা (let, const)
- শুধু ব্লকের ভিতরে দেখা যায়

```js
if (true) {
  let blockVar = "I am block scoped";
  console.log(blockVar); // "I am block scoped"
}
// console.log(blockVar); // Error
```

## ২. ক্লোজার (Closure)

- ফাংশন তার বাইরের ভ্যারিয়েবল "মনে রাখে"
- Lexical scope

```js
function outer() {
  let name = "Anis";
  function inner() {
    console.log(name); // "Anis"
  }
  return inner;
}
const greet = outer();
greet(); // "Anis"
```

### বাস্তব উদাহরণ:

```js
function counter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}
const inc = counter();
console.log(inc()); // 1
console.log(inc()); // 2
```

## ৩. Analogies

- **স্কোপ = "ঘরের boundary"**: কোন ঘরে কোন জিনিস দেখা যাবে
- **ক্লোজার = "ঘরের বাইরে থেকে চাবি রেখে আসা"**: দরকার হলে পরে ব্যবহার

## ৪. সাধারণ ভুল ও সমাধান

- **var দিয়ে ব্লক স্কোপ আশা করা**: var ব্লক স্কোপড নয়
- **ক্লোজার ভুলভাবে ব্যবহার**: ভ্যারিয়েবল "লিক" হতে পারে

## ৫. চ্যালেঞ্জ

```js
function makeAdder(x) {
  return function(y) {
    return x + y;
  };
}
const add5 = makeAdder(5);
console.log(add5(10)); // 15
```

## উপসংহার

স্কোপ ও ক্লোজার আয়ত্ত করলে, জাভাস্ক্রিপ্টে ডাটা নিরাপদ, কোড মডুলার, এবং ফাংশন শক্তিশালী হয়।
