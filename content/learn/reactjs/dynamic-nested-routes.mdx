## ভূমিকা

জটিল ওয়েব অ্যাপ্লিকেশনে ডাইনামিক এবং নেস্টেড রাউটিং অত্যন্ত গুরুত্বপূর্ণ। আসুন একটি সহজ উপমার মাধ্যমে এটি বুঝি।

### লাইব্রেরির উপমা
কল্পনা করুন একটি বড় লাইব্রেরি:
- **মূল হল** = রুট রাউট (`/`)
- **বিভাগ** = নেস্টেড রাউট (`/science`, `/literature`)
- **বইয়ের তাক** = সাব-রাউট (`/science/physics`, `/science/chemistry`)
- **বইয়ের আইডি** = ডাইনামিক রাউট (`/books/123`, `/books/456`)

## ডাইনামিক রাউটিং

### ১. URL প্যারামিটার
```jsx
import { useParams } from 'react-router-dom';

function App() {
  return (
    <Routes>
      <Route path="/products/:category" element={<ProductList />} />
      <Route path="/product/:id" element={<ProductDetails />} />
    </Routes>
  );
}

function ProductDetails() {
  const { id } = useParams();
  const [product, setProduct] = useState(null);

  useEffect(() => {
    // প্রোডাক্ট ডেটা লোড
    fetchProduct(id).then(setProduct);
  }, [id]);

  if (!product) return <div>লোড হচ্ছে...</div>;

  return (
    <div>
      <h1>{product.name}</h1>
      <p>মূল্য: ৳{product.price}</p>
    </div>
  );
}
```

### ২. অপশনাল প্যারামিটার

## উন্নত প্যাটার্ন

### ১. ডাইনামিক নেস্টিং
```jsx
const routes = {
  admin: {
    path: 'admin',
    children: {
      users: {
        path: 'users',
        children: {
          user: {
            path: ':userId',
            children: {
              edit: { path: 'edit' },
              delete: { path: 'delete' }
            }
          }
        }
      }
    }
  }
};

function generateRoutes(routeObj, parentPath = '') {
  return Object.entries(routeObj).map(([key, config]) => {
    const path = `${parentPath}/${config.path}`;
    
    return (
      <Route key={path} path={path} element={config.element}>
        {config.children && generateRoutes(config.children, path)}
      </Route>
    );
  });
}
```

### ২. রাউট ম্যাচিং
```jsx
function RouteWithSubRoutes(route) {
  return (
    <Route
      path={route.path}
      element={
        route.protected ? (
          <ProtectedRoute>
            <route.element />
          </ProtectedRoute>
        ) : (
          <route.element />
        )
      }
    >
      {route.routes?.map((subRoute, i) => (
        <RouteWithSubRoutes key={i} {...subRoute} />
      ))}
    </Route>
  );
}
```

## প্র্যাকটিক্যাল উদাহরণ

### ১. ই-কমার্স রাউটিং সিস্টেম
```jsx
function EcommerceRoutes() {
  return (
    <Routes>
      <Route path="/" element={<Layout />}>
        {/* প্রোডাক্ট রাউটস */}
        <Route path="products" element={<ProductLayout />}>
          <Route index element={<ProductList />} />
          <Route path=":category" element={<CategoryProducts />} />
          <Route path=":category/:id" element={<ProductDetails />} />
        </Route>

        {/* অর্ডার রাউটস */}
        <Route path="orders" element={<ProtectedRoute />}>
          <Route index element={<OrderList />} />
          <Route path=":orderId" element={<OrderDetails />} />
          <Route path=":orderId/track" element={<TrackOrder />} />
        </Route>

        {/* ইউজার রাউটস */}
        <Route path="account" element={<ProtectedRoute />}>
          <Route index element={<AccountOverview />} />
          <Route path="profile" element={<Profile />} />
          <Route path="addresses" element={<AddressList />} />
          <Route path="addresses/new" element={<AddAddress />} />
          <Route path="addresses/:id/edit" element={<EditAddress />} />
        </Route>
      </Route>
    </Routes>
  );
}
```

## বেস্ট প্র্যাকটিস

১. **রাউট অর্গানাইজেশন**
- লজিক্যাল গ্রুপিং করুন
- রাউট কনস্ট্যান্ট ব্যবহার করুন
- কোড স্প্লিটিং করুন

২. **পারফরম্যান্স**
- লেজি লোডিং ব্যবহার করুন
- রাউট প্রি-ফেচিং
- অপ্টিমাইজড কম্পোনেন্ট লোডিং

৩. **মেইনটেইনেবিলিটি**
- মডুলার রাউট ফাইল
- ডকুমেন্টেশন
- টাইপ চেকিং

## সতর্কতা

১. **রাউট প্যারামিটার**
- ভ্যালিড প্যারামিটার চেক করুন
- XSS প্রতিরোধ করুন
- এরর হ্যান্ডলিং করুন

২. **নেস্টিং ডেপথ**
- অতিরিক্ত নেস্টিং এড়িয়ে চলুন
- পারফরম্যান্স বিবেচনা করুন
- ইউজার এক্সপেরিয়েন্স মাথায় রাখুন

## উপসংহার

ডাইনামিক এবং নেস্টেড রাউটিং আপনার অ্যাপ্লিকেশনকে আরও শক্তিশালী ও সুসংগঠিত করে। সঠিক প্যাটার্ন অনুসরণ করে আপনি জটিল রাউটিং স্ট্রাকচারও সহজে পরিচালনা করতে পারবেন। পরবর্তী পাঠে আমরা শিখব কীভাবে লিঙ্ক এবং ন্যাভিগেশন ব্যবহার করতে হয়।
