## `.gitignore` কার্যকরভাবে ব্যবহার

`.gitignore` ফাইল গিটে এমন ফাইল/ফোল্ডার নির্দিষ্ট করতে ব্যবহৃত হয়, যেগুলো ট্র্যাক বা কমিট করা উচিত নয় (যেমন: লোগ, বিল্ট ফাইল, সিক্রেট, ডিপেন্ডেন্সি ইত্যাদি)।

### `.gitignore` ফাইল কোথায়?
- রিপোজিটরির রুটে `.gitignore` নামে ফাইল রাখুন
- চাইলে সাব-ডিরেক্টরিতেও `.gitignore` রাখা যায়

### সাধারণ `.gitignore` এন্ট্রি
```
# বিল্ট ফাইল
/dist
/build

# নোড মডিউল
node_modules/

# লোগ ফাইল
*.log

# অপারেটিং সিস্টেম ফাইল
.DS_Store
Thumbs.db

# এনভ ফাইল
.env
```

### নির্দিষ্ট ফাইল/ফোল্ডার বাদ দেওয়া
- `folder/` : পুরো ফোল্ডার বাদ
- `*.ext` : নির্দিষ্ট এক্সটেনশনের সব ফাইল বাদ
- `!important.js` : আগের নিয়মে বাদ দেওয়া হলেও এই ফাইল আবার ইনক্লুড

### নতুন `.gitignore` যোগ করলে
- আগে ট্র্যাক হওয়া ফাইল বাদ দিতে:
  ```bash
  git rm --cached <file>
  ```

### টেমপ্লেট
- [gitignore.io](https://www.toptal.com/developers/gitignore) থেকে বিভিন্ন প্ল্যাটফর্মের জন্য টেমপ্লেট জেনারেট করুন

### টিপস
- সিক্রেট, কনফিগ, বিল্ট ফাইল কখনোই গিটে রাখবেন না
- টিমে `.gitignore` শেয়ার করুন

### উপসংহার
`.gitignore` সঠিকভাবে ব্যবহার করলে রিপোজিটরি পরিষ্কার, নিরাপদ ও পোর্টেবল থাকে।
