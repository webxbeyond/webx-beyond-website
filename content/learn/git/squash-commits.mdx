## ক্লিন হিস্টরির জন্য কমিট স্কোয়াশ

কমিট স্কোয়াশ (Squash) হল একাধিক কমিটকে একত্রে একটি কমিটে পরিণত করার প্রক্রিয়া। এটি সাধারণত তখনই করা হয় যখন আপনি অনেক ছোট ছোট কমিট করেছেন, কিন্তু মূল ব্রাঞ্চে মার্জ করার সময় একটি পরিষ্কার ও অর্থবহ কমিট হিস্টরি রাখতে চান।

### কেন স্কোয়াশ করবেন?
- অনেক ছোট ছোট ফিক্স/টেস্ট কমিটকে একত্রে করে মূল কাজের সারাংশ রাখতে
- ইতিহাস পরিষ্কার ও সহজবোধ্য রাখতে
- কোড রিভিউ সহজ করতে

### স্কোয়াশ করার উপায়

#### ১. ইন্টার‌্যাকটিভ রিবেস দিয়ে স্কোয়াশ

```bash
git rebase -i HEAD~N
```
এখানে `N` হল কতটি কমিট স্কোয়াশ করতে চান তার সংখ্যা।

- কমান্ড দিলে একটি টেক্সট এডিটর খুলবে।
- প্রথম কমিটের পাশে `pick` রেখে, পরবর্তী কমিটগুলোর পাশে `squash` বা সংক্ষেপে `s` লিখুন।
- ফাইল সেভ ও ক্লোজ করুন।
- এরপর কমিট মেসেজ এডিট করতে বলবে।

#### উদাহরণ:
ধরা যাক, শেষ ৩টি কমিট স্কোয়াশ করতে চান:

```bash
git rebase -i HEAD~3
```

এডিটরে:
```
pick 1234abcd ফিচার যোগ
pick 5678efgh টাইপো ফিক্স
pick 9abcdeff টেস্ট ফিক্স
```
এখন:
```
pick 1234abcd ফিচার যোগ
squash 5678efgh টাইপো ফিক্স
squash 9abcdeff টেস্ট ফিক্স
```

#### ২. মার্জ করার সময় স্কোয়াশ

ফিচার ব্রাঞ্চ মার্জ করার সময় স্কোয়াশ করতে পারেন:

```bash
git merge --squash feature-branch
```

এতে ফিচার ব্রাঞ্চের সব পরিবর্তন এক কমিটে মার্জ হবে।

### সতর্কতা
- স্কোয়াশ করলে কমিট আইডি বদলে যায়, তাই শেয়ার করা ব্রাঞ্চে স্কোয়াশ করার আগে সতর্ক থাকুন।
- স্কোয়াশ করা কমিট পুশ করার আগে টিমের সাথে আলোচনা করুন।

### উপসংহার
কমিট স্কোয়াশ গিট হিস্টরি পরিষ্কার ও পেশাদার রাখতে সাহায্য করে। তবে কখন স্কোয়াশ করবেন এবং কখন করবেন না, তা বুঝে ব্যবহার করুন।
