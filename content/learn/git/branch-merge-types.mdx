# ব্রাঞ্চ মার্জিং: ফাস্ট-ফরওয়ার্ড বনাম থ্রি-ওয়ে মার্জ

গিটে দুইটি ব্রাঞ্চ একত্রে মিশিয়ে (merge) কোডের ইতিহাস একীভূত করা যায়। মার্জিংয়ের প্রধান দুই ধরনের পদ্ধতি হলো: ফাস্ট-ফরওয়ার্ড এবং থ্রি-ওয়ে মার্জ।

## ফাস্ট-ফরওয়ার্ড মার্জ

- যখন টার্গেট ব্রাঞ্চে কোনো নতুন কমিট নেই, শুধু সোর্স ব্রাঞ্চে নতুন কমিট আছে, তখন ফাস্ট-ফরওয়ার্ড মার্জ হয়।
- এতে মূলত টার্গেট ব্রাঞ্চ সরাসরি সোর্স ব্রাঞ্চের শেষ কমিটে চলে যায়।

### উদাহরণ:
```sh
git checkout main
git merge feature/login
```

- যদি `main`-এ কোনো নতুন কমিট না থাকে, তাহলে ফাস্ট-ফরওয়ার্ড মার্জ হবে।

## থ্রি-ওয়ে মার্জ

- যখন দুই ব্রাঞ্চেই নতুন কমিট আছে, তখন গিট একটি নতুন মার্জ কমিট তৈরি করে, যাকে বলে থ্রি-ওয়ে মার্জ।
- এতে তিনটি কমিটের মধ্যে পার্থক্য দেখে মার্জ হয়: common ancestor, বর্তমান ব্রাঞ্চ, এবং মার্জ করতে চাওয়া ব্রাঞ্চ।

### উদাহরণ:
```sh
git checkout main
git merge feature/login
```

- যদি `main` ও `feature/login` দুইটিতেই নতুন কমিট থাকে, তাহলে থ্রি-ওয়ে মার্জ হবে এবং একটি নতুন মার্জ কমিট তৈরি হবে।

## সংক্ষেপে
- **ফাস্ট-ফরওয়ার্ড:** শুধু সোর্স ব্রাঞ্চে নতুন কমিট থাকলে, সরাসরি মার্জ
- **থ্রি-ওয়ে মার্জ:** দুই ব্রাঞ্চেই নতুন কমিট থাকলে, নতুন মার্জ কমিট তৈরি

মার্জিংয়ের ধরন বুঝলে কনফ্লিক্ট ও ইতিহাস পরিষ্কার রাখা সহজ হবে।
