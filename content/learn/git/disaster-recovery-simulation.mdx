ডিজাস্টার রিকভারি সিমুলেশন মানে ইচ্ছাকৃতভাবে গিট রিপোজিটরিতে সমস্যা তৈরি করে (যেমন কমিট হারানো, ব্রাঞ্চ ডিলিট, কনফ্লিক্ট) সেগুলো সমাধান অনুশীলন করা। এতে টিমের প্রস্তুতি ও দক্ষতা বাড়ে।

## ১. কেন দরকার?
- রিয়েল লাইফে ভুল বা দুর্ঘটনা ঘটতেই পারে
- আগেভাগে অনুশীলন করলে ভয় কমে, সমাধান জানা থাকে

## ২. কমন সিমুলেশন আইডিয়া
- ভুল করে ব্রাঞ্চ ডিলিট
- কমিট হারানো (reset, rebase, amend)
- মার্জ/রিবেস কনফ্লিক্ট
- রিপো করাপ্ট/ব্রোকেন

## ৩. উদাহরণ: হারানো কমিট উদ্ধার
```sh
git commit --amend # ভুলে আগের কমিট ওভাররাইট
# এখন আগের কমিট হারিয়ে গেছে মনে হচ্ছে

git reflog
# হারানো কমিটের রেফারেন্স খুঁজে বের করুন

git checkout <commit-hash>
```

## ৪. উদাহরণ: ভুলে ব্রাঞ্চ ডিলিট
```sh
git branch -d feature-x # ভুলে ডিলিট

git reflog
# ডিলিটেড ব্রাঞ্চের লাস্ট কমিট খুঁজে বের করুন

git checkout -b feature-x <commit-hash>
```

## ৫. মার্জ কনফ্লিক্ট সিমুলেশন
- দুইজন একই ফাইলে আলাদা চেঞ্জ করে push/pull করুন
- মার্জ কনফ্লিক্ট হলে সমাধান অনুশীলন করুন

## ৬. টিমে সিমুলেশন আয়োজন
- Dummy রিপো তৈরি করুন
- টাস্ক/সিনারিও লিখে দিন
- সবাইকে সমাধান করতে বলুন
- শেষে আলোচনা ও বেস্ট প্র্যাকটিস শেয়ার করুন

## ৭. টিপস
- সিমুলেশন শেষে রিপো ক্লিন করুন
- রিয়েল প্রজেক্টে না করে ডেমো রিপো ব্যবহার করুন
- সমাধান ডকুমেন্ট করুন
