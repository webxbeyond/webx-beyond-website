# গার্বেজ কালেকশন ও রিপো ক্লিনআপ (`git gc`)

গিট রিপোজিটরিতে অপ্রয়োজনীয় অবজেক্ট, পুরনো রিফলগ, ডিলিটেড ব্রাঞ্চ ইত্যাদি জমা হতে থাকে। এগুলো ক্লিনআপ করতে গিটের গার্বেজ কালেকশন (Garbage Collection) ব্যবহৃত হয়।

## গার্বেজ কালেকশন কী?
- অপ্রয়োজনীয় অবজেক্ট, ড্যাংলিং কমিট, পুরনো রিফলগ, টেম্প ফাইল—সবকিছু মুছে ফেলে
- রিপো ছোট ও দ্রুত হয়
- স্বয়ংক্রিয়ভাবে মাঝে মাঝে চলে, তবে ম্যানুয়ালি চালানো যায়

## `git gc` কমান্ড
```sh
git gc
```
- সব অবজেক্ট প্যাক করে, অপ্রয়োজনীয় ডিলিট করে
- `.git/objects/pack/`-এ অবজেক্ট প্যাক হয়

## আরও কিছু কমান্ড
- শুধু লুজ অবজেক্ট ক্লিনআপ: `git prune`
- রিফলগ ক্লিনআপ: `git reflog expire --expire=30.days.ago --all`

## কখন চালাবেন?
- অনেক ব্রাঞ্চ/কমিট ডিলিট করলে
- রিপো সাইজ অস্বাভাবিক বড় হলে
- স্পেস বাঁচাতে

## সতর্কতা
- গার্বেজ কালেকশন চালালে হারানো কমিট, পুরনো রিফলগ, ড্যাংলিং অবজেক্ট মুছে যেতে পারে
- নিশ্চিত হয়ে চালান

## সংক্ষেপে
- `git gc` = রিপো ক্লিনআপ ও অপ্টিমাইজ
- স্পেস বাঁচাতে ও পারফরম্যান্স বাড়াতে
- সাবধানে ব্যবহার করুন

এগুলো জানলে গিট রিপো মেইনটেন্যান্স আরও সহজ হবে।
