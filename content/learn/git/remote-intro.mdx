# রিমোট রিপোজিটরি পরিচিতি: `origin`, `upstream`

রিমোট রিপোজিটরি হলো অনলাইনে (যেমন: GitHub, GitLab) রাখা গিট রিপোজিটরির কপি, যেখানে টিমের সবাই একসাথে কাজ করতে পারে।

## `origin` কী?
- `origin` হলো ডিফল্ট রিমোট রিপোজিটরির নাম, যেখান থেকে আপনি ক্লোন করেছেন বা যেটিতে কোড পুশ করেন।
- সাধারণত আপনার নিজের রিপোজিটরি বা মূল কাজের জায়গা।

## `upstream` কী?
- `upstream` হলো মূল সোর্স রিপোজিটরি, যেটি থেকে আপনি ফর্ক করেছেন বা যেটি থেকে আপডেট নিতে চান।
- ওপেন সোর্স কন্ট্রিবিউশনে `upstream` খুব গুরুত্বপূর্ণ।

## কিভাবে চেক করবেন?
```sh
git remote -v
```
এতে সব রিমোটের নাম ও URL দেখাবে।

## কিভাবে `upstream` যোগ করবেন?
```sh
git remote add upstream https://github.com/owner/repo.git
```

## সংক্ষেপে
- `origin`: আপনার কাজের রিমোট রিপো
- `upstream`: মূল সোর্স বা ফর্কের মূল রিপো
- `git remote -v` দিয়ে সব রিমোট দেখুন

রিমোট কনসেপ্ট বুঝলে টিমওয়ার্ক ও ওপেন সোর্স কন্ট্রিবিউশন সহজ হয়।
