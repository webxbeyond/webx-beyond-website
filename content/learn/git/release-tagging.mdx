# রিলিজ ট্যাগিং: লাইটওয়েট বনাম অ্যানোটেটেড ট্যাগ

প্রজেক্টের নির্দিষ্ট ভার্সন বা রিলিজ চিহ্নিত করতে গিটে ট্যাগ ব্যবহার করা হয়। ট্যাগ দুই ধরনের: লাইটওয়েট ও অ্যানোটেটেড।

## লাইটওয়েট ট্যাগ
- শুধু নির্দিষ্ট কমিটে একটি নাম (রেফারেন্স) তৈরি করে
- কোনো অতিরিক্ত তথ্য, মেসেজ বা সিগনেচার থাকে না
- দ্রুত ও সাধারণ ট্যাগিংয়ের জন্য

```sh
git tag v1.0.0
```

## অ্যানোটেটেড ট্যাগ
- ট্যাগের সাথে নাম, ইমেইল, তারিখ, মেসেজ ও GPG সিগনেচার যুক্ত হয়
- অফিসিয়াল রিলিজ বা গুরুত্বপূর্ণ ভার্সনের জন্য

```sh
git tag -a v1.0.0 -m "প্রথম রিলিজ"
```

## ট্যাগ লিস্ট দেখা
```sh
git tag
```

## ট্যাগ পুশ করা
- ডিফল্টভাবে ট্যাগ পুশ হয় না, আলাদাভাবে করতে হয়:
```sh
git push origin v1.0.0
```
- সব ট্যাগ পুশ করতে:
```sh
git push origin --tags
```

## ট্যাগ ডিলিট করা
```sh
git tag -d v1.0.0
# রিমোট থেকেও ডিলিট:
git push origin --delete v1.0.0
```

## কখন কোনটা?
- **লাইটওয়েট:** দ্রুত, অস্থায়ী বা ব্যক্তিগত ট্যাগ
- **অ্যানোটেটেড:** অফিসিয়াল রিলিজ, পাবলিক ভার্সন, ইতিহাস সংরক্ষণ

## সংক্ষেপে
- `git tag` = লাইটওয়েট
- `git tag -a` = অ্যানোটেটেড
- ট্যাগ পুশ করতে `git push origin <tag>`

ট্যাগ ব্যবহার করলে রিলিজ ম্যানেজমেন্ট ও ভার্সনিং সহজ হয়।
