# ধাপে ধাপে মার্জ কনফ্লিক্ট সমাধান

গিটে মার্জ করার সময় দুই ব্রাঞ্চে একই লাইনে পরিবর্তন থাকলে "মার্জ কনফ্লিক্ট" হয়। নিচে কনফ্লিক্ট সমাধানের ধাপগুলো দেওয়া হলো:

## ১. মার্জ শুরু করুন
```sh
git merge <branch-name>
```

## ২. কনফ্লিক্ট হলে গিট বার্তা দেখাবে
- `git status` দিয়ে কোন ফাইলে কনফ্লিক্ট হয়েছে দেখুন।

## ৩. কনফ্লিক্ট চিহ্নিত করুন
- কনফ্লিক্ট হওয়া ফাইলে এইরকম চিহ্ন দেখবেন:
  ```
  <<<<<<< HEAD
  ...আপনার ব্রাঞ্চের কোড...
  =======
  ...মার্জ করতে চাওয়া ব্রাঞ্চের কোড...
  >>>>>>> branch-name
  ```

## ৪. কনফ্লিক্ট ম্যানুয়ালি ঠিক করুন
- প্রয়োজনীয় কোড রেখে বাকি চিহ্ন মুছে দিন।

## ৫. ফাইল স্টেজ করুন
```sh
git add <conflicted-file>
```

## ৬. মার্জ কমিট সম্পন্ন করুন
```sh
git commit
```

## ৭. সবকিছু ঠিক আছে কিনা যাচাই করুন
```sh
git status
```

## সংক্ষেপে
- মার্জ কনফ্লিক্ট হলে ফাইল খুলে চিহ্নিত অংশ ঠিক করুন
- ফাইল স্টেজ করে মার্জ কমিট দিন

এভাবে ধাপে ধাপে গিট মার্জ কনফ্লিক্ট সমাধান করতে পারবেন।
