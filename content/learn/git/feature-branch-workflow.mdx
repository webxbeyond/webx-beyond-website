## ফিচার ব্রাঞ্চ ওয়ার্কফ্লো

ফিচার ব্রাঞ্চ ওয়ার্কফ্লো (Feature Branch Workflow) হল গিটে কাজ করার একটি জনপ্রিয় ও পেশাদার প্যাটার্ন, যেখানে প্রতিটি নতুন ফিচার বা বাগফিক্সের জন্য আলাদা ব্রাঞ্চ তৈরি করা হয়।

### কিভাবে কাজ করে?
- মূল ব্রাঞ্চ (সাধারণত `main` বা `master`) সবসময় প্রোডাকশন-রেডি থাকে।
- নতুন ফিচার বা ফিক্সের জন্য আলাদা ব্রাঞ্চ তৈরি হয় (যেমন `feature/login`, `bugfix/navbar`)।
- কাজ শেষে ফিচার ব্রাঞ্চ মার্জ/পুল রিকোয়েস্টের মাধ্যমে মূল ব্রাঞ্চে যোগ হয়।

### ধাপসমূহ
1. **নতুন ফিচার ব্রাঞ্চ তৈরি করুন:**
   ```bash
   git checkout -b feature/feature-name
   ```
2. **কাজ করুন ও কমিট করুন:**
   ```bash
   git add .
   git commit -m "ফিচার যোগ: ..."
   ```
3. **রিমোটে পুশ করুন:**
   ```bash
   git push origin feature/feature-name
   ```
4. **পুল রিকোয়েস্ট/মার্জ রিকোয়েস্ট তৈরি করুন:**
   - GitHub/GitLab-এ ফিচার ব্রাঞ্চ থেকে `main`-এ মার্জের জন্য PR/MR খুলুন।
5. **কোড রিভিউ ও মার্জ:**
   - রিভিউয়ের পর ফিচার ব্রাঞ্চ `main`-এ মার্জ করুন।

### সুবিধা
- মূল ব্রাঞ্চ সবসময় স্থিতিশীল থাকে
- একাধিক ফিচার/বাগফিক্স একসাথে আলাদাভাবে ডেভেলপ করা যায়
- কোড রিভিউ সহজ হয়

### সীমাবদ্ধতা
- অনেক ব্রাঞ্চ থাকলে ম্যানেজমেন্ট একটু জটিল হতে পারে
- ফিচার ব্রাঞ্চ পুরনো হলে মার্জ কনফ্লিক্ট হতে পারে

### টিপস
- নিয়মিত `main` থেকে ফিচার ব্রাঞ্চে রিবেস/মার্জ করুন
- ফিচার শেষ হলে দ্রুত মার্জ করুন

### উপসংহার
ফিচার ব্রাঞ্চ ওয়ার্কফ্লো টিমে সমন্বিত ও নিরাপদভাবে কাজ করার জন্য অত্যন্ত কার্যকর। এটি স্কেলেবল ও প্রফেশনাল গিট ব্যবহারের অন্যতম সেরা প্যাটার্ন।
