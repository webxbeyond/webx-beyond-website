## গিট হুক লেখা ও ব্যবহার

গিট হুক (Git Hooks) হল স্ক্রিপ্ট, যা গিটের নির্দিষ্ট ইভেন্টে (যেমন commit, push, merge) স্বয়ংক্রিয়ভাবে চলে। এতে আপনি কাস্টম টাস্ক, চেক বা অটোমেশন যোগ করতে পারেন।

### গিট হুক কোথায় থাকে?
প্রতিটি রিপোজিটরির `.git/hooks/` ডিরেক্টরিতে ডিফল্ট কিছু স্যাম্পল হুক থাকে (যেমন `pre-commit.sample`)।

### হুক সক্রিয় করার নিয়ম
১. `.sample` এক্সটেনশন সরিয়ে দিন (যেমন `pre-commit.sample` → `pre-commit`)
২. স্ক্রিপ্ট ফাইল এক্সিকিউটেবল করুন (Linux/macOS: `chmod +x pre-commit`)

### জনপ্রিয় গিট হুক
- **pre-commit**: কমিটের আগে চলে
- **commit-msg**: কমিট মেসেজ চেক করে
- **pre-push**: পুশের আগে চলে
- **post-merge**: মার্জের পর চলে

### উদাহরণ: pre-commit হুক
`pre-commit` ফাইলে নিচের মতো স্ক্রিপ্ট লিখুন:

```bash
#!/bin/sh
# কোড ফরম্যাট চেক
npm run lint
if [ $? -ne 0 ]; then
  echo "লিন্টিং ফেইল! কমিট বাতিল।"
  exit 1
fi
```

### হুক ব্যবহারের সুবিধা
- কোড স্টাইল, টেস্ট, সিকিউরিটি চেক অটোমেট করা
- ভুল কমিট, খারাপ কোড পুশ প্রতিরোধ

### হুক শেয়ার করার টিপস
`.git/hooks/` ডিরেক্টরি গিটে ট্র্যাক হয় না। তাই শেয়ার করতে [Husky](https://typicode.github.io/husky) বা [lefthook](https://evilmartians.com/chronicles/lefthook) ব্যবহার করুন।

### উপসংহার
গিট হুক দিয়ে রিপোজিটরিতে স্বয়ংক্রিয় চেক ও অটোমেশন যোগ করা যায়, যা টিমওয়ার্ক ও কোড কোয়ালিটি নিশ্চিত করে।
