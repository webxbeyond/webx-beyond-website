## ওপেন সোর্সের জন্য ফর্কিং ওয়ার্কফ্লো

ফর্কিং ওয়ার্কফ্লো (Forking Workflow) ওপেন সোর্স প্রজেক্টে অবদান রাখার সবচেয়ে জনপ্রিয় ও নিরাপদ পদ্ধতি। এতে মূল রিপোজিটরির উপর সরাসরি লেখার অনুমতি না থাকলেও অবদান রাখা যায়।

### কিভাবে কাজ করে?
1. **মূল রিপোজিটরি ফর্ক করুন:**
   - GitHub/GitLab-এ মূল রিপোতে গিয়ে “Fork” বাটনে ক্লিক করুন।
2. **নিজের ফর্ক ক্লোন করুন:**
   ```bash
   git clone https://github.com/আপনার-ইউজারনেম/প্রজেক্ট.git
   ```
3. **নতুন ফিচার/ফিক্স ব্রাঞ্চ তৈরি করুন:**
   ```bash
   git checkout -b feature/feature-name
   ```
4. **কাজ করুন, কমিট করুন ও নিজের ফর্কে পুশ করুন:**
   ```bash
   git add .
   git commit -m "ফিচার/বাগফিক্স"
   git push origin feature/feature-name
   ```
5. **মূল রিপোজিটরিতে Pull Request (PR) পাঠান:**
   - GitHub/GitLab-এ গিয়ে “New Pull Request” খুলুন।
6. **কোড রিভিউ ও মার্জ:**
   - মেইনটেইনার রিভিউ করে PR মার্জ করবেন।

### আপস্ট্রিম রিপো আপডেট রাখা
মূল রিপোজিটরিকে `upstream` নামে যোগ করুন:
```bash
git remote add upstream https://github.com/মূল-অর্গ/প্রজেক্ট.git
```

আপডেট আনতে:
```bash
git fetch upstream
# আপনার ব্রাঞ্চে মার্জ/রিবেস করুন
```

### সুবিধা
- মূল কোডবেস নিরাপদ থাকে
- অবদানকারীরা স্বাধীনভাবে কাজ করতে পারে
- ওপেন সোর্স কমিউনিটিতে সহজ অবদান

### সীমাবদ্ধতা
- ফর্ক ও ব্রাঞ্চ ম্যানেজমেন্ট জানতে হয়
- নিয়মিত আপডেট না নিলে কনফ্লিক্ট হতে পারে

### উপসংহার
ফর্কিং ওয়ার্কফ্লো ওপেন সোর্সে অবদান রাখার জন্য আদর্শ। এতে নিরাপদ ও স্কেলেবলভাবে হাজারো কন্ট্রিবিউটর একসাথে কাজ করতে পারে।
