# কাজের অগ্রগতি স্ট্যাশ করা (`git stash`)

কখনো কখনো কাজের মাঝখানে ব্রাঞ্চ পরিবর্তন বা অন্য টাস্ক করতে হয়, কিন্তু পরিবর্তনগুলো কমিট করতে চান না। তখন `git stash` দিয়ে অস্থায়ীভাবে কাজের অগ্রগতি সংরক্ষণ করা যায়।

## `git stash` কী?
- স্টেজিং ও ওয়ার্কিং ডিরেক্টরির পরিবর্তনগুলো অস্থায়ীভাবে সরিয়ে রাখে
- রিপো পরিষ্কার হয়, অন্য ব্রাঞ্চে যেতে বা টাস্ক করতে সুবিধা হয়

## স্ট্যাশ করা
```sh
git stash
```
- সব পরিবর্তন স্ট্যাশে চলে যাবে
- রিপো আগের অবস্থায় ফিরে যাবে

## নির্দিষ্ট মেসেজসহ স্ট্যাশ
```sh
git stash save "feature-x কাজ"
```

## স্ট্যাশ লিস্ট দেখা
```sh
git stash list
```

## স্ট্যাশ প্রয়োগ করা
```sh
git stash apply
```
- সর্বশেষ স্ট্যাশ প্রয়োগ হবে
- নির্দিষ্ট স্ট্যাশ প্রয়োগ:
  ```sh
  git stash apply stash@{2}
  ```

## স্ট্যাশ ডিলিট করা
```sh
git stash drop stash@{0}
```
- সব স্ট্যাশ ডিলিট:
  ```sh
  git stash clear
  ```

## সংক্ষেপে
- `git stash` = অস্থায়ীভাবে কাজ সংরক্ষণ
- `git stash list` = স্ট্যাশ লিস্ট
- `git stash apply` = স্ট্যাশ প্রয়োগ
- `git stash drop/clear` = স্ট্যাশ ডিলিট

এভাবে স্ট্যাশ ব্যবহার করলে কাজের মাঝখানে ব্রাঞ্চ পরিবর্তন বা টাস্ক সুইচ করা সহজ হয়।
