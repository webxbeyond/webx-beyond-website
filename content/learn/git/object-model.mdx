# ব্লব, ট্রি, কমিট, ট্যাগ

গিটের ভিতরের কাঠামো বোঝার জন্য অবজেক্ট মডেল জানা জরুরি। গিট মূলত চার ধরনের অবজেক্ট ব্যবহার করে: ব্লব, ট্রি, কমিট, ও ট্যাগ।

## ১. ব্লব (Blob)
- ফাইলের কনটেন্ট সংরক্ষণ করে
- কোনো ফাইল যোগ বা পরিবর্তন করলে নতুন ব্লব তৈরি হয়
- কমপ্রেসড ও ইউনিক (SHA-1 হ্যাশ)

## ২. ট্রি (Tree)
- ডিরেক্টরি বা ফোল্ডার স্ট্রাকচার ধরে রাখে
- কোন ফাইলে কোন ব্লব আছে, কোন সাব-ডিরেক্টরি আছে—এসব তথ্য সংরক্ষণ করে
- একাধিক ট্রি একসাথে পুরো প্রজেক্টের স্ন্যাপশট তৈরি করে

## ৩. কমিট (Commit)
- একটি নির্দিষ্ট সময়ের পুরো রিপোজিটরির অবস্থা সংরক্ষণ করে
- কমিট অবজেক্টে থাকে: ট্রি রেফারেন্স, প্যারেন্ট কমিট, অথর, মেসেজ
- প্রতিটি কমিট ইউনিক (SHA-1 হ্যাশ)

## ৪. ট্যাগ (Tag)
- নির্দিষ্ট কমিট বা অবজেক্টকে চিহ্নিত করে
- অ্যানোটেটেড ট্যাগ হলে নাম, মেসেজ, অথর, তারিখ, সিগনেচার থাকে

## অবজেক্ট মডেল কিভাবে কাজ করে?
- ফাইল যোগ করলে ব্লব তৈরি হয়
- ফোল্ডার বা প্রজেক্টের জন্য ট্রি তৈরি হয়
- সবকিছু একসাথে কমিটে সংরক্ষিত হয়
- গুরুত্বপূর্ণ ভার্সনের জন্য ট্যাগ ব্যবহার হয়

## চিত্র (সাধারণ ধারণা):

```
কমিট
 |
ট্রি
 |------ ব্লব (file1)
 |------ ব্লব (file2)
 |------ ট্রি (subfolder)
             |-- ব্লব (file3)
```

## কমান্ড দিয়ে দেখা
- অবজেক্ট লিস্ট: `git cat-file -t <hash>`
- অবজেক্ট কনটেন্ট: `git cat-file -p <hash>`
- সব অবজেক্ট: `.git/objects` ফোল্ডারে

## সংক্ষেপে
- ব্লব = ফাইল কনটেন্ট
- ট্রি = ফোল্ডার স্ট্রাকচার
- কমিট = স্ন্যাপশট
- ট্যাগ = চিহ্নিতকরণ

এগুলো জানলে গিটের ভিতরের কাজ আরও পরিষ্কার হবে।
