গিট কমিট ও পুল রিকোয়েস্ট (PR) মেসেজ শুধু ইতিহাস নয়, টিম কমিউনিকেশনের গুরুত্বপূর্ণ মাধ্যম। স্পষ্ট, কনভেনশনাল ও তথ্যবহুল মেসেজ টিমের কাজ সহজ করে এবং ভুল কমায়।

## ১. কমিট মেসেজে কীভাবে টিম কমিউনিকেশন করবেন?
- **স্পষ্ট ও সংক্ষিপ্ত টাইটেল:**
  - কী পরিবর্তন হয়েছে, এক লাইনে লিখুন।
- **বিস্তারিত বডি (প্রয়োজনে):**
  - কেন পরিবর্তন দরকার, কী সমস্যা সমাধান, ইত্যাদি ব্যাখ্যা করুন।
- **কনভেনশনাল কমিট স্টাইল:**
  - `feat:`, `fix:`, `docs:`, `refactor:` ইত্যাদি prefix ব্যবহার করুন।
- **রেফারেন্স ও টিকিট নম্বর:**
  - সংশ্লিষ্ট ইস্যু/টাস্ক নম্বর দিন (e.g., `Fixes #42`)
- **উদাহরণ:**
```sh
git commit -m "fix: লগইন বাগ ফিক্স (#123)"
```

## ২. PR মেসেজে কী লিখবেন?
- **সংক্ষিপ্ত সারাংশ:**
  - PR কী পরিবর্তন এনেছে, এক-দুই লাইনে লিখুন।
- **ডিটেইলড বর্ণনা:**
  - ইমপ্লিমেন্টেশন, ব্রেকিং চেঞ্জ, টেস্ট কভারেজ ইত্যাদি ব্যাখ্যা করুন।
- **রিভিউয়ারের জন্য টাস্ক/নোট:**
  - কী ফোকাস করতে হবে, কোন ফাইল গুরুত্বপূর্ণ, ইত্যাদি লিখুন।
- **স্ক্রিনশট/লিঙ্ক:**
  - UI পরিবর্তন হলে স্ক্রিনশট দিন।
- **উদাহরণ:**
```markdown
### Summary
- লগইন বাগ ফিক্স
- নতুন ভ্যালিডেশন যোগ

### Issue
Fixes #123

### Test
- [x] লগইন টেস্ট পাস
```

## ৩. টিম কমিউনিকেশনের বেস্ট প্র্যাকটিস
- কমিট ও PR মেসেজে ইংরেজি বা টিমের সুবিধাজনক ভাষা ব্যবহার করুন
- টেমপ্লেট ব্যবহার করুন (PR টেমপ্লেট)
- নিয়মিত রিভিউ ও ফিডব্যাক দিন
- ছোট ও ফোকাসড কমিট/PR করুন

## ৪. টেমপ্লেট উদাহরণ
- `.github/PULL_REQUEST_TEMPLATE.md` ফাইল ব্যবহার করুন

```markdown
### Summary

### Issue

### Checklist
- [ ] টেস্ট পাস
- [ ] ডকুমেন্টেশন আপডেট
```
