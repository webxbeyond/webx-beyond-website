# ট্র্যাকিং ব্রাঞ্চ বোঝা

টিমে কাজ করার সময় লোকাল ও রিমোট রিপোজিটরির মধ্যে সমন্বয় রাখতে ট্র্যাকিং ব্রাঞ্চ গুরুত্বপূর্ণ।

## ট্র্যাকিং ব্রাঞ্চ কী?

- ট্র্যাকিং ব্রাঞ্চ হলো এমন একটি লোকাল ব্রাঞ্চ, যা সরাসরি কোনো নির্দিষ্ট রিমোট ব্রাঞ্চের সাথে যুক্ত থাকে।
- সাধারণত, `git clone` করলে `main` বা `master` ব্রাঞ্চ লোকাল ও রিমোট (origin/main) এর মধ্যে ট্র্যাকিং সেট হয়।

## কিভাবে চেনা যায়?

```sh
git branch -vv
```
এতে দেখাবে কোন লোকাল ব্রাঞ্চ কোন রিমোট ব্রাঞ্চকে ট্র্যাক করছে।

## নতুন ট্র্যাকিং ব্রাঞ্চ তৈরি

```sh
git checkout -b feature-x origin/feature-x
```
অথবা
```sh
git switch --track origin/feature-x
```

## লোকাল ব্রাঞ্চে ট্র্যাকিং সেট করা

```sh
git branch --set-upstream-to=origin/feature-x
```

## কেন দরকার?
- `git pull` বা `git push` কমান্ডে রিমোট ব্রাঞ্চের নাম না দিলেও কাজ হয়, কারণ ট্র্যাকিং ব্রাঞ্চ সেট করা থাকে।
- টিমে কাজের সময় কনফ্লিক্ট কম হয় এবং সিঙ্ক রাখা সহজ হয়।

## সংক্ষেপে
- ট্র্যাকিং ব্রাঞ্চ = লোকাল ও রিমোট ব্রাঞ্চের সংযোগ
- `git branch -vv` দিয়ে চেক করুন
- `git switch --track origin/branch` দিয়ে নতুন ট্র্যাকিং ব্রাঞ্চ নিন
- `git branch --set-upstream-to` দিয়ে পুরনো ব্রাঞ্চে ট্র্যাকিং যুক্ত করুন

এগুলো জানলে টিমে কাজ করা আরও সহজ হবে।
