# পরিবর্তন ট্র্যাকিং: `git status`, `git add`, `git commit`

গিটে কাজ করার সময় ফাইলের পরিবর্তন ট্র্যাক করা খুবই গুরুত্বপূর্ণ। এজন্য তিনটি প্রধান কমান্ড ব্যবহার হয়: `git status`, `git add`, এবং `git commit`।

## ১. git status

এই কমান্ডটি দিয়ে আপনি জানতে পারবেন কোন ফাইল পরিবর্তিত হয়েছে, কোন ফাইল স্টেজিং এরিয়াতে আছে, আর কোন ফাইল কমিটের জন্য প্রস্তুত নয়।

```sh
git status
```

## ২. git add

যে ফাইল বা পরিবর্তনগুলো আপনি পরবর্তী কমিটে রাখতে চান, সেগুলো স্টেজিং এরিয়াতে যোগ করতে হয়।

```sh
git add <file>
```

সব ফাইল যোগ করতে:
```sh
git add .
```

## ৩. git commit

স্টেজিং এরিয়াতে থাকা পরিবর্তনগুলো রিপোজিটরিতে স্থায়ীভাবে সংরক্ষণ করতে `git commit` ব্যবহার হয়।

```sh
git commit -m "কমিট মেসেজ"
```

কমিট মেসেজে সংক্ষেপে কী পরিবর্তন করেছেন তা লিখুন।

## উদাহরণ

1. ফাইল পরিবর্তন করুন বা নতুন ফাইল যোগ করুন
2. পরিবর্তন স্টেজ করুন:
   ```sh
   git add index.html
   ```
3. কমিট করুন:
   ```sh
   git commit -m "Add index.html"
   ```

## সংক্ষেপে
- `git status` — বর্তমান অবস্থা দেখুন
- `git add` — পরিবর্তন স্টেজ করুন
- `git commit` — পরিবর্তন সংরক্ষণ করুন

এই তিনটি কমান্ড গিটে পরিবর্তন ট্র্যাকিংয়ের মূল ভিত্তি।
