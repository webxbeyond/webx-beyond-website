# `.git` ডিরেক্টরি এক্সপ্লোর

গিট রিপোজিটরির সবচেয়ে গুরুত্বপূর্ণ অংশ হলো `.git` ডিরেক্টরি। এখানে গিটের সব ইতিহাস, অবজেক্ট, কনফিগারেশন ও রেফারেন্স থাকে।

## `.git` ডিরেক্টরির প্রধান ফোল্ডার ও ফাইল

- **objects/**: সব ব্লব, ট্রি, কমিট, ট্যাগ অবজেক্ট সংরক্ষণ করে
- **refs/**: ব্রাঞ্চ, ট্যাগ, রিমোট রেফারেন্স
- **HEAD**: বর্তমানে কোন ব্রাঞ্চ চেকআউট করা আছে
- **config**: লোকাল রিপোর কনফিগারেশন
- **index**: স্টেজিং এরিয়ার তথ্য
- **logs/**: রেফ ও HEAD-এর ইতিহাস (রিফলগ)
- **description**: রিপোর বর্ণনা (সার্ভারে ব্যবহৃত)
- **hooks/**: গিট হুক স্ক্রিপ্ট
- **info/**: এক্সক্লুড ফাইল ও অন্যান্য তথ্য

## কিভাবে দেখবেন?
```sh
ls -al .git
```

## কিছু গুরুত্বপূর্ণ ফাইল ও ফোল্ডার
- `.git/objects/`: গিটের আসল ডেটাবেস
- `.git/refs/heads/`: সব লোকাল ব্রাঞ্চ
- `.git/refs/remotes/`: রিমোট ব্রাঞ্চ
- `.git/refs/tags/`: ট্যাগ
- `.git/logs/HEAD`: HEAD-এর ইতিহাস
- `.git/config`: লোকাল কনফিগ
- `.git/index`: স্টেজিং এরিয়া

## `.git` ডিরেক্টরি মুছে গেলে কী হবে?
- পুরো ইতিহাস, ব্রাঞ্চ, ট্যাগ, সবকিছু হারিয়ে যাবে
- তাই `.git` ডিরেক্টরি কখনোই ডিলিট করবেন না

## সংক্ষেপে
- `.git` = গিট রিপোর হার্ট
- এখানে সব ইতিহাস ও অবজেক্ট থাকে
- সাবধানে ব্যবহার করুন

এগুলো জানলে গিটের ভিতরের স্ট্রাকচার ও ডেটা ম্যানেজমেন্ট আরও পরিষ্কার হবে।
