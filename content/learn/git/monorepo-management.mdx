## মনোরেপো কী?

মনোরেপো (Monorepo) মানে—একই রিপোজিটরিতে একাধিক প্রজেক্ট/প্যাকেজ রাখা। বড় টিম বা কোম্পানিতে কোড শেয়ার, ডিপেন্ডেন্সি ও টুলিং সহজ করতে ব্যবহৃত হয়।

## মনোরেপো ব্যবহারের সুবিধা
- একাধিক প্রজেক্টে কোড শেয়ার সহজ
- একসাথে টেস্ট, বিল্ড, রিলিজ
- ডিপেন্ডেন্সি ও কনফিগারেশন এক জায়গায়

## মনোরেপো ব্যবস্থাপনার চ্যালেঞ্জ
- বড় রিপোতে পারফরম্যান্স সমস্যা
- CI/CD কনফিগার জটিল
- অ্যাক্সেস কন্ট্রোল ও রিভিউ কঠিন

## গিটে মনোরেপো সেটআপ
- মূল রিপোতে প্রতিটি প্রজেক্ট/প্যাকেজ আলাদা ফোল্ডারে রাখুন
- `.gitignore` দিয়ে অপ্রয়োজনীয় ফাইল বাদ দিন

## মনোরেপো টুলিং
- **Nx, Turborepo, Lerna**: মনোরেপো ম্যানেজমেন্ট, বিল্ড, টেস্ট, ডিপেন্ডেন্সি গ্রাফ
- **Yarn Workspaces, pnpm Workspaces**: প্যাকেজ ম্যানেজমেন্ট

## গিট কমান্ড টিপস
- নির্দিষ্ট ফোল্ডারের হিস্টরি দেখতে:
  ```sh
  git log -- path/to/project
  ```
- সাবডিরেক্টরি থেকে সাবমডিউল বা আলাদা রিপোতে মাইগ্রেট:
  ```sh
  git subtree split -P path/to/project -b new-branch
  ```

## মনোরেপো বনাম মাল্টিরেপো
- মনোরেপো: এক রিপোতে সব কোড
- মাল্টিরেপো: প্রতিটি প্রজেক্ট আলাদা রিপো
- টিম ও স্কেল অনুযায়ী সিদ্ধান্ত নিন
