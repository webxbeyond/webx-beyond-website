## কোড অ্যানোটেট করা: `git blame`, `git annotate`

কোনো ফাইলের কোন লাইনে কে, কখন, কোন কমিটে পরিবর্তন এনেছে—এ তথ্য জানতে `git blame` ও `git annotate` কমান্ড ব্যবহার করা হয়। এতে কোড রিভিউ, বাগ ট্র্যাকিং ও ইতিহাস বোঝা সহজ হয়।

### `git blame` কী?
`git blame` কমান্ড দিয়ে ফাইলের প্রতিটি লাইনের পাশে সংশ্লিষ্ট কমিট, লেখক ও সময় দেখা যায়।

#### ব্যবহারবিধি
```bash
git blame <file>
```

#### উদাহরণ:
```bash
git blame app.js
```

প্রতিটি লাইনের পাশে কমিট আইডি, লেখক ও তারিখ দেখাবে।

#### নির্দিষ্ট লাইনের জন্য:
```bash
git blame -L 10,20 <file>
```
এতে ১০ থেকে ২০ নম্বর লাইনের তথ্য দেখাবে।

### অতিরিক্ত অপশন
- `-e` : ইমেইলসহ লেখক দেখাবে
- `-p` : আরও ডিটেইলড আউটপুট

### `git annotate`

কিছু গিট সংস্করণে `git annotate` ও `git blame` একই কাজ করে। তবে মূলত `git blame`-ই বেশি ব্যবহৃত হয়।

### ব্যবহারিক প্রয়োগ
- কে কোন কোড লিখেছে বা পরিবর্তন করেছে, তা দ্রুত জানা
- বাগি লাইনের জন্য দায়ী কমিট খুঁজে বের করা
- ইতিহাস বোঝা ও রিভিউ সহজ করা

### উপসংহার
`git blame`/`git annotate` দিয়ে কোডের ইতিহাস ও দায়িত্ব দ্রুত জানা যায়, যা টিমওয়ার্ক ও ডিবাগিংয়ে খুবই কার্যকর।
