# পুল রিকোয়েস্ট তৈরি ও রিভিউ ম্যানেজ

ওপেন সোর্স বা টিম প্রজেক্টে অবদান রাখতে পুল রিকোয়েস্ট (Pull Request/PR) তৈরি ও রিভিউ ম্যানেজ করা অত্যন্ত গুরুত্বপূর্ণ।

## পুল রিকোয়েস্ট (PR) কী?
- PR হলো আপনার পরিবর্তনগুলো মূল রিপোজিটরিতে মার্জ করার জন্য একটি অনুরোধ।
- এতে টিম মেম্বাররা আপনার কোড রিভিউ করতে পারে, আলোচনা করতে পারে, ও প্রয়োজন হলে পরিবর্তন চাইতে পারে।

## PR তৈরি করার ধাপ
1. **নতুন ব্রাঞ্চে কাজ করুন:**
   ```sh
   git checkout -b feature-x
   # কোড পরিবর্তন
   git add .
   git commit -m "নতুন ফিচার"
   git push origin feature-x
   ```
2. **গিটহাব/গিটল্যাবে গিয়ে 'Pull Request' বাটন চাপুন।**
3. **PR-এর টাইটেল ও বিবরণ লিখুন।**
4. **রিভিউয়ার অ্যাসাইন করুন (প্রয়োজনে)।**
5. **PR সাবমিট করুন।**

## PR রিভিউ ম্যানেজমেন্ট
- **কোড রিভিউ:** টিম মেম্বাররা PR-এ কমেন্ট, সাজেশন বা চেঞ্জ রিকোয়েস্ট করতে পারে।
- **ডিসকাশন:** PR-এ আলোচনা করে সমস্যার সমাধান করা যায়।
- **চেঞ্জ আপডেট:** দরকার হলে নতুন কমিট করে PR-এ পুশ করুন।
- **মার্জ:** রিভিউয়াররা সব ঠিক থাকলে PR মার্জ করে দেয়।
- **ক্লোজ:** প্রয়োজনে PR ক্লোজ করা যায় (যদি আর দরকার না হয়)।

## ভালো PR-এর জন্য টিপস
- ছোট ও স্পষ্ট পরিবর্তন করুন
- বিস্তারিত ও অর্থবহ টাইটেল ও বিবরণ লিখুন
- প্রয়োজনীয় স্ক্রিনশট/লিঙ্ক দিন
- টেস্ট পাস হচ্ছে কিনা দেখুন

## সংক্ষেপে
- নতুন ব্রাঞ্চে কাজ → পুশ → PR তৈরি → রিভিউ → মার্জ
- PR-এ আলোচনা ও রিভিউ টিমওয়ার্ক বাড়ায়

এভাবে পুল রিকোয়েস্ট ও রিভিউ ম্যানেজ করলে টিমে কোড কোয়ালিটি ও সহযোগিতা বাড়ে।
