# চেরি-পিকিং কমিট (`git cherry-pick`)

কখনো কখনো অন্য ব্রাঞ্চের নির্দিষ্ট একটি বা একাধিক কমিট আপনার বর্তমান ব্রাঞ্চে আনতে হয়। এই কাজের জন্য `git cherry-pick` ব্যবহার হয়।

## চেরি-পিক কী?
- নির্দিষ্ট কমিট/কমিটসমূহ বর্তমান ব্রাঞ্চে কপি করে নতুন কমিট হিসেবে যোগ করে
- মার্জ বা রিবেসের মতো পুরো ইতিহাস নয়, শুধু নির্দিষ্ট কমিট

## চেরি-পিক করার ধাপ
1. টার্গেট ব্রাঞ্চে যান:
   ```sh
   git switch main
   ```
2. চেরি-পিক করুন:
   ```sh
   git cherry-pick <commit-hash>
   ```
- একাধিক কমিট:
   ```sh
   git cherry-pick <hash1> <hash2>
   ```
- ধারাবাহিক কমিট:
   ```sh
   git cherry-pick <start-hash>^..<end-hash>
   ```

## কনফ্লিক্ট হলে
- চেরি-পিক করতে গিয়ে কনফ্লিক্ট হলে ফাইল ঠিক করে `git add` করুন
- তারপর:
   ```sh
   git cherry-pick --continue
   ```
- বাতিল করতে:
   ```sh
   git cherry-pick --abort
   ```

## কখন ব্যবহার করবেন?
- ফিচার/বাগফিক্স অন্য ব্রাঞ্চেও দরকার হলে
- মার্জ না করে নির্দিষ্ট কমিট কপি করতে

## সংক্ষেপে
- `git cherry-pick <hash>` = নির্দিষ্ট কমিট কপি
- কনফ্লিক্ট হলে ঠিক করে `--continue`
- বাতিল করতে `--abort`

এভাবে চেরি-পিক ব্যবহার করলে নির্দিষ্ট পরিবর্তন সহজে অন্য ব্রাঞ্চে আনা যায়।
