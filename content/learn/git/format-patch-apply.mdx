## প্যাচ তৈরি ও প্রয়োগ (`git format-patch`, `git apply`)

গিটে প্যাচ তৈরি ও প্রয়োগের মাধ্যমে আপনি নির্দিষ্ট কমিট বা পরিবর্তন অন্য রিপোজিটরিতে বা ব্রাঞ্চে সহজে পাঠাতে পারেন। এটি ওপেন সোর্স কন্ট্রিবিউশন, কোড রিভিউ বা অফলাইন কোড শেয়ারিংয়ের জন্য খুবই কার্যকর।

### প্যাচ কী?
প্যাচ হল টেক্সট ফাইল, যেখানে নির্দিষ্ট পরিবর্তনের ডিফ (diff) সংরক্ষিত থাকে।

### প্যাচ তৈরি: `git format-patch`

নির্দিষ্ট কমিট বা রেঞ্জের জন্য প্যাচ ফাইল তৈরি করতে:

```bash
git format-patch <commit-range>
```

#### উদাহরণ:
শেষ ১টি কমিটের জন্য:
```bash
git format-patch -1
```

শেষ ৩টি কমিটের জন্য:
```bash
git format-patch -3
```

নির্দিষ্ট দুই কমিটের মধ্যে:
```bash
git format-patch <commit1>..<commit2>
```

এতে `.patch` এক্সটেনশনের এক বা একাধিক ফাইল তৈরি হবে।

### প্যাচ প্রয়োগ: `git apply`

প্রাপ্ত `.patch` ফাইল প্রয়োগ করতে:

```bash
git apply <patch-file>
```

#### উদাহরণ:
```bash
git apply 0001-some-change.patch
```

### প্যাচ প্রয়োগের সময় সতর্কতা
- প্রয়োগের আগে `git status` ও `git diff` দিয়ে পরিবর্তন দেখে নিন।
- কনফ্লিক্ট হলে ম্যানুয়ালি ঠিক করতে হবে।

### প্যাচ ইমেইল থেকে প্রয়োগ: `git am`

ইমেইল-স্টাইল প্যাচ প্রয়োগ করতে:
```bash
git am <patch-file>
```

### উপসংহার
প্যাচের মাধ্যমে গিটে পরিবর্তন শেয়ার ও প্রয়োগ করা সহজ হয়। বিশেষ করে যখন সরাসরি পুশ/পুল সম্ভব নয়, তখন প্যাচ আদান-প্রদান কার্যকর সমাধান।
