## প্রি-কমিট ও প্রি-পুশ হুক সেটআপ

প্রি-কমিট (pre-commit) ও প্রি-পুশ (pre-push) গিট হুক দিয়ে কমিট বা পুশের আগে স্বয়ংক্রিয়ভাবে কোড চেক, টেস্ট, লিন্ট ইত্যাদি চালানো যায়। এতে কোডবেসে ভুল, বাগ বা খারাপ কোড ঢোকার ঝুঁকি কমে।

### প্রি-কমিট হুক সেটআপ
1. `.git/hooks/pre-commit` নামে স্ক্রিপ্ট তৈরি করুন:
   ```bash
   #!/bin/sh
   npm run lint
   if [ $? -ne 0 ]; then
     echo "লিন্টিং ফেইল! কমিট বাতিল।"
     exit 1
   fi
   ```
2. স্ক্রিপ্ট এক্সিকিউটেবল করুন:
   ```bash
   chmod +x .git/hooks/pre-commit
   ```

### প্রি-পুশ হুক সেটআপ
1. `.git/hooks/pre-push` নামে স্ক্রিপ্ট তৈরি করুন:
   ```bash
   #!/bin/sh
   npm test
   if [ $? -ne 0 ]; then
     echo "টেস্ট ফেইল! পুশ বাতিল।"
     exit 1
   fi
   ```
2. স্ক্রিপ্ট এক্সিকিউটেবল করুন:
   ```bash
   chmod +x .git/hooks/pre-push
   ```

### Husky দিয়ে হুক ম্যানেজ (রেকমেন্ডেড)
1. ইনস্টল করুন:
   ```bash
   npm install husky --save-dev
   npx husky install
   ```
2. হুক যোগ করুন:
   ```bash
   npx husky add .husky/pre-commit "npm run lint"
   npx husky add .husky/pre-push "npm test"
   ```

### সুবিধা
- কোডবেসে ভুল, বাগ, ফরম্যাট সমস্যা কমে
- টিমে কনসিসটেন্ট কোয়ালিটি
- CI/CD-র আগেই লোকাল চেক

### উপসংহার
প্রি-কমিট ও প্রি-পুশ হুক দিয়ে কোড কোয়ালিটি ও টিম স্ট্যান্ডার্ড নিশ্চিত করুন। Husky দিয়ে হুক শেয়ার ও ম্যানেজ করা আরও সহজ।
