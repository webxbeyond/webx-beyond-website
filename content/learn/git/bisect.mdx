## বাগ খুঁজতে `git bisect` ব্যবহার

`git bisect` গিটের একটি শক্তিশালী টুল, যা বাইনারি সার্চ পদ্ধতিতে দ্রুত কোন কমিটে বাগ এসেছে তা খুঁজে বের করতে সাহায্য করে।

### কিভাবে কাজ করে?
আপনি জানেন, কোন কমিটে কোড ঠিক ছিল (good) এবং কোন কমিটে বাগ দেখা দিয়েছে (bad)। `git bisect` এই দুই পয়েন্টের মধ্যে স্বয়ংক্রিয়ভাবে কমিটগুলো ভাগ করে আপনাকে বারবার পরীক্ষা করতে বলে, যতক্ষণ না নির্দিষ্ট বাগি কমিট পাওয়া যায়।

### ব্যবহারবিধি

#### ১. বিআইসেক্ট শুরু করুন
```bash
git bisect start
```

#### ২. খারাপ (bad) কমিট চিহ্নিত করুন
```bash
git bisect bad
```

#### ৩. ভালো (good) কমিট চিহ্নিত করুন
```bash
git bisect good <commit-hash>
```

#### ৪. প্রতিটি ধাপে টেস্ট করুন
গিট আপনাকে মাঝামাঝি একটি কমিটে নিয়ে যাবে। এখানে কোড টেস্ট করুন—বাগ আছে কি না দেখুন।
- যদি বাগ থাকে:
  ```bash
  git bisect bad
  ```
- যদি বাগ না থাকে:
  ```bash
  git bisect good
  ```

এভাবে কয়েক ধাপে গিট নিজে নিজে কমিট রেঞ্জ ছোট করবে।

#### ৫. শেষ হলে বিআইসেক্ট বন্ধ করুন
```bash
git bisect reset
```

### স্বয়ংক্রিয় টেস্ট স্ক্রিপ্ট দিয়ে
আপনার টেস্ট স্ক্রিপ্ট থাকলে:
```bash
git bisect run <script>
```

### উপকারিতা
- বড় রিপোজিটরিতে দ্রুত বাগি কমিট খুঁজে পাওয়া যায়
- ম্যানুয়াল ও স্বয়ংক্রিয় দুইভাবেই ব্যবহার করা যায়

### উপসংহার
`git bisect` ব্যবহার করে বাগ ট্র্যাকিং অনেক সহজ ও সময় সাশ্রয়ী হয়।
