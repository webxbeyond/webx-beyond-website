# ভুল সংশোধন: ফাইলের জন্য `git checkout`, বেসিক `git reset`

কখনো কখনো গিটে কাজ করার সময় ভুল হয়ে যেতে পারে। গিটের কিছু কমান্ড দিয়ে সহজেই এই ভুলগুলো সংশোধন করা যায়।

## ১. ফাইলের জন্য git checkout

যদি কোনো ফাইলে পরিবর্তন করে ফেলেন এবং আগের অবস্থায় ফিরিয়ে আনতে চান:

```sh
git checkout -- <filename>
```

এতে ফাইলটি সর্বশেষ কমিটের অবস্থায় ফিরে যাবে। (নোট: নতুন গিট ভার্সনে `git restore <filename>` ব্যবহার করা হয়)

## ২. বেসিক git reset

স্টেজিং এরিয়াতে যোগ করা কোনো পরিবর্তন যদি কমিট করতে না চান, তাহলে:

```sh
git reset <file>
```

এতে ফাইলটি স্টেজিং থেকে আনস্টেজড হয়ে যাবে, কিন্তু ওয়ার্কিং ডিরেক্টরিতে পরিবর্তন থাকবে।

## উদাহরণ

1. ফাইল পরিবর্তন করেছেন, কিন্তু আগের অবস্থায় ফিরতে চান:
   ```sh
   git checkout -- index.html
   ```
2. ভুল করে ফাইল স্টেজ করেছেন, আনস্টেজ করতে চান:
   ```sh
   git reset index.html
   ```

## সতর্কতা
- `git checkout -- <filename>` ফাইলের সব আনসেভড পরিবর্তন মুছে দেবে।
- `git reset` শুধু স্টেজিং থেকে সরায়, ফাইল ডিলিট করে না।

## সংক্ষেপে
- ফাইল আগের অবস্থায় ফেরাতে: `git checkout -- <filename>`
- স্টেজিং থেকে সরাতে: `git reset <file>`

এই কমান্ডগুলো দিয়ে গিটে ছোটখাটো ভুল সহজেই সংশোধন করা যায়।
