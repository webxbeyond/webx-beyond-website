# রিমোট সহযোগিতায় কনফ্লিক্ট ম্যানেজমেন্ট

রিমোট রিপোজিটরিতে একাধিক ডেভেলপার কাজ করলে প্রায়ই কনফ্লিক্ট হয়। এগুলো বুঝে ও সঠিকভাবে ম্যানেজ করলে টিমওয়ার্ক সহজ হয়।

## কনফ্লিক্ট কীভাবে হয়?
- দুইজন একই ফাইলের একই লাইনে পরিবর্তন করলে
- রিমোটে নতুন কমিট থাকলে এবং আপনি লোকাল থেকে পুশ করলে
- মার্জ, রিবেস, চেরি-পিক ইত্যাদিতে

## কনফ্লিক্ট চিহ্নিত করা
- `git pull` বা `git merge` করলে কনফ্লিক্ট হলে গিট বার্তা দেয়
- কনফ্লিক্টেড ফাইলে `<<<<<<<`, `=======`, `>>>>>>>` চিহ্ন থাকে

## কনফ্লিক্ট সমাধানের ধাপ
1. **কনফ্লিক্টেড ফাইল খুলুন**
2. `<<<<<<<`, `=======`, `>>>>>>>` অংশ দেখে ঠিক করুন কোন অংশ রাখতে চান
3. ফাইল সেভ করুন
4. কনফ্লিক্ট রিজলভের পর:
   ```sh
   git add <file>
   git commit
   ```

## রিমোটে কনফ্লিক্ট হলে
- পুশ করতে গেলে রিমোটে নতুন কমিট থাকলে পুশ রিজেক্ট হয়
- সমাধান:
   1. `git pull --rebase` বা `git pull` করুন
   2. কনফ্লিক্ট রিজলভ করুন
   3. আবার পুশ করুন

## ভিজ্যুয়াল টুল ব্যবহার
- VSCode, Meld, KDiff3, Beyond Compare ইত্যাদি দিয়ে সহজে কনফ্লিক্ট রিজলভ করা যায়

## টিপস
- পুশের আগে `git fetch` ও `git status` দেখে নিন
- ছোট ও ফোকাসড কমিট করুন
- টিমে নিয়মিত কমিউনিকেশন রাখুন

## সংক্ষেপে
- কনফ্লিক্ট হলে ফাইল খুলে ম্যানুয়ালি ঠিক করুন
- `git add` ও `git commit` দিয়ে সমাধান করুন
- প্রয়োজনে ভিজ্যুয়াল টুল ব্যবহার করুন

এভাবে কনফ্লিক্ট ম্যানেজ করলে টিমে কাজ করা সহজ ও ঝামেলাহীন হয়।
