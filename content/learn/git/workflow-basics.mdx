# গিট ওয়ার্কফ্লো বেসিকস

গিটে কাজ করার সময় তিনটি প্রধান অংশ থাকে: ওয়ার্কিং ডিরেক্টরি, স্টেজিং এরিয়া (ইন্ডেক্স), এবং রিপোজিটরি।

## ওয়ার্কিং ডিরেক্টরি
এটি আপনার প্রকল্পের ফোল্ডার, যেখানে আপনি ফাইল তৈরি, পরিবর্তন বা ডিলিট করেন। এখানে করা পরিবর্তন সরাসরি গিটে সংরক্ষিত হয় না।

## স্টেজিং এরিয়া (Index)
স্টেজিং এরিয়া হলো একটি মধ্যবর্তী জায়গা, যেখানে আপনি কোন কোন পরিবর্তন কমিট করতে চান তা নির্বাচন করেন।

- `git add <file>` কমান্ড দিয়ে ফাইল স্টেজিং এরিয়াতে যোগ করা হয়।
- স্টেজিং এরিয়াতে থাকা ফাইলগুলোই পরবর্তী কমিটে যুক্ত হবে।

## রিপোজিটরি
রিপোজিটরি হলো গিটের ডাটাবেস, যেখানে আপনার কমিট করা সমস্ত ইতিহাস সংরক্ষিত থাকে।

- `git commit` কমান্ড দিলে স্টেজিং এরিয়ার পরিবর্তন রিপোজিটরিতে স্থায়ীভাবে সংরক্ষিত হয়।

## ওয়ার্কফ্লো চিত্র

```
ওয়ার্কিং ডিরেক্টরি → (git add) → স্টেজিং এরিয়া → (git commit) → রিপোজিটরি
```

## উদাহরণ

1. ফাইল তৈরি বা পরিবর্তন করুন:
   ```sh
   echo "Hello World" > hello.txt
   ```
2. ফাইল স্টেজ করুন:
   ```sh
   git add hello.txt
   ```
3. কমিট করুন:
   ```sh
   git commit -m "Add hello.txt"
   ```

## সংক্ষেপে
- ওয়ার্কিং ডিরেক্টরি: যেখানে আপনি কাজ করেন
- স্টেজিং এরিয়া: কোন পরিবর্তন কমিট হবে তা নির্বাচন
- রিপোজিটরি: চূড়ান্ত ইতিহাস সংরক্ষণ

এই তিনটি অংশ বুঝলে গিটের বেসিক ওয়ার্কফ্লো পরিষ্কার হবে।
