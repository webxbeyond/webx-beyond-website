# রিফলগ দিয়ে আনডু ও রিকভারি

গিটে ভুল কমিট, ব্রাঞ্চ ডিলিট বা HEAD মুভ হয়ে গেলে রিফলগ (reflog) দিয়ে সহজেই উদ্ধার করা যায়।

## রিফলগ (reflog) কী?
- গিটে HEAD ও ব্রাঞ্চের সব মুভমেন্টের ইতিহাস সংরক্ষণ করে
- লোকাল রিপোতে সীমাবদ্ধ (রিমোটে থাকে না)
- `.git/logs/HEAD` ও `.git/logs/refs/heads/`-এ সংরক্ষিত

## রিফলগ দেখা
```sh
git reflog
```
- এতে HEAD-এর সব পরিবর্তনের লিস্ট পাবেন
- প্রতিটি এন্ট্রির সাথে হ্যাশ ও অ্যাকশন দেখা যায়

## হারানো কমিট উদ্ধার
1. `git reflog` দিয়ে কাঙ্ক্ষিত কমিটের হ্যাশ বের করুন
2. সেই হ্যাশে চেকআউট করুন:
   ```sh
   git checkout <hash>
   ```
3. দরকার হলে নতুন ব্রাঞ্চ তৈরি করুন:
   ```sh
   git checkout -b recovered-branch
   ```

## ভুল মার্জ/রিবেস/রিসেট Undo
- রিফলগ থেকে আগের অবস্থার হ্যাশ বের করে `git reset --hard <hash>` করুন

## টিপস
- রিফলগ লোকাল, তাই ক্লোন করা রিপোতে পাওয়া যাবে না
- পুরনো এন্ট্রি গার্বেজ কালেকশনে মুছে যেতে পারে

## সংক্ষেপে
- `git reflog` = সব HEAD মুভমেন্টের ইতিহাস
- হারানো কমিট, ব্রাঞ্চ, HEAD সহজে উদ্ধার
- ভুল Undo ও রিকভারিতে অপরিহার্য

এগুলো জানলে গিটে ভুল থেকে উদ্ধার হওয়া অনেক সহজ হবে।
