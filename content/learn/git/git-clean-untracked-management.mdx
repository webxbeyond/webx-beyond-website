গিট রিপোজিটরিতে অনেক সময় এমন ফাইল তৈরি হয় যেগুলো গিট ট্র্যাক করে না (untracked files)। আবার, build বা temp ফাইলের মতো অপ্রয়োজনীয় ফাইলও জমা হতে পারে। এই পাঠে শিখব কীভাবে গিট দিয়ে এসব ফাইল ম্যানেজ ও ক্লিন করা যায়।

## ১. আনট্র্যাকড ফাইল কী?

- যেসব ফাইল গিটে add/commit করা হয়নি, সেগুলো আনট্র্যাকড।
- `git status` কমান্ডে এগুলো আলাদা করে দেখায়।

## ২. আনট্র্যাকড ফাইল দেখার উপায়

```sh
git status
```

- "Untracked files" সেকশনে সব আনট্র্যাকড ফাইল দেখাবে।

## ৩. গিট ক্লিন: অপ্রয়োজনীয় ফাইল ডিলিট

- `git clean` কমান্ড দিয়ে আনট্র্যাকড ফাইল ও ডিরেক্টরি সরানো যায়।
- **সতর্কতা:** এটি ডিলিট করে, রিকভার করা যায় না!

### (ক) শুধু ফাইল ডিলিট:

```sh
git clean -f
```

### (খ) ফোল্ডারসহ ডিলিট:

```sh
git clean -fd
```

### (গ) dry-run (কী কী ডিলিট হবে আগে দেখে নিন):

```sh
git clean -n
```

### (ঘ) নির্দিষ্ট টাইপের ফাইল ডিলিট:

```sh
git clean -f -e !*.md
```

- উপরের কমান্ডে `.md` বাদে সব আনট্র্যাকড ফাইল ডিলিট হবে।

## ৪. `.gitignore` দিয়ে ফাইল ম্যানেজমেন্ট

- যেসব ফাইল/ফোল্ডার গিটে ট্র্যাক করতে চান না, সেগুলো `.gitignore`-এ লিখুন।
- উদাহরণ:

```
node_modules/
build/
*.log
```

- `.gitignore`-এ থাকা ফাইল গিট ট্র্যাক করবে না, কিন্তু আগে ট্র্যাক করা ফাইল হলে `git rm --cached` দিয়ে আনট্র্যাক করতে হবে।

## ৫. সতর্কতা ও বেস্ট প্র্যাকটিস

- `git clean` চালানোর আগে `-n` দিয়ে dry-run করুন।
- গুরুত্বপূর্ণ ফাইল কখনো আনট্র্যাকড অবস্থায় না রাখুন।
- `.gitignore` নিয়মিত আপডেট করুন।

---

**সংক্ষেপে:**
- `git status` দিয়ে আনট্র্যাকড ফাইল দেখুন, `git clean` দিয়ে ক্লিন করুন।
- `.gitignore` ব্যবহার করে অপ্রয়োজনীয় ফাইল গিটে আসা বন্ধ করুন।
- ডিলিট করার আগে সবসময় double-check করুন।
