ওপেন সোর্স বা টিম প্রজেক্টে কার্যকর কন্ট্রিবিউশন গাইডলাইন (CONTRIBUTING.md) নতুন কন্ট্রিবিউটরদের জন্য পথনির্দেশিকা হিসেবে কাজ করে। এতে কিভাবে ইস্যু খুলবেন, PR করবেন, কোড স্টাইল, টেস্টিং, রিভিউ—সবকিছু স্পষ্টভাবে উল্লেখ থাকে।

## ১. CONTRIBUTING.md ফাইল কী?
- প্রজেক্টের রুটে রাখা হয়।
- গিটহাব/গিটল্যাব স্বয়ংক্রিয়ভাবে এটি দেখায়।

## ২. কী কী থাকতে পারে?
- কন্ট্রিবিউট করার ধাপ (fork, branch, commit, PR)
- কোড স্টাইল ও লিন্টিং গাইডলাইন
- টেস্টিং ও লোকাল রান করার নিয়ম
- ইস্যু/বাগ রিপোর্ট করার টেমপ্লেট
- PR রিভিউ প্রসেস
- কমিউনিটি আচরণবিধি (Code of Conduct)

## ৩. উদাহরণ কাঠামো
```
# কন্ট্রিবিউশন গাইডলাইন

## কিভাবে কন্ট্রিবিউট করবেন
1. রিপো ফর্ক করুন
2. নতুন ব্রাঞ্চ তৈরি করুন
3. কোড পরিবর্তন করুন
4. টেস্ট রান করুন
5. PR সাবমিট করুন

## কোড স্টাইল
- Prettier/Linter ব্যবহার করুন
- কমিট মেসেজ কনভেনশন ফলো করুন

## টেস্টিং
- `npm test` দিয়ে টেস্ট রান করুন

## ইস্যু/বাগ রিপোর্ট
- স্পষ্টভাবে বাগ/ফিচার বর্ণনা করুন

## রিভিউ প্রসেস
- PR-এ ফিডব্যাক পেলে ঠিক করুন
- সব টেস্ট পাস হলে মার্জ হবে
```

## ৪. বেস্ট প্র্যাকটিস
- গাইডলাইন সংক্ষিপ্ত, স্পষ্ট ও আপডেট রাখুন
- নতুন কন্ট্রিবিউটরদের জন্য উদাহরণ দিন
- কমিউনিটি আচরণবিধি যুক্ত করুন

---

**সংক্ষেপে:**
- কার্যকর CONTRIBUTING.md নতুন কন্ট্রিবিউটরদের জন্য স্পষ্ট নির্দেশনা দেয়
- এতে টিমওয়ার্ক, কোড কোয়ালিটি ও কমিউনিটি কালচার উন্নত হয়
