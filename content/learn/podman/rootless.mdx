পডম্যানের সবচেয়ে বড় বৈশিষ্ট্য হলো **daemonless এবং rootless mode**। কনটেইনার চালানো যায় root ব্যবহার করে বা root ছাড়াই। এই লেসনে আমরা শিখব Rootful এবং Rootless কনটেইনারের মধ্যে পার্থক্য এবং তাদের সুবিধা ও সীমাবদ্ধতা।  

## ১. Rootful কনটেইনার

**Rootful কনটেইনার** হচ্ছে সেই কনটেইনার যা **root ইউজারের অধীনে চলে**।  

### বৈশিষ্ট্য:

- কনটেইনারে সম্পূর্ণ root access থাকে।  
- সকল সিস্টেম রিসোর্স ব্যবহার করা সম্ভব।  
- সহজে নেটওয়ার্ক ও ফাইল সিস্টেম কনফিগার করা যায়।  

### সীমাবদ্ধতা:

- নিরাপত্তা ঝুঁকি বেশি।  
- যদি কনটেইনার কমান্ডে ভুল হয়, পুরো হোস্ট সিস্টেমের নিরাপত্তা বিপন্ন হতে পারে।  

> **উপমা:** Rootful কনটেইনার হলো “মালিকের চাবি নিয়ে সম্পূর্ণ দোকান চালানো”।  

## ২. Rootless কনটেইনার

**Rootless কনটেইনার** হচ্ছে সেই কনটেইনার যা **সাধারণ ইউজারের অধীনে চলে**, root না হয়ে।  

### বৈশিষ্ট্য:

- নিরাপদ, হোস্ট সিস্টেমে কম ঝুঁকি।  
- হোস্টের রুট প্রিভিলেজ ছাড়াই কনটেইনার চালানো যায়।  
- Podman-এর মূল শক্তি এই Rootless mode-এ।  

### সীমাবদ্ধতা:

- কিছু সিস্টেম রিসোর্স সীমিত।  
- কিছু নেটওয়ার্ক অপারেশন root ছাড়া করা যায় না।  

> **উপমা:** Rootless কনটেইনার হলো “দোকানের কাজ শুধু অনুমতি নিয়ে করা, চাবি ছাড়া”।  

## ৩. Rootful বনাম Rootless তুলনা

| বৈশিষ্ট্য | Rootful | Rootless |
|-----------|---------|----------|
| ইউজার প্রিভিলেজ | Root | Normal user |
| নিরাপত্তা | কম | বেশি |
| ফাইল/নেটওয়ার্ক অ্যাক্সেস | পূর্ণ | সীমিত |
| Podman daemon | Optional | নেই |
| ব্যবহারের সুবিধা | সহজ কনফিগারেশন | নিরাপদ |

## ৪. Rootless কনটেইনার চালানো

Rootless কনটেইনার চালানোর জন্য শুধু সাধারণ ইউজার হিসেবে কমান্ড চালাও:

```bash
podman run -it --rm alpine sh
````

> যদি তুমি root ইউজার হও, rootless mode হবে না।

## ৫. সহজ উপমা

* Rootful → “দোকানের চাবি নিয়ে পুরো নিয়ন্ত্রণ”
* Rootless → “অনুমতি নিয়ে সীমিত নিয়ন্ত্রণ, নিরাপদ”

## ৬. প্রাকটিক্যাল এক্সারসাইজ

  - সাধারণ ইউজার হিসেবে একটি Rootless কনটেইনার চালাও।  
  - root ইউজার হিসেবে একটি Rootful কনটেইনার চালাও।  
  - উভয় কনটেইনারের নেটওয়ার্ক এবং ফাইল সিস্টেম এক্সেস পার্থক্য লক্ষ্য করো।  
  - Rootless কনটেইনারে কোন কাজ সীমিত তা পরীক্ষা করো।
