# পডম্যান ইমেজ ব্যবস্থাপনা

কনটেইনার চালানোর জন্য ইমেজ অপরিহার্য। পডম্যানে ইমেজ ব্যবস্থাপনা করা খুবই সহজ। এই লেসনে আমরা শিখব কীভাবে **ইমেজ সার্চ, ডাউনলোড, তৈরি, পরীক্ষা এবং মুছে ফেলা যায়।**

## ১. ইমেজ সার্চ করা

নতুন ইমেজ পেতে চাইলে সার্চ করতে পারো:

```bash
podman search <image_name>
````

উদাহরণ:

```bash
podman search nginx
```

> এটি ডিফল্ট রেজিস্ট্রি থেকে ইমেজের তালিকা দেখাবে। এখানে তুমি ইমেজের নাম, স্টার, অফিসিয়াল বা আর্ন্তজাতিক চিহ্ন দেখতে পারবে।

## ২. ইমেজ ডাউনলোড (Pull)

যে ইমেজ চালাতে চাও সেটি ডাউনলোড করতে:

```bash
podman pull <image_name>:<tag>
```

উদাহরণ:

```bash
podman pull nginx:latest
```

> যদি ট্যাগ না দাও, ডিফল্টভাবে `latest` ব্যবহার হবে।

## ৩. ইমেজ তৈরি (Build)

নিজের ডিরেক্টরি থেকে কাস্টম ইমেজ তৈরি করতে ডকারফাইল বা Containerfile ব্যবহার করো:

```bash
podman build -t <image_name>:<tag> .
```

উদাহরণ:

```bash
podman build -t myapp:1.0 .
```

> এখানে `.` মানে বর্তমান ডিরেক্টরি।

## ৪. ইমেজ পরীক্ষা (Inspect)

ইমেজের বিস্তারিত তথ্য জানতে:

```bash
podman inspect <image_name_or_id>
```

উদাহরণ:

```bash
podman inspect nginx:latest
```

> এটি ইমেজের লেয়ার, ক্রিয়েটেড টাইম, কমান্ড এবং অন্যান্য মেটাডেটা দেখায়।

## ৫. ইমেজ মুছে ফেলা (Remove)

অপ্রয়োজনীয় ইমেজ মুছে দিতে:

```bash
podman rmi <image_name_or_id>
```

> **ফোর্স দিয়ে মুছে ফেলা:**

```bash
podman rmi -f <image_name_or_id>
```

> **নোট:** যদি ইমেজটি কোনো কনটেইনার দ্বারা ব্যবহৃত হয়, তাহলে `-f` ব্যবহার করতে হবে।

## ৬. সহজ উপমা

ইমেজকে তুমি ভাবতে পারো **কোনও রান্নার রেসিপি** হিসেবে:

* **Search** → বাজারে রেসিপি খোঁজা
* **Pull** → রেসিপি অনুযায়ী উপকরণ আনানো
* **Build** → নিজস্ব কাস্টম রেসিপি তৈরি করা
* **Inspect** → রেসিপির বিবরণ দেখা
* **Remove** → পুরোনো বা অপ্রয়োজনীয় রেসিপি ফেলে দেওয়া

এই উপমা মনে রাখলে ইমেজ ব্যবস্থাপনা সহজ মনে হবে।

## ৭. প্রাকটিক্যাল এক্সারসাইজ

  - একটি নতুন ইমেজ সার্চ করো, উদাহরণস্বরূপ `ubuntu`।
  - সার্চ থেকে ইমেজ পুল করো।
  - পুল করা ইমেজটি ইনস্পেক্ট করো।
  - নিজের একটি ছোট কাস্টম ইমেজ বিল্ড করো।
  - অপ্রয়োজনীয় ইমেজ মুছে ফেলো।
