# পডম্যান আর্কিটেকচার: Daemonless ও Rootless Mode

পডম্যান শুধু একটি **কনটেইনার রানটাইম** নয়, এটি একটি আধুনিক, নিরাপদ, এবং ব্যবহারকারী-বান্ধব আর্কিটেকচার ফলো করে। ডকারের মতো, পডম্যানও কনটেইনার তৈরি ও পরিচালনা করে, কিন্তু এর **ভিত্তি মূলত আলাদা**। চলুন সহজ ভাষায় ধাপে ধাপে বোঝা যাক।

---

## ১. Daemonless আর্কিটেকচার

ডকারে আমরা যাকে “Docker Daemon” বলি, সেটি একটি ব্যাকগ্রাউন্ড সার্ভিস যা সব কনটেইনার ও ইমেজ পরিচালনা করে। কিন্তু পডম্যান **daemonless**, অর্থাৎ এর জন্য আলাদা সার্ভিস চালানোর দরকার নেই। 

**কেন গুরুত্বপূর্ণ?**  
- **নিরাপত্তা:** ডকারের মতো একটি চলমান ডিমন সার্ভিস থাকলে, যদি সেটি কমপ্রোমাইজ হয়, পুরো সিস্টেম ঝুঁকিতে পড়তে পারে।  
- **সহজ রিসোর্স ব্যবহার:** কোন ব্যাকগ্রাউন্ড প্রক্রিয়া সর্বদা চলবে না।  
- **সরাসরি CLI ইন্টারঅ্যাকশন:** পডম্যান কমান্ড চালালে এটি সরাসরি OCI runtime (যেমন runc বা crun) কে কল করে।  

**চিত্রগতভাবে ভাবুন:**  
ডকার হলো “মধ্যবর্তী কর্মকর্তা” যিনি সব কাজের জন্য অনুমতি চায়। পডম্যান হলো “স্বাধীন কর্মী” যিনি সরাসরি কাজ সম্পন্ন করে, কোন মধ্যবর্তী ছাড়া।  

---

## ২. Rootless Mode

পডম্যানকে ব্যবহার করা যায় **root ছাড়া**, অর্থাৎ সাধারণ ব্যবহারকারী হিসাবেও। এটি অনেক নিরাপদ করে তোলে, কারণ কনটেইনার চালানোর জন্য সিস্টেম-লেভেল root অনুমতি প্রয়োজন হয় না।  

**Rootless এর সুবিধা:**  
- ব্যবহারকারী **সীমিত অনুমতি** থাকায় সিস্টেম ক্ষতি হওয়ার সম্ভাবনা কমে।  
- হোস্ট সিস্টেমের সাথে কম ঝুঁকি।  
- ক্লাউড বা মাল্টি-ইউজার পরিবেশে আদর্শ।  

**কিভাবে কাজ করে:**  
- পডম্যান ব্যবহারকারীর UID/GID অনুযায়ী ফাইল সিস্টেম ও নেটওয়ার্ক নেমস্পেস তৈরি করে।  
- প্রতিটি কনটেইনার কার্যকর হয় ব্যবহারকারীর সীমাবদ্ধতার মধ্যে।  
- কিছু ফিচার (যেমন কিছু পোর্ট binding) সীমাবদ্ধ হতে পারে rootless মোডে।  

**উপমা:**  
ধরা যাক আপনি একটি ছোট দোকানের মালিক। আপনার দোকানের সমস্ত কাজ আপনি নিজেই করছেন, কিন্তু কোন বড় ম্যানেজার বা নিরাপত্তা প্রহরীর অনুমতি প্রয়োজন নেই। আপনি নিজের নিয়ন্ত্রণে থাকছেন, কিন্তু নিরাপদে কাজ করছেন। এটাই rootless পডম্যানের ধারণা।

## ৩. পডম্যান আর্কিটেকচারের মূল উপাদান

- **CLI:** ব্যবহারকারীর সঙ্গে সরাসরি ইন্টারঅ্যাকশন।  
- **OCI Runtime (runc/crun):** কনটেইনার প্রসেস চালায়।  
- **CNI Networking:** কনটেইনার নেটওয়ার্কিং সমর্থন করে।  
- **Storage:** OverlayFS বা অন্যান্য স্টোরেজ ব্যাকএন্ড ব্যবহার করে।  

**কাজের ধারা:**  
1. ব্যবহারকারী কমান্ড দেয়।  
2. পডম্যান CLI তা রিড করে।  
3. OCI Runtime কে কল করে কনটেইনার চালায়।  
4. ফাইল সিস্টেম ও নেটওয়ার্ক ব্যবহারকারীর অনুমতি অনুযায়ী পরিচালনা করা হয়।

## ৪. প্র্যাক্টিক্যাল এক্সারসাইজ

**লক্ষ্য:** আপনার সিস্টেমে daemonless এবং rootless পডম্যান অনুশীলন করা।  

1. **পডম্যান ইনস্টল করুন:**
```bash
sudo apt install podman
````

2. **Rootless মোডে একটি কনটেইনার চালান:**

```bash
podman run --rm -it alpine sh
```

3. **ডকারের মতো একটি ইমেজ তালিকা দেখুন:**

```bash
podman images
```

4. **নতুন কনটেইনার তৈরি ও নাম দিন:**

```bash
podman run -dt --name mytest alpine sleep 3600
```

5. **চেক করুন এটি root ছাড়া চলছে:**

```bash
podman ps
```

> একবার এই ধাপগুলো শেষ করলে আপনি বুঝতে পারবেন কিভাবে পডম্যান **daemonless এবং rootless** কাজ করে।
