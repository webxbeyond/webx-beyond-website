## পরিচিতি

এই লেসনে আমরা খুব সহজভাবে প্রথম কমান্ডগুলো দিয়ে একটি `Hello World` কনটেইনার চালাবো। উদ্দেশ্য হলো — ভাঙা না করে ধাপে ধাপে দেখানো যাতে নতুনরা সহজে ধরতে পারে: কিভাবে ইমেজ ডাউনলোড হয়, কিভাবে কনটেইনার রান হয়, এবং চালানোর পরে কী চেক করতে হবে।

লক্ষ্য:

- `podman pull` ও `podman run` কমান্ডের ধারণা পাওয়া
- রান সময় আউটপুট, লিস্টিং ও লোগস যাচাই করা
- ছোটখাটো ডিবাগ কিভাবে করবেন সেই ধারণা পাওয়া

নোট: এই লেসনে আমরা ধারণা-ভিত্তিক হ্যান্ডস-অন দিই; ইনস্টলেশন ও কনফিগারেশন আলাদা লেসনে আছে—এখানে ধরে নেওয়া হয়েছে আপনি `podman` সিস্টেমে ইনস্টল করে ফেলেছেন।

---

## সহজ উপমা (Analogy)

কল্পনা করুন কনটেইনার হলো একটি ছোট পপ-আপ ক্যাফে। ইমেজ হলো সেই ক্যাফের প্রস্তুত খাবারের রেসিপি ও উপকরণ; `podman run` হলো সেই ক্যাফে-ট্রাক পার্কিং করে খাবার পরিবেশন শুরু করার মতো। `podman pull` করলে আপনি সরবরাহকারী থেকে রেডি প্যাকেজ নিয়ে আসেন; `podman run --rm` দিলে ব্যবসা শেষ হতেই ট্রাক সরিয়ে নেয়া হবে।

এই উপমাটি সাহায্য করবে: ইমেজ (রেসিপি) আলাদা; কনটেইনার (চলিত দোকান) তখন তৈরি করে চালানো হয়।

## ধাপে ধাপে: Hello World রান করা

1) ইমেজ টানুন (pull)

```bash
podman pull hello-world
```

কী হচ্ছে: `hello-world` ইমেজ ডাউনলোড হবে আপনার ডিফল্ট রেজিস্ট্রি থেকে (প্রায়শই Docker Hub)।

2) কনটেইনার চালান এবং আউটপুট দেখুন

```bash
podman run --rm hello-world
```

কি ঘটবে: কনটেইনার স্টার্ট হবে, একটি ছোটটুকু টেক্সট প্রিন্ট করবে এবং `--rm` থাকায় কনটেইনার শেষ হলে নিজেই মুছে যাবে।

3) আউটপুট না আসে — কিভাবে চেক করবেন

- `podman images` — আপনার ইমেজ তালিকা দেখুন
- `podman ps -a` — শেষ হওয়া কন্টেইনারগুলো দেখুন
- `podman logs <container-id>` — যদি কনটেইনার চলতেই থাকে বা শেষ হয়েছে কিন্তু লোগস দরকার

উদাহরণ:

```bash
podman images
podman ps -a
podman logs <container-id>
```

টিপ: `podman ps -a` থেকে container-id নিন, তারপর `podman logs` চালান।

## বেশ কিছু সাধারণ বিকল্প (common flags)

- `--rm`: কনটেইনার শেষ হলে নিজে-মুছে যাবে (temporary runs)।
- `-it`: interactive টার্মিনাল (একটি shell/interactive app চালাতে)।
- `-d`: detached mode, ব্যাকগ্রাউন্ডে চালাতে।
- `-p HOST:CONTAINER`: পোর্ট বাইন্ডিং (local হোস্টে সার্ভিস এক্সপোজ করতে)।

উদাহরণ: একটি ইন্টারেকটিভ আলপাইন শেল

```bash
podman run --rm -it alpine sh
```

এখানে আপনি কনটেইনারের ভিতরে শেলে প্রবেশ করবেন, কমান্ড চালিয়ে দেখতে পারবেন।

## ছোটখাটো ডিবাগিং চেকলিস্ট

যদি কিছু না চলে, দ্রুত যাচাই করার জন্য:

1. `podman --version` — ইনস্টল আছে কি না
2. `podman info` — রuntime ও rootless/VM ইত্যাদি কনফিগারেশন দেখুন
3. পারমিশন ইস্যু? (rootless হলে `subuid/subgid` চেক করুন)
4. নেটওয়ার্ক প্রবলেম? macOS/Windows-এ Podman Machine চালু আছে কি না দেখুন (`podman machine list`)

## প্র্যাকটিক্যাল এক্সারসাইজ (Hands-on)

উদ্দেশ্য: Hello World থেকে একটু এগিয়ে কনটেইনার ইন্টারঅ্যাকশন ও লোগস দেখার অভ্যাস করা।

ধাপ ১ — Pull ও Run

```bash
podman pull hello-world
podman run --rm hello-world
```

নোট: আউটপুট স্ক্রীনে দেখা উচিত "Hello from Docker!" বা অনুরূপ বার্তা (imagen নির্ভর করে)।

ধাপ ২ — Interactive shell

```bash
podman run --rm -it alpine sh
# তারপর কনটেইনারের মধ্যে থেকে
echo "আমি কনটেইনারে আছি"
exit
```

ধাপ ৩ — ছোট ওয়েব সার্ভার করে দেখুন (পোর্ট এক্সপোজ)

```bash
podman run -d --name my-nginx -p 8080:80 nginx:alpine
podman ps
# ব্রাউজারে http://localhost:8080 খুলে Nginx ডিফল্ট পেজ দেখুন
podman logs my-nginx
podman rm -f my-nginx
```

ধাপ ৪ — ফলাফল নোট করুন ও কয়েক লাইন রিফ্লেকশন লিখুন

- কী দেখা গেল? কোন কমান্ডটি সবচেয়ে সহজ লাগলো? কোথায় বাধা পেলেন?

## নিরাপত্তা নোট

- প্রোডাকশনে কনটেইনার চালানোর আগে rootless বা systemd integration নিয়ে ভাবুন।
- آزمایش (testing) ও ডেভেলপমেন্টে `--rm` ব্যবহার করুন যাতে পরে অপ্রয়োজনীয় কনটেইনার ময়নাতদন্ত না জমে।

## সারমর্ম

এই লেসনে আমরা `podman pull`, `podman run`, ইন্টারঅ্যাকটিভ শেল ও একটি ছোট Nginx সার্ভার চালিয়ে দেখেছি। পরের লেসনে আমরা পডম্যান আর্কিটেকচার (daemonless, rootless) গভীরে দেখব এবং কিভাবে systemd-সহ কাজ করে তা হ্যান্ডস-অন করব।
