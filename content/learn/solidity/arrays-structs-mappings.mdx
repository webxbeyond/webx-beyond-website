### ১. Array

**একই ধরনের ডাটা একসাথে রাখার জন্য Array ব্যবহার করা হয়।**
- ধারাবাহিকভাবে ডাটা সংরক্ষণ (যেমন: রোল নম্বর, টোকেন আইডি)
- অ্যারে দুই ধরনের: Dynamic (আকার পরিবর্তনযোগ্য) ও Fixed (নির্দিষ্ট আকার)

**ব্যবহার:**
- ইউজার তালিকা, টোকেন আইডি, স্কোর

**উদাহরণ:**
```solidity
uint[] public scores = [10, 20, 30]; // স্কোরের তালিকা
string[] public names; // নামের তালিকা
```

**টিপস:**
- অ্যারে ইনডেক্স ০ থেকে শুরু হয়
- অ্যারে লুপ করে সব ডাটা পড়া যায়

### ২. Struct

**Struct দিয়ে একাধিক প্রপার্টি নিয়ে কাস্টম ডাটা টাইপ তৈরি করা যায়।**
- যেমন: ইউজার প্রোফাইল, প্রোডাক্ট ইনফো

**ব্যবহার:**
- ইউজার, প্রোডাক্ট, ট্রান্সাকশন

**উদাহরণ:**
```solidity
struct User {
    string name; // নাম
    uint age;    // বয়স
}
User public user = User("Anis", 25); // একজন ইউজার
```

**টিপস:**
- struct-এর ভিতরে array/mapping রাখা যায়
- struct initialize না করলে ডিফল্ট value থাকে

### ৩. Mapping

**Mapping হলো Key-Value ডাটা স্টোর—dictionary বা hash table-এর মতো।**
- যেমন: address থেকে balance, রোল নম্বর থেকে নাম

**ব্যবহার:**
- ব্যালেন্স ট্র্যাকিং, মালিকানা, vote count

**উদাহরণ:**
```solidity
mapping(address => uint) public balances; // address অনুযায়ী ব্যালেন্স
balances[msg.sender] = 100; // নিজের address-এ ১০০ সেট
```

**টিপস:**
- mapping-এ সব key এক্সিস্ট করে, ডিফল্ট value ০
- mapping-এ লুপ করা যায় না (iterate করা যায় না)

### ৪. পার্থক্য

| বিষয়         | Array (অ্যারে)    | Struct (স্ট্রাক্ট) | Mapping (ম্যাপিং) |
|--------------|-------------------|-------------------|-------------------|
| ডাটা টাইপ    | একটাই             | একাধিক            | Key-Value         |
| অ্যাক্সেস    | ইনডেক্স           | প্রপার্টি         | Key               |
| ব্যবহার      | তালিকা            | প্রোফাইল          | ব্যালেন্স/মালিকানা |

### ৫. কেন গুরুত্বপূর্ণ?

**কেন গুরুত্বপূর্ণ?**
- ডাটা সংগঠন ও অপ্টিমাইজেশন
- কনট্রাক্টের লজিক সহজ করা
- স্টোরেজ efficient রাখা

### ৬. Common Pitfalls

**সাধারণ ভুল:**
- Array Out of Bounds (ইনডেক্স ভুল হলে error)
- Struct initialize না করলে ডিফল্ট value
- Mapping-এ key না থাকলে ডিফল্ট value (০) পাওয়া যায়


## উদাহরণ ও এনালজি

**তুলনা ও উদাহরণ:**
- অ্যারে—"রোল নম্বরের তালিকা" (একই ধরনের ডাটা, যেমন: ছাত্রদের রোল নম্বর)
    - যেমন: একটি স্কুলে ১০০ জন ছাত্রের রোল নম্বর ধারাবাহিকভাবে রাখা হয়েছে।
- স্ট্রাক্ট—"ছাত্রের তথ্য" (একাধিক প্রপার্টি, যেমন: নাম, বয়স, গ্রেড)
    - যেমন: একজন ছাত্রের নাম, বয়স, ক্লাস, গ্রেড—সব তথ্য একসাথে struct-এ রাখা যায়।
- ম্যাপিং—"রোল নম্বর থেকে নাম খোঁজা" (key-value, যেমন: রোল নম্বর দিয়ে ছাত্রের নাম বের করা)
    - যেমন: dictionary-তে শব্দ দিয়ে অর্থ খোঁজা, address দিয়ে balance খোঁজা।

**আরও উদাহরণ:**
- অ্যারে: টোকেন আইডির তালিকা, ভোটারদের ভোটের হিস্ট্রি
- স্ট্রাক্ট: প্রোডাক্টের তথ্য (নাম, দাম, স্টক), ইউজার প্রোফাইল
- ম্যাপিং: address থেকে balance খোঁজা, ভোটার আইডি থেকে ভোটের সংখ্যা

## Quick Reference

## দ্রুত রেফারেন্স
- অ্যারে: `type[] name;` // অ্যারে ডিক্লেয়ার, যেমন: `uint[] scores;`
- স্ট্রাক্ট: `struct Name { ... }` // স্ট্রাক্ট ডিক্লেয়ার, যেমন: `struct User { string name; uint age; }`
- ম্যাপিং: `mapping(keyType => valueType) name;` // ম্যাপিং ডিক্লেয়ার, যেমন: `mapping(address => uint) balances;`

**টিপস:**
- অ্যারে ও ম্যাপিং একসাথে ব্যবহার করলে key list রাখা সহজ হয়
- স্ট্রাক্টের ভিতরে ম্যাপিং বা অ্যারে রাখা যায়

## উপসংহার

Array, Struct, Mapping—Solidity-তে ডাটা সংগঠনের মূল হাতিয়ার। প্রতিটি সেকশন ভালোভাবে বুঝে, সঠিকভাবে ব্যবহার করলে কনট্রাক্ট সহজ, দ্রুত ও নিরাপদ হয়।
ভুলত্রুটি এড়াতে:
- ইনডেক্স/কী ভুল হলে error বা ডিফল্ট value আসতে পারে
- সব ডাটা initialize করুন
- নামকরণ ও access control পরিষ্কার রাখুন
সেরা চর্চা অনুসরণ করলে Solidity-তে ডাটা স্ট্রাকচারে দক্ষতা অর্জন করবেন।
