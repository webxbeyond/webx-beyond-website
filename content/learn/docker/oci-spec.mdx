## মূল ধারণা

### ১. OCI Image Specification Explained

- **OCI:** Open Container Initiative—industry standard image format/specification।
- **Image structure:** Manifest, config, layer—standardized format।
- **Benefit:** Universal compatibility, security, automation।

### ২. Image Structure: Manifest, Config, Layer

- **Manifest:** Image metadata—layer info, config, digest।
- **Config:** Runtime config—env, cmd, entrypoint, user।
- **Layer:** Filesystem changes—each Dockerfile instruction-এ new layer।
- **Example:**
  - Manifest.json
  - Config.json
  - Layer.tar.gz

### ৩. Workflow: OCI Image Build & Deploy

1. **Dockerfile লিখুন—OCI compatible instruction।**
2. **docker build—OCI image create।**
3. **Image registry-তে push—OCI format enforced।**
4. **Any runtime-এ deploy—Docker, Kubernetes, Podman।**

### ৪. Validation & Security

- **Image validation:** OCI spec compatible—tool দিয়ে check করুন।
- **Security scan:** Standardized format—supply chain vulnerability detect।
- **Automation:** CI/CD pipeline-এ OCI image build/test/deploy enforced।

### ৫. সহজ এনালজি

- "OCI image spec মানে—একটি আন্তর্জাতিক পাসপোর্ট, যেটা সব দেশে (runtime/platform) accept হয়।"

### ৬. Best Practices

- **Always OCI compatible image build করুন।**
- **Manifest/config/layer—standard format enforced।**
- **Validation tool:** OCI image validator/tool use করুন।

- **Multi-cloud deploy:** Same image—multiple platform-এ run করুন।
