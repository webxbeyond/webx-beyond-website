## ১। কি উদ্দেশ্য

`docker run hello-world` হলো Docker ইনস্টলেশন সঠিক আছে কি না তা দ্রুত যাচাই করার সবচেয়ে সহজ উপায়। এটি একটি ছোট টেস্ট কনটেইনার রান করে এবং একটি প্রোবিং মেসেজ প্রিন্ট করে।

## ২। কিভাবে চালাবেন — ধাপে ধাপে

1. নিশ্চিত করুন Docker ইনস্টল ও Daemon চলছে (`docker --version`, `docker info`)।
2. টার্মিনাল খুলে নিচের কমান্ড দিন:

```bash
docker run hello-world
```

3. আপনি দেখতে পাবেন কিছুটা এমন আউটপুট:

```
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

এই কমান্ডের কাজ: যদি লোকাল মেশিনে `hello-world` ইমেজ না থাকে, Docker ইমেজটি registry থেকে pull করে এবং কনটেইনার চালায়; কনটেইনারটি নিজে থেকে চেক করে একটি মেইসেজ প্রিন্ট করে এবং শেষ হয়ে যায়।

## ৩। বিকল্প এবং বেশি তথ্যপূর্ণ রান

- `docker run --rm hello-world` — কনটেইনার শেষ হলে নিজে থেকেই মুছে যাবে।
- `docker run -it alpine sh` — ইন্টারেক্টিভ শেল চালিয়ে কনটেইনার ইনসাইট দেখতে পারবেন (প্রায়শই ডিবাগিংয়ে কাজে লাগে)।

## ৪। কীভাবে এটা কাজ করে (আর্কিটেকচারিক্যাল সংক্ষিপ্ত)

- CLI: আপনি `docker run` টাইপ করেন।
- Daemon: যদি লোকাল ইমেজ না থাকে, Daemon ইমেজ pull করে।
- Engine: কনটেইনার তৈরি ও রান করে, আউটপুট stdout এ প্রিন্ট হয়।

## ৫। সাধারণ সমস্যা ও সমাধান

- Permission denied (socket): `Got permission denied while trying to connect to the Docker daemon socket` — Linux-এ আপনার ইউজার `docker` গ্রুপে আছে কি না চেক করুন বা `sudo` দিয়ে চালান।
- Daemon not running: `Cannot connect to the Docker daemon` — `sudo systemctl start docker` বা Docker Desktop চালু করুন।
- Image pull fails: নেটওয়ার্ক / proxy চেক করুন; `docker pull hello-world` চালিয়ে error message দেখুন।

## ৬। কেন এই টেস্টটি গুরুত্বপূর্ণ

- দ্রুত নিশ্চয়তা দেয় যে Docker CLI ও Daemon একসাথে কাজ করছে।
- নতুন সিস্টেম বা CI agent-এ অটোমেটেড healthcheck হিসেবে ব্যবহার করা যায়।

## ৭। পরবর্তী ধাপ

এই টেস্ট পাস করলে: `Docker Images: Pull, Build, Tag` লেসনে যান যেখানে আমরা কিভাবে ইমেজ বানাবেন ও ট্যাগ করবেন তা দেখব।
