# Macvlan, IPVLAN, Custom DNS ও IPAM: ডকার নেটওয়ার্কিং-এর উন্নত কৌশল

## মূল ধারণা

### ১. Macvlan Explained

- **Macvlan:** Physical network-এ container-কে direct IP assign করে।
- **Each container—own MAC address, own IP—like real device।**
- **Benefit:** Legacy network integration, direct access, high performance।

#### Workflow

1. **Macvlan network create করুন:**
   ```bash
   docker network create -d macvlan \
     --subnet=192.168.1.0/24 \
     --gateway=192.168.1.1 \
     -o parent=eth0 my_macvlan
   ```
2. **Container run করুন—network attach করুন:**
   ```bash
   docker run -d --name myapp --network my_macvlan nginx
   ```

#### সহজ এনালজি

- "Macvlan মানে—একটি নতুন ফোন নম্বর, যেটা দিয়ে সরাসরি কল করা যায়—no extension needed।"

### ২. IPVLAN Explained

- **IPVLAN:** Physical network-এ multiple container—same MAC, different IP—assign করে।
- **Benefit:** Efficient, less network overhead, cloud/VM-friendly।

#### Workflow

1. **IPVLAN network create করুন:**
   ```bash
   docker network create -d ipvlan \
     --subnet=192.168.2.0/24 \
     --gateway=192.168.2.1 \
     -o parent=eth0 my_ipvlan
   ```
2. **Container run করুন—network attach করুন:**
   ```bash
   docker run -d --name myapp2 --network my_ipvlan nginx
   ```

#### সহজ এনালজি

- "IPVLAN মানে—একটি অফিসে সবাই একই ল্যান্ডলাইন (MAC), কিন্তু আলাদা extension (IP) পায়।"

### ৩. Custom DNS Configuration

- **Custom DNS:** Container-এ নিজের DNS server set করা যায়—service discovery, internal/external resolution।
- **Benefit:** Microservices, hybrid cloud—flexible name resolution।

#### Workflow

1. **Container run করুন—custom DNS set করুন:**
   ```bash
   docker run -d --name web --dns 8.8.8.8 nginx
   ```
2. **docker-compose.yml-এ DNS set করুন:**
   ```yaml
   services:
     app:
       image: nginx
       dns:
         - 8.8.8.8
         - 1.1.1.1
   ```

### ৪. IPAM (IP Address Management)

- **IPAM:** Custom IP allocation—subnet, gateway, IP range—define করা যায়।
- **Benefit:** Complex topology, multi-subnet, overlapping network—manage করা সহজ।

#### Workflow

1. **Custom IPAM network create করুন:**
   ```bash
   docker network create -d bridge \
     --subnet=10.10.0.0/16 \
     --ip-range=10.10.1.0/24 \
     --gateway=10.10.0.1 my_custom_net
   ```

### ৫. Best Practices

- **Production-এ Macvlan/IPVLAN—direct access, high performance।**
- **Custom DNS/IPAM—complex topology, microservices-এ must-have।**
- **Network monitoring—performance/security ensure করুন।**
- **Documentation—network config clearly maintain করুন।**

### ৬. সাধারণ ভুলত্রুটি

- Overlapping subnet—IP conflict।
- Custom DNS/IPAM ভুল—service discovery break।
- Security config miss—exposed network risk।
