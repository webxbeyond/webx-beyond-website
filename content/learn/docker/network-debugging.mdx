## মূল ধারণা

### ১. netshoot: All-in-one Network Debugging Container

- **netshoot:** Special Docker image—network troubleshooting tool (ping, curl, dig, traceroute, etc.) pre-installed।
- **Benefit:** Any container/network—quick debug—no extra install needed।

#### Workflow

1. **netshoot container run করুন:**
   ```bash
   docker run -it --rm --network my_network nicolaka/netshoot
   ```
2. **Tools available:** ping, curl, dig, traceroute, nslookup, ip, etc.
3. **Example:**
   ```bash
   ping google.com
   curl http://my-service:8080
   dig my-service
   traceroute my-service
   ```

#### সহজ এনালজি

- "netshoot মানে—একটি সুপার টুলবক্স, যেখানে সব network tool একসাথে আছে।"

### ২. ping: Connectivity Test

- **ping:** Container থেকে অন্য container/service-এ connectivity test—packet loss, latency check।
- **Benefit:** Network reachability, firewall issue, DNS resolution—quick test।

#### Workflow

1. **Container-এ ping install করুন (Debian/Ubuntu):**
   ```bash
   apt-get update && apt-get install iputils-ping
   ping my-service
   ```
2. **netshoot-এ ping use করুন:**
   ```bash
   ping my-service
   ```

#### সহজ এনালজি

- "ping মানে—একটি ডাক পাঠানো, উত্তর এলে জানবেন যোগাযোগ হচ্ছে।"

### ৩. curl: HTTP Connectivity & API Test

- **curl:** Container/service-এ HTTP request—API, web service test—response, latency, error code check।
- **Benefit:** Web/API connectivity, authentication, SSL—debug করা সহজ।

#### Workflow

1. **Container-এ curl install করুন (Debian/Ubuntu):**
   ```bash
   apt-get update && apt-get install curl
   curl http://my-service:8080
   ```
2. **netshoot-এ curl use করুন:**
   ```bash
   curl http://my-service:8080
   ```

#### সহজ এনালজি

- "curl মানে—একটি চিঠি পাঠানো, উত্তর এলে জানবেন service ঠিক আছে।"

### ৪. Common Debugging Scenarios

- **DNS issue:** dig/nslookup/netshoot-এ test করুন।
- **Firewall issue:** ping/curl—connectivity test করুন।
- **Service discovery:** ping/curl—service name-based communication test করুন।
- **Latency/performance:** ping/traceroute—delay/source detect করুন।

### ৫. Best Practices

- **Always netshoot use করুন—quick debug, all tool available।**
- **Minimal container—ping/curl install করুন—debug সহজ করুন।**
- **Network monitoring—performance bottleneck detect করুন।**
- **Documentation—debugging steps clearly maintain করুন।**

### ৬. সাধারণ ভুলত্রুটি

- netshoot না use করা—tool install করতে সময় নষ্ট।
- ping/curl install miss—debugging fail।
- Wrong network attach—service unreachable।
- Debugging steps/documentation miss—issue reproduce কঠিন।
