# Image Build & Push Automation: Docker Workflow-এ পূর্ণ অটোমেশন

## প্রস্তাবনা

Google, Netflix, Uber—modern app-এ code change থেকে production deploy—speed, reliability, security—critical। Docker image build & push—automation—CI/CD pipeline-এ industry standard। Manual build/push—slow, error-prone; automation—fast, repeatable, secure।

- **Speed:** Manual build/push—slow; automation—seconds/minutes-এ complete।
- **Reliability:** Human error কমে; same workflow—every deploy-এ।
- **Security:** Automated scan/test—production-এ only safe image push হয়।

### ২. Typical Workflow

1. **Code change—push করুন (GitHub/GitLab)।**
2. **CI/CD pipeline—auto trigger—Docker image build।**
3. **Build complete—auto push—Docker Hub/Private Registry-তে।**
4. **Automated test/security scan—production-এ only safe image।**

### ৩. Example: GitHub Actions-এ Build & Push Automation

1. **.github/workflows/docker.yml লিখুন:**
   ```yaml
   name: Build & Push Docker Image
   on:
     push:
       branches: [ main ]
   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v2
         - name: Build Docker image
           run: docker build -t myapp:${{ github.sha }} .
         - name: Login to Docker Hub
           run: echo ${{ secrets.DOCKER_PASSWORD }} | docker login -u ${{ secrets.DOCKER_USERNAME }} --password-stdin
         - name: Push Docker image
           run: docker push myapp:${{ github.sha }}
   ```
2. **Push করলে—workflow auto trigger—build/push।**

### ৪. Example: Jenkins-এ Build & Push Automation

1. **Jenkinsfile লিখুন:**
   ```groovy
   pipeline {
     agent any
     stages {
       stage('Build') {
         steps {
           sh 'docker build -t myapp:${BUILD_NUMBER} .'
         }
       }
       stage('Push') {
         steps {
           withCredentials([usernamePassword(credentialsId: 'docker-hub', usernameVariable: 'DOCKER_USER', passwordVariable: 'DOCKER_PASS')]) {
             sh 'echo $DOCKER_PASS | docker login -u $DOCKER_USER --password-stdin'
             sh 'docker push myapp:${BUILD_NUMBER}'
           }
         }
       }
     }
   }
   ```
2. **Build/push—auto trigger—every code change-এ।**

### ৫. Best Practices

- **Unique tag/version—every build-এ use করুন।**
- **Secrets—environment variable/secret manager-এ store করুন।**
- **Automated test/security scan—pipeline-এ enforce করুন।**
- **Rollback strategy—deploy fail হলে restore করুন।**
- **Documentation—workflow config clear রাখুন।**

### ৬. সাধারণ ভুলত্রুটি

- Latest tag—production-এ use—rollback/trace কঠিন।
- Manual build/push—automation miss—bug detect late।
- Secrets—codebase-এ store—security risk।
- Pipeline config/documentation miss—debugging কঠিন।
