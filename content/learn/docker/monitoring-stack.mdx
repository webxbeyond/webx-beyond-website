# Prometheus + Grafana দিয়ে মনিটরিং: Docker Performance Visualization

## প্রস্তাবনা

Google, Netflix, Uber—সব বড় কোম্পানি production-এ হাজার হাজার container চালায়। Container-এ কী হচ্ছে, resource usage, error, latency—সবকিছু জানতে real-time monitoring অপরিহার্য। Prometheus + Grafana—industry-standard monitoring stack—Docker performance visualize ও alert করতে ব্যবহৃত হয়।

## কেন গুরুত্বপূর্ণ?

- **Real-time Monitoring:** Container performance, resource usage, error—সবকিছু live track করা যায়।
- **Visualization:** Metrics graph/chart-এ দেখে bottleneck, anomaly, trend detect করা যায়।
- **Alerting:** Threshold cross করলে automated alert trigger হয়।
- **Capacity Planning:** Historical data দেখে future resource estimate করা যায়।

## বাস্তব উদাহরণ

- **Netflix:** Prometheus + Grafana দিয়ে millions of container monitor ও visualize হয়।
- **Uber:** Real-time alerting দিয়ে latency, error rate detect ও fix হয়।
- **Google:** Kubernetes/Docker monitoring—Prometheus/Grafana industry standard।

## মূল ধারণা

### ১. Prometheus: Metrics Collector

- **Prometheus:** Open-source metrics collector ও time-series database।
- **Exporter:** Docker metrics collect করতে exporter (cAdvisor, node-exporter) ব্যবহার হয়।
- **Scraping:** Prometheus regular interval-এ metrics collect করে।

- **Docker Compose Example:**

```yaml
version: '3.8'
services:
  prometheus:
    image: prom/prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
  cadvisor:
    image: gcr.io/google-containers/cadvisor:latest
    ports:
      - "8080:8080"
  grafana:
    image: grafana/grafana
    ports:
      - "3000:3000"
```

### ২. Grafana: Visualization Tool

- **Grafana:** Open-source visualization tool—Prometheus metrics graph/chart-এ দেখায়।
- **Dashboard:** Custom dashboard create করে CPU, memory, network, error visualize করা যায়।
- **Alerting:** Dashboard-এ alert rule set করা যায়।

### ৩. Monitoring Workflow

1. **Exporter Deploy:** cAdvisor/node-exporter container run করুন।
2. **Prometheus Configure:** prometheus.yml-এ exporter endpoint add করুন।
3. **Grafana Setup:** Prometheus data source add করুন, dashboard create করুন।
4. **Alerting Configure:** Threshold set করুন, email/slack alert enable করুন।

### ৪. Advanced Features

- **Query Language (PromQL):** Custom metrics query করা যায়।
- **Retention Policy:** Historical data কতদিন store থাকবে তা set করা যায়।
- **Integration:** Slack, email, webhook—alert integration করা যায়।

## সাধারণ ভুলত্রুটি

- Exporter/container run না করা—metrics collect হয় না।
- Prometheus scrape interval ভুল set করা—data delay/overload হয়।
- Grafana dashboard customize না করা—important metrics miss হয়।
- Alerting configure না করা—critical issue unnoticed থাকে।

## সেরা অনুশীলন (Best Practices)

- Exporter/container properly run ও configure করুন।
- Prometheus scrape interval optimize করুন।
- Grafana dashboard customize ও regularly review করুন।
- Alerting rule set ও test করুন।
- Historical data retention policy configure করুন।
