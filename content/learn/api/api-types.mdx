# পাবলিক, প্রাইভেট, পার্টনার, ওপেন, কম্পোজিট

## এপিআই টাইপস কী?

API-এর বিভিন্ন ধরন আছে, প্রতিটি নির্দিষ্ট উদ্দেশ্য ও ব্যবহারের জন্য। নিচে জনপ্রিয় API টাইপগুলোর ব্যাখ্যা ও উদাহরণ দেয়া হলো:

## ১. পাবলিক API (Public API)
- **সংজ্ঞা:** সকল ডেভেলপার ও কোম্পানির জন্য উন্মুক্ত।
- **ব্যবহার:** থার্ড-পার্টি অ্যাপ, ইন্টিগ্রেশন, ওপেন ইনোভেশন।
- **উদাহরণ:** Twitter API, Google Maps API

## ২. প্রাইভেট API (Private API)
- **সংজ্ঞা:** শুধুমাত্র একটি কোম্পানির অভ্যন্তরীণ ব্যবহারের জন্য।
- **ব্যবহার:** ইন্টারনাল টুল, ডেটা শেয়ারিং, অটোমেশন।
- **উদাহরণ:** একটি ব্যাংকের অভ্যন্তরীণ অ্যাকাউন্ট ম্যানেজমেন্ট API

## ৩. পার্টনার API (Partner API)
- **সংজ্ঞা:** নির্দিষ্ট পার্টনার বা কোম্পানির জন্য সীমিত অ্যাক্সেস।
- **ব্যবহার:** বিজনেস টু বিজনেস (B2B) ইন্টিগ্রেশন, কাস্টমার/ভেন্ডর ইন্টিগ্রেশন।
- **উদাহরণ:** পেমেন্ট গেটওয়ে API, যেখানে নির্দিষ্ট মার্চেন্টদের অ্যাক্সেস দেয়া হয়।

## ৪. ওপেন API (Open API)
- **সংজ্ঞা:** পাবলিক API-এর মতো, তবে ওপেন স্ট্যান্ডার্ড ও ডকুমেন্টেশনসহ।
- **ব্যবহার:** ওপেন সোর্স, কমিউনিটি ডেভেলপমেন্ট, স্ট্যান্ডার্ডাইজেশন।
- **উদাহরণ:** OpenWeatherMap API, OpenAPI Specification (Swagger)

## ৫. কম্পোজিট API (Composite API)
- **সংজ্ঞা:** একাধিক API কলকে একত্রে একটি রিকোয়েস্টে কম্বাইন করা যায়।
- **ব্যবহার:** মাইক্রোসার্ভিস আর্কিটেকচারে একাধিক ডেটা সোর্স থেকে একসাথে ডেটা আনা।
- **উদাহরণ:** একটি কম্পোজিট API কল দিয়ে ইউজার প্রোফাইল, অর্ডার হিস্ট্রি ও পেমেন্ট ডিটেইল একসাথে পাওয়া।

## তুলনামূলক টেবিল

| টাইপ         | অ্যাক্সেস | ব্যবহার | উদাহরণ |
|--------------|----------|---------|---------|
| পাবলিক      | উন্মুক্ত  | থার্ড-পার্টি, ওপেন | Google Maps |
| প্রাইভেট    | সীমিত    | ইন্টারনাল | ব্যাংক API |
| পার্টনার    | পার্টনার | B2B      | পেমেন্ট গেটওয়ে |
| ওপেন        | উন্মুক্ত  | ওপেন সোর্স | OpenWeatherMap |
| কম্পোজিট    | নির্ভর করে | মাইক্রোসার্ভিস | কম্বো API |

## কখন কোন API ব্যবহার করবেন?
- **পাবলিক:** যখন আপনি চান সবাই আপনার ডেটা/সার্ভিস ব্যবহার করুক।
- **প্রাইভেট:** শুধুমাত্র নিজের কোম্পানির জন্য।
- **পার্টনার:** নির্দিষ্ট পার্টনারদের জন্য।
- **ওপেন:** ওপেন সোর্স ও স্ট্যান্ডার্ডাইজড ইন্টিগ্রেশন দরকার হলে।
- **কম্পোজিট:** একাধিক অপারেশন একসাথে করতে হলে।

> পরবর্তী অধ্যায়ে: API কমিউনিকেশন—রিকোয়েস্ট-রেসপন্স, এন্ডপয়েন্ট, প্যারাম ইত্যাদি।
