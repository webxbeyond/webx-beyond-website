# প্রোটোকল বাফার, স্ট্রিমিং, পারফরম্যান্স

gRPC হলো Google-এর তৈরি একটি হাই-পারফরম্যান্স, ওপেন সোর্স RPC (Remote Procedure Call) ফ্রেমওয়ার্ক।

## ১. প্রোটোকল বাফার (Protocol Buffers)
- **সংজ্ঞা:** ডেটা সিরিয়ালাইজেশনের জন্য Google-এর তৈরি বাইনারি ফরম্যাট।
- **সুবিধা:** দ্রুত, কম ব্যান্ডউইথ, টাইপ-সেফ।
- **উদাহরণ:**
    ```proto
    message User {
      int32 id = 1;
      string name = 2;
      string email = 3;
    }
    ```

## ২. স্ট্রিমিং (Streaming)
- **সংজ্ঞা:** ক্লায়েন্ট ও সার্ভার উভয় দিক থেকে একাধিক মেসেজ পাঠানো যায় (bidirectional streaming)।
- **টাইপ:**
  - Unary (একটি রিকোয়েস্ট, একটি রেসপন্স)
  - Server streaming (একটি রিকোয়েস্ট, একাধিক রেসপন্স)
  - Client streaming (একাধিক রিকোয়েস্ট, একটি রেসপন্স)
  - Bidirectional streaming (উভয় দিকেই একাধিক মেসেজ)
- **উদাহরণ:**
    ```proto
    service Chat {
      rpc ChatStream(stream Message) returns (stream Message);
    }
    ```

## ৩. পারফরম্যান্স (Performance)
- **বৈশিষ্ট্য:**
  - REST/JSON-এর তুলনায় অনেক দ্রুত
  - কম লেটেন্সি, কম ডেটা সাইজ
  - মাইক্রোসার্ভিস আর্কিটেকচারে জনপ্রিয়

## gRPC বনাম REST
- gRPC বাইনারি ফরম্যাট (Protobuf), REST সাধারণত JSON
- gRPC স্ট্রং টাইপড, REST ফ্লেক্সিবল
- gRPC স্ট্রিমিং সাপোর্ট করে, REST সাধারণত করে না

## সংক্ষেপে
- gRPC API-তে প্রোটোকল বাফার, স্ট্রিমিং ও পারফরম্যান্স—তিনটি মূল কনসেপ্ট
- মডার্ন, স্কেলেবল ও হাই-পারফরম্যান্স API ডিজাইন

> পরবর্তী অধ্যায়ে: WebSocket—রিয়েল-টাইম, ইভেন্ট-ড্রিভেন ডিজাইন
