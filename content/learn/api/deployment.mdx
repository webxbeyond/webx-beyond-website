## সার্ভার, সার্ভারলেস, কনটেইনার

API ডিপ্লয়মেন্টের জন্য তিনটি প্রধান মডেল—সার্ভার, সার্ভারলেস ও কনটেইনার। প্রতিটির সুবিধা, চ্যালেঞ্জ ও ব্যবহার ক্ষেত্র আলাদা।

## ১. সার্ভার (Traditional Server)
- **বৈশিষ্ট্য:**
  - নিজস্ব VM/Physical Server-এ API হোস্ট
  - OS, আপডেট, স্কেলিং নিজে ম্যানেজ করতে হয়
- **ডিপ্লয় টুল:** Nginx, Apache, PM2, Supervisor
- **ব্যবহার:** ছোট/মাঝারি প্রজেক্ট, কাস্টম কনফিগ দরকার হলে

## ২. সার্ভারলেস (Serverless)
- **বৈশিষ্ট্য:**
  - কোড ফাংশন হিসেবে রান হয় (Function-as-a-Service)
  - সার্ভার ম্যানেজমেন্ট, স্কেলিং, বিলিং—সব ক্লাউডে অটোমেটেড
- **ডিপ্লয় টুল:** AWS Lambda, Azure Functions, Google Cloud Functions, Vercel, Netlify
- **ব্যবহার:** ইভেন্ট-ড্রিভেন, স্কেলেবল, কস্ট-এফিশিয়েন্ট API

## ৩. কনটেইনার (Container)
- **বৈশিষ্ট্য:**
  - API ও ডিপেন্ডেন্সি একসাথে প্যাকেজ (Docker)
  - সহজ স্কেলিং, ইমেজ রিপ্লিকেশন, ক্লাস্টারিং
- **ডিপ্লয় টুল:** Docker, Kubernetes, AWS ECS, GCP Cloud Run
- **ব্যবহার:** মাইক্রোসার্ভিস, স্কেলেবল ও ক্লাউড-নেটিভ API

## উদাহরণ: Docker-এ API ডিপ্লয়
```dockerfile
FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["npm", "start"]
```
- রান: `docker build -t myapi . && docker run -p 3000:3000 myapi`

## বেস্ট প্র্যাকটিস
- এনভায়রনমেন্ট ভ্যারিয়েবল দিয়ে কনফিগ ম্যানেজ করুন
- অটো-স্কেলিং ও মনিটরিং এনাবল করুন
- CI/CD দিয়ে ডিপ্লয়মেন্ট অটোমেট করুন
- ডকুমেন্টেশন ও ব্যাকআপ রাখুন

## টিপস
- ছোট প্রজেক্টে Vercel/Netlify-তে সার্ভারলেস ডিপ্লয় সহজ
- বড় প্রজেক্টে Docker/Kubernetes দিয়ে স্কেল করুন
- ক্লাউড প্ল্যাটফর্মের ফ্রি টায়ার ট্রাই করুন
