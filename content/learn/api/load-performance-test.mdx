লোড/পারফরম্যান্স টেস্ট API-র স্কেল, রেসপন্স টাইম ও স্ট্রেস কেমন হ্যান্ডল করে তা যাচাইয়ের জন্য করা হয়। এতে প্রোডাকশন ইস্যু, বটলনেক ও স্কেলিং প্ল্যান সহজ হয়।

## লোড/পারফরম্যান্স টেস্ট কী?
- অনেকগুলো কনকারেন্ট রিকোয়েস্ট দিয়ে API-এর পারফরম্যান্স, লেটেন্সি, থ্রুপুট, এরর রেট মাপা
- লোড, স্ট্রেস, স্পাইক, সাসটেইনড লোড—বিভিন্ন ধরনের টেস্ট

## জনপ্রিয় টুল
- **k6:** JavaScript-ভিত্তিক, CLI ও ক্লাউড সাপোর্ট
- **JMeter:** GUI ও CLI, এক্সটেনসিভ কনফিগ

## উদাহরণ: k6
```js
import http from 'k6/http';
import { check, sleep } from 'k6';
export default function () {
  let res = http.get('http://localhost:3000/api');
  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}
```
- রান: `k6 run script.js`

## উদাহরণ: JMeter
- GUI দিয়ে টেস্ট প্ল্যান তৈরি
- Thread Group, HTTP Request, Assertion কনফিগার
- রান: CLI বা GUI

## বেস্ট প্র্যাকটিস
- প্রোডাকশন-লাইক এনভায়রনমেন্টে টেস্ট করুন
- গ্র্যাজুয়াল লোড বাড়ান (ramp-up)
- রেসপন্স টাইম, এরর রেট, থ্রুপুট মনিটর করুন
- টেস্ট রিপোর্ট ও গ্রাফ বিশ্লেষণ করুন

## টিপস
- CI/CD-তে লোড টেস্ট অটোমেট করুন
- k6 Cloud বা JMeter Dashboard দিয়ে রিপোর্ট শেয়ার করুন
- API Gateway/Server log মনিটর করুন
