## JSON, XML, প্যাজিনেশন, ফিল্টারিং

REST API-তে ডেটা কীভাবে পাঠানো ও গ্রহণ করা হয়, এবং বড় ডেটাসেট কিভাবে প্যাজিনেশন ও ফিল্টারিং করা হয়, তা জানা জরুরি।

## ১. ডেটা ফরম্যাট: JSON ও XML
- **JSON (JavaScript Object Notation):**
  - সবচেয়ে জনপ্রিয়, হিউম্যান-রিডেবল, লাইটওয়েট
  - উদাহরণ:
    ```json
    {
      "id": 123,
      "name": "Rahim"
    }
    ```
- **XML (eXtensible Markup Language):**
  - পুরনো ও কিছু এন্টারপ্রাইজ সিস্টেমে ব্যবহৃত
  - উদাহরণ:
    ```xml
    <user>
      <id>123</id>
      <name>Rahim</name>
    </user>
    ```

## ২. প্যাজিনেশন (Pagination)
- **কেন দরকার:** বড় ডেটাসেট একবারে পাঠালে পারফরম্যান্স খারাপ হয়, তাই ছোট ছোট ভাগে পাঠানো হয়।
- **পদ্ধতি:**
  - **Offset-based:** `?page=2&limit=10`
  - **Cursor-based:** `?cursor=abc123&limit=10`
- **উদাহরণ:**
  - `/users?page=2&limit=20`
- **রেসপন্সে মেটাডেটা:**
    ```json
    {
      "data": [...],
      "page": 2,
      "total": 100
    }
    ```

## ৩. ফিল্টারিং (Filtering)
- **কেন দরকার:** নির্দিষ্ট ডেটা খুঁজে বের করতে।
- **পদ্ধতি:**
  - Query Parameter: `/products?category=mobile&price_lt=20000`
- **বেস্ট প্র্যাকটিস:**
  - মাল্টিপল ফিল্টার সাপোর্ট
  - স্পষ্ট naming (যেমন: `price_lt`, `created_after`)

## সংক্ষেপে
- JSON সবচেয়ে জনপ্রিয় ডেটা ফরম্যাট, XML এখনও কিছু জায়গায় ব্যবহৃত
- প্যাজিনেশন ও ফিল্টারিং API-কে স্কেলেবল ও ইউজার-ফ্রেন্ডলি করে তোলে
