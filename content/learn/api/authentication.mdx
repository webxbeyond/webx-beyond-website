# Basic, API Key, OAuth2, JWT, SSO

API অথেন্টিকেশন মানে হচ্ছে ইউজার বা অ্যাপ্লিকেশন কে যাচাই করা—তারা কে এবং তাদের কি অনুমতি আছে। নিচে জনপ্রিয় অথেন্টিকেশন মেথডগুলোর ব্যাখ্যা ও উদাহরণ দেওয়া হলো:

## ১. Basic Authentication
- ইউজারনেম ও পাসওয়ার্ড Base64 এনকোড করে HTTP Header-এ পাঠানো হয়।
- উদাহরণ:
  ```http
  Authorization: Basic dXNlcjpwYXNz
  ```
- সহজ, কিন্তু নিরাপত্তা কম (TLS ছাড়া ব্যবহার করা ঠিক নয়)।

## ২. API Key
- সার্ভার থেকে জেনারেট করা ইউনিক কী ক্লায়েন্টকে দেওয়া হয়।
- ক্লায়েন্ট প্রতিটি রিকোয়েস্টে কী পাঠায় (Header/Query Param)।
- উদাহরণ:
  ```http
  GET /api/data
  X-API-Key: 123456abcdef
  ```
- সহজ ইমপ্লিমেন্টেশন, কিন্তু কী লিক হলে নিরাপত্তা ঝুঁকি।

## ৩. OAuth2
- থার্ড-পার্টি অথেন্টিকেশন ও অথরাইজেশনের জন্য স্ট্যান্ডার্ড ফ্লো।
- ইউজার, ক্লায়েন্ট, অথরাইজেশন সার্ভার ও রিসোর্স সার্ভার ইনভলভড।
- উদাহরণ: Google, Facebook Login
- ফ্লো: Authorization Code, Implicit, Client Credentials, Resource Owner Password
- নিরাপদ ও স্কেলেবল, কিন্তু ইমপ্লিমেন্টেশন জটিল।

## ৪. JWT (JSON Web Token)
- ইউজার লগইন করলে সার্ভার থেকে সাইনড টোকেন (JWT) পাওয়া যায়।
- ক্লায়েন্ট প্রতিটি রিকোয়েস্টে টোকেন পাঠায় (Header-এ)।
- উদাহরণ:
  ```http
  Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  ```
- টোকেনে ইউজার ইনফো ও এক্সপায়ারি থাকে।
- স্ট্যাটলেস অথেন্টিকেশন, স্কেলেবল ও দ্রুত।

## ৫. SSO (Single Sign-On)
- একবার লগইন করলেই একাধিক অ্যাপ/সার্ভিসে এক্সেস পাওয়া যায়।
- OAuth2, SAML, OpenID Connect ইত্যাদির মাধ্যমে ইমপ্লিমেন্ট হয়।
- কর্পোরেট ও এন্টারপ্রাইজ অ্যাপে বেশি ব্যবহৃত।

## তুলনামূলক টেবিল
| মেথড         | নিরাপত্তা | ইমপ্লিমেন্টেশন | স্কেল | ব্যবহারের ক্ষেত্র |
|--------------|-----------|----------------|-------|-------------------|
| Basic        | কম        | সহজ           | ছোট  | টেস্ট, ইন্টারনাল  |
| API Key      | মাঝারি    | সহজ           | ছোট/মাঝারি | পাবলিক API |
| OAuth2       | বেশি      | জটিল          | বড়   | পাবলিক/এন্টারপ্রাইজ|
| JWT          | বেশি      | মাঝারি         | বড়   | ওয়েব/মোবাইল API  |
| SSO          | বেশি      | জটিল          | বড়   | এন্টারপ্রাইজ      |

## টিপস
- সবসময় HTTPS ব্যবহার করুন।
- API Key/Token কখনোই ক্লায়েন্ট-সাইড কোডে এক্সপোজ করবেন না।
- টোকেন এক্সপায়ারি ও রিভোকেশন সিস্টেম রাখুন।
- প্রয়োজন অনুযায়ী অথেন্টিকেশন মেথড বেছে নিন।
