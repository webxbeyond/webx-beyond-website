# নেমিং ও কনভেনশন

API ডিজাইনে কনসিস্টেন্ট ও অর্থবহ নামকরণ (naming) এবং কনভেনশন ফলো করা খুব গুরুত্বপূর্ণ। এতে API পড়া, ব্যবহার ও মেইনটেইন করা সহজ হয়।

## ১. রিসোর্স নেমিং
- **প্লুরাল (বহুবচন) ব্যবহার করুন:** `/users`, `/orders`
- **নাউন (noun) ব্যবহার করুন:** `/products`, `/categories`
- **ক্রিয়াপদ (verb) এড়িয়ে চলুন:** `/getUser` নয়, `/users` লিখুন

## ২. কেস কনভেনশন
- **কেবাব কেস (kebab-case):** `/user-orders`
- **স্নেক কেস (snake_case):** `/user_orders`
- **ক্যামেল কেস (camelCase):** সাধারণত query param বা JSON property-তে

## ৩. এন্ডপয়েন্ট কনভেনশন
- **নেস্টেড রিসোর্স:** `/users/123/orders/456`
- **ফিল্টারিং ও প্যাজিনেশন:** `/products?category=mobile&page=2`
- **অ্যাকশন:** `/users/123/activate` (যদি বিশেষ অ্যাকশন দরকার হয়)

## ৪. HTTP মেথড কনভেনশন
- **GET:** ডেটা পড়া
- **POST:** নতুন তৈরি
- **PUT/PATCH:** আপডেট
- **DELETE:** ডিলিট

## ৫. কনসিস্টেন্সি
- সব API-তে একই কনভেনশন ফলো করুন
- ডকুমেন্টেশনে স্পষ্টভাবে উল্লেখ করুন

## উদাহরণ
```
GET /api/v1/users/123/orders?status=active
```

## সংক্ষেপে
- API নেমিং ও কনভেনশন মানলে API হবে পরিষ্কার, কনসিস্টেন্ট ও ইউজার-ফ্রেন্ডলি
