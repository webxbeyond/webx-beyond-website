# HTTP/HTTPS, gRPC, WebSocket, MQTT, GraphQL

API-তে ডেটা আদান-প্রদানের জন্য বিভিন্ন প্রোটোকল ব্যবহৃত হয়। প্রতিটির নিজস্ব সুবিধা ও ব্যবহার ক্ষেত্র আছে।

## ১. HTTP/HTTPS
- **সংজ্ঞা:** ওয়েবের মূল প্রোটোকল। HTTP (HyperText Transfer Protocol), HTTPS (HTTP Secure—TLS/SSL দিয়ে এনক্রিপ্টেড)।
- **ব্যবহার:** REST API, ওয়েব সার্ভিস, ব্রাউজার-ভিত্তিক অ্যাপ।
- **বৈশিষ্ট্য:** সহজ, স্ট্যান্ডার্ড, GET/POST/PUT/DELETE ইত্যাদি মেথড।
- **উদাহরণ:**
  - `GET https://api.example.com/user/123`

## ২. gRPC
- **সংজ্ঞা:** Google-এর তৈরি হাই-পারফরম্যান্স RPC (Remote Procedure Call) ফ্রেমওয়ার্ক।
- **ব্যবহার:** মাইক্রোসার্ভিস, ইন্টার-সার্ভিস কমিউনিকেশন, রিয়েল-টাইম অ্যাপ।
- **বৈশিষ্ট্য:** Protocol Buffers (Protobuf) দিয়ে ডেটা সিরিয়ালাইজেশন, বাইনারি ফরম্যাট, স্ট্রিমিং সাপোর্ট।
- **উদাহরণ:**
  - `service UserService { rpc GetUser (UserRequest) returns (UserResponse); }`

## ৩. WebSocket
- **সংজ্ঞা:** ক্লায়েন্ট-সার্ভার মধ্যে দুই-দিকের (bidirectional) রিয়েল-টাইম কমিউনিকেশন।
- **ব্যবহার:** চ্যাট অ্যাপ, লাইভ নোটিফিকেশন, গেমিং।
- **বৈশিষ্ট্য:** একবার সংযোগ হলে উভয় দিক থেকে ডেটা পাঠানো যায়।
- **উদাহরণ:**
  - `ws://example.com/socket`

## ৪. MQTT
- **সংজ্ঞা:** লাইটওয়েট মেসেজিং প্রোটোকল, IoT (Internet of Things)-এর জন্য জনপ্রিয়।
- **ব্যবহার:** সেন্সর ডেটা, স্মার্ট ডিভাইস, রিয়েল-টাইম মনিটরিং।
- **বৈশিষ্ট্য:** Publish/Subscribe মডেল, কম ব্যান্ডউইথ, লো পাওয়ার।
- **উদাহরণ:**
  - `topic/sensor/temperature`

## ৫. GraphQL
- **সংজ্ঞা:** Facebook-এর তৈরি API কুয়েরি ভাষা। ক্লায়েন্ট নির্দিষ্ট করে কী ডেটা চায়।
- **ব্যবহার:** ফ্লেক্সিবল ডেটা ফেচিং, কম নেটওয়ার্ক কল, মডার্ন ওয়েব/মোবাইল অ্যাপ।
- **বৈশিষ্ট্য:** এক রিকোয়েস্টে একাধিক রিসোর্স, টাইপ সিস্টেম, স্কিমা।
- **উদাহরণ:**
  ```graphql
  query {
    user(id: 123) {
      name
      email
    }
  }
  ```

## তুলনামূলক টেবিল
| প্রোটোকল   | প্রধান ব্যবহার | বৈশিষ্ট্য | উদাহরণ |
|------------|---------------|-----------|---------|
| HTTP/HTTPS | REST, ওয়েব   | স্ট্যান্ডার্ড, সহজ | GET/POST |
| gRPC       | মাইক্রোসার্ভিস | ফাস্ট, বাইনারি, স্ট্রিমিং | Protobuf |
| WebSocket  | রিয়েল-টাইম   | দুই-দিকের, পার্সিস্টেন্ট | চ্যাট |
| MQTT       | IoT           | লাইটওয়েট, Pub/Sub | সেন্সর |
| GraphQL    | ফ্লেক্সিবল API | কুয়েরি, স্কিমা | query |

## কখন কোন প্রোটোকল ব্যবহার করবেন?
- **HTTP/HTTPS:** সাধারণ REST API, ওয়েব অ্যাপ
- **gRPC:** হাই-পারফরম্যান্স, মাইক্রোসার্ভিস
- **WebSocket:** রিয়েল-টাইম, চ্যাট, লাইভ আপডেট
- **MQTT:** IoT, সেন্সর ডেটা
- **GraphQL:** ক্লায়েন্ট-ড্রিভেন ডেটা ফেচিং
