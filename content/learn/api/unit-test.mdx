# ইউনিট টেস্ট

ইউনিট টেস্ট API ডেভেলপমেন্টে ছোট ছোট ফাংশন, মেথড বা কম্পোনেন্টের সঠিকতা যাচাইয়ের জন্য লেখা হয়। এতে বাগ কমে, কোড রিফ্যাক্টর সহজ হয় এবং ডিপ্লয়মেন্টে আত্মবিশ্বাস বাড়ে।

## ইউনিট টেস্ট কী?
- কোডের ক্ষুদ্রতম অংশ (ফাংশন/মেথড) আলাদাভাবে টেস্ট করা
- একেকটি টেস্ট কেস নির্দিষ্ট ইনপুট ও প্রত্যাশিত আউটপুট চেক করে

## API-তে ইউনিট টেস্টের গুরুত্ব
- বিজনেস লজিক, ডেটা প্রসেসিং, ভ্যালিডেশন ইত্যাদি টেস্ট করা যায়
- API endpoint-এর সরাসরি টেস্ট নয়, বরং ফাংশন/সার্ভিস লেয়ারের টেস্ট

## জনপ্রিয় টেস্টিং ফ্রেমওয়ার্ক
- **Node.js:** Jest, Mocha, Chai
- **Python:** unittest, pytest
- **Java:** JUnit
- **Go:** testing

## উদাহরণ: Node.js (Jest)
```js
function sum(a, b) { return a + b; }
test('sum adds numbers', () => {
  expect(sum(2, 3)).toBe(5);
});
```

## উদাহরণ: Python (pytest)
```python
def add(a, b):
    return a + b
def test_add():
    assert add(2, 3) == 5
```

## বেস্ট প্র্যাকটিস
- প্রতিটি ফাংশনের জন্য টেস্ট লিখুন
- টেস্ট কেসে ইনপুট-আউটপুট স্পষ্ট রাখুন
- টেস্ট ফোল্ডার/ফাইল আলাদা রাখুন (e.g., `tests/`)
- CI/CD-তে টেস্ট অটোমেট করুন

## টিপস
- Mocking/Stub দিয়ে ডাটাবেস/এক্সটার্নাল সার্ভিস আলাদা করুন
- টেস্ট কাভারেজ রিপোর্ট দেখুন
- ফেইল হলে দ্রুত ফিক্স করুন
