# রিকোয়েস্ট-রেসপন্স, এন্ডপয়েন্ট, প্যারাম, স্ট্যাটাস কোড

API কমিউনিকেশন মানে কীভাবে ক্লায়েন্ট (যেমন: ওয়েব/মোবাইল অ্যাপ) ও সার্ভার (API) একে অপরের সাথে তথ্য আদান-প্রদান করে।

## ১. রিকোয়েস্ট-রেসপন্স (Request-Response)
- **রিকোয়েস্ট:** ক্লায়েন্ট API-তে নির্দিষ্ট তথ্য/অ্যাকশন চায়।
- **রেসপন্স:** সার্ভার সেই রিকোয়েস্ট প্রসেস করে ফলাফল পাঠায়।

**উদাহরণ:**
- আপনি ওয়েদার অ্যাপে "ঢাকা" শহরের আবহাওয়া জানতে চান। অ্যাপ API-তে রিকোয়েস্ট পাঠায়, API থেকে রেসপন্সে আবহাওয়ার ডেটা আসে।

## ২. এন্ডপয়েন্ট (Endpoint)
- **সংজ্ঞা:** API-র নির্দিষ্ট URL, যেখানে রিকোয়েস্ট পাঠানো হয়।
- **ফরম্যাট:** `https://api.example.com/weather?city=Dhaka`
- **প্রতিটি এন্ডপয়েন্ট একটি নির্দিষ্ট ফাংশন/ডেটা সার্ভ করে।**

## ৩. প্যারামিটার (Parameter)
- **সংজ্ঞা:** রিকোয়েস্টে অতিরিক্ত তথ্য, যা সার্ভারকে বলে কী ডেটা বা অ্যাকশন দরকার।
- **টাইপ:**
  - **Query Parameter:** URL-এর ? এর পরে (যেমন: `?city=Dhaka`)
  - **Path Parameter:** URL-এর অংশ (যেমন: `/user/123`)
  - **Header Parameter:** রিকোয়েস্ট হেডারে (যেমন: `Authorization: Bearer ...`)
  - **Body Parameter:** POST/PUT রিকোয়েস্টে ডেটা (যেমন: JSON body)

## ৪. স্ট্যাটাস কোড (Status Code)
- **সংজ্ঞা:** সার্ভার রেসপন্সে একটি কোড দেয়, যা বলে রিকোয়েস্ট সফল হয়েছে কিনা।
- **গুরুত্বপূর্ণ কিছু স্ট্যাটাস কোড:**
  - **200 OK:** সফল
  - **201 Created:** নতুন কিছু তৈরি হয়েছে
  - **400 Bad Request:** রিকোয়েস্ট ভুল
  - **401 Unauthorized:** অথরাইজেশন নেই
  - **404 Not Found:** ডেটা নেই
  - **500 Internal Server Error:** সার্ভার সমস্যা

## উদাহরণ: একটি API কল

```
GET https://api.example.com/user/123
Header: Authorization: Bearer <token>
```

- **এন্ডপয়েন্ট:** `/user/123`
- **Method:** GET
- **Header:** Authorization
- **রেসপন্স:**
```json
{
  "id": 123,
  "name": "Rahim",
  "email": "rahim@example.com"
}
```
- **স্ট্যাটাস কোড:** 200

## সংক্ষেপে
- API কমিউনিকেশনের মূল ভিত্তি: রিকোয়েস্ট-রেসপন্স, এন্ডপয়েন্ট, প্যারামিটার, স্ট্যাটাস কোড।
- এগুলো না জানলে API ব্যবহার বা তৈরি—কোনোটাই সহজ হবে না।

> পরবর্তী অধ্যায়ে: API প্রোটোকল—HTTP/HTTPS, gRPC, WebSocket, MQTT, GraphQL
