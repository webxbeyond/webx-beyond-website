## SQL Injection, XSS, WAF, OWASP Top 10

API-তে নিরাপত্তা নিশ্চিত করতে হলে বিভিন্ন থ্রেট ও আক্রমণ থেকে রক্ষা পেতে হবে। এখানে গুরুত্বপূর্ণ কিছু থ্রেট ও প্রতিরোধের উপায় আলোচনা করা হলো:

## ১. SQL Injection
- আক্রমণকারী ইনপুটের মাধ্যমে ম্যালিশিয়াস SQL কোড ইনজেক্ট করে ডাটাবেসে এক্সেস নিতে পারে।
- **প্রতিরোধ:**
  - Prepared Statement/Parameterized Query ব্যবহার করুন
  - ইনপুট ভ্যালিডেশন ও স্যানিটাইজেশন
  - ORM ব্যবহার করলে ডিফল্টভাবে সুরক্ষিত

## ২. XSS (Cross-Site Scripting)
- আক্রমণকারী স্ক্রিপ্ট ইনজেক্ট করে ইউজারের ব্রাউজারে রান করাতে পারে
- **প্রতিরোধ:**
  - ইনপুট ও আউটপুট স্যানিটাইজেশন
  - Content Security Policy (CSP) এনাবল করুন
  - HTTPOnly ও Secure কুকি ব্যবহার করুন

## ৩. WAF (Web Application Firewall)
- WAF API-তে আসা রিকোয়েস্ট ফিল্টার করে ম্যালিশিয়াস ট্রাফিক ব্লক করে
- **প্রতিরোধ:**
  - Cloudflare, AWS WAF, Azure WAF ইত্যাদি ব্যবহার করুন
  - কাস্টম রুল সেট করুন

## ৪. OWASP Top 10
- ওয়েব অ্যাপ্লিকেশন সিকিউরিটির সবচেয়ে কমন ১০টি ঝুঁকি
- SQLi, XSS ছাড়াও: Broken Auth, Sensitive Data Exposure, Security Misconfiguration, CSRF, SSRF, ইত্যাদি
- **প্রতিরোধ:**
  - OWASP Top 10 ডকুমেন্টেশন পড়ুন
  - কোড রিভিউ, পেন-টেস্ট, অটোমেটেড টুল ব্যবহার করুন

## বেস্ট প্র্যাকটিস
- ইনপুট ভ্যালিডেশন ও স্যানিটাইজেশন বাধ্যতামূলক
- API রেসপন্সে সেনসিটিভ ডেটা এক্সপোজ করবেন না
- Rate Limiting, Logging, Monitoring চালু রাখুন
- সর্বশেষ লাইব্রেরি ও ফ্রেমওয়ার্ক ব্যবহার করুন
- Security Header (CSP, X-Frame-Options, X-Content-Type-Options) যুক্ত করুন

## টিপস
- API Gateway-এ থ্রেট প্রোটেকশন ফিচার এনাবল করুন
- OWASP ZAP, Burp Suite দিয়ে API টেস্ট করুন
- ডকুমেন্টেশনে সিকিউরিটি রিকমেন্ডেশন দিন
