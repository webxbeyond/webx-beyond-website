# এনভায়রনমেন্ট ও সিক্রেট ম্যানেজমেন্ট

API ডেভেলপমেন্টে এনভায়রনমেন্ট কনফিগারেশন ও সিক্রেট (API Key, Password, Token) নিরাপদে সংরক্ষণ করা অত্যন্ত গুরুত্বপূর্ণ।

## এনভায়রনমেন্ট কনফিগারেশন
- ডেভেলপমেন্ট, টেস্ট, প্রোডাকশন—প্রতিটি এনভায়রনমেন্টের জন্য আলাদা কনফিগ
- ডাটাবেস URI, API Key, থার্ড-পার্টি সার্ভিসের ক্রেডেনশিয়াল আলাদা রাখুন
- `.env` ফাইল ব্যবহার করুন (Node.js: dotenv, Python: python-dotenv)

### উদাহরণ: Node.js-এ dotenv
```js
require('dotenv').config();
const db = process.env.DB_URI;
```

### উদাহরণ: Python-এ python-dotenv
```python
from dotenv import load_dotenv
import os
load_dotenv()
db = os.getenv('DB_URI')
```

## সিক্রেট ম্যানেজমেন্ট
- সিক্রেট কখনোই সোর্স কোডে রাখবেন না
- `.env` ফাইল গিটে কমিট করবেন না (`.gitignore`-এ রাখুন)
- ক্লাউডে Secret Manager ব্যবহার করুন (AWS Secrets Manager, Azure Key Vault, GCP Secret Manager)

### উদাহরণ: AWS Secrets Manager
- সিক্রেট ক্লাউডে স্টোর করুন
- অ্যাপ্লিকেশন থেকে API দিয়ে রিড করুন

## বেস্ট প্র্যাকটিস
- `.env.example` ফাইল দিয়ে ডকুমেন্টেশন দিন
- সিক্রেট রোটেশন (নিয়মিত পরিবর্তন) করুন
- প্রোডাকশনে এনক্রিপ্টেড স্টোরেজ ব্যবহার করুন
- সিক্রেট অ্যাক্সেস লগ ও মনিটরিং রাখুন

## টিপস
- ডেভেলপমেন্টে ফেক/ডামি সিক্রেট ব্যবহার করুন
- CI/CD-তে এনভায়রনমেন্ট ভ্যারিয়েবল কনফিগার করুন
- সিক্রেট লিক হলে দ্রুত রিভোক করুন
