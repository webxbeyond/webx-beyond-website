## সংজ্ঞা

**API** (Application Programming Interface) হলো এমন একটি ইন্টারফেস, যার মাধ্যমে দুটি সফটওয়্যার বা সিস্টেম একে অপরের সাথে যোগাযোগ করতে পারে। সহজভাবে, API হচ্ছে এক ধরনের চুক্তি—যেখানে নির্দিষ্ট নিয়ম ও ফরম্যাটে ডেটা আদান-প্রদান হয়।

- **A**pplication: সফটওয়্যার বা অ্যাপ্লিকেশন
- **P**rogramming: কোডিং বা প্রোগ্রামিং
- **I**nterface: সংযোগ বা ইন্টারফেস

## ইতিহাস

- ১৯৬০-৭০ দশকে API ধারণা প্রথম আসে, মূলত অপারেটিং সিস্টেম ও লাইব্রেরি ফাংশন ব্যবহারের জন্য।
- ১৯৯০-এর দশকে ওয়েব API জনপ্রিয় হয়, বিশেষ করে SOAP ও XML-RPC দিয়ে।
- ২০০০-এর পর RESTful API, JSON, GraphQL ইত্যাদির মাধ্যমে API আরও সহজ ও জনপ্রিয় হয়।

## কেন API দরকার?

- **কমিউনিকেশন:** বিভিন্ন অ্যাপ্লিকেশন/সার্ভিস একে অপরের সাথে সহজে কথা বলতে পারে।
- **রিইউজেবিলিটি:** একবার API বানালে অনেক অ্যাপ্লিকেশন সেটি ব্যবহার করতে পারে।
- **ইন্টিগ্রেশন:** থার্ড-পার্টি অ্যাপ, মোবাইল অ্যাপ, ওয়েব অ্যাপ—সব জায়গায় একই API ব্যবহার করা যায়।
- **স্কেলেবিলিটি:** বড় সিস্টেম সহজে মডিউলার করা যায়।

## বাস্তব উদাহরণ

### ১. রেস্টুরেন্ট মেনু (Analogical Example)
আপনি রেস্টুরেন্টে গেলে ওয়েটারকে অর্ডার দেন—ওয়েটার হচ্ছে API, আপনি ক্লায়েন্ট, কিচেন হচ্ছে সার্ভার। আপনি কী চান (রিকোয়েস্ট) বলেন, ওয়েটার সেটি কিচেনে পাঠায়, কিচেন খাবার (রেসপন্স) তৈরি করে ওয়েটার আপনার কাছে নিয়ে আসে।

### ২. ওয়েদার অ্যাপ
আপনার মোবাইলের ওয়েদার অ্যাপ মূলত ওয়েদার API-তে রিকোয়েস্ট পাঠায়, এবং API থেকে ডেটা নিয়ে আপনাকে দেখায়।

### ৩. পেমেন্ট গেটওয়ে
ই-কমার্স সাইটে পেমেন্ট করার সময় API-এর মাধ্যমে ব্যাংক/পেমেন্ট সার্ভিসের সাথে যোগাযোগ হয়।

### ৪. সোশ্যাল মিডিয়া লগইন
"Login with Google/Facebook"—এগুলো API ব্যবহার করে আপনার তথ্য যাচাই করে।

## API কিভাবে কাজ করে?

১. **রিকোয়েস্ট:** ক্লায়েন্ট (যেমন: ওয়েব/মোবাইল অ্যাপ) API-তে নির্দিষ্ট ফরম্যাটে রিকোয়েস্ট পাঠায়।
২. **প্রসেসিং:** সার্ভার রিকোয়েস্ট প্রসেস করে।
৩. **রেসপন্স:** সার্ভার নির্দিষ্ট ফরম্যাটে (JSON/XML) ডেটা পাঠায়।

## সংক্ষেপে
- API হচ্ছে সফটওয়্যার কমিউনিকেশনের সেতুবন্ধন।
- এটি সফটওয়্যার ইকোসিস্টেমকে আরও ওপেন, স্কেলেবল ও ইন্টিগ্রেটেড করে তোলে।
