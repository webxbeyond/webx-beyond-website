# রেট লিমিটিং

রেট লিমিটিং API-তে নির্দিষ্ট সময়ের মধ্যে কতগুলো রিকোয়েস্ট আসতে পারবে তা নিয়ন্ত্রণ করে। এটি API-কে ম্যালিশিয়াস ইউজ, স্প্যাম, DDoS অ্যাটাক ও অতিরিক্ত লোড থেকে রক্ষা করে।

## কেন রেট লিমিটিং দরকার?
- সার্ভার রিসোর্স অপ্টিমাইজ করা
- ম্যালিশিয়াস অ্যাক্টিভিটি ও বট প্রতিরোধ
- ফেয়ার ইউজার এক্সপেরিয়েন্স
- DDoS/স্প্যাম অ্যাটাক প্রতিরোধ

## কিভাবে কাজ করে?
- প্রতি ইউজার/ক্লায়েন্ট/আইপি-র জন্য নির্দিষ্ট সময়ের মধ্যে নির্দিষ্ট সংখ্যক রিকোয়েস্ট অনুমোদন
- সীমা অতিক্রম করলে 429 Too Many Requests রেসপন্স

## কমন রেট লিমিটিং স্ট্র্যাটেজি
- **Fixed Window:** নির্দিষ্ট সময় (যেমন ১ মিনিট) ধরে কাউন্ট
- **Sliding Window:** চলমান সময়ের উইন্ডো ধরে কাউন্ট
- **Token Bucket:** প্রতিটি রিকোয়েস্টে টোকেন খরচ হয়, নির্দিষ্ট সময়ে টোকেন রিফিল
- **Leaky Bucket:** নির্দিষ্ট হারে রিকোয়েস্ট প্রসেস হয়, অতিরিক্ত রিকোয়েস্ট কিউতে যায়

## উদাহরণ: Express.js-এ রেট লিমিটিং
```js
const rateLimit = require('express-rate-limit');
const limiter = rateLimit({
  windowMs: 1 * 60 * 1000, // ১ মিনিট
  max: 100, // প্রতি মিনিটে ১০০ রিকোয়েস্ট
});
app.use(limiter);
```

## HTTP Header
- `X-RateLimit-Limit`: সর্বোচ্চ রিকোয়েস্ট সংখ্যা
- `X-RateLimit-Remaining`: বাকি রিকোয়েস্ট সংখ্যা
- `X-RateLimit-Reset`: কখন রিসেট হবে

## বেস্ট প্র্যাকটিস
- রেট লিমিটিং সব API-তে এনাবল করুন
- ক্লায়েন্টকে স্পষ্টভাবে লিমিট ও রিসেট টাইম জানিয়ে দিন
- ডিফারেন্ট ইউজার টাইপের জন্য আলাদা লিমিট সেট করুন (free/premium)
- রেট লিমিটিং বাইপাস প্রতিরোধে API Key/IP/Token ভিত্তিক লিমিট ব্যবহার করুন

## টিপস
- API Gateway (Kong, AWS API Gateway) দিয়ে সহজেই রেট লিমিটিং ইমপ্লিমেন্ট করা যায়
- লগ ও মনিটরিং রাখুন—কে কতবার লিমিট হিট করছে
- ডকুমেন্টেশনে রেট লিমিট স্পষ্টভাবে উল্লেখ করুন
