## কুয়েরি, স্কিমা, ফ্র্যাগমেন্ট

GraphQL হলো Facebook-এর তৈরি একটি API কুয়েরি ভাষা, যা REST-এর বিকল্প হিসেবে জনপ্রিয়।

## ১. কুয়েরি (Query)
- **সংজ্ঞা:** ক্লায়েন্ট নির্দিষ্ট করে কী ডেটা চায়, শুধু সেটাই রিটার্ন হয়।
- **উদাহরণ:**
    ```graphql
    query {
      user(id: 123) {
        name
        email
      }
    }
    ```
- **সুবিধা:** কম নেটওয়ার্ক কল, কম ডেটা ট্রান্সফার, ফ্লেক্সিবল ফেচিং।

## ২. স্কিমা (Schema)
- **সংজ্ঞা:** API-র ডেটা টাইপ, রিলেশন ও অপারেশন সংজ্ঞায়িত করে।
- **উদাহরণ:**
    ```graphql
    type User {
      id: ID!
      name: String!
      email: String!
    }
    type Query {
      user(id: ID!): User
    }
    ```
- **সুবিধা:** টাইপ সেফটি, অটো-ডকুমেন্টেশন, ইন্টেলিজেন্ট টুলিং।

## ৩. ফ্র্যাগমেন্ট (Fragment)
- **সংজ্ঞা:** রিপিটেড কুয়েরি অংশকে রিইউজেবল ব্লকে রাখা।
- **উদাহরণ:**
    ```graphql
    fragment userFields on User {
      name
      email
    }
    query {
      user(id: 123) {
        ...userFields
      }
    }
    ```
- **সুবিধা:** কোড রিইউজ, বড় কুয়েরি সহজে মেইনটেইন।

## GraphQL বনাম REST
- এক রিকোয়েস্টে একাধিক রিসোর্স ফেচ করা যায়
- ক্লায়েন্ট নির্ধারণ করে কী ডেটা লাগবে
- স্ট্রং টাইপড স্কিমা

## সংক্ষেপে
- GraphQL API-তে কুয়েরি, স্কিমা ও ফ্র্যাগমেন্ট—তিনটি মূল কনসেপ্ট
- ফ্লেক্সিবল, টাইপ-সেফ ও মডার্ন API ডিজাইন

> পরবর্তী অধ্যায়ে: gRPC—প্রোটোকল বাফার, স্ট্রিমিং, পারফরম্যান্স
