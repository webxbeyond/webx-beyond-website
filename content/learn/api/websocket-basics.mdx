# রিয়েল-টাইম, ইভেন্ট-ড্রিভেন

WebSocket হলো ক্লায়েন্ট ও সার্ভারের মধ্যে দুই-দিকের (bidirectional) রিয়েল-টাইম কমিউনিকেশন প্রোটোকল।

## ১. রিয়েল-টাইম কমিউনিকেশন
- **সংজ্ঞা:** ক্লায়েন্ট ও সার্ভার উভয় দিক থেকে ডেটা পাঠানো ও গ্রহণ করা যায়, সংযোগ একবার স্থাপন হলে বারবার HTTP রিকোয়েস্টের দরকার হয় না।
- **ব্যবহার:** চ্যাট অ্যাপ, লাইভ নোটিফিকেশন, গেমিং, স্টক মার্কেট ড্যাশবোর্ড।
- **উদাহরণ:**
    ```js
    const socket = new WebSocket('ws://example.com/socket');
    socket.onmessage = (event) => {
      console.log(event.data);
    };
    socket.send('Hello Server!');
    ```

## ২. ইভেন্ট-ড্রিভেন ডিজাইন
- **সংজ্ঞা:** নির্দিষ্ট ইভেন্ট (যেমন: message, open, close, error) হলে নির্দিষ্ট কোড রান হয়।
- **উদাহরণ:**
    ```js
    socket.onopen = () => { /* সংযোগ স্থাপন */ };
    socket.onclose = () => { /* সংযোগ বন্ধ */ };
    socket.onerror = (err) => { /* এরর হ্যান্ডলিং */ };
    ```

## ৩. WebSocket বনাম HTTP
- WebSocket: একবার সংযোগ হলে বারবার ডেটা পাঠানো যায় (low latency)
- HTTP: প্রতিবার নতুন রিকোয়েস্ট-রেসপন্স

## বেস্ট প্র্যাকটিস
- সংযোগ ব্যবস্থাপনা (reconnect, heartbeat/ping)
- অথেন্টিকেশন (token-based)
- স্কেলিং (Pub/Sub, message broker)

## সংক্ষেপে
- WebSocket API-তে রিয়েল-টাইম, ইভেন্ট-ড্রিভেন কমিউনিকেশন সহজ
- চ্যাট, লাইভ আপডেট, গেমিং ইত্যাদিতে অপরিহার্য

> পরবর্তী অধ্যায়ে: SOAP—XML, WSDL, লিগ্যাসি API
