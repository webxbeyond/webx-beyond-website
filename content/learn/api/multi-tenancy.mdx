মাল্টি-টেন্যান্সি মানে হচ্ছে এক API বা অ্যাপ্লিকেশন একাধিক কাস্টমার/টিম/অর্গানাইজেশনের জন্য আলাদা ডেটা ও কনফিগারেশন মেইনটেইন করে। SaaS, B2B API-তে এটি খুবই গুরুত্বপূর্ণ।

## মাল্টি-টেন্যান্সি কী?
- এক সার্ভার/কোডবেসে একাধিক টেন্যান্ট (কাস্টমার/অর্গানাইজেশন)
- প্রতিটি টেন্যান্টের ডেটা, ইউজার, কনফিগ আলাদা
- অথেন্টিকেশন, অথরাইজেশন, ডেটা আইসোলেশন

## মাল্টি-টেন্যান্সি আর্কিটেকচার
- **Database-per-tenant:** প্রতিটি টেন্যান্টের জন্য আলাদা ডাটাবেস
- **Schema-per-tenant:** এক ডাটাবেস, আলাদা স্কিমা
- **Shared Database, Tenant ID:** এক ডাটাবেস, টেন্যান্ট আইডি দিয়ে ডেটা আলাদা

## চ্যালেঞ্জ ও সমাধান
- **ডেটা আইসোলেশন:** টেন্যান্টের ডেটা অন্য টেন্যান্ট না পায়
- **স্কেলিং:** টেন্যান্ট বাড়লে পারফরম্যান্স
- **কাস্টমাইজেশন:** টেন্যান্টভিত্তিক ফিচার/কনফিগ
- **বিলিং ও usage tracking**

## উদাহরণ: টেন্যান্ট আইডি দিয়ে ডেটা ফিল্টার
```js
// Express.js
app.get('/users', (req, res) => {
  const tenantId = req.headers['x-tenant-id'];
  User.find({ tenantId }).then(users => res.json(users));
});
```

## বেস্ট প্র্যাকটিস
- টেন্যান্ট আইডি সব রিকোয়েস্টে বাধ্যতামূলক
- ডেটা আইসোলেশন ও এনক্রিপশন
- টেন্যান্টভিত্তিক usage, billing, কনফিগ
- টেন্যান্ট ডিলিট/ডাটা মাইগ্রেশন সাপোর্ট

## টিপস
- SaaS API-তে মাল্টি-টেন্যান্সি মাস্ট
- টেন্যান্ট অনবোর্ডিং/অফবোর্ডিং অটোমেট করুন
- ডকুমেন্টেশনে মাল্টি-টেন্যান্সি গাইড দিন
