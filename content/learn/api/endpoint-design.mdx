# নাউন-ভিত্তিক URL, CRUD, ভার্সনিং

RESTful API ডিজাইনের অন্যতম গুরুত্বপূর্ণ অংশ হলো এন্ডপয়েন্ট ডিজাইন। ভালো এন্ডপয়েন্ট ডিজাইন API-কে সহজ, স্পষ্ট ও স্কেলেবল করে তোলে।

## ১. নাউন-ভিত্তিক URL (Noun-based URL)
- **সংজ্ঞা:** URL-এ রিসোর্সের নাম (নাউন) ব্যবহার করা।
- **ভুল:** `/getUser`, `/createOrder`
- **সঠিক:** `/users`, `/orders`
- **উদাহরণ:**
  - `/products` (সব প্রোডাক্ট)
  - `/products/123` (একটি নির্দিষ্ট প্রোডাক্ট)

## ২. CRUD অপারেশন
- **CRUD:** Create, Read, Update, Delete
- **HTTP Method Mapping:**
  - **POST** `/users` → নতুন ইউজার তৈরি
  - **GET** `/users` → সব ইউজার দেখুন
  - **GET** `/users/123` → নির্দিষ্ট ইউজার
  - **PUT/PATCH** `/users/123` → ইউজার আপডেট
  - **DELETE** `/users/123` → ইউজার ডিলিট

## ৩. ভার্সনিং (Versioning)
- **কেন দরকার:** API আপডেট হলে পুরনো ক্লায়েন্ট যেন ভেঙে না যায়।
- **পদ্ধতি:**
  - **URL ভার্সনিং:** `/v1/users`, `/v2/users`
  - **Header ভার্সনিং:** `Accept: application/vnd.example.v1+json`
- **উদাহরণ:**
  - `/api/v1/products`
  - `/api/v2/products`

## বেস্ট প্র্যাকটিস
- URL-এ ক্রিয়াপদ (verb) নয়, নাউন ব্যবহার করুন
- প্লুরাল (বহুবচন) রিসোর্স নাম (যেমন: `/users`, `/orders`)
- কনসিস্টেন্ট ও স্পষ্ট naming
- ভার্সনিং স্পষ্টভাবে উল্লেখ করুন

## উদাহরণ: একটি RESTful এন্ডপয়েন্ট
```
GET /api/v1/orders/789
```
- **Method:** GET
- **Resource:** orders/789
- **Version:** v1

## সংক্ষেপে
- RESTful API-তে এন্ডপয়েন্ট ডিজাইনের মূল: নাউন-ভিত্তিক URL, CRUD অপারেশন, ভার্সনিং
- এগুলো মানলে API হবে সহজ, maintainable ও future-proof

> পরবর্তী অধ্যায়ে: রিকোয়েস্ট ও রেসপন্স ফরম্যাট—JSON, XML, প্যাজিনেশন, ফিল্টারিং
