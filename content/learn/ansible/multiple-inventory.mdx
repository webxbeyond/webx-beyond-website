বড় ও জটিল ইনফ্রাস্ট্রাকচারে অনেক সময় একাধিক ইনভেন্টরি ফাইল বা সোর্স একসাথে ব্যবহার করতে হয়। যেমন, কিছু হোস্ট স্ট্যাটিক ফাইলে, কিছু ডাইনামিক সোর্সে, আবার কিছু স্পেশাল কনফিগে। আনসিবল আপনাকে একাধিক ইনভেন্টরি একত্রে ব্যবহার করার সুবিধা দেয়, যাতে স্কেলিং, ফ্লেক্সিবিলিটি ও মেইনটেনেন্স সহজ হয়।

## অ্যানালজি: মাল্টি-শপ ম্যানেজমেন্ট

ধরা যাক, আপনি একটি চেইন শপের ম্যানেজার। কিছু শপের তালিকা আপনার হাতে লেখা, কিছু শপের ডেটা অনলাইনে, কিছু শপের ডেটা অন্য সফটওয়্যারে। সব শপের ডেটা একসাথে ম্যানেজ করতে হলে, সব সোর্স একত্রে আনতে হবে।

আনসিবলে একাধিক ইনভেন্টরি একত্রে ব্যবহার ঠিক এমনই—বিভিন্ন সোর্স থেকে হোস্ট একত্রে এনে এক প্লেবুকে রান করা যায়।

## একাধিক ইনভেন্টরি যুক্ত করার নিয়ম

### ১. কমা দিয়ে ইনভেন্টরি ফাইল যুক্ত করা

```bash
ansible-playbook -i inventory/hosts,inventory/aws_ec2.yml playbook.yml
```

এভাবে কমা দিয়ে একাধিক ইনভেন্টরি ফাইল যুক্ত করা যায়।

### ২. ইনভেন্টরি ডিরেক্টরি ব্যবহার

আপনি একটি ডিরেক্টরি তৈরি করে সেখানে একাধিক ইনভেন্টরি ফাইল রাখতে পারেন।

```bash
ansible-playbook -i inventory/ playbook.yml
```

এতে ডিরেক্টরির সব ইনভেন্টরি ফাইল একত্রে লোড হবে।

### ৩. YAML ইনভেন্টরি ফাইলের মধ্যে include

YAML ইনভেন্টরি ফাইলে `compose` বা `include` ব্যবহার করে অন্য ইনভেন্টরি ফাইল যুক্ত করা যায়।

**inventory/main.yml**
```yaml
plugin: constructed
include:
  - hosts.yml
  - aws_ec2.yml
```

### ৪. Static + Dynamic ইনভেন্টরি একত্রে

স্ট্যাটিক ও ডাইনামিক ইনভেন্টরি একসাথে ব্যবহার করতে চাইলে:

```bash
ansible-playbook -i inventory/hosts -i inventory/aws_ec2.yml playbook.yml
```

এভাবে `-i` ফ্ল্যাগ একাধিকবার ব্যবহার করা যায়।

## কনফ্লিক্ট ও ওভাররাইড

একাধিক ইনভেন্টরি ফাইলে একই হোস্ট থাকলে, শেষের ইনভেন্টরি ফাইলের ভেরিয়েবল ও কনফিগারেশন ওভাররাইড করবে।

## বেস্ট প্র্যাকটিস

- ইনভেন্টরি ফাইল/ডিরেক্টরি version control এ রাখুন
- ইনভেন্টরি ফাইলের naming convention স্পষ্ট রাখুন
- Sensitive data vault দিয়ে এনক্রিপ্ট করুন
- ডকুমেন্টেশন রাখুন—কোন ইনভেন্টরি কোথা থেকে এসেছে

## বাস্তব উদাহরণ

ধরা যাক, আপনার কিছু হোস্ট স্ট্যাটিক ফাইলে, কিছু AWS EC2 তে।

**inventory/hosts**
```
[web]
web01
web02
```

**inventory/aws_ec2.yml**
```yaml
plugin: aws_ec2
regions:
  - ap-south-1
filters:
  instance.state-name: running
```

**প্লেবুক রান:**
```bash
ansible-playbook -i inventory/hosts -i inventory/aws_ec2.yml playbook.yml
```

## উপসংহার

একাধিক ইনভেন্টরি একত্রে ব্যবহার করলে আনসিবল দিয়ে বড়, ডাইনামিক, মাল্টি-সোর্স ইনফ্রাস্ট্রাকচার সহজে ও ফ্লেক্সিবলি ম্যানেজ করা যায়।
