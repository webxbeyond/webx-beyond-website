আনসিবল দিয়ে বড় ও জটিল ইনফ্রাস্ট্রাকচার ম্যানেজ করতে গেলে ভেরিয়েবল ব্যবস্থাপনা অত্যন্ত গুরুত্বপূর্ণ। প্লেবুক, রোল, টাস্ক—সব জায়গায় ভিন্ন ভিন্ন ভেরিয়েবল দরকার হয়। কিন্তু যদি প্রতিটি হোস্ট বা গ্রুপের জন্য আলাদা আলাদা ভেরিয়েবল রাখতে হয়, তাহলে কীভাবে সহজে ও সুন্দরভাবে সেটি করা যায়? এখানেই আসে group_vars ও host_vars এর ব্যবহার।

## বাস্তব উদাহরণ ও অ্যানালজি

ধরা যাক, আপনি একটি স্কুলের আইটি অ্যাডমিন। স্কুলে তিনটি বিভাগ—সায়েন্স, আর্টস, কমার্স। প্রতিটি বিভাগে অনেক কম্পিউটার আছে। কিছু সেটিংস সব বিভাগে এক, আবার কিছু সেটিংস শুধু সায়েন্স বিভাগে, কিছু শুধু কমার্সে।

এখন, যদি আপনাকে প্রতিটি কম্পিউটারে আলাদা আলাদা কনফিগারেশন দিতে হয়, তাহলে কাজটা হবে খুবই কষ্টকর। কিন্তু যদি আপনি "সায়েন্স" গ্রুপের জন্য একবারেই সেটিংস লিখে দেন, তাহলে সেই গ্রুপের সব কম্পিউটারে সেটিংস অটো-এপ্লাই হবে।

এভাবেই group_vars ও host_vars কাজ করে—গ্রুপভিত্তিক ও হোস্টভিত্তিক কনফিগারেশন সহজ করে দেয়।

## group_vars ও host_vars কী?

- **group_vars**: একটি গ্রুপের জন্য কমন ভেরিয়েবল।
- **host_vars**: নির্দিষ্ট হোস্টের জন্য স্পেশাল ভেরিয়েবল।

## ফোল্ডার স্ট্রাকচার

আপনার ইনভেন্টরি ফোল্ডারে সাধারণত এভাবে ফোল্ডার থাকে:

```
inventory/
  hosts
  group_vars/
    science.yml
    arts.yml
    commerce.yml
  host_vars/
    pc01.yml
    pc02.yml
```

## group_vars ব্যবহার

ধরা যাক, science গ্রুপের সব কম্পিউটারে Python 3.10 লাগবে। তাহলে:

**inventory/group_vars/science.yml**
```yaml
python_version: "3.10"
```

## host_vars ব্যবহার

ধরা যাক, pc01 কম্পিউটারে বিশেষভাবে Java লাগবে:

**inventory/host_vars/pc01.yml**
```yaml
java_version: "17"
```

## প্লেবুকে কিভাবে কাজ করে?

আপনি যখন প্লেবুক রান করবেন, আনসিবল আগে group_vars, তারপর host_vars থেকে ভেরিয়েবল লোড করবে।

- গ্রুপের জন্য কমন ভেরিয়েবল group_vars থেকে আসবে।
- নির্দিষ্ট হোস্টের জন্য host_vars থেকে আসবে।
- host_vars এর ভেরিয়েবল group_vars কে ওভাররাইড করতে পারে।

## বাস্তব প্লেবুক উদাহরণ

**inventory/hosts**
```
[science]
pc01
pc02
[arts]
pc03
pc04
```

**playbook.yml**
```yaml
- hosts: science
  tasks:
    - name: Show python version
      debug:
        msg: "Python version is {{ python_version }}"
    - name: Show java version (if exists)
      debug:
        msg: "Java version is {{ java_version | default('Not required') }}"
```

## কেন group_vars ও host_vars ব্যবহার করবেন?

- বড় প্রজেক্টে কনফিগারেশন সহজ হয়
- কোড রিপিট হয় না
- স্কেলিং সহজ
- মেইনটেনেন্স সহজ

## টিপস ও বেস্ট প্র্যাকটিস

- ভেরিয়েবল নাম স্পষ্ট রাখুন
- Sensible default ব্যবহার করুন
- Sensitive data (password, secret) vault দিয়ে এনক্রিপ্ট করুন
- ফোল্ডার স্ট্রাকচার পরিষ্কার রাখুন

## উপসংহার

group_vars ও host_vars ব্যবহার করলে আনসিবল প্লেবুক আরও ডাইনামিক, স্কেলেবল ও মেইনটেনেবল হয়। বড় ইনফ্রাস্ট্রাকচার ম্যানেজ করতে গেলে এটি মাস্টার করার মতো একটি কনসেপ্ট।
