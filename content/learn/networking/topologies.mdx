## স্টার, মেশ, বাস, রিং, হাইব্রিড

নেটওয়ার্ক টপোলজি হলো নেটওয়ার্কের ভৌত (physical) ও যৌক্তিক (logical) বিন্যাস—অর্থাৎ, ডিভাইসগুলো কিভাবে সংযুক্ত এবং তথ্য কিভাবে প্রবাহিত হয়। বিভিন্ন টপোলজির সুবিধা-অসুবিধা, ব্যবহারিক দিক ও চিত্রসহ এখানে বিস্তারিত আলোচনা করা হলো।

## ১. স্টার টপোলজি (Star Topology)

**বর্ণনা:**
সব ডিভাইস (নোড) একটি কেন্দ্রীয় ডিভাইস (সুইচ/হাব)-এর সাথে সংযুক্ত।

**চিত্র:**
```
    PC1   PC2
      \   /
      [Switch]
      /   \
    PC3   PC4
```

**বৈশিষ্ট্য:**
- কেন্দ্রীয় ডিভাইসের মাধ্যমে সব যোগাযোগ
- সহজে সমস্যা নির্ণয়
- একটি লিঙ্ক/নোড নষ্ট হলে বাকি নেটওয়ার্ক সচল

**সুবিধা:**
- ম্যানেজমেন্ট সহজ
- স্কেল করা সহজ

**অসুবিধা:**
- কেন্দ্রীয় ডিভাইস নষ্ট হলে পুরো নেটওয়ার্ক অচল
- বেশি কেবল লাগে

**ব্যবহার:**
- আধুনিক LAN, Wi-Fi

---

## ২. বাস টপোলজি (Bus Topology)

**বর্ণনা:**
সব ডিভাইস একটি একক কেবল (ব্যাকবোন)-এর সাথে সংযুক্ত।

**চিত্র:**
```
PC1---+---PC2---+---PC3---+---PC4
         |
      [Backbone]
```

**বৈশিষ্ট্য:**
- একক কেবল দিয়ে সব ডেটা প্রবাহ
- টার্মিনেটর দরকার

**সুবিধা:**
- কম কেবল লাগে
- ছোট নেটওয়ার্কে সাশ্রয়ী

**অসুবিধা:**
- কেবল নষ্ট হলে পুরো নেটওয়ার্ক অচল
- স্কেল করা কঠিন
- কনজেশন বেশি

**ব্যবহার:**
- পুরনো ইথারনেট

---

## ৩. রিং টপোলজি (Ring Topology)

**বর্ণনা:**
সব ডিভাইস একটি বৃত্তে সংযুক্ত, ডেটা একদিকে বা দুইদিকে ঘুরে যায়।

**চিত্র:**
```
PC1---PC2
 |      |
PC4---PC3
```

**বৈশিষ্ট্য:**
- ডেটা এক/দুই দিকে প্রবাহিত
- টোকেন পাসিং ব্যবহৃত

**সুবিধা:**
- ডেটা সংঘর্ষ কম
- নির্ধারিত পারফরম্যান্স

**অসুবিধা:**
- একটি লিঙ্ক/নোড নষ্ট হলে সমস্যা
- ট্রাবলশুটিং কঠিন

**ব্যবহার:**
- FDDI, Token Ring

---

## ৪. মেশ টপোলজি (Mesh Topology)

**বর্ণনা:**
প্রতিটি ডিভাইস অন্য সব ডিভাইসের সাথে সংযুক্ত (ফুল মেশ) বা কিছু ডিভাইসের সাথে (পার্শিয়াল মেশ)।

**চিত্র:**
```
PC1-----PC2
 | \   / |
 |  PC3  |
 | /   \ |
PC4-----PC5
```

**বৈশিষ্ট্য:**
- একাধিক পথ
- উচ্চ রিডান্ডেন্সি

**সুবিধা:**
- ফেইলওভার ভালো
- উচ্চ পারফরম্যান্স

**অসুবিধা:**
- কেবল ও পোর্ট বেশি লাগে
- কনফিগারেশন জটিল

**ব্যবহার:**
- WAN, ইন্টারনেট ব্যাকবোন

---

## ৫. হাইব্রিড টপোলজি (Hybrid Topology)

**বর্ণনা:**
দুই বা ততোধিক টপোলজির সংমিশ্রণ।

**চিত্র:**
```
Star + Bus, Mesh + Star ইত্যাদি
```

**বৈশিষ্ট্য:**
- কাস্টমাইজড ডিজাইন
- বড় নেটওয়ার্কে ব্যবহৃত

**সুবিধা:**
- ফ্লেক্সিবল
- স্কেলেবল

**অসুবিধা:**
- ডিজাইন ও মেইনটেনেন্স জটিল
- খরচ বেশি

**ব্যবহার:**
- বড় কর্পোরেট নেটওয়ার্ক, ডেটা সেন্টার

## তুলনামূলক টেবিল

| টপোলজি | সুবিধা | অসুবিধা | ব্যবহার |
|---------|--------|----------|---------|
| স্টার   | সহজ ম্যানেজ, স্কেল | কেন্দ্রীয় ডিভাইস ফেইল | আধুনিক LAN |
| বাস     | কম কেবল, সাশ্রয়ী | কেবল ফেইল, স্কেল কঠিন | পুরনো LAN |
| রিং     | সংঘর্ষ কম | লিঙ্ক ফেইল, ট্রাবলশুট কঠিন | Token Ring |
| মেশ     | রিডান্ডেন্সি, পারফ | খরচ, জটিলতা | WAN, ব্যাকবোন |
| হাইব্রিড| ফ্লেক্সিবল, স্কেল | ডিজাইন জটিল | বড় নেটওয়ার্ক |

---

## উপসংহার

নেটওয়ার্ক টপোলজি বাছাই নেটওয়ার্কের আকার, বাজেট, নির্ভরযোগ্যতা ও ব্যবহারের উপর নির্ভর করে। সঠিক টপোলজি ডিজাইন করলে নেটওয়ার্ক পারফরম্যান্স, স্কেলেবিলিটি ও রিলায়েবিলিটি অনেক বাড়ে।
