লোড ব্যালান্সার হলো এমন একটি নেটওয়ার্ক ডিভাইস বা সফটওয়্যার, যা একাধিক সার্ভারের মধ্যে নেটওয়ার্ক ট্রাফিক ভাগ করে দেয়। এর ফলে সার্ভারের উপর চাপ কমে, পারফরম্যান্স বাড়ে এবং সার্ভিস ডাউন হওয়ার ঝুঁকি কমে যায়।

---

## লোড ব্যালান্সার কীভাবে কাজ করে?

1. ক্লায়েন্ট অনুরোধ পাঠায় (যেমন: ওয়েব ব্রাউজার থেকে)
2. লোড ব্যালান্সার অনুরোধ গ্রহণ করে
3. উপলব্ধ সার্ভারগুলোর মধ্যে একটি নির্বাচন করে
4. অনুরোধটি সেই সার্ভারে পাঠায়
5. সার্ভার রেসপন্স দিলে, লোড ব্যালান্সার সেটি ক্লায়েন্টকে ফেরত পাঠায়

---

## লোড ব্যালান্সারের সুবিধা

- **উচ্চ অ্যাভেইলেবিলিটি:** একাধিক সার্ভার থাকায় সার্ভিস ডাউন হওয়ার সম্ভাবনা কম
- **স্কেলেবিলিটি:** সহজে নতুন সার্ভার যোগ করা যায়
- **পারফরম্যান্স:** ট্রাফিক ভাগ করে দ্রুত রেসপন্স
- **নিরাপত্তা:** কিছু আক্রমণ (DDoS) প্রতিরোধে সহায়ক

|-------------|----------------------------------------|

## L4 বনাম L7 লোড ব্যালান্সার

লোড ব্যালান্সার OSI মডেলের বিভিন্ন লেয়ারে কাজ করতে পারে।

### L4 (Transport Layer) লোড ব্যালান্সার

- TCP/UDP লেভেলে কাজ করে
- পোর্ট ও আইপি অ্যাড্রেস দেখে ট্রাফিক ভাগ
- দ্রুত, কম ফিচার
- উদাহরণ: Linux IPVS, AWS NLB, HAProxy (TCP mode)

### L7 (Application Layer) লোড ব্যালান্সার

- HTTP/HTTPS, SMTP, FTP ইত্যাদি অ্যাপ্লিকেশন লেভেলে কাজ করে
- URL, হোস্টনেম, কুকি, হেডার দেখে সিদ্ধান্ত নেয়
- অ্যাডভান্সড ফিচার: SSL টার্মিনেশন, কন্টেন্ট-ভিত্তিক রাউটিং
- উদাহরণ: NGINX, AWS ALB, HAProxy (HTTP mode)

---

## তুলনামূলক টেবিল: L4 বনাম L7

| বৈশিষ্ট্য         | L4 লোড ব্যালান্সার         | L7 লোড ব্যালান্সার           |
|-------------------|----------------------------|------------------------------|
| OSI লেয়ার         | Transport (Layer 4)        | Application (Layer 7)        |
| প্রোটোকল           | TCP, UDP                   | HTTP, HTTPS, SMTP, FTP       |
| সিদ্ধান্তের ভিত্তি | IP, Port                   | URL, Cookie, Header, Host    |
| গতি               | দ্রুত                      | তুলনামূলক ধীর               |
| ফিচার             | সীমিত                      | অ্যাডভান্সড                  |
| SSL টার্মিনেশন    | সাধারণত না                 | হ্যাঁ                        |
| উদাহরণ            | AWS NLB, IPVS, TCP HAProxy | NGINX, AWS ALB, HTTP HAProxy |

## লোড ব্যালান্সিং অ্যালগরিদম

| অ্যালগরিদম         | বর্ণনা                                 |
|--------------------|----------------------------------------|
| Round Robin        | একের পর এক সার্ভারে অনুরোধ পাঠানো      |
| Least Connections  | যেসব সার্ভারে কম কানেকশন, সেখানে পাঠানো|
| IP Hash            | ক্লায়েন্ট আইপি অনুযায়ী সার্ভার নির্ধারণ|
| Weighted           | শক্তিশালী সার্ভারে বেশি অনুরোধ পাঠানো   |

## জনপ্রিয় লোড ব্যালান্সার সফটওয়্যার/সার্ভিস

| নাম         | ধরন        | বৈশিষ্ট্য                      |
|-------------|------------|-------------------------------|
| NGINX       | সফটওয়্যার  | L4 ও L7, SSL, কন্টেন্ট রাউটিং  |
| HAProxy     | সফটওয়্যার  | হাই পারফরম্যান্স, TCP/HTTP     |
| AWS ELB     | ক্লাউড      | অটো-স্কেল, হাই অ্যাভেইলেবিলিটি|
| F5 BIG-IP   | হার্ডওয়্যার | এন্টারপ্রাইজ ফিচার             |
| Azure LB    | ক্লাউড      | ইন্টিগ্রেটেড উইথ Azure        |

## কনফিগারেশন উদাহরণ (NGINX)

```nginx
# HTTP লোড ব্যালান্সিং
http {
  upstream backend {
    server 192.168.1.10;
    server 192.168.1.11;
  }
  server {
    listen 80;
    location / {
      proxy_pass http://backend;
    }
  }
}
```

- ভুল কনফিগারেশন হলে সার্ভিস ডাউন হতে পারে

## সংক্ষেপে
- লোড ব্যালান্সার সার্ভারের উপর চাপ ভাগ করে পারফরম্যান্স ও অ্যাভেইলেবিলিটি বাড়ায়
- L4 দ্রুত, L7 বেশি ফিচার সমৃদ্ধ
- আধুনিক ওয়েব আর্কিটেকচারে অপরিহার্য

লোড ব্যালান্সার ছাড়া স্কেলেবল ও রিলায়েবল সার্ভিস কল্পনা করা যায় না।
