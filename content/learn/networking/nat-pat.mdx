নেটওয়ার্কিং-এ নিরাপত্তা, IP ব্যবস্থাপনা ও ইন্টারনেট সংযোগের জন্য NAT (Network Address Translation) ও PAT (Port Address Translation) অত্যন্ত গুরুত্বপূর্ণ। এখানে এই দুই প্রযুক্তির ধারণা, কাজের পদ্ধতি, ব্যবহারিক দিক ও উদাহরণ নিয়ে বিস্তারিত আলোচনা করা হলো।

---

## NAT (Network Address Translation) কী?

**সংজ্ঞা:**
NAT হলো এমন একটি প্রযুক্তি, যা এক নেটওয়ার্কের IP অ্যাড্রেসকে অন্য নেটওয়ার্কের IP-তে রূপান্তর করে। সাধারণত, প্রাইভেট নেটওয়ার্কের IP-কে পাবলিক IP-তে রূপান্তর করতে ব্যবহৃত হয়।

### কেন দরকার?
- IPv4 অ্যাড্রেস সংকট সমাধান
- প্রাইভেট নেটওয়ার্ককে ইন্টারনেটের সাথে সংযুক্ত করা
- নিরাপত্তা বৃদ্ধি (ইন্টারনাল IP হাইড করা)

### কিভাবে কাজ করে?
- রাউটার/ফায়ারওয়াল ইনবাউন্ড ও আউটবাউন্ড ট্রাফিকের IP অ্যাড্রেস পরিবর্তন করে
- একাধিক প্রাইভেট IP এক বা একাধিক পাবলিক IP-তে রূপান্তর

### NAT-এর ধরন
| ধরন | বর্ণনা |
|------|---------|
| Static NAT | নির্দিষ্ট এক প্রাইভেট IP ↔ নির্দিষ্ট এক পাবলিক IP |
| Dynamic NAT | একাধিক প্রাইভেট IP ↔ একাধিক পাবলিক IP (পুল থেকে) |
| PAT (NAT Overload) | একাধিক প্রাইভেট IP ↔ এক পাবলিক IP (পোর্ট নম্বর ব্যবহার করে) |

---

## PAT (Port Address Translation) কী?

**সংজ্ঞা:**
PAT হলো NAT-এর একটি বিশেষ রূপ, যেখানে একাধিক প্রাইভেট IP একটিমাত্র পাবলিক IP ব্যবহার করে ইন্টারনেটে যোগাযোগ করে, কিন্তু প্রত্যেকটি সংযোগের জন্য আলাদা পোর্ট নম্বর ব্যবহার হয়। একে NAT Overload-ও বলা হয়।

### কিভাবে কাজ করে?
- রাউটার প্রতিটি সংযোগের জন্য ইউনিক পোর্ট নম্বর অ্যাসাইন করে
- Source IP + Source Port → Public IP + Unique Port
- রিটার্ন ট্রাফিক সঠিক প্রাইভেট IP-তে পাঠানো হয়

### সুবিধা
- এক পাবলিক IP দিয়ে শত শত ডিভাইস ইন্টারনেট ব্যবহার করতে পারে
- IPv4 অ্যাড্রেসের অপ্টিমাল ব্যবহার

## উদাহরণ চিত্র

```
[PC1: 192.168.1.10]   \
[PC2: 192.168.1.20] ----> [Router: NAT/PAT] ----> [Internet: 203.0.113.5]
[PC3: 192.168.1.30]   /
```

- সবগুলো প্রাইভেট IP এক পাবলিক IP (203.0.113.5) ব্যবহার করছে
- PAT হলে, প্রতিটি সংযোগের জন্য আলাদা পোর্ট নম্বর ব্যবহৃত

---

## কনফিগারেশন (Cisco উদাহরণ)

### Static NAT
```bash
Router(config)# ip nat inside source static 192.168.1.10 203.0.113.10
```

### Dynamic NAT
```bash
Router(config)# access-list 1 permit 192.168.1.0 0.0.0.255
Router(config)# ip nat pool MYPOOL 203.0.113.20 203.0.113.30 netmask 255.255.255.0
Router(config)# ip nat inside source list 1 pool MYPOOL
```

### PAT (NAT Overload)
```bash
Router(config)# access-list 1 permit 192.168.1.0 0.0.0.255
Router(config)# ip nat inside source list 1 interface GigabitEthernet0/0 overload
```

## তুলনামূলক টেবিল

| বৈশিষ্ট্য | NAT | PAT |
|-----------|-----|-----|
| IP Mapping | একাধিক প্রাইভেট ↔ একাধিক পাবলিক | একাধিক প্রাইভেট ↔ এক পাবলিক (পোর্ট) |
| পোর্ট নম্বর | ব্যবহৃত হয় না | ব্যবহৃত হয় |
| স্কেলেবিলিটি | সীমিত | অত্যন্ত বেশি |
| সাধারণ নাম | NAT | NAT Overload |

## উপসংহার

NAT ও PAT ছাড়া আধুনিক নেটওয়ার্কিং, বিশেষ করে IPv4-এ, কল্পনা করা যায় না। এগুলো IP ব্যবস্থাপনা, নিরাপত্তা ও ইন্টারনেট সংযোগে অপরিহার্য। বাস্তবিক কনফিগারেশন ও ধারণা ভালোভাবে জানলে নেটওয়ার্ক ডিজাইন সহজ হয়।
