নেটওয়ার্কিং-এ ডেটা এক স্থান থেকে অন্য স্থানে পৌঁছানোর জন্য রাউটিং অপরিহার্য। রাউটিং হচ্ছে এমন একটি প্রক্রিয়া, যার মাধ্যমে রাউটার বা নেটওয়ার্ক ডিভাইস নির্ধারণ করে কোন পথে ডেটা পাঠানো হবে। রাউটিং দুইভাবে করা যায়: **স্ট্যাটিক (Static) রাউটিং** এবং **ডায়নামিক (Dynamic) রাউটিং**।

---

## স্ট্যাটিক রাউটিং

স্ট্যাটিক রাউটিং হল এমন একটি পদ্ধতি, যেখানে নেটওয়ার্ক অ্যাডমিন ম্যানুয়ালি রাউটিং টেবিলে রুট এন্ট্রি যোগ করেন।

### বৈশিষ্ট্য
- রাউটিং টেবিল ম্যানুয়ালি কনফিগার করতে হয়
- ছোট ও সহজ নেটওয়ার্কে বেশি ব্যবহৃত
- কম রিসোর্স ব্যবহার করে
- স্বয়ংক্রিয়ভাবে পরিবর্তন হয় না

### সুবিধা
- নিরাপদ: অপ্রত্যাশিত রুট পরিবর্তন হয় না
- সহজ: ছোট নেটওয়ার্কে কনফিগারেশন সহজ
- কম CPU ও মেমোরি ব্যবহার

### অসুবিধা
- বড় নেটওয়ার্কে পরিচালনা কঠিন
- লিঙ্ক ডাউন হলে ম্যানুয়ালি আপডেট করতে হয়
- স্কেলেবল নয়

### কনফিগারেশন উদাহরণ (Cisco)
```bash
Router(config)# ip route [গন্তব্য নেটওয়ার্ক] [সাবনেট মাস্ক] [নেক্সট-হপ IP]
Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
```

## ডায়নামিক রাউটিং

ডায়নামিক রাউটিং-এ রাউটাররা একে অপরের সাথে রাউটিং তথ্য শেয়ার করে এবং স্বয়ংক্রিয়ভাবে রাউটিং টেবিল আপডেট করে।

### বৈশিষ্ট্য
- রাউটাররা রাউটিং প্রোটোকল ব্যবহার করে তথ্য আদান-প্রদান করে
- বড় ও জটিল নেটওয়ার্কে বেশি ব্যবহৃত
- স্বয়ংক্রিয়ভাবে রুট পরিবর্তন ও আপডেট

### সুবিধা
- স্কেলেবল: বড় নেটওয়ার্কে সহজে পরিচালনা
- স্বয়ংক্রিয় ব্যাকআপ রুট
- দ্রুত কনভার্জেন্স (নেটওয়ার্ক পরিবর্তনে দ্রুত আপডেট)

### অসুবিধা
- বেশি CPU ও মেমোরি ব্যবহার
- কনফিগারেশন তুলনামূলক জটিল
- নিরাপত্তা ঝুঁকি (ভুল কনফিগারেশনে রুট লিক হতে পারে)

### জনপ্রিয় ডায়নামিক রাউটিং প্রোটোকল
| প্রোটোকল | টাইপ | বৈশিষ্ট্য |
|----------|------|-----------|
| RIP      | Distance Vector | সহজ, ছোট নেটওয়ার্কে উপযোগী |
| OSPF     | Link State     | বড় নেটওয়ার্ক, দ্রুত কনভার্জেন্স |
| EIGRP    | Hybrid         | Cisco-তে জনপ্রিয় |
| BGP      | Path Vector    | ইন্টারনেট ও বড় ISP-তে ব্যবহৃত |

### কনফিগারেশন উদাহরণ (RIP)
```bash
Router(config)# router rip
Router(config-router)# network 192.168.1.0
Router(config-router)# network 10.0.0.0
```

## তুলনামূলক টেবিল

| বৈশিষ্ট্য         | স্ট্যাটিক রাউটিং         | ডায়নামিক রাউটিং         |
|-------------------|--------------------------|--------------------------|
| কনফিগারেশন       | ম্যানুয়াল               | স্বয়ংক্রিয়              |
| স্কেলেবিলিটি      | কম                       | বেশি                     |
| রিসোর্স ব্যবহার   | কম                       | বেশি                     |
| পরিবর্তন          | ম্যানুয়াল               | স্বয়ংক্রিয়              |
| নিরাপত্তা         | বেশি                     | তুলনামূলক কম            |
| ব্যবহারের ক্ষেত্র  | ছোট নেটওয়ার্ক           | বড় ও জটিল নেটওয়ার্ক     |

---

## কখন কোনটি ব্যবহার করবেন?
- **স্ট্যাটিক রাউটিং:** ছোট, নির্ভরযোগ্য, কম পরিবর্তনশীল নেটওয়ার্কে
- **ডায়নামিক রাউটিং:** বড়, জটিল, দ্রুত পরিবর্তনশীল নেটওয়ার্কে

---

## সংক্ষেপে
- স্ট্যাটিক রাউটিং সহজ, নিরাপদ, কিন্তু স্কেলেবল নয়
- ডায়নামিক রাউটিং স্কেলেবল, স্বয়ংক্রিয়, কিন্তু কনফিগারেশন ও রিসোর্স ব্যবহারে বেশি

নেটওয়ার্ক ডিজাইন করার সময় আপনার চাহিদা ও নেটওয়ার্কের আকার অনুযায়ী সঠিক রাউটিং পদ্ধতি বেছে নিন।
