ইথেরিয়াম স্মার্ট কন্ট্র্যাক্টে "ইভেন্ট" ও "লগ" ব্যবহার করে চেইনে তথ্য প্রকাশ করা হয়। ড্যাপ, ফ্রন্টএন্ড, ও ডেভেলপাররা এই ইভেন্ট ও লগ ব্যবহার করে কন্ট্র্যাক্টের কার্যকলাপ ট্র্যাক করতে পারে।

## ইভেন্ট কী?

ইভেন্ট হল স্মার্ট কন্ট্র্যাক্টে একটি বিশেষ ফাংশন, যা চেইনে নির্দিষ্ট তথ্য "লগ" করে।

- **ফ্রন্টএন্ড অ্যাপ:** ইভেন্ট শুনে UI আপডেট করে
- **ডেভেলপার:** ট্রান্সাকশন ট্র্যাক করে

## ইভেন্ট ডিক্লেয়ার ও এমিট

```solidity
pragma solidity ^0.8.0;

contract EventDemo {
    event MessageChanged(address indexed user, string newMessage);
    string public message;

    function setMessage(string memory _msg) public {
        message = _msg;
        emit MessageChanged(msg.sender, _msg);
    }
}
```

এখানে, যখনই `setMessage` ফাংশন কল হবে, একটি ইভেন্ট চেইনে "লগ" হবে।

## ইভেন্টের সুবিধা

- **ড্যাপ ইন্টিগ্রেশন:** ফ্রন্টএন্ডে রিয়েল-টাইম আপডেট
- **ডিবাগিং:** কন্ট্র্যাক্টের কার্যকলাপ ট্র্যাক
- **অডিটিং:** ট্রান্সাকশন হিস্ট্রি

## লগ কী?

লগ হল চেইনে সংরক্ষিত ইভেন্টের তথ্য। Web3.js, Ethers.js দিয়ে লগ পড়া যায়।

## Web3.js দিয়ে ইভেন্ট পড়া

```javascript
contract.events.MessageChanged({})
.on('data', event => {
  console.log(event.returnValues);
});
```

## বাস্তব উদাহরণ

ধরা যাক, আপনি একটি ব্যাংকে টাকা জমা দিলেন। ব্যাংক আপনাকে একটি রিসিট দিল—এটাই "ইভেন্ট"। পরে আপনি রিসিট দেখে হিসাব মিলাতে পারবেন—এটাই "লগ"।

## নিরাপত্তা ও পারফরম্যান্স টিপস

- ইভেন্টে সংবেদনশীল তথ্য প্রকাশ করবেন না
- Indexed প্যারামিটার ব্যবহার করুন, যাতে সহজে ফিল্টার করা যায়
- ইভেন্টের নাম ও স্ট্রাকচার স্পষ্ট রাখুন
