বিটকয়েনের লেনদেনের নিরাপত্তা ও নিয়ন্ত্রণের জন্য "স্ক্রিপ্টিং ল্যাঙ্গুয়েজ" ব্যবহৃত হয়। এটি একটি সহজ, স্ট্যাক-ভিত্তিক ভাষা, যা লেনদেনের শর্ত নির্ধারণ করে।

## স্ক্রিপ্ট কী?

বিটকয়েন স্ক্রিপ্ট হল ছোট ছোট কমান্ডের সমষ্টি, যা বলে দেয় কিভাবে একটি UTXO খরচ করা যাবে। উদাহরণস্বরূপ, "শুধুমাত্র এই ব্যক্তির সিগনেচার থাকলে টাকা খরচ হবে"।

## স্ক্রিপ্টের গঠন

প্রতিটি বিটকয়েন লেনদেনে দুটি স্ক্রিপ্ট থাকে:
- **Locking Script (ScriptPubKey):** আউটপুটে থাকে, কে টাকা খরচ করতে পারবে তা নির্ধারণ করে
- **Unlocking Script (ScriptSig):** ইনপুটে থাকে, খরচ করার জন্য প্রমাণ দেয়

## বাস্তব উদাহরণ: Pay-to-PubKey-Hash (P2PKH)

সবচেয়ে সাধারণ স্ক্রিপ্ট:
- **Locking Script:** `OP_DUP OP_HASH160 <PubKeyHash> OP_EQUALVERIFY OP_CHECKSIG`
- **Unlocking Script:** `<Signature> <PublicKey>`

এটি বলে দেয়, নির্দিষ্ট পাবলিক কী-এর মালিক সঠিক সিগনেচার দিলে UTXO খরচ হবে।

## স্ক্রিপ্ট অপকোড

- `OP_DUP`: স্ট্যাকের টপ আইটেম ডুপ্লিকেট করে
- `OP_HASH160`: পাবলিক কী-কে হ্যাশ করে
- `OP_EQUALVERIFY`: দুইটি আইটেম সমান কিনা যাচাই
- `OP_CHECKSIG`: সিগনেচার যাচাই

## বাস্তব জীবনের তুলনা

ধরা যাক, একটি লকার আছে, যার চাবি শুধু নির্দিষ্ট ব্যক্তির কাছে। লকার খুলতে হলে সেই ব্যক্তির সঠিক চাবি (সিগনেচার) লাগবে।

## কোড উদাহরণ: স্ক্রিপ্ট ইমুলেশন

```python
# বিটকয়েন স্ক্রিপ্ট ইমুলেশন (সিম্পল)

def p2pkh_script(pubkey_hash, signature, pubkey):
    # Step 1: OP_DUP
    stack = [pubkey, pubkey]
    # Step 2: OP_HASH160
    stack[-1] = hash160(stack[-1])
    # Step 3: OP_EQUALVERIFY
    if stack[-1] != pubkey_hash:
        return False
    # Step 4: OP_CHECKSIG
    return verify_signature(signature, pubkey)

# hash160 ও verify_signature ফাংশন ইমপ্লিমেন্ট করতে হবে
```

## স্ক্রিপ্টের সীমাবদ্ধতা

- Turing-complete নয়, অর্থাৎ লুপ বা জটিল লজিক নেই
- নিরাপত্তার জন্য সীমিত অপকোড

## নিরাপত্তা ও পারফরম্যান্স টিপস

- স্ক্রিপ্ট লেখার সময় ভুল অপকোড এড়িয়ে চলুন
- ScriptSig ও ScriptPubKey সঠিকভাবে যাচাই করুন
- Multisig, timelock ইত্যাদি অ্যাডভান্সড স্ক্রিপ্ট ব্যবহার শিখুন
