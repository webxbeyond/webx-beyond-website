বিটকয়েনের সবচেয়ে গুরুত্বপূর্ণ ধারণাগুলোর একটি হল UTXO (Unspent Transaction Output) মডেল। এটি বিটকয়েনের লেনদেনের ভিত্তি, যা অন্যান্য ব্লকচেইন (যেমন ইথেরিয়াম) থেকে আলাদা।

## লেনদেন কীভাবে কাজ করে?

বিটকয়েনে "অ্যাকাউন্ট" নেই, বরং "UTXO" আছে। প্রতিটি লেনদেনের আউটপুট, যা এখনও খরচ হয়নি, সেটাই UTXO।

- **ইনপুট:** পূর্ববর্তী লেনদেনের UTXO, যা খরচ করা হবে
- **আউটপুট:** নতুন UTXO, যা ভবিষ্যতে খরচ করা যাবে

## বাস্তব উদাহরণ

ধরা যাক, আপনার কাছে ৫০০ টাকার দুটি নোট আছে (দুটি UTXO)। আপনি কাউকে ৭০০ টাকা পাঠাতে চান। আপনি দুটি নোট (৫০০+৫০০=১০০০) ব্যবহার করবেন, ৭০০ পাঠাবেন, আর ৩০০ টাকা "চেঞ্জ" হিসেবে নিজের কাছে ফেরত পাবেন।

## UTXO মডেলের সুবিধা

- **নিরাপত্তা:** প্রতিটি আউটপুট আলাদা, তাই ডাবল-স্পেন্ডিং রোধ করা সহজ
- **স্কেলেবিলিটি:** প্যারালাল ভেরিফিকেশন সম্ভব
- **প্রাইভেসি:** একাধিক আউটপুট ব্যবহার করে ট্রান্সাকশন বিভাজন করা যায়

## UTXO বনাম অ্যাকাউন্ট মডেল

- **UTXO:** বিটকয়েন, Litecoin, Bitcoin Cash
- **অ্যাকাউন্ট:** Ethereum, Solana

UTXO-তে প্রতিটি লেনদেন "নোট"-এর মতো, আর অ্যাকাউন্ট মডেলে "ব্যালেন্স" থাকে।

## কোড উদাহরণ: সহজ UTXO ট্রান্সাকশন

```python
# UTXO ট্রান্সাকশন মডেল

class UTXO:
    def __init__(self, txid, amount):
        self.txid = txid
        self.amount = amount

# পূর্ববর্তী UTXO
inputs = [UTXO('txid1', 500), UTXO('txid2', 500)]

# আউটপুট: ৭০০ পাঠানো, ৩০০ চেঞ্জ
outputs = [
    {'address': 'receiver', 'amount': 700},
    {'address': 'sender', 'amount': 300}
]

# ট্রান্সাকশন তৈরি
transaction = {
    'inputs': inputs,
    'outputs': outputs
}
print(transaction)
```

## নিরাপত্তা ও পারফরম্যান্স টিপস

- UTXO খরচ করার সময় সব ইনপুট সঠিকভাবে সাইন করুন
- চেইন স্ক্যান করে নিজের UTXO খুঁজে নিন
- "Dust" UTXO (খুব ছোট) এড়িয়ে চলুন, কারণ ফি বেশি লাগতে পারে
