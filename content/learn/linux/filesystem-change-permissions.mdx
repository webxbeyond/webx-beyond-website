## chmod, chown, chgrp

লিনাক্সে ফাইল ও ডিরেক্টরির পারমিশন ও মালিকানা পরিবর্তনের জন্য তিনটি গুরুত্বপূর্ণ কমান্ড আছে: `chmod`, `chown`, ও `chgrp`। এগুলোর মাধ্যমে আপনি নির্দিষ্ট করতে পারেন কে কোন ফাইল পড়তে, লিখতে, বা চালাতে পারবে এবং কে মালিক বা কোন গ্রুপের অধীনে থাকবে।

---

## ১. chmod: পারমিশন পরিবর্তন

`chmod` (change mode) দিয়ে ফাইল/ডিরেক্টরির পারমিশন পরিবর্তন করা যায়।

### Symbolic Method

```bash
chmod u+x script.sh   # owner-এর জন্য execute পারমিশন যোগ
chmod g-w file.txt    # group-এর জন্য write পারমিশন বাদ
chmod o+r file.txt    # others-এর জন্য read পারমিশন যোগ
```

### Numeric (Octal) Method

```bash
chmod 755 file.sh     # owner: rwx, group: r-x, others: r-x
chmod 644 document.txt # owner: rw, group: r, others: r
```

### Recursive

```bash
chmod -R 755 myfolder/   # ফোল্ডার ও ভিতরের সব ফাইল/ফোল্ডারে
```

---

## ২. chown: মালিকানা পরিবর্তন

`chown` (change owner) দিয়ে ফাইল/ডিরেক্টরির মালিক (owner) ও গ্রুপ পরিবর্তন করা যায়।

```bash
chown anis file.txt           # owner পরিবর্তন
chown anis:devs file.txt      # owner ও group পরিবর্তন
chown -R root:root /var/www   # recursive
```

**শুধুমাত্র root ইউজার chown চালাতে পারে।**

---

## ৩. chgrp: গ্রুপ পরিবর্তন

`chgrp` (change group) দিয়ে শুধু গ্রুপ পরিবর্তন করা যায়।

```bash
chgrp devs file.txt
chgrp -R staff myfolder/
```

---

## ৪. প্র্যাকটিক্যাল টিপস

- স্ক্রিপ্ট চালাতে `chmod +x script.sh` দিন
- টিমওয়ার্কে গ্রুপ পারমিশন ও মালিকানা ঠিক রাখুন
- ভুলে গেলে `ls -l` দিয়ে চেক করুন
- রুট ইউজার ছাড়া chown/chgrp চালানো যাবে না

## চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. একটি ফাইলের পারমিশন 755 করুন
২. ফাইলের মালিকানা নিজের নামে পরিবর্তন করুন
৩. ফাইলের গ্রুপ পরিবর্তন করুন
৪. একটি ফোল্ডারে recursive ভাবে পারমিশন/মালিকানা পরিবর্তন করুন

## উপসংহার

`chmod`, `chown`, ও `chgrp`—এই তিনটি কমান্ড ছাড়া লিনাক্সে ফাইল নিরাপত্তা, মালিকানা, ও টিমওয়ার্ক কল্পনাই করা যায় না। এগুলো ভালোভাবে আয়ত্ত করলে আপনি সিস্টেম অ্যাডমিনিস্ট্রেশন ও সিকিউরিটি ম্যানেজমেন্টে দক্ষ হবেন।

**প্র্যাকটিস করুন, ফাইল কন্ট্রোলের মাস্টার হয়ে উঠুন!**
