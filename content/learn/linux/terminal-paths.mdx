## Absolute বনাম Relative

টার্মিনালে ফাইল বা ডিরেক্টরি নিয়ে কাজ করার সময় "পাথ" (Path) খুব গুরুত্বপূর্ণ। পাথ হচ্ছে কোনো ফাইল বা ফোল্ডারের অবস্থান নির্দেশ করার উপায়। লিনাক্সে দুই ধরনের পাথ আছে: Absolute Path ও Relative Path।

---

## ১. Absolute Path (পূর্ণাঙ্গ পাথ)

Absolute Path হচ্ছে ফাইল/ফোল্ডারের সম্পূর্ণ ঠিকানা, যা রুট (/) থেকে শুরু হয়।

**উদাহরণ:**

```
/home/anis/Documents/file.txt
/etc/hosts
/var/log/syslog
```

- সবসময় `/` দিয়ে শুরু হয়
- যেকোনো অবস্থান থেকে একইভাবে অ্যাক্সেস করা যায়

**কেন দরকার?**
- স্ক্রিপ্ট, কনফিগারেশন, বা রিমোট অ্যাক্সেসে নির্ভুলভাবে ফাইল লোকেট করতে

---

## ২. Relative Path (আপেক্ষিক পাথ)

Relative Path হচ্ছে বর্তমান ডিরেক্টরি (Current Directory) থেকে গন্তব্যের পথ। `/` দিয়ে শুরু হয় না।

**উদাহরণ:**

```
Documents/file.txt
../Downloads/music.mp3
./script.sh
```

- `.` (dot) মানে বর্তমান ডিরেক্টরি
- `..` (double dot) মানে এক ধাপ উপরের ডিরেক্টরি
- বর্তমান অবস্থান অনুযায়ী পাথ পরিবর্তন হয়

**কেন দরকার?**
- দ্রুত নেভিগেশন, স্ক্রিপ্টে পোর্টেবিলিটি, ও ডাইনামিক ফাইল অ্যাক্সেসে

---

## ৩. কিছু কমান্ডে পাথ ব্যবহার

- `cd /home/anis/Documents` (Absolute)
- `cd ../Downloads` (Relative)
- `cp ./file.txt /home/anis/backup/` (Relative to Absolute)

---

## ৪. প্র্যাকটিক্যাল টিপস

- `pwd` দিয়ে বর্তমান পাথ জেনে নিন
- Absolute Path সবসময় নির্ভুল, কিন্তু Relative Path দ্রুত
- স্ক্রিপ্ট লেখার সময় Relative Path দিলে অন্যদের জন্যও সহজ হয়

---

## চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. Absolute Path দিয়ে Desktop-এ যান
২. Relative Path দিয়ে এক ডিরেক্টরি থেকে অন্য ডিরেক্টরিতে যান
৩. `pwd` দিয়ে চেক করুন, কোথায় আছেন

---

## উপসংহার

Absolute ও Relative Path—দুটোই দরকারি। Absolute Path নির্ভুল, Relative Path দ্রুত ও ফ্লেক্সিবল। দুটোই ভালোভাবে আয়ত্ত করলে টার্মিনালে কাজ হবে আরও সহজ ও দক্ষ।

**প্র্যাকটিস করুন, পাথের জাদু শিখুন!**
