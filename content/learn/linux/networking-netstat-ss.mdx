# netstat ও ss দিয়ে কানেকশন দেখা: নেটওয়ার্ক ট্রাবলশুটিংয়ের মাস্টারি

লিনাক্সে কোন কোন কানেকশন ওপেন, কোন পোর্টে কোন সার্ভিস চলছে, TCP/UDP স্টেটাস—এসব জানতে সবচেয়ে বেশি ব্যবহৃত টুল হলো `netstat` ও `ss`। এখানে আমরা এই দুই টুলের ব্যবহার, পার্থক্য, কমান্ড, ও প্র্যাকটিক্যাল টিপস জানব।

---

## ১. netstat: ক্লাসিক নেটওয়ার্ক টুল

### বেসিক ব্যবহার:
```bash
netstat
```

### সব কানেকশন ও লিসেনিং পোর্ট:
```bash
netstat -tuln
```
- `-t`: TCP
- `-u`: UDP
- `-l`: Listening
- `-n`: Numeric (DNS রেজলভ ছাড়া)

### নির্দিষ্ট পোর্টে কি চলছে?
```bash
netstat -tulnp | grep 80
```

### রাউটিং টেবিল:
```bash
netstat -r
```

### ইন্টারফেস স্ট্যাটাস:
```bash
netstat -i
```

---

## ২. ss: আধুনিক ও দ্রুত টুল

- netstat-এর বিকল্প, আরও দ্রুত ও ফিচারড

### বেসিক ব্যবহার:
```bash
ss
```

### সব TCP কানেকশন:
```bash
ss -t
```

### সব UDP কানেকশন:
```bash
ss -u
```

### লিসেনিং পোর্ট:
```bash
ss -lnt
```

### নির্দিষ্ট পোর্টে কি চলছে?
```bash
ss -lntp | grep 80
```

### সার্ভিস ও PID সহ:
```bash
ss -lntp
```

---

## ৩. netstat বনাম ss

| বৈশিষ্ট্য | netstat | ss |
|-----------|---------|----|
| গতি | কম | বেশি |
| ফিচার | কম | বেশি |
| ডিফল্ট ইনস্টল | পুরনো | নতুন |
| রিলায়েবিলিটি | ভালো | খুব ভালো |

---

## ৪. প্র্যাকটিক্যাল টিপস

- ss ব্যবহার করুন, netstat পুরনো
- sudo দিয়ে চালালে সব কানেকশন দেখা যায়
- grep দিয়ে নির্দিষ্ট পোর্ট/সার্ভিস ফিল্টার করুন
- man page পড়ুন: `man netstat`, `man ss`

---

## ৫. চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. ss দিয়ে সব TCP/UDP কানেকশন দেখুন
২. কোন পোর্টে কোন সার্ভিস চলছে বের করুন
৩. netstat ও ss-এর আউটপুট তুলনা করুন

---

## উপসংহার

netstat ও ss—এই দুইটি টুল ছাড়া লিনাক্সে নেটওয়ার্ক কানেকশন ও পোর্ট মনিটরিং অসম্ভব। ss-এ দক্ষ হোন, নেটওয়ার্ক ট্রাবলশুটিংয়ে এক্সপার্ট হোন।

**প্র্যাকটিস করুন, নেটওয়ার্ক মনিটরিংয়ে মাস্টার হোন!**
