# diff দিয়ে ফাইল তুলনা

লিনাক্সে দুইটি ফাইল বা ডিরেক্টরির মধ্যে পার্থক্য (difference) খুঁজে বের করার জন্য সবচেয়ে বেশি ব্যবহৃত টুল হলো `diff`। এটি কোড, কনফিগারেশন, বা যেকোনো টেক্সট ফাইলের তুলনা করতে কাজে লাগে। এখানে আমরা diff-এর বেসিক থেকে অ্যাডভান্সড ব্যবহার, অপশন, ও টিপস জানব।

---

## ১. diff-এর বেসিক ব্যবহার

```bash
diff file1.txt file2.txt
```

এতে দুই ফাইলের পার্থক্য লাইন বাই লাইন দেখাবে।

---

## ২. diff আউটপুট বোঝা

- `<` : প্রথম ফাইলের কনটেন্ট
- `>` : দ্বিতীয় ফাইলের কনটেন্ট
- সংখ্যা ও চিহ্ন দিয়ে কোন লাইনে পার্থক্য, তা দেখায়

---

## ৩. ইউনিফাইড ফরম্যাট (-u)

`-u` অপশন দিলে ইউনিফাইড (git-style) আউটপুট পাবেন, যা কোড রিভিউ বা প্যাচ তৈরিতে কাজে লাগে।

```bash
diff -u old.py new.py
```

---

## ৪. রিকার্সিভ ডিরেক্টরি তুলনা (-r)

ডিরেক্টরি তুলনা করতে:

```bash
diff -r dir1/ dir2/
```

---

## ৫. side-by-side তুলনা (-y)

`-y` অপশন দিলে দুই ফাইল পাশাপাশি দেখাবে।

```bash
diff -y file1.txt file2.txt
```

---

## ৬. কেস-ইনসেনসিটিভ (-i)

```bash
diff -i file1.txt file2.txt
```

---

## ৭. diff আউটপুট প্যাচ হিসেবে সংরক্ষণ

```bash
diff -u old.txt new.txt > changes.patch
```

পরে প্যাচ অ্যাপ্লাই করতে:

```bash
patch old.txt < changes.patch
```

---

## ৮. প্র্যাকটিক্যাল টিপস

- কোড/কনফিগারেশন পরিবর্তন ট্র্যাক করতে diff ব্যবহার করুন
- ইউনিফাইড (-u) আউটপুট git-এ প্যাচ/রিভিউতে কাজে লাগে
- ডিরেক্টরি তুলনা করে ব্যাকআপ/মাইগ্রেশন চেক করুন

---

## চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. দুইটি টেক্সট ফাইলের পার্থক্য দেখুন
২. ইউনিফাইড ফরম্যাটে diff চালান
৩. দুইটি ফোল্ডার তুলনা করুন
৪. diff আউটপুট প্যাচ হিসেবে সংরক্ষণ ও অ্যাপ্লাই করুন

---

## উপসংহার

`diff` ছাড়া লিনাক্সে ফাইল/ডিরেক্টরি তুলনা, কোড রিভিউ, ও কনফিগারেশন ম্যানেজমেন্ট অসম্ভব। এটি ভালোভাবে আয়ত্ত করলে আপনি সিস্টেম অ্যাডমিন, ডেভেলপার, ও পাওয়ার ইউজার হিসেবে আরও দক্ষ হবেন।

**প্র্যাকটিস করুন, diff-এর জাদু শিখুন!**
