## ;, &&, ||

লিনাক্স টার্মিনালে একাধিক কমান্ড এক লাইনে চালানোর জন্য চেইনিং অপারেটর ব্যবহার করা হয়। এতে কাজ হয় দ্রুত, স্মার্ট ও অটোমেটেড। এখানে আমরা তিনটি গুরুত্বপূর্ণ চেইনিং অপারেটর নিয়ে বিস্তারিত আলোচনা করব: `;`, `&&`, ও `||`।

---

## ১. ; (সেমিকোলন): সব কমান্ড চালাবে

`;` দিয়ে একাধিক কমান্ড এক লাইনে লিখলে, প্রতিটি কমান্ড একে একে চালানো হবে—আগেরটা সফল/ব্যর্থ যাই হোক না কেন।

```bash
command1 ; command2 ; command3
```

**উদাহরণ:**

```bash
cd ~/Downloads ; ls ; echo "ডাউনলোড ফোল্ডারে আছি"
```

---

## ২. && (AND): আগেরটা সফল হলে পরেরটা

`&&` দিয়ে চেইন করলে, আগের কমান্ড সফল (exit status 0) হলে তবেই পরেরটা চলবে।

```bash
command1 && command2
```

**উদাহরণ:**

```bash
mkdir test && cd test && touch file.txt
```

এখানে, `mkdir test` সফল হলে তবেই `cd test` হবে, তারপর `touch file.txt`।

---

## ৩. || (OR): আগেরটা ব্যর্থ হলে পরেরটা

`||` দিয়ে চেইন করলে, আগের কমান্ড ব্যর্থ (exit status non-zero) হলে তবেই পরেরটা চলবে।

```bash
command1 || command2
```

**উদাহরণ:**

```bash
cd myfolder || echo "ফোল্ডার নেই!"
```

এখানে, `cd myfolder` ব্যর্থ হলে (ফোল্ডার না থাকলে) "ফোল্ডার নেই!" প্রিন্ট হবে।

---

## ৪. চেইনিং কম্বিনেশন

তিনটি অপারেটর একসাথে ব্যবহার করা যায়:

```bash
mkdir test && cd test || echo "ফোল্ডার তৈরি/প্রবেশ ব্যর্থ"
```

---

## ৫. প্র্যাকটিক্যাল টিপস

- স্ক্রিপ্টে চেইনিং দিয়ে error handling সহজ করুন
- && দিয়ে কন্ডিশনাল কাজ করুন (যেমন: build && deploy)
- || দিয়ে fallback বা error message দিন
- ; দিয়ে একসাথে অনেক কাজ করুন

---

## চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. এক লাইনে তিনটি কমান্ড চালান
২. && দিয়ে কন্ডিশনাল ফাইল তৈরি করুন
৩. || দিয়ে error message দেখান

---

## উপসংহার

`;`, `&&`, `||`—এই চেইনিং অপারেটরগুলো ছাড়া টার্মিনালে স্মার্ট ও দ্রুত কাজ কল্পনাই করা যায় না। এগুলো ভালোভাবে আয়ত্ত করলে আপনার স্ক্রিপ্টিং ও ডেইলি টার্মিনাল ইউজ অনেক সহজ হবে।

**প্র্যাকটিস করুন, চেইনিংয়ের জাদু শিখুন!**
