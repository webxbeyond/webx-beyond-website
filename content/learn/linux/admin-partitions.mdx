## লিনাক্সে স্টোরেজ ম্যানেজমেন্টের হাতেখড়ি

ডিস্ক পার্টিশনিং হলো—স্টোরেজ ডিভাইসকে একাধিক অংশে ভাগ করা, যাতে ডেটা আলাদা রাখা, মাল্টিবুট, ব্যাকআপ, ইত্যাদি সহজ হয়। এখানে আমরা fdisk ও lsblk দিয়ে পার্টিশন দেখা, তৈরি, মুছে ফেলা, ও মাউন্ট—সবকিছু হাতে-কলমে শিখব।

---

## ১. পার্টিশন কী ও কেন?

- ডিস্ককে লজিক্যাল অংশে ভাগ করা
- আলাদা OS, ডেটা, ব্যাকআপ, সুইপ, ইত্যাদির জন্য

---

## ২. পার্টিশন টাইপ

- Primary, Extended, Logical
- ফাইল সিস্টেম: ext4, xfs, ntfs, fat32

---

## ৩. ডিস্ক ও পার্টিশন দেখতে lsblk

```bash
lsblk
lsblk -f # ফাইল সিস্টেমসহ
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT
```

---

## ৪. ডিস্ক ও পার্টিশন দেখতে fdisk

```bash
sudo fdisk -l
```

---

## ৫. নতুন পার্টিশন তৈরি (fdisk)

**সতর্কতা: ডেটা হারানোর ঝুঁকি!**

```bash
sudo fdisk /dev/sdb
```

- `m` : help
- `n` : নতুন পার্টিশন
- `p` : প্রাইমারি
- `e` : এক্সটেন্ডেড
- `d` : ডিলিট
- `w` : লিখে বের হওয়া
- `q` : সেভ না করে বের হওয়া

---

## ৬. পার্টিশন ফরম্যাট ও মাউন্ট

```bash
sudo mkfs.ext4 /dev/sdb1
sudo mkdir /mnt/mydata
sudo mount /dev/sdb1 /mnt/mydata
```

---

## ৭. পার্মানেন্ট মাউন্ট (fstab)

`/etc/fstab` ফাইলে এন্ট্রি দিন:
```
/dev/sdb1  /mnt/mydata  ext4  defaults  0  2
```

---

## ৮. পার্টিশন রিসাইজ ও মুছে ফেলা

- রিসাইজ: `resize2fs`, `parted`
- ডিলিট: fdisk দিয়ে `d`

---

## ৯. পার্টিশন UUID ও লেবেল

```bash
lsblk -o NAME,UUID,LABEL
sudo e2label /dev/sdb1 mylabel
```

---

## ১০. পার্টিশন টেবিল: MBR বনাম GPT

- MBR: পুরনো, ২TB সীমা, ৪টি প্রাইমারি
- GPT: আধুনিক, ১২৮+ পার্টিশন, ৯.৪ZB পর্যন্ত

---

## ১১. কমন সমস্যা ও সতর্কতা

- ভুল পার্টিশন ডিলিট করলে ডেটা হারাতে পারেন
- ব্যাকআপ ছাড়া পার্টিশনিং করবেন না
- মাউন্ট/আনমাউন্টের সময় ফাইল ব্যবহার করবেন না

---

## ১২. বাস্তব উদাহরণ: নতুন হার্ডডিস্ক পার্টিশন

১. `sudo fdisk /dev/sdb` দিয়ে নতুন পার্টিশন
২. `sudo mkfs.ext4 /dev/sdb1` দিয়ে ফরম্যাট
৩. `sudo mount /dev/sdb1 /mnt/mydata`
৪. `/etc/fstab` এ এন্ট্রি
