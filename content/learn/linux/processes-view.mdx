# ps, top, htop দিয়ে প্রসেস দেখা: সিস্টেম মনিটরিংয়ের হাতেখড়ি

লিনাক্সে কোন কোন প্রোগ্রাম বা প্রসেস চলছে, কতটা রিসোর্স ব্যবহার করছে—এসব জানতে `ps`, `top`, ও `htop` কমান্ড অপরিহার্য। এখানে আমরা এই তিনটি টুলের ব্যবহার, পার্থক্য, ও প্র্যাকটিক্যাল টিপস জানব।

---

## ১. ps (Process Status)

`ps` কমান্ড দিয়ে সিস্টেমে চলমান প্রসেসের তালিকা দেখা যায়।

### বেসিক ব্যবহার:
```bash
ps
```
- কারেন্ট শেলের প্রসেস দেখাবে

### সব প্রসেস দেখতে:
```bash
ps aux
```
- `a`: সব ইউজারের
- `u`: ইউজার ফরম্যাট
- `x`: কন্ট্রোল টার্মিনাল ছাড়াও

### নির্দিষ্ট প্রসেস খুঁজতে:
```bash
ps aux | grep firefox
```

### নির্দিষ্ট ইউজারের প্রসেস:
```bash
ps -u anis
```

---

## ২. top: লাইভ প্রসেস মনিটর

`top` কমান্ড দিয়ে রিয়েল-টাইমে প্রসেস, CPU, RAM, লোড ইত্যাদি দেখা যায়।

### চালানো:
```bash
top
```

### top-এর ইন্টারফেস:
- উপরে: সিস্টেম ইনফো (CPU, RAM, লোড)
- নিচে: প্রসেস লিস্ট

### top-এ কী-বাইন্ডিং:
- `P`: CPU usage অনুযায়ী সাজানো
- `M`: মেমোরি usage অনুযায়ী সাজানো
- `k`: প্রসেস kill
- `q`: বের হওয়া

### নির্দিষ্ট ইউজারের প্রসেস:
```bash
top -u anis
```

---

## ৩. htop: গ্রাফিক্যাল ও ইন্টারেক্টিভ

`htop` হলো top-এর উন্নত ও গ্রাফিক্যাল ভার্সন।

### ইন্সটল:
```bash
sudo apt install htop
```

### চালানো:
```bash
htop
```

### htop-এর ফিচার:
- কালারফুল ও ইন্টারেক্টিভ
- মাউস সাপোর্ট
- প্রসেস kill/renice সহজ
- ফিল্টার ও সার্চ

---

## ৪. প্র্যাকটিক্যাল টিপস

- `ps aux | less` দিয়ে বড় লিস্ট সহজে দেখুন
- top/htop-এ PID দেখে kill করুন
- htop-এ F6 দিয়ে সাজানোর অপশন
- প্রসেসের ডিটেইল জানতে man page পড়ুন: `man ps`, `man top`, `man htop`

---

## ৫. চ্যালেঞ্জ: নিজে চেষ্টা করুন

১. ps দিয়ে নিজের ও অন্য ইউজারের প্রসেস দেখুন
২. top ও htop চালিয়ে লাইভ প্রসেস মনিটর করুন
৩. htop দিয়ে প্রসেস kill করুন

---

## উপসংহার

`ps`, `top`, ও `htop`—এই তিনটি টুল ছাড়া লিনাক্স সিস্টেম মনিটরিং অসম্ভব। এগুলো ভালোভাবে আয়ত্ত করলে আপনি সিস্টেমের রিসোর্স ও প্রসেস সহজে ম্যানেজ করতে পারবেন।

**প্র্যাকটিস করুন, সিস্টেম মনিটরিংয়ে এক্সপার্ট হোন!**
