Nextcloud হলো জনপ্রিয় ওপেন-সোর্স ফাইল, ক্যালেন্ডার, কন্টাক্ট ও ডকুমেন্ট শেয়ারিং প্ল্যাটফর্ম। নিজস্ব সার্ভারে এটি চালিয়ে আপনি নিজের ডেটার পূর্ণ নিয়ন্ত্রণ পাবেন।

## ১. সার্ভার প্রস্তুতি
- VPS/Cloud/Local server (Ubuntu 22.04 LTS সুপারিশকৃত)
- SSH access
- Firewall (UFW) enable করুন

## ২. Docker দিয়ে Nextcloud ডিপ্লয়


### Docker Compose ফাইল:
```yaml title="docker-compose.yml"
version: '3.7'
services:
  db:
    image: mariadb
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - db_data:/var/lib/mysql
  app:
    image: nextcloud
    restart: always
    ports:
      - 8080:80
    environment:
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_HOST: db
    volumes:
      - nextcloud_data:/var/www/html
volumes:
  db_data:
  nextcloud_data:
```

- ফাইলটি `docker-compose.yml` নামে সেভ করুন।
- একই ডিরেক্টরিতে `.env` ফাইল তৈরি করুন এবং সেখানে সব sensitive ডাটা রাখুন:

```bash title=".env"
MYSQL_ROOT_PASSWORD=yourStrongRootPassword
MYSQL_DATABASE=nextcloud
MYSQL_USER=nextcloud
MYSQL_PASSWORD=yourStrongDbPassword
```

- `docker compose up -d` কমান্ড চালান।

**নোট:** sensitive ডাটা কখনোই সরাসরি compose ফাইলে লিখবেন না, `.env` ফাইল ব্যবহার করুন এবং গিটে push করার আগে `.env` কে `.gitignore`-এ রাখুন।

## ৩. Initial Setup
- ব্রাউজারে `http://server-ip:8080` খুলুন
- Nextcloud ইউজার তৈরি করুন
- Database info দিন (user: nextcloud, pass: example)

## ৪. Security Best Practices
- **HTTPS:**
  - Caddy/Nginx/Traefik দিয়ে reverse proxy সেট করুন
  - Let's Encrypt দিয়ে SSL enable করুন
- **Firewall:**
  - শুধু 80/443/SSH port খুলুন
  - UFW: `ufw allow 80,443/tcp`
- **Strong Passwords:**
  - Admin ও database ইউজারের জন্য শক্তিশালী পাসওয়ার্ড ব্যবহার করুন
- **Updates:**
  - Docker images ও Nextcloud app নিয়মিত আপডেট করুন
- **External Exposure:**
  - Public IP-তে এক্সপোজ করলে Fail2Ban, CrowdSec, Authelia/Keycloak SSO ব্যবহার করুন
- **Data Backup:**
  - volumes backup করুন (cron + rsync/S3)

## ৫. Extra Hardening
- Two-factor authentication enable করুন
- External storage integration (S3/MinIO) ব্যবহার করুন
- Rate limiting ও brute-force protection

## ৬. Common Issues
- Permission error: volumes-এর owner ঠিক করুন (`chown -R www-data:www-data`)
- Slow performance: Redis cache enable করুন
