## ভূমিকা
**Nginx Proxy Manager (NPM)** একটি জনপ্রিয় টুল যা দিয়ে সহজেই ওয়েবসাইট বা ওয়েবঅ্যাপের জন্য **Reverse Proxy** কনফিগার করা যায়। এটি ব্যবহার করে আপনি খুব সহজেই:

- SSL সার্টিফিকেট (Let's Encrypt) সেটআপ করতে পারবেন।
- ডোমেইন/সাবডোমেইন রাউট করতে পারবেন।
- ওয়েব সার্ভিসগুলোকে নিরাপদভাবে এক্সপোজ করতে পারবেন।
- একটি সুন্দর ওয়েব ইন্টারফেস থেকে সবকিছু ম্যানেজ করতে পারবেন।

### কেন নিজে হোস্ট করবেন?
- তৃতীয় পক্ষের উপর নির্ভরতা কমানো।
- ফ্রি SSL সার্টিফিকেট (Let's Encrypt) স্বয়ংক্রিয়ভাবে নবায়ন।
- এক জায়গা থেকে সব ওয়েব সার্ভিস ম্যানেজ করার সুবিধা।
- সেল্ফ-হোস্টিং প্রোজেক্টগুলো (যেমন Vaultwarden, Nextcloud ইত্যাদি) সহজে ইন্টারনেটে এক্সপোজ করা।

## ইনস্টলেশন পদ্ধতি

Nginx Proxy Manager সাধারণত Docker ব্যবহার করে সবচেয়ে সহজে সেটআপ করা যায়।

### ধাপ ১: প্রয়োজনীয় সফটওয়্যার ইন্সটল
- **Docker** এবং **Docker Compose** ইনস্টল করতে হবে।

Ubuntu/Debian এ:
```bash
sudo apt update
sudo apt install docker.io docker-compose -y
```

---

### ধাপ ২: প্রোজেক্ট ফোল্ডার তৈরি

```bash
mkdir ~/nginx-proxy-manager
cd ~/nginx-proxy-manager
```

---

### ধাপ ৩: `.env` ফাইল তৈরি করুন

```bash title=".env"
DB_SQLITE_FILE=/data/database.sqlite
TZ=Asia/Dhaka
```

> **নোট:** চাইলে MariaDB বা MySQL ব্যবহার করতে পারেন, তবে সহজতার জন্য SQLite যথেষ্ট।

---

### ধাপ ৪: `docker-compose.yml` ফাইল তৈরি

```yaml title="docker-compose.yml"
services:
  app:
    image: jc21/nginx-proxy-manager:latest
    restart: unless-stopped
    ports:
      - "80:80"
      - "81:81"
      - "443:443"
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
    environment:
      - DB_SQLITE_FILE=${DB_SQLITE_FILE}
      - TZ=${TZ}
```

---

### ধাপ ৫: কন্টেইনার চালু করা

```bash
docker-compose up -d
```

কয়েক মিনিট সময় লাগতে পারে। সবকিছু সঠিকভাবে চলছে কিনা দেখতে:

```bash
docker ps
```

## লগইন তথ্য

প্রথমবার ওয়েব ইন্টারফেস অ্যাক্সেস করতে:

* URL: `http://your-server-ip:81`
* **ডিফল্ট ইউজারনেম:** `admin@example.com`
* **ডিফল্ট পাসওয়ার্ড:** `changeme`

প্রথমবার লগইন করার পর পাসওয়ার্ড পরিবর্তন করা জরুরি।

## ব্যবহার

### নতুন Proxy Host যোগ করা

1. ড্যাশবোর্ডে যান → **Hosts** → **Proxy Hosts** → **Add Proxy Host**
2. **Domain Names** এ আপনার ডোমেইন/সাবডোমেইন লিখুন (যেমন: `app.example.com`)
3. **Forward Hostname/IP** এ সার্ভিসের লোকাল আইপি দিন (যেমন: `172.20.0.5`)
4. **Forward Port** দিন (যেমন: `8080`)
5. **SSL** ট্যাবে গিয়ে **Request a new SSL Certificate** নির্বাচন করুন।

## টিপস ও সমস্যা সমাধান

* **পোর্ট কনফ্লিক্ট হলে:** নিশ্চিত হোন সার্ভারে আরেকটি সার্ভিস (যেমন Apache/Nginx) 80/443 পোর্ট ব্যবহার করছে না।
* **SSL সার্টিফিকেট ব্যর্থ হলে:** ডোমেইনটি আপনার সার্ভারের পাবলিক আইপিতে পয়েন্ট করা আছে কিনা চেক করুন।
* **ডেটা ব্যাকআপ:**

  ```bash
  tar -czvf npm-backup.tar.gz data letsencrypt
  ```

## উপসংহার

Nginx Proxy Manager সেল্ফ-হোস্টিং করলে আপনি সহজে ওয়েব অ্যাপস এবং সার্ভিসগুলো নিরাপদভাবে এক্সপোজ করতে পারবেন। Docker-এর মাধ্যমে এটি খুব সহজে সেটআপ ও ম্যানেজ করা যায়।
