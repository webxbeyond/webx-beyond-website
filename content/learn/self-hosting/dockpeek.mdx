# ভূমিকা (Introduction)

**Dockpeek** একটি হালকা এবং ব্যবহারবান্ধব টুল যা Docker কনটেইনারের লগ, স্ট্যাটাস, এবং রিসোর্স ব্যবহার পর্যবেক্ষণ করতে সাহায্য করে। এটি বিশেষভাবে ছোট থেকে মাঝারি আকারের সার্ভারে কাজ করার জন্য ডিজাইন করা হয়েছে।

**কেন নিজে হোস্ট করবেন:**
- আপনার ডেটা সম্পূর্ণ নিয়ন্ত্রণে থাকবে।
- ক্লাউডে ডিপেন্ডেন্সি কমানো যাবে।
- যেকোনো নিরাপত্তা বা প্রাইভেসি উদ্বেগ এড়ানো যাবে।

**প্রধান ব্যবহার ক্ষেত্রসমূহ:**
- কনটেইনার লগ রিয়েল-টাইমে দেখা।
- Docker কনটেইনারের স্ট্যাটাস মনিটর করা।
- কনটেইনারের রিসোর্স ব্যবহার (CPU, RAM) পর্যবেক্ষণ।

# ইনস্টলেশন পদ্ধতি (Installation Methods)

Dockpeek হোস্ট করার জন্য সবচেয়ে সহজ উপায় হলো **Docker Compose** ব্যবহার করা। নিচে ধাপে ধাপে নির্দেশনা দেওয়া হলো।

## ১. প্রয়োজনীয়তা
- Docker & Docker Compose ইনস্টল করা থাকতে হবে
- একটি সার্ভার বা লোকাল মেশিন
- প্রয়োজন অনুযায়ী `.env` ফাইল ব্যবহারের জন্য

## ২. প্রজেক্ট ফোল্ডার তৈরি
```bash
mkdir -p ~/dockpeek
cd ~/dockpeek
```

## ৩. `.env` ফাইল তৈরি (সেনসিটিভ ডেটা)

```bash title=".env"
# Dockpeek Configuration
SECRET_KEY=my_secret_key
DCK_USER=admin
DCK_PASSWORD=strongpassword
DCK_PORT=8080
```

> **নোট:** `.env` ফাইলটি আপনার লগইন ক্রেডেনশিয়াল সংরক্ষণ করবে।

## ৪. `docker-compose.yml` তৈরি

```yml title="docker-compose.yml"
services:
  dockpeek:
    image: ghcr.io/dockpeek/dockpeek:latest
    container_name: dockpeek
    restart: unless-stopped
    ports:
      - "${DCK_PORT}:8000"
    environment:
      - SECRET_KEY=${SECRET_KEY}
      - USERNAME=${DCK_USER}
      - PASSWORD=${DCK_PASSWORD}
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
```

## ৫. Docker Compose দিয়ে কনটেইনার চালানো

```bash
docker-compose up -d
```

> **টিপ:** `docker-compose logs -f` দিয়ে রিয়েল-টাইম লগ দেখতে পারেন।

## ৬. ব্রাউজারে প্রবেশ

* URL: `http://<আপনার সার্ভার আইপি>:8080`
* লগইন: `.env`-এ দেওয়া `DCK_USER` এবং `DCK_PASSWORD`


# অতিরিক্ত Docker হোস্ট যুক্ত করা (Add Additional Docker Hosts)

Dockpeek-এর মাধ্যমে একাধিক Docker হোস্ট একত্রে পরিচালনা করা সম্ভব।

**সেরা পদ্ধতি:**

* প্রতিটি Docker হোস্টে **Docker Socket Proxy** ইনস্টল করুন।
* এটি Docker API HTTP পোর্টের মাধ্যমে এক্সপোজ করবে (যেমন: 2375) এবং নিরাপদভাবে দূরবর্তী এক্সেস দেবে।

**Environment ভেরিয়েবল উদাহরণ:**

```yaml
environment:
  # Docker Host 1
  - DOCKER_HOST_1_URL=unix:///var/run/docker.sock
  - DOCKER_HOST_1_NAME=MyServer1
  - DOCKER_HOST_1_PUBLIC_HOSTNAME=
  
  # Docker Host 2
  - DOCKER_HOST_2_URL=tcp://192.168.1.168:2375
  - DOCKER_HOST_2_NAME=Synology
  - DOCKER_HOST_2_PUBLIC_HOSTNAME=NAS
```

> **নোট:** `unix:///var/run/docker.sock` ব্যবহার করতে হলে অবশ্যই volumes এ `/var/run/docker.sock:/var/run/docker.sock` মাউন্ট করতে হবে।

### Environment Variables ব্যাখ্যা

| ভেরিয়েবল                         | ব্যাখ্যা                                                                     |
| --------------------------------- | ---------------------------------------------------------------------------- |
| SECRET\_KEY                       | একটি শক্তিশালী এবং অনন্য সিক্রেট।                                            |
| USERNAME                          | Dockpeek লগইনের জন্য ব্যবহারকারী নাম।                                        |
| PASSWORD                          | লগইনের জন্য পাসওয়ার্ড।                                                      |
| DOCKER\_HOST                      | Docker Socket URL (local: unix:///var/run/docker.sock, TCP: tcp\://IP:2375)। |
| DOCKER\_HOST\_NAME                | UI-তে প্রদর্শিত নাম (ডিফল্ট: local)।                                         |
| DOCKER\_HOST\_PUBLIC\_HOSTNAME    | পাবলিক হোস্টনেম বা IP (ঐচ্ছিক)।                                              |
| DOCKER\_HOST\_N\_URL              | অতিরিক্ত Docker হোস্টের URL।                                                 |
| DOCKER\_HOST\_N\_NAME             | UI-তে প্রদর্শিত নাম।                                                         |
| DOCKER\_HOST\_N\_PUBLIC\_HOSTNAME | পাবলিক হোস্টনেম বা IP (ঐচ্ছিক)।                                              |

> **নোট:** DOCKER\_HOST\_N\_\* ভেরিয়েবল শুধুমাত্র সরাসরি এক্সেসযোগ্য হোস্টের জন্য কাজ করে। `host.docker.internal` ব্যবহার করা যাবে না।

# সমস্যার সমাধান ও টিপস (Troubleshooting & Tips)

* **পোর্ট সমস্যা:** অন্য কোন সার্ভিস একই পোর্ট ব্যবহার করলে `.env`-এ পোর্ট পরিবর্তন করুন।
* **কনটেইনার রিস্টার্ট হচ্ছে না:** `docker-compose logs` দিয়ে এরর চেক করুন।
* **লগ দেখায় না:** নিশ্চিত করুন `/var/run/docker.sock` ঠিকমত মাউন্ট করা হয়েছে।
* **Docker Compose আপডেট:** নতুন ভার্সন ব্যবহার করলে `docker-compose pull` করে আপডেট নিতে হবে।
