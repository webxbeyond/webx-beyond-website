# Host-based Routing (Subdomains, Wildcards)

## পরিচিতি

Modern web architecture-এ একাধিক subdomain, app, বা tenant একসাথে চালাতে হয়। Host-based routing মানে—user কোন domain বা subdomain-এ request পাঠাচ্ছে, তার উপর ভিত্তি করে Traefik সেই request-কে নির্দিষ্ট backend-এ পাঠায়। এই পাঠে জানব, host-based routing কী, কেন দরকার, কীভাবে Traefik-এ সেটআপ করবেন, এবং real-world example ও pitfalls।

---

## Host-based Routing কী?

- **Host-based Routing** হলো—request-এর Host header দেখে নির্দিষ্ট service-এ পাঠানো
- উদাহরণ: `api.example.com`-এ গেলে API server, `blog.example.com`-এ গেলে blog app
- Multi-tenant SaaS, microservices, বা multi-app platform-এ এটি must-have

### Real-life Use Case
Google, Netflix—`mail.google.com`, `drive.google.com`-এর মতো subdomain-এ আলাদা service চালায়

---

## Traefik-এ Host-based Routing কিভাবে কাজ করে?

Traefik-এর router config-এ "rule" define করতে হয়—যেখানে Host match করলে কোন service-এ request যাবে

### YAML Config Example

```yaml
http:
  routers:
    api-router:
      rule: "Host(`api.example.com`)"
      service: api-service
      entryPoints:
        - web
    blog-router:
      rule: "Host(`blog.example.com`)"
      service: blog-service
      entryPoints:
        - web
```

- **Host:** নির্দিষ্ট domain/subdomain match করলে
- **service:** কোন backend-এ request যাবে

### Docker Labels Example

```yaml
labels:
  - "traefik.http.routers.api-router.rule=Host(`api.example.com`)"
  - "traefik.http.routers.api-router.service=api-service"
  - "traefik.http.routers.blog-router.rule=Host(`blog.example.com`)"
  - "traefik.http.routers.blog-router.service=blog-service"
```

## Wildcard Subdomain Routing

- Wildcard দিয়ে multiple subdomain match করা যায়—যেমন `*.example.com`
- Rule: `HostRegexp(`{subdomain:[a-z]+}.example.com`)`

### Example

```yaml
http:
  routers:
    tenant-router:
      rule: "HostRegexp(`{tenant:[a-z]+}.example.com`)"
      service: tenant-service
      entryPoints:
        - web
```

## Common Pitfalls (সাধারণ ভুল)

- **DNS config ভুল:** Subdomain pointing না করলে routing কাজ করবে না
- **Rule overlap:** Multiple host rule থাকলে ভুল service-এ request যেতে পারে
- **Wildcard overuse:** Security risk—unintended subdomain serve হতে পারে

---

## Best Practices

- Specific host rule আগে define করুন—priority ঠিক রাখুন
- Wildcard ব্যবহার করলে security validate করুন
- DNS config ও SSL certificate ঠিক রাখুন
- Config change/staging-এ test করুন
