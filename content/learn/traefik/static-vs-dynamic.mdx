# স্ট্যাটিক বনাম ডাইনামিক কনফিগারেশন: Traefik-এ মৌলিক ধারণা

## পরিচিতি

Traefik-এর কনফিগারেশন দুইভাবে করা যায়—স্ট্যাটিক (Static) ও ডাইনামিক (Dynamic)। এই দুই ধরনের কনফিগারেশন বুঝলে আপনি সহজেই production-ready, scalable, ও automated রিভার্স প্রক্সি প্ল্যাটফর্ম তৈরি করতে পারবেন।

---

## ১. স্ট্যাটিক কনফিগারেশন (Static Configuration)

### কী?

স্ট্যাটিক কনফিগারেশন মানে—Traefik চালু হওয়ার সময় যেসব সেটিংস নির্ধারণ করা হয়, যেমন EntryPoints, Providers, API, Dashboard ইত্যাদি।

### কোথায় থাকে?
- `traefik.yml` বা `traefik.toml` ফাইলে
- Docker/Kubernetes-এ environment variable বা command-line flag

### কেন গুরুত্বপূর্ণ?
- **Initial setup:** EntryPoints, Providers, Dashboard—সব Traefik চালুর সময় নির্ধারণ হয়
- **Security:** TLS/SSL, API access—স্ট্যাটিক কনফিগে সেট করা হয়

### বাস্তব উদাহরণ
- Production server-এ HTTPS EntryPoint, Docker provider
- Dashboard access control

### সহজ উদাহরণ: স্কুলের নিয়ম

যেমন স্কুলে ভর্তি হওয়ার সময় কিছু নিয়ম নির্ধারণ হয়—Traefik-এর static config মানে, শুরুতেই নির্ধারিত নিয়ম।

---

## ২. ডাইনামিক কনফিগারেশন (Dynamic Configuration)

### কী?

ডাইনামিক কনফিগারেশন মানে—Traefik চলার সময় যেসব সেটিংস পরিবর্তন/যোগ/বাতিল করা যায়, যেমন Routers, Services, Middleware ইত্যাদি।

### কোথায় থাকে?
- File provider (YAML/TOML)
- Docker/Kubernetes label/annotation
- API call

### কেন গুরুত্বপূর্ণ?
- **Automation:** নতুন সার্ভিস এলে অটোমেটিক রাউটিং
- **Scalability:** সার্ভিস যোগ/বাতিল, middleware পরিবর্তন
- **Zero-downtime deployment:** production-এ কোনো interruption ছাড়াই config আপডেট

### বাস্তব উদাহরণ
- Docker container চালু হলে অটোমেটিক router/service তৈরি
- Kubernetes pod এলে অটোমেটিক middleware যোগ

### সহজ উদাহরণ: ক্লাসের রুটিন

যেমন ক্লাসের রুটিন মাঝে মাঝে পরিবর্তন হয়—Traefik-এর dynamic config মানে, চলার সময় পরিবর্তনযোগ্য নিয়ম।

---

## কখন কোনটা ব্যবহার করবেন?

- **Static:** EntryPoints, Providers, Dashboard, TLS/SSL
- **Dynamic:** Routers, Services, Middleware, Automation

## কমন পিটফল

- Static config ভুল হলে Traefik চালু হবে না
- Dynamic config ভুল হলে রাউটিং/সার্ভিস inaccessible
- Static config পরিবর্তন করতে হলে restart করতে হয়

## বেস্ট প্র্যাকটিস

- Static config version control-এ রাখুন
- Dynamic config automation-friendly রাখুন
- Config file/comment/documentation ভালোভাবে লিখুন

## ইন্ডাস্ট্রি ট্রেন্ডস

- Automation ও GitOps—dynamic config জনপ্রিয় হচ্ছে
- Cloud-native deployment—static config minimal, dynamic config automated
- Production-ready platform—static config security, dynamic config scalability

## উপসংহার

Traefik-এর static ও dynamic configuration—দুইটাই production-ready, scalable, automated রিভার্স প্রক্সি প্ল্যাটফর্মের জন্য অপরিহার্য।

পরবর্তী পাঠে আমরা EntryPoints (HTTP, HTTPS, TCP, UDP) নিয়ে বিস্তারিতভাবে দেখব।
