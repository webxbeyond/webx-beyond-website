মাইক্রোটিক রাউটার কনফিগার করতে হলে নেটওয়ার্কিংয়ের কিছু মৌলিক ধারণা জানা জরুরি। এখানে IP, সাবনেটিং, রাউটিং ও NAT নিয়ে সহজ ভাষায় আলোচনা করা হলো।

## IP (Internet Protocol) কী?
- **IP Address** হলো প্রতিটি ডিভাইসের জন্য ইউনিক নম্বর, যা নেটওয়ার্কে ডেটা পাঠাতে ও পেতে ব্যবহৃত হয়।
- দুই ধরনের IP: IPv4 (উদাহরণ: 192.168.1.1), IPv6 (উদাহরণ: 2400:abcd::1)
- **Private IP:** লোকাল নেটওয়ার্কে ব্যবহৃত (192.168.x.x, 10.x.x.x, 172.16.x.x)
- **Public IP:** ইন্টারনেটে ইউনিক, ISP থেকে পাওয়া যায়

## সাবনেটিং (Subnetting)
- বড় নেটওয়ার্ককে ছোট ছোট ভাগে ভাগ করার পদ্ধতি
- **Subnet Mask:** কোন অংশ নেটওয়ার্ক, কোন অংশ হোস্ট তা নির্ধারণ করে (উদাহরণ: 255.255.255.0)
- **CIDR নোটেশন:** 192.168.1.0/24 (24 বিট নেটওয়ার্ক, 8 বিট হোস্ট)
- সাবনেটিংয়ের মাধ্যমে নিরাপত্তা, ট্রাফিক ম্যানেজমেন্ট ও স্কেলেবিলিটি বাড়ানো যায়

### মাইক্রোটিকে সাবনেট কনফিগার
```bash
/ip address add address=192.168.10.1/24 interface=ether1
```

## রাউটিং (Routing)
- এক নেটওয়ার্ক থেকে অন্য নেটওয়ার্কে ডেটা পাঠানোর নিয়ম
- **Static Routing:** ম্যানুয়ালি রুট নির্ধারণ (ছোট নেটওয়ার্কে সহজ)
- **Dynamic Routing:** রাউটার নিজেরা রুট শিখে নেয় (OSPF, BGP, RIP)

### Static Route উদাহরণ (মাইক্রোটিক)
```bash
/ip route add dst-address=10.10.20.0/24 gateway=192.168.10.2
```

## NAT (Network Address Translation)
- লোকাল (প্রাইভেট) IP কে পাবলিক IP-তে রূপান্তর
- ইন্টারনেট শেয়ারিং, নিরাপত্তা ও IP কনজারভেশনের জন্য ব্যবহৃত
- দুই ধরনের: Source NAT (Masquerade), Destination NAT (Port Forwarding)

### Source NAT (Masquerade) উদাহরণ
```bash
/ip firewall nat add chain=srcnat action=masquerade out-interface=ether1
```

### Destination NAT (Port Forwarding) উদাহরণ
```bash
/ip firewall nat add chain=dstnat action=dst-nat to-addresses=192.168.10.10 to-ports=80 protocol=tcp dst-port=8080
```

## বাস্তব উদাহরণ
- **ISP:** একাধিক সাবনেট, রাউটিং, NAT দিয়ে ইন্টারনেট শেয়ার
- **হোম/অফিস:** WiFi, LAN, গেস্ট নেটওয়ার্কে আলাদা সাবনেট
- **সার্ভার হোস্টিং:** Port Forwarding দিয়ে ওয়েব সার্ভার এক্সপোজ

## উপসংহার
IP, সাবনেটিং, রাউটিং ও NAT-এর বেসিক ধারণা ছাড়া মাইক্রোটিক কনফিগারেশন অসম্পূর্ণ। এই অধ্যায়ের কনসেপ্টগুলো ভালোভাবে আয়ত্ত করলে পরবর্তী অ্যাডভান্সড কনফিগারেশন সহজ হবে।
