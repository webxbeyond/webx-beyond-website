নেটওয়ার্ক ডিজাইন ও ম্যানেজমেন্টে Bridge, VLAN ও Routing—এই তিনটি কনসেপ্ট অত্যন্ত গুরুত্বপূর্ণ। কখন কোনটি ব্যবহার করবেন, কী সুবিধা-অসুবিধা, এবং মাইক্রোটিকে কনফিগারেশন কেমন হবে—সবকিছু এখানে বিশ্লেষণ করা হলো।

## ১. Bridge

### কী?
- একাধিক ইন্টারফেসকে একসাথে গ্রুপ করে এক নেটওয়ার্কে পরিণত করা
- Layer 2 (Data Link) লেভেলে কাজ করে

### কখন ব্যবহার করবেন?
- ছোট নেটওয়ার্কে, যেখানে সব ডিভাইস একই সাবনেটে
- ওয়্যারলেস ও ওয়্যারড ডিভাইস একসাথে রাখতে

### সুবিধা
- সহজ কনফিগারেশন
- ব্রডকাস্ট/মাল্টিকাস্ট সহজ

### অসুবিধা
- বড় নেটওয়ার্কে ব্রডকাস্ট ট্রাফিক বেশি
- আইসোলেশন/সিকিউরিটি কম

### মাইক্রোটিক কনফিগারেশন
```bash
/interface bridge add name=bridge1
/interface bridge port add bridge=bridge1 interface=ether2
/interface bridge port add bridge=bridge1 interface=ether3
```

## ২. VLAN (Virtual LAN)

### কী?
- এক ফিজিক্যাল নেটওয়ার্কে লজিক্যালভাবে আলাদা নেটওয়ার্ক তৈরি
- Layer 2-এ আইসোলেশন

### কখন ব্যবহার করবেন?
- একাধিক ডিপার্টমেন্ট/গ্রুপ আলাদা করতে
- এক ইন্টারফেসে একাধিক সাবনেট চালাতে
- নিরাপত্তা ও ট্রাফিক কন্ট্রোল বাড়াতে

### সুবিধা
- আইসোলেশন, স্কেলেবিলিটি
- ব্রডকাস্ট ডোমেইন ছোট

### অসুবিধা
- VLAN ট্যাগিং/কনফিগারেশন জটিল
- VLAN-aware switch দরকার

### মাইক্রোটিক কনফিগারেশন
```bash
/interface vlan add name=HR_vlan vlan-id=10 interface=ether2
/interface vlan add name=IT_vlan vlan-id=20 interface=ether2
/ip address add address=192.168.10.1/26 interface=HR_vlan
/ip address add address=192.168.10.129/26 interface=IT_vlan
```

## ৩. Routing

### কী?
- একাধিক নেটওয়ার্ক/সাবনেটের মধ্যে ট্রাফিক ফরোয়ার্ড করা
- Layer 3 (Network) লেভেলে কাজ করে

### কখন ব্যবহার করবেন?
- একাধিক সাবনেট/ভিন্ন নেটওয়ার্কের মধ্যে কমিউনিকেশন
- ইন্টারনেট/ওয়াইডার নেটওয়ার্কে কানেক্ট করতে

### সুবিধা
- সাবনেট আইসোলেশন
- ফায়ারওয়াল/ACL দিয়ে কন্ট্রোল

### অসুবিধা
- কনফিগারেশন তুলনামূলক জটিল
- রাউটিং টেবিল ম্যানেজমেন্ট দরকার

### মাইক্রোটিক কনফিগারেশন
```bash
/ip route add dst-address=192.168.20.0/24 gateway=192.168.10.2
```

## ৪. তুলনামূলক টেবিল

| ফিচার         | Bridge         | VLAN           | Routing         |
|---------------|---------------|----------------|----------------|
| লেয়ার        | 2             | 2              | 3              |
| আইসোলেশন     | কম            | বেশি           | সর্বোচ্চ        |
| স্কেলেবিলিটি  | কম            | বেশি           | সর্বোচ্চ        |
| কনফিগ সহজ    | সহজ           | মাঝারি         | তুলনামূলক কঠিন |
| ব্রডকাস্ট     | বেশি          | কম             | কম             |
| ব্যবহার       | ছোট নেটওয়ার্ক | মিড-লেভেল      | বড় নেটওয়ার্ক  |

## ৫. কখন কোনটি?
- **Bridge:** ছোট, সিম্পল নেটওয়ার্কে
- **VLAN:** মিড-লেভেল, মাল্টি-ডিপার্টমেন্ট, আইসোলেশন দরকার হলে
- **Routing:** বড়, মাল্টি-সাবনেট, ইন্টারনেট/ওয়াইডার নেটওয়ার্কে

## ৬. বাস্তব কেস স্টাডি

### কেস ১: ছোট অফিস
- সব ডিভাইস এক ব্রিজে

### কেস ২: মিড-সাইজ অফিস
- VLAN দিয়ে HR, IT আলাদা

### কেস ৩: বড় অফিস/ISP
- Routing দিয়ে একাধিক ব্রাঞ্চ/নেটওয়ার্ক কানেক্ট

## উপসংহার
Bridge, VLAN ও Routing—তিনটি কনসেপ্টের পার্থক্য ও ব্যবহারিক দিক জানা থাকলে নেটওয়ার্ক ডিজাইন ও ম্যানেজমেন্ট সহজ হয়। পরবর্তী অধ্যায়ে IP Alias ও মাল্টিপল IP নিয়ে আলোচনা করা হবে।
