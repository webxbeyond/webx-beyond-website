একই ইন্টারফেসে একাধিক IP অ্যাড্রেস অ্যাসাইন করার জন্য মাইক্রোটিকে IP Alias বা মাল্টিপল IP কনফিগারেশন ব্যবহার করা হয়। এতে নেটওয়ার্ক ফ্লেক্সিবিলিটি, ভার্চুয়াল হোস্টিং, সাবনেট ট্রানজিশন ইত্যাদি সহজ হয়। এখানে হাতে-কলমে কনফিগারেশন ও ব্যবহারিক দিক নিয়ে আলোচনা করা হলো।

## ১. IP Alias কী?
- একটি ইন্টারফেসে একাধিক IP অ্যাড্রেস অ্যাসাইন
- ভিন্ন সাবনেট/নেটওয়ার্কে একসাথে কানেক্ট
- ভার্চুয়াল হোস্টিং, সাবনেট মাইগ্রেশন, Policy Routing ইত্যাদিতে ব্যবহৃত

## ২. কবে ব্যবহার করবেন?
- এক ইন্টারফেসে একাধিক নেটওয়ার্ক/সাবনেট দরকার হলে
- পুরনো ও নতুন সাবনেট একসাথে চালাতে
- Policy Routing, NAT, Port Forwarding ইত্যাদিতে

## ৩. কনফিগারেশন (Winbox/WebFig)
- IP > Addresses > +
- Address: 192.168.10.1/24 (Primary)
- Address: 10.10.10.1/24 (Alias)
- Interface: ether2/bridge

## ৪. কনফিগারেশন (CLI)
```bash
/ip address add address=192.168.10.1/24 interface=ether2
/ip address add address=10.10.10.1/24 interface=ether2
```

## ৫. মাল্টিপল IP-এর ব্যবহারিক দিক
- **Policy Routing:** ভিন্ন ট্রাফিক ভিন্ন গেটওয়ে
- **NAT:** একাধিক পাবলিক IP ব্যবহারে
- **Port Forwarding:** আলাদা IP-তে আলাদা সার্ভিস
- **DHCP Relay:** এক ইন্টারফেসে একাধিক DHCP Pool

## ৬. সমস্যা ও সমাধান
- **IP Conflict:** একই সাবনেটে ডুপ্লিকেট IP দেবেন না
- **Routing সমস্যা:** রাউটিং টেবিল চেক করুন
- **Firewall:** নতুন IP-র জন্য রুল আপডেট করুন

## ৭. বাস্তব কেস স্টাডি

### কেস: পুরনো ও নতুন সাবনেট একসাথে চালানো
- পুরনো: 192.168.10.0/24
- নতুন: 10.10.10.0/24
- কনফিগারেশন:
```bash
/ip address add address=192.168.10.1/24 interface=ether2
/ip address add address=10.10.10.1/24 interface=ether2
```

## ৮. টিপস
- Alias IP-তে DHCP Server চালাতে চাইলে pool/network আলাদা দিন
- NAT/Port Forwarding-এ IP স্পেসিফাই করুন

## উপসংহার
IP Alias ও মাল্টিপল IP কনফিগারেশন নেটওয়ার্ক ফ্লেক্সিবিলিটি ও মাইগ্রেশনে অত্যন্ত কার্যকর। পরবর্তী মডিউলে NAT ও ফায়ারওয়াল নিয়ে আলোচনা করা হবে।
