ডাইনামিক রাউটিং হলো এমন একটি পদ্ধতি যেখানে রাউটারগুলো স্বয়ংক্রিয়ভাবে একে অপরের সাথে রাউটিং তথ্য আদান-প্রদান করে এবং নেটওয়ার্ক পরিবর্তনের সাথে সাথে রাউটিং টেবিল আপডেট করে। বড়, জটিল ও পরিবর্তনশীল নেটওয়ার্কে ডাইনামিক রাউটিং অপরিহার্য। এখানে আমরা OSPF, BGP ও RIP নিয়ে বিস্তারিত আলোচনা করব।

## ডাইনামিক রাউটিং কী?

ডাইনামিক রাউটিং প্রোটোকল রাউটারগুলোকে স্বয়ংক্রিয়ভাবে নেটওয়ার্ক টপোলজি শিখতে ও রাউটিং টেবিল আপডেট করতে সাহায্য করে। এতে ম্যানুয়াল কনফিগারেশনের ঝামেলা কমে এবং রিডান্ডেন্সি ও ফেইলওভারে দ্রুত রেসপন্স পাওয়া যায়।

## OSPF (Open Shortest Path First)

OSPF হলো একটি লিংক-স্টেট ডাইনামিক রাউটিং প্রোটোকল, যা বড় ও এন্টারপ্রাইজ নেটওয়ার্কে বহুল ব্যবহৃত।

### OSPF-এর বৈশিষ্ট্য
- দ্রুত কনভার্জেন্স
- Area ভিত্তিক নেটওয়ার্ক বিভাজন
- Cost ভিত্তিক রাউটিং
- VLSM ও CIDR সাপোর্ট

### OSPF কনফিগারেশন (মাইক্রোটিক)

ধরা যাক, আপনার রাউটারের IP: 10.0.0.1/24

```shell
/routing ospf instance
set [ find default=yes ] router-id=10.0.0.1

/routing ospf network
add network=10.0.0.0/24 area=backbone
```

## BGP (Border Gateway Protocol)

BGP হলো ইন্টারনেটের মূল রাউটিং প্রোটোকল, যা বিভিন্ন ISP, ডাটা সেন্টার ও বড় নেটওয়ার্কের মধ্যে ব্যবহৃত হয়।

### BGP-এর বৈশিষ্ট্য
- Path Vector প্রোটোকল
- Policy-based রাউটিং
- ইন্টারনেট স্কেল
- AS (Autonomous System) ভিত্তিক

### BGP কনফিগারেশন (মাইক্রোটিক)

ধরা যাক, আপনার AS: 65001, পিয়ার AS: 65002, পিয়ার IP: 203.0.113.2

```shell
/routing bgp instance
set default as=65001 router-id=10.0.0.1

/routing bgp peer
add remote-address=203.0.113.2 remote-as=65002
```

## RIP (Routing Information Protocol)

RIP হলো একটি সহজ ডিস্ট্যান্স-ভেক্টর ডাইনামিক রাউটিং প্রোটোকল, ছোট নেটওয়ার্কে ব্যবহৃত হয়।

### RIP-এর বৈশিষ্ট্য
- Distance Vector
- 15 hop limit
- সহজ কনফিগারেশন

### RIP কনফিগারেশন (মাইক্রোটিক)

```shell
/routing rip interface
add interface=ether1
add interface=ether2
```

## ডাইনামিক রাউটিং-এর সুবিধা ও সীমাবদ্ধতা

### সুবিধা
- স্বয়ংক্রিয় রাউট আপডেট
- দ্রুত ফেইলওভার
- বড় নেটওয়ার্কে স্কেলেবল

### সীমাবদ্ধতা
- কনফিগারেশন তুলনামূলক জটিল
- ভুল কনফিগারেশনে নেটওয়ার্ক ডাউন হতে পারে
- রিসোর্স ব্যবহার বেশি

## OSPF, BGP ও RIP – কখন কোনটি ব্যবহার করবেন?
- ছোট নেটওয়ার্ক: RIP
- এন্টারপ্রাইজ/ISP: OSPF, BGP
- ইন্টারনেট পিয়ারিং: BGP

## বাস্তব কেস স্টাডি

ধরা যাক, দুটি ব্রাঞ্চ অফিস OSPF দিয়ে কানেক্টেড, আর ইন্টারনেটের জন্য BGP ব্যবহার হচ্ছে।

```shell
/routing ospf instance
set [ find default=yes ] router-id=10.1.1.1

/routing ospf network
add network=10.1.1.0/24 area=backbone

/routing bgp instance
set default as=65010 router-id=10.1.1.1

/routing bgp peer
add remote-address=198.51.100.2 remote-as=65020
```

## টিপস
- OSPF Area design ভালোভাবে করুন
- BGP-তে ফিল্টার ও পলিসি ব্যবহার করুন
- RIP শুধুমাত্র ছোট নেটওয়ার্কে ব্যবহার করুন

## উপসংহার

ডাইনামিক রাউটিং বড় ও জটিল নেটওয়ার্কের জন্য অপরিহার্য। OSPF, BGP ও RIP কনফিগারেশন ও ব্যবহারের ধারণা থাকলে নেটওয়ার্ক ম্যানেজমেন্ট অনেক সহজ হবে।
