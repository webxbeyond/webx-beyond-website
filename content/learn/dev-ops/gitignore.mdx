---

`.gitignore` হলো একটি বিশেষ ফাইল, যেখানে আপনি নির্দিষ্ট ফাইল/ফোল্ডার গিট ট্র্যাক না করার জন্য তালিকা দেন। এতে অপ্রয়োজনীয়, সিক্রেট, বা বিল্ড ফাইল গিটে যোগ হয় না।

## কেন .gitignore দরকার?
- **বিল্ড ফাইল:** যেমন `node_modules/`, `dist/`, `.env` ইত্যাদি গিটে রাখা উচিত নয়।
- **সিক্রেট/কনফিগ:** API key, password, config ফাইল গিটে গেলে নিরাপত্তা ঝুঁকি হয়।
- **লোকাল সেটিংস:** IDE, OS-specific ফাইল (যেমন `.DS_Store`, `Thumbs.db`) গিটে রাখা উচিত নয়।

## .gitignore ফাইল তৈরি

প্রজেক্টের root-এ `.gitignore` নামে ফাইল তৈরি করুন।

**উদাহরণ:**
```
# Node.js
node_modules/
dist/
.env

# IDE
.vscode/
.idea/

# OS
.DS_Store
Thumbs.db
```

## কাস্টম .gitignore

প্রয়োজনে নিজের প্রয়োজন অনুযায়ী ফাইল/ফোল্ডার যুক্ত করুন।

**Python প্রজেক্ট:**
```
__pycache__/
*.pyc
.env
```

**Java:**
```
target/
*.class
```

## .gitignore কাজ করে কীভাবে?
- গিট কমিট করার সময় `.gitignore`-এ থাকা ফাইল/ফোল্ডার ট্র্যাক করবে না।
- আগে গিটে যোগ করা ফাইল `.gitignore`-এ দিলেও, সেগুলো ট্র্যাক থাকবে—তাই প্রথমেই .gitignore ব্যবহার করুন।

## বাস্তব উদাহরণ

> "আমরা টিমে কাজ করি, সবাই .gitignore ব্যবহার করে। এতে অপ্রয়োজনীয় ফাইল গিটে যায় না, প্রজেক্ট পরিষ্কার থাকে। একবার ভুলে `.env` গিটে চলে গেলে, সিক্রেট exposed হয়ে যায়—তাই .gitignore খুব গুরুত্বপূর্ণ।"

## টিপস
- GitHub-এ বিভিন্ন টেমপ্লেট পাওয়া যায়: https://github.com/github/gitignore
- নতুন প্রজেক্ট শুরুতেই .gitignore তৈরি করুন।
- সিক্রেট/কনফিগ ফাইল কখনো গিটে যোগ করবেন না।

## সংক্ষেপে
- `.gitignore` গিটে অপ্রয়োজনীয়/সিক্রেট ফাইল যোগ হওয়া থেকে রক্ষা করে।
- প্রজেক্ট পরিষ্কার, নিরাপদ, ও গোছালো রাখে।
- টিমওয়ার্কে .gitignore অপরিহার্য।
