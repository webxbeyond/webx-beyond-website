Jenkins-এ ক্রেডেনশিয়াল ব্যবস্থাপনা মানে—password, API key, SSH key, token ইত্যাদি নিরাপদে সংরক্ষণ ও pipeline-এ ব্যবহার করা। এই টিউটোরিয়ালে credentials add, bind, usage, real-life example, এবং best practices দেখানো হয়েছে।

## ১. Credentials Management কী?
- Secure storage of secrets
- Centralized management
- Pipeline/job-এ safe usage

## ২. Credentials Add করা
1. Jenkins UI > Credentials > (global/domain)
2. "Add Credentials" ক্লিক করুন
3. Type select করুন (Username/Password, Secret text, SSH key, Certificate)
4. ID, description, value দিন

## ৩. Credentials Usage in Pipeline
- Declarative pipeline-এ:
```groovy
environment {
  SECRET = credentials('my-secret-id')
}
```
- Scripted pipeline-এ:
```groovy
withCredentials([string(credentialsId: 'my-secret-id', variable: 'SECRET')]) {
  sh 'echo $SECRET'
}
```

## ৪. Real-life Example: Docker Registry Login
1. Credentials add: Username/password
2. Pipeline-এ:
```groovy
environment {
  DOCKER_USER = credentials('docker-user')
  DOCKER_PASS = credentials('docker-pass')
}
stages {
  stage('Login') {
    steps {
      sh 'docker login -u $DOCKER_USER -p $DOCKER_PASS registry.example.com'
    }
  }
}
```

## ৫. Credentials Scope
- Global: সব job/pipeline-এ
- Domain: নির্দিষ্ট job/agent/source

## ৬. Best Practices
- Secrets plaintext-এ না রাখুন
- Credentials manager-এ সব secret
- ID naming convention
- Regular rotation/update
- Access control (RBAC)

## ৭. Team Collaboration
- Shared credentials (RBAC)
- Credentials documentation
- Audit trail

## সংক্ষেপে
Jenkins-এ credentials management করলে pipeline/job-এ secrets নিরাপদে, সহজে ও efficiently ব্যবহার করা যায়—DevOps টিমের security ও productivity বাড়ে।
