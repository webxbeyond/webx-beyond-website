ইনভাইরোনমেন্ট ভেরিএবল (Environment Variable) হচ্ছে এমন কিছু ভেরিএবল, যা অপারেটিং সিস্টেম বা শেল দ্বারা ব্যবহৃত হয় এবং স্ক্রিপ্ট, সফটওয়্যার, বা কমান্ডের জন্য গুরুত্বপূর্ণ তথ্য সরবরাহ করে।

## ইনভাইরোনমেন্ট ভেরিএবল কী?

এগুলো মূলত কী-ভ্যালু পেয়ার, যা সিস্টেমে বিভিন্ন জায়গায় ব্যবহার হয়। যেমন: PATH, HOME, USER, SHELL ইত্যাদি।

## কেন দরকার?

- সফটওয়্যার কোথায় ইনস্টল আছে, তা জানাতে
- ইউজার/সার্ভার কনফিগারেশন
- স্ক্রিপ্টে ডাইনামিক তথ্য ব্যবহার
- সিক্রেট/কনফিগ সংরক্ষণ

## সাধারণ ইনভাইরোনমেন্ট ভেরিএবল

| ভেরিএবল | কাজ |
|----------|--------------------------|
| PATH     | এক্সিকিউটেবল ফাইলের লোকেশন |
| HOME     | ইউজারের হোম ডিরেক্টরি |
| USER     | ইউজারের নাম |
| SHELL    | কোন শেল ব্যবহার হচ্ছে |

## কিভাবে চেক করবেন?

```bash
echo $PATH
echo $HOME
echo $USER
echo $SHELL
```

## কিভাবে সেট করবেন?

স্ক্রিপ্ট বা টার্মিনালে:

```bash
export MY_VAR="Hello World"
echo $MY_VAR
```

## স্ক্রিপ্টে ব্যবহার

```bash
#!/bin/bash
export BACKUP_DIR="/backup"
cp /home/user/data.txt $BACKUP_DIR
```

## রিয়েল লাইফ উদাহরণ

### Example 1: ডাটাবেজ ইউজার/পাসওয়ার্ড

```bash
export DB_USER="admin"
export DB_PASS="secret"
python app.py
```

### Example 2: ডাইনামিক স্ক্রিপ্ট কনফিগ

```bash
#!/bin/bash
export ENV="production"
if [ "$ENV" == "production" ]; then
  echo "Production server deploy হচ্ছে..."
else
  echo "Test server deploy হচ্ছে..."
fi
```

## টেম্পোরারি vs পার্মানেন্ট

- **টেম্পোরারি:** টার্মিনালে export করলে, সেই সেশন পর্যন্ত থাকবে
- **পার্মানেন্ট:** `.bashrc`, `.zshrc` বা `/etc/environment`-এ লিখলে, সবসময় থাকবে

## শেষ কথা

ইনভাইরোনমেন্ট ভেরিএবল DevOps, স্ক্রিপ্টিং, এবং সার্ভার কনফিগারেশনের জন্য অত্যন্ত গুরুত্বপূর্ণ। এগুলো ঠিকভাবে ব্যবহার করলে অটোমেশন, সিকিউরিটি, এবং স্কেলেবিলিটি অনেক সহজ হয়।
