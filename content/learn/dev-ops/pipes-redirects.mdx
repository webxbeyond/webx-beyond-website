# বেসিক লিনাক্স কমান্ড - Pipes & Redirects

Pipes ও Redirects হলো লিনাক্সের CLI-র সবচেয়ে শক্তিশালী ফিচার। এগুলো দিয়ে একাধিক কমান্ডকে একসাথে চেইন করে ডেটা প্রসেস, ফিল্টার, এবং ফাইল/আউটপুট ম্যানেজ করা যায়। DevOps-এ অটোমেশন, লগ প্রসেসিং, এবং ডেটা ফিল্টারিংয়ে এগুলোর ব্যবহার অপরিহার্য।

## Pipe (|) কী?

Pipe দিয়ে এক কমান্ডের আউটপুট সরাসরি অন্য কমান্ডে পাঠানো যায়।

### উদাহরণ:
```bash
ls -l | grep ".txt"
```
এখানে ls -l কমান্ডের আউটপুট grep-এ যায়, grep শুধু .txt ফাইল দেখায়।

## Redirect `(>, >>, <)` কী?

- `**>**:` আউটপুট নতুন ফাইলে লিখে
- `**>>**:` আউটপুট পুরনো ফাইলে যোগ করে
- `**<**:` ইনপুট ফাইল থেকে নেয়

### উদাহরণ:
```bash
ls > files.txt
cat files.txt >> allfiles.txt
sort < names.txt
```

## চেইনিং ও ফিল্টারিং

- একাধিক কমান্ড চেইন করা যায়:
  ```bash
  cat logs.txt | grep error | sort | uniq > errors.txt
  ```
- এতে error খুঁজে, সাজিয়ে, ইউনিক করে errors.txt-তে লেখা হয়।

## Pipes ও Redirects-এর কমন কমান্ড

- **grep:** টেক্সট খুঁজে বের করা
- **sort:** সাজানো
- **uniq:** ইউনিক
- **wc:** কাউন্ট
- **awk, sed:** টেক্সট প্রসেসিং

## বাস্তব উদাহরণ

ধরা যাক, সার্ভারের লগ ফাইলে error খুঁজে বের করতে হবে:
```bash
grep "error" /var/log/syslog | wc -l
```
এতে error-এর সংখ্যা পাওয়া যাবে।

## DevOps-এ Pipes ও Redirects

- লগ প্রসেসিং
- অটোমেশন স্ক্রিপ্ট
- ডেটা ফিল্টারিং
- রিপোর্ট জেনারেশন

## শুরু করার জন্য

- ls, cat, grep, sort, uniq, wc, awk, sed কমান্ড প্র্যাকটিস করুন
- Pipe ও Redirect দিয়ে চেইনিং করুন
- আউটপুট ফাইলে লিখুন, ইনপুট ফাইল থেকে নিন
