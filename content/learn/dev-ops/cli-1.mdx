CLI (Command Line Interface) হলো লিনাক্সের সবচেয়ে শক্তিশালী ও প্রয়োজনীয় টুল। GUI-র বাইরে CLI-তে কাজ করলে আপনি দ্রুত, স্ক্রিপ্টেবল, এবং অটোমেটেডভাবে সার্ভার/সিস্টেম ম্যানেজ করতে পারবেন।

## CLI কী?

CLI হলো এমন একটি ইন্টারফেস, যেখানে আপনি কমান্ড লিখে কম্পিউটারকে নির্দেশ দেন। Windows-এর Command Prompt, Mac-এর Terminal, আর লিনাক্সের Bash—সব CLI।

## কেন CLI শিখবেন?

- **Speed:** GUI-র চেয়ে CLI-তে কাজ দ্রুত হয়
- **Automation:** স্ক্রিপ্ট লিখে কাজ অটোমেট করা যায়
- **Remote Management:** SSH দিয়ে রিমোট সার্ভার ম্যানেজ করা যায়
- **Resource Friendly:** কম রিসোর্সে চলে

### বাস্তব উদাহরণ:

ধরা যাক, আপনাকে ১০০টি ফাইল কপি করতে হবে। GUI-তে এক এক করে করতে হবে, CLI-তে এক লাইনে:

```bash
cp /source/* /destination/
```

## CLI-র মূল অংশ

- **Shell:** Bash, Zsh, Fish
- **Prompt:** যেখানে কমান্ড লিখবেন
- **Command:** ls, cd, pwd, mkdir, rm, cp, mv, touch, echo
- **Arguments:** কমান্ডের সাথে অতিরিক্ত তথ্য (ls -l, rm -rf)
- **Pipe & Redirect:** আউটপুট অন্য কমান্ডে পাঠানো (ls | grep txt)

## বেসিক কমান্ড

| কমান্ড | কাজ |
|--------|-----|
| ls     | ফাইল/ফোল্ডার দেখার জন্য |
| cd     | ডিরেক্টরি পরিবর্তন |
| pwd    | বর্তমান অবস্থান |
| mkdir  | নতুন ফোল্ডার |
| rm     | ফাইল/ফোল্ডার ডিলিট |
| cp     | কপি |
| mv     | মুভ/রিনেম |
| touch  | নতুন ফাইল তৈরি |
| echo   | টেক্সট/ভ্যারিয়েবল দেখানো |

## CLI-তে নেভিগেশন

- cd /home/user
- cd .. (এক ধাপ পিছনে)
- cd ~ (ইউজার হোমে)
- ls -l (ডিটেইল দেখুন)

## Pipe ও Redirect

- **Pipe (|):** এক কমান্ডের আউটপুট অন্য কমান্ডে পাঠানো
  - উদাহরণ: ls | grep .txt
- **Redirect (>):** আউটপুট ফাইলে লেখা
  - উদাহরণ: ls > files.txt

## CLI-তে হেল্প

- `man <command>` (কমান্ডের ম্যানুয়াল)
- `<command> --help`

## DevOps-এ CLI

- সার্ভার/ক্লাউড/ডকার—সবকিছু CLI-তে ম্যানেজ হয়
- অটোমেশন স্ক্রিপ্ট, কনফিগ, ডিপ্লয়—all CLI
- SSH দিয়ে রিমোট সার্ভার ম্যানেজ

## শুরু করার জন্য

- Terminal ওপেন করুন
- ls, cd, pwd, mkdir, rm, cp, mv, touch, echo কমান্ড প্র্যাকটিস করুন
- man ও --help দিয়ে কমান্ডের ডিটেইল জানুন
