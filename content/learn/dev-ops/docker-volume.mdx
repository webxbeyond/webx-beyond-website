ডকার কন্টেইনার ব্যবহার করলে অনেক সময় আমাদের অ্যাপ্লিকেশনের ডেটা কন্টেইনারের বাইরে সংরক্ষণ করতে হয়। কারণ, কন্টেইনার ডিলিট বা রিস্টার্ট হলে কন্টেইনারের ভিতরের ডেটা হারিয়ে যায়। এই সমস্যার সমাধান হলো **ডকার ভলিউম**।

## ভলিউম কী?
ডকার ভলিউম হচ্ছে ডকারের নিজস্ব ডেটা ম্যানেজমেন্ট সিস্টেম, যা হোস্ট মেশিনে ডেটা সংরক্ষণ করে এবং কন্টেইনারের সাথে শেয়ার করে।

- কন্টেইনার ডিলিট হলেও ভলিউমের ডেটা থাকে
- একাধিক কন্টেইনার একই ভলিউম ব্যবহার করতে পারে
- ব্যাকআপ, রিস্টোর, মাইগ্রেশন সহজ

## কেন ভলিউম ব্যবহার করবেন?
- **ডেটা পার্সিস্টেন্স:** কন্টেইনার ডিলিট হলেও ডেটা থাকবে
- **শেয়ারড ডেটা:** একাধিক কন্টেইনারে একই ডেটা ব্যবহার
- **ব্যাকআপ:** সহজে ডেটা ব্যাকআপ ও রিস্টোর
- **সিকিউরিটি:** হোস্ট ফাইল সিস্টেম থেকে আলাদা

## ভলিউম তৈরি ও ব্যবহার

### ১. ভলিউম তৈরি
```bash
docker volume create mydata
```

### ২. ভলিউম লিস্ট
```bash
docker volume ls
```

### ৩. ভলিউম ইনফো
```bash
docker volume inspect mydata
```

### ৪. ভলিউম যুক্ত করে কন্টেইনার চালানো
```bash
docker run -d --name myapp -v mydata:/app/data nginx
```
এখানে `/app/data` হচ্ছে কন্টেইনারের ভিতরের ফোল্ডার, আর `mydata` হচ্ছে ভলিউম।

## রিয়েল লাইফ উদাহরণ: ডাটাবেজ ডেটা পার্সিস্টেন্স
ধরা যাক, আপনি একটি MySQL কন্টেইনার চালাচ্ছেন। ডেটাবেজের ডেটা যাতে কন্টেইনার ডিলিট হলেও থাকে, তার জন্য ভলিউম ব্যবহার করুন:

```bash
docker volume create mysql-data
docker run -d --name mysql-server -e MYSQL_ROOT_PASSWORD=secret -v mysql-data:/var/lib/mysql mysql:latest
```

এখন কন্টেইনার ডিলিট করলেও `mysql-data` ভলিউমে ডেটা থাকবে।

## একাধিক কন্টেইনারে একই ভলিউম

```bash
docker run -d --name app1 -v shared-data:/data busybox
docker run -d --name app2 -v shared-data:/data busybox
```

এভাবে `app1` ও `app2` কন্টেইনার একই ডেটা শেয়ার করতে পারে।

## ভলিউম ব্যাকআপ ও রিস্টোর

### ব্যাকআপ:
```bash
docker run --rm -v mydata:/data -v $(pwd):/backup busybox tar czf /backup/mydata.tar.gz -C /data .
```

### রিস্টোর:
```bash
docker run --rm -v mydata:/data -v $(pwd):/backup busybox tar xzf /backup/mydata.tar.gz -C /data
```

## ভলিউম ক্লিনআপ

### ভলিউম ডিলিট:
```bash
docker volume rm mydata
```

### অপ্রয়োজনীয় ভলিউম ডিলিট:
```bash
docker volume prune
```

## ভলিউম বনাম বাইন্ড মাউন্ট
- **ভলিউম:** ডকার নিজে ম্যানেজ করে, সিকিউর, ব্যাকআপ সহজ
- **বাইন্ড মাউন্ট:** হোস্টের নির্দিষ্ট ফোল্ডার কন্টেইনারে যুক্ত হয়, কনফিগারেশন বেশি

## ডকার কম্পোজে ভলিউম

`docker-compose.yml`:
```yaml
version: '3.8'
services:
  db:
    image: mysql:latest
    environment:
      MYSQL_ROOT_PASSWORD: secret
    volumes:
      - dbdata:/var/lib/mysql
volumes:
  dbdata:
```

## সিকিউরিটি ও বেস্ট প্র্যাকটিস
- প্রয়োজন ছাড়া ভলিউম শেয়ার করবেন না
- সেনসিটিভ ডেটা এনক্রিপ্ট করুন
- নিয়মিত ব্যাকআপ নিন
- অপ্রয়োজনীয় ভলিউম ডিলিট করুন

## সংক্ষেপে
ডকার ভলিউম ডেটা পার্সিস্টেন্স, শেয়ারিং, ব্যাকআপ ও রিস্টোরের জন্য অপরিহার্য। ডকারে ডেটা ম্যানেজমেন্টের জন্য ভলিউম ব্যবহার করুন এবং আপনার অ্যাপ্লিকেশনকে আরও রিলায়েবল ও স্কেলেবল করুন।
