# ব্রাঞ্চ ডিলিট করা কেন দরকার?

গিটে ব্রাঞ্চ তৈরি হয় নতুন ফিচার, বাগ ফিক্স, বা এক্সপেরিমেন্টের জন্য। কাজ শেষ হলে, ব্রাঞ্চ রেখে দিলে রেপোজিটরি অগোছালো হয়। তাই অপ্রয়োজনীয় ব্রাঞ্চ ডিলিট করা ভালো অভ্যাস।

## কখন ব্রাঞ্চ ডিলিট করবেন?
- ফিচার/বাগ ফিক্স ব্রাঞ্চ মার্জ হয়ে গেলে
- এক্সপেরিমেন্টাল ব্রাঞ্চ বাতিল হলে
- পুরনো/অপ্রয়োজনীয় ব্রাঞ্চ

## লোকাল ব্রাঞ্চ ডিলিট

লোকাল (নিজের কম্পিউটার) থেকে ব্রাঞ্চ ডিলিট করতে:

```bash
git branch -d branch-name
```

- `-d` (delete): ব্রাঞ্চ মার্জ হয়ে গেলে ডিলিট করবে।
- `-D` (force delete): মার্জ না হলেও জোর করে ডিলিট করবে।

**উদাহরণ:**
```bash
git branch -d feature/login-page
```

## রিমোট ব্রাঞ্চ ডিলিট

রিমোট (GitHub/GitLab) থেকে ব্রাঞ্চ ডিলিট করতে:

```bash
git push origin --delete branch-name
```

**উদাহরণ:**
```bash
git push origin --delete feature/login-page
```

## বাস্তব উদাহরণ

ধরা যাক, `feature/login-page` ব্রাঞ্চে কাজ শেষ।
1. মার্জ রিকোয়েস্ট দিয়ে মূল ব্রাঞ্চে মার্জ হয়েছে।
2. এখন লোকাল ও রিমোট থেকে ব্রাঞ্চ ডিলিট করুন:
   ```bash
   git branch -d feature/login-page
   git push origin --delete feature/login-page
   ```

## সতর্কতা
- ডিলিট করার আগে নিশ্চিত হন, ব্রাঞ্চের কাজ/কমিট প্রয়োজন নেই।
- মার্জ না হলে `-d` কাজ করবে না, `-D` ব্যবহার করলে সাবধান থাকুন।
- রিমোটে ডিলিট করলে অন্যদেরও ব্রাঞ্চটি আর থাকবে না।

## টিপস
- পুরনো ব্রাঞ্চ নিয়মিত ডিলিট করুন, রেপোজিটরি পরিষ্কার থাকবে।
- টিমে কাজ করলে, সবাইকে জানিয়ে ব্রাঞ্চ ডিলিট করুন।

## সংক্ষেপে
- ব্রাঞ্চ ডিলিট করা গিট রেপোজিটরি পরিষ্কার ও গোছালো রাখে।
- লোকাল ও রিমোট—দুই জায়গা থেকেই ডিলিট করতে হয়।
- সতর্কভাবে ডিলিট করুন, যাতে প্রয়োজনীয় কাজ হারিয়ে না যায়।
