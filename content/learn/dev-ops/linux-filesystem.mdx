লিনাক্স ফাইল সিস্টেম হলো ডেটা সংরক্ষণ, সংগঠিতকরণ, এবং অ্যাক্সেস করার পদ্ধতি। Windows-এর মতো ড্রাইভ (C:, D:) নয়, লিনাক্সে সবকিছু একটি "রুট" (/) থেকে শুরু হয় এবং সব ফাইল/ফোল্ডার সেই রুটের অধীনে থাকে।

## ফাইল সিস্টেমের মূল ধারণা

- **Root (/):** সবকিছুর শুরু
- **Directory:** ফোল্ডার, যেখানে ফাইল/ফোল্ডার থাকে
- **File:** ডেটা, কনফিগ, স্ক্রিপ্ট, ইত্যাদি
- **Path:** ফাইল/ফোল্ডারের অবস্থান (যেমন: /home/user/file.txt)

## গুরুত্বপূর্ণ ডিরেক্টরি

| ডিরেক্টরি | কাজ |
|-----------|-----|
| /          | রুট, সবকিছুর মূল |
| /home      | ইউজারদের ফোল্ডার |
| /etc       | কনফিগারেশন ফাইল |
| /var       | লগ, চেঞ্জিং ডেটা |
| /usr       | ইউটিলিটি, অ্যাপ |
| /bin       | বেসিক কমান্ড |
| /sbin      | সিস্টেম কমান্ড |
| /tmp       | টেম্পরারি ফাইল |
| /opt       | এক্সট্রা সফটওয়্যার |

## ফাইল সিস্টেম টাইপ

- **ext4:** সবচেয়ে কমন, ফাস্ট ও রিলায়েবল
- **xfs:** বড় ডেটা, সার্ভার
- **btrfs:** অ্যাডভান্সড ফিচার

## Permission ও Ownership

- **Owner:** কে ফাইল/ফোল্ডার ব্যবহার করতে পারবে
- **Group:** কোন গ্রুপ ব্যবহার করতে পারবে
- **Permission:** Read (r), Write (w), Execute (x)
- **Command:** chmod, chown, chgrp

### উদাহরণ:

```bash
ls -l /home/user
```

এতে দেখবেন: owner, group, permission

## ফাইল সিস্টেমে কমান্ড

- **ls:** ফাইল/ফোল্ডার দেখার জন্য
- **cd:** ডিরেক্টরি পরিবর্তন
- **pwd:** বর্তমান অবস্থান
- **mkdir:** নতুন ফোল্ডার
- **rm:** ফাইল/ফোল্ডার ডিলিট
- **cp:** কপি
- **mv:** মুভ/রিনেম

## বাস্তব উদাহরণ

ধরা যাক, আপনি /home/anis ফোল্ডারে একটি স্ক্রিপ্ট রাখলেন।
- আপনি চাইলে শুধু নিজে রান করতে পারবেন (chmod 700)
- চাইলে টিমের সবাইকে রান করার পারমিশন দিতে পারবেন (chmod 770)

## DevOps-এ ফাইল সিস্টেমের গুরুত্ব

- কনফিগ ফাইল, লগ, স্ক্রিপ্ট—সবকিছু সঠিক ডিরেক্টরিতে রাখতে হয়
- পারমিশন ঠিক না থাকলে সার্ভার/অ্যাপ্লিকেশন চলবে না
- অটোমেশন স্ক্রিপ্টে ফাইল/ডিরেক্টরি ম্যানেজমেন্ট অপরিহার্য

## ফাইল সিস্টেম চেক ও মেইনটেন্যান্স

- **df -h:** ডিস্ক ইউজেজ
- **du -sh /path:** ফোল্ডার সাইজ
- **fsck:** ফাইল সিস্টেম চেক
- **mount/umount:** ড্রাইভ/ডিরেক্টরি মাউন্ট/আনমাউন্ট

## শুরু করার জন্য

- ls, cd, pwd, mkdir, rm, cp, mv কমান্ড প্র্যাকটিস করুন
- ফাইল/ফোল্ডার permission ও ownership বুঝুন
- কনফিগ ফাইল কোথায় থাকে, শিখুন (/etc)
