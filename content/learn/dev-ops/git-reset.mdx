গিট রিসেট হলো এমন একটি কমান্ড, যা দিয়ে আপনি কমিট, ফাইল, বা ব্রাঞ্চকে আগের অবস্থায় ফিরিয়ে আনতে পারেন। এটি ভুল কমিট, ভুল পরিবর্তন, বা অপ্রয়োজনীয় কাজ "উল্টে" দিতে কাজে লাগে।

## গিট রিসেটের ধরন

### ১. Soft Reset
```bash
git reset --soft <commit-SHA>
```
- HEAD (ব্রাঞ্চ পয়েন্টার) আগের কমিটে যাবে, কিন্তু পরিবর্তনগুলো স্টেজড থাকবে।
- **ব্যবহার:** ভুল কমিট করলে, কিন্তু পরিবর্তন রাখতে চাইলে।

### ২. Mixed Reset (Default)
```bash
git reset <commit-SHA>
```
- HEAD আগের কমিটে যাবে, পরিবর্তনগুলো unstaged থাকবে।
- **ব্যবহার:** কমিট "উল্টে" দিয়ে, পরিবর্তন আবার এডিট করতে চাইলে।

### ৩. Hard Reset
```bash
git reset --hard <commit-SHA>
```
- HEAD আগের কমিটে যাবে, পরিবর্তনগুলো হারিয়ে যাবে।
- **ব্যবহার:** সবকিছু আগের অবস্থায় ফিরিয়ে আনতে, কিন্তু সাবধান—হারানো পরিবর্তন ফেরত পাওয়া যাবে না।

## বাস্তব উদাহরণ

> "আমরা টিমে কাজ করি, ভুল কমিট হলে soft reset করি—পরিবর্তন থাকে, আবার কমিট করি। কখনো বড় ভুল হলে hard reset করি, কিন্তু আগে ব্যাকআপ রাখি।"

## নির্দিষ্ট ফাইল রিসেট
```bash
git checkout HEAD path/to/file
```
- এতে নির্দিষ্ট ফাইল আগের কমিটের অবস্থায় ফিরে যাবে।

## আগের কমিট ডিলিট (Uncommit)
```bash
git reset HEAD~1
```
- শেষ কমিট "উল্টে" যাবে, পরিবর্তন unstaged থাকবে।

## সতর্কতা
- Hard reset করলে পরিবর্তন হারিয়ে যেতে পারে—ব্যাকআপ রাখুন।
- পাবলিক/রিমোট ব্রাঞ্চে reset করলে টিমের অন্যদের সমস্যা হতে পারে।
- Soft/Mixed reset নিরাপদ, Hard reset সাবধানে ব্যবহার করুন।

## টিপস
- Visual tool (VS Code, GitHub Desktop) দিয়েও reset করা যায়।
- ভুল হলে, reset করার আগে সবকিছু ব্যাকআপ রাখুন।

## সংক্ষেপে
- গিট রিসেট দিয়ে কমিট, ফাইল, বা ব্রাঞ্চ আগের অবস্থায় ফিরিয়ে আনা যায়।
- Soft/Mixed/Hard—তিন ধরনের reset আছে, প্রয়োজন অনুযায়ী ব্যবহার করুন।
- সতর্কভাবে reset ব্যবহার করুন, বিশেষ করে টিমওয়ার্কে।
