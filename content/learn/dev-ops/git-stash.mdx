description: গিটে স্ট্যাশের ব্যবহার, সুবিধা, ও বাস্তব উদাহরণ
---

গিট স্ট্যাশ হলো এমন একটি ফিচার, যা আপনার বর্তমান কাজ (আনকমিটেড পরিবর্তন) সাময়িকভাবে "স্টোর" করে রাখে, যাতে আপনি অন্য ব্রাঞ্চে বা টাস্কে সুইচ করতে পারেন। পরে আবার সেই কাজ ফিরিয়ে আনতে পারেন।

## কখন গিট স্ট্যাশ ব্যবহার করবেন?
- আপনি কোনো ফিচার/বাগ ফিক্সে কাজ করছেন, হঠাৎ অন্য জরুরি টাস্কে সুইচ করতে হবে।
- আপনার পরিবর্তনগুলো এখনই কমিট করতে চান না, কিন্তু হারাতে চান না।

## গিট স্ট্যাশের কমান্ড

### কাজ স্ট্যাশ করুন
```bash
git stash
```

### স্ট্যাশের নাম দিন
```bash
git stash save "work-in-progress: login page"
```

### স্ট্যাশের তালিকা দেখুন
```bash
git stash list
```

### স্ট্যাশ ফিরিয়ে আনুন
```bash
git stash apply
```

### নির্দিষ্ট স্ট্যাশ ফিরিয়ে আনুন
```bash
git stash apply stash@{2}
```

### স্ট্যাশ ডিলিট করুন
```bash
git stash drop stash@{0}
```

### সব স্ট্যাশ ডিলিট করুন
```bash
git stash clear
```

## বাস্তব উদাহরণ

ধরা যাক, আপনি `feature/login-page` ব্রাঞ্চে কাজ করছেন। হঠাৎ `main` ব্রাঞ্চে জরুরি বাগ ফিক্স করতে হবে।

1. কাজ স্ট্যাশ করুন:
   ```bash
   git stash save "login page WIP"
   ```
2. `main` ব্রাঞ্চে সুইচ করুন, বাগ ফিক্স করুন।
3. আবার `feature/login-page`-এ ফিরে এসে:
   ```bash
   git stash list
   git stash apply stash@{0}
   ```

## টিপস
- স্ট্যাশে শুধু আনকমিটেড পরিবর্তন (staged/unstaged) থাকে, নতুন ফাইল (untracked) রাখতে চাইলে:
  ```bash
  git stash -u
  ```
- একাধিক কাজ স্ট্যাশ করা যায়, পরে যেটা দরকার, সেটা ফিরিয়ে আনুন।
- স্ট্যাশের নাম দিন, যাতে বুঝতে সুবিধা হয়।

## সতর্কতা
- স্ট্যাশে রাখা কাজ ভুলে গেলে হারিয়ে যেতে পারে, তাই সময়মতো ফিরিয়ে আনুন।
- স্ট্যাশ apply করার পরও, স্ট্যাশ লিস্টে থাকে—ডিলিট করতে `git stash drop` বা `clear` ব্যবহার করুন।

## সংক্ষেপে
- গিট স্ট্যাশ সাময়িকভাবে কাজ "স্টোর" করে রাখে।
- জরুরি টাস্কে সুইচ করতে, বা কাজ হারাতে না চাইলে স্ট্যাশ ব্যবহার করুন।
- টিমওয়ার্কে, মাল্টিপল টাস্কে স্ট্যাশ খুবই কার্যকর।
