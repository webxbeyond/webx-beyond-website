title: রিবেস (Git Rebase)

---

গিটে রিবেস হলো একটি ব্রাঞ্চের পরিবর্তনকে অন্য ব্রাঞ্চের উপর "পুনরায় লেখা"। এটি মার্জের মতোই, কিন্তু ইতিহাসকে আরও পরিষ্কার ও সরল করে।

## কেন রিবেস ব্যবহার করবেন?
- **পরিষ্কার ইতিহাস:** কমিটগুলো সরাসরি মূল ব্রাঞ্চের পরে চলে আসে, মার্জ কমিট হয় না।
- **লিনিয়ার হিস্ট্রি:** ইতিহাসে কমিটগুলো এক লাইনে থাকে, ব্রাঞ্চিং কমপ্লেক্সিটি কমে।
- **কনফ্লিক্ট সহজে সমাধান:** একাধিক ফিচার ব্রাঞ্চে কাজ করলে, কনফ্লিক্ট সহজে দেখা যায় ও সমাধান করা যায়।

## রিবেসের ধাপ

ধরা যাক, আপনি `feature/login-page` ব্রাঞ্চে কাজ করছেন, আর `main` ব্রাঞ্চে নতুন কমিট এসেছে। আপনার ব্রাঞ্চকে আপডেট রাখতে চাইলে:

1. **মূল ব্রাঞ্চে সুইচ করুন ও আপডেট নিন:**
   ```bash
   git checkout main
   git pull origin main
   ```
2. **ফিচার ব্রাঞ্চে ফিরে যান:**
   ```bash
   git checkout feature/login-page
   ```
3. **রিবেস করুন:**
   ```bash
   git rebase main
   ```
   এতে আপনার ব্রাঞ্চের কমিটগুলো `main`-এর সর্বশেষ কমিটের পরে চলে আসবে।

## কনফ্লিক্ট হলে কী করবেন?
- রিবেস চলাকালীন কনফ্লিক্ট হলে, গিট জানিয়ে দেবে কোন ফাইল কনফ্লিক্ট হয়েছে।
- কনফ্লিক্ট ম্যানুয়ালি ঠিক করুন, তারপর:
  ```bash
  git add <conflicted-file>
  git rebase --continue
  ```
- সব কনফ্লিক্ট ঠিক হলে, রিবেস শেষ হবে।

## ইন্টার‍্যাক্টিভ রিবেস

ইতিহাস সাজাতে, কমিট স্কোয়াশ/রিনেম/ডিলিট করতে ইন্টার‍্যাক্টিভ রিবেস ব্যবহার হয়:
```bash
git rebase -i main
```
- এখানে কমিটগুলো নিয়ে নানা অপশন পাবেন (pick, squash, edit, drop)।

## বাস্তব উদাহরণ

> "আমাদের টিমে সবাই আলাদা ফিচার ব্রাঞ্চে কাজ করে। কাজ শেষে মার্জের আগে রিবেস করে, যাতে ইতিহাস পরিষ্কার থাকে। এতে কনফ্লিক্ট কম হয়, কোড রিভিউ সহজ হয়।"

## সতর্কতা
- রিবেস ইতিহাস পরিবর্তন করে, তাই পাবলিক/শেয়ার করা ব্রাঞ্চে রিবেস না করাই ভালো।
- লোকাল/নিজের ব্রাঞ্চে রিবেস নিরাপদ।
- রিবেসের পর force push (`git push --force`) লাগতে পারে।

## মার্জ বনাম রিবেস
| বিষয়         | মার্জ                | রিবেস                |
|--------------|----------------------|----------------------|
| ইতিহাস        | ব্রাঞ্চিং, মার্জ কমিট | সরল, লিনিয়ার         |
| কনফ্লিক্ট    | মার্জের সময়          | রিবেসের সময়          |
| ব্যবহার       | পাবলিক/টিম           | লোকাল/নিজের কাজ      |

## সংক্ষেপে
- রিবেস ইতিহাস পরিষ্কার রাখে, কনফ্লিক্ট সহজে সমাধান হয়।
- মার্জের মতোই, কিন্তু ইতিহাস "পুনরায় লেখা" হয়।
- সতর্কভাবে ব্যবহার করুন, বিশেষ করে শেয়ার করা ব্রাঞ্চে।
